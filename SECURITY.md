# 安全漏洞报告

## 私下报告

请使用仓库的 GitHub **Security Advisories** 创建私有报告：

https://github.com/laoliu-463/saas/security/advisories/new

不要为安全漏洞创建 public Issue，也不要在 PR、日志、截图或 evidence 中提交可利用细节、真实 Token、密码、密钥、OAuth code、个人信息或业务敏感数据。

报告建议包含：

- 受影响的版本或完整 commit SHA；
- 可复现步骤和最小复现；
- 影响范围与利用前提；
- 已脱敏的日志、请求响应或截图；
- 建议修复或缓解方式（如有）。

维护者完成复现和影响评估后，会通过 Security Advisory 协调修复、验证和披露。不要在修复合并前公开漏洞细节。

## 非安全问题

普通缺陷、功能建议和协作流程问题请使用对应的 GitHub Issue Form。若无法确认是否属于安全问题，优先使用 Security Advisories 私下报告。
