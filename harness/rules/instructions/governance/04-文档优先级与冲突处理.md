# 文档优先级与冲突处理

## 文档优先级

出现冲突时按以下顺序判断：

1. 当前源码与真实运行环境。
2. V1 交付合同。
3. 当前项目状态。
4. real-pre 证据。
5. 七个领域当前版文档。
6. 技术落地设计。
7. V2.2 完整方案，仅作历史背景。

## 明确冲突口径

- 旧文档中的 FastAPI / Celery 不代表当前实际技术栈。
- 旧文档中的 Python 爬虫式方案不代表当前运行事实。
- 旧文档中的独家达人 / 独家商家不代表 V1 必做。
- 旧文档中的复杂财务毛利扩展不代表 V1 必做；2026-06-05 起经营毛利（Y-04）按 `服务费收益 - 招商提成 - 渠道/媒介提成` 纳入 V1 指标。
- 旧文档中的自动物流不代表 V1 必做。
- 旧文档中的外部抖店快速寄样不代表 V1 必做。

## 冲突处理流程

1. 不直接删除冲突文档。
2. 在新文档中只引用当前可执行事实。
3. 在 `04-state/05-文档债务与冲突台账.md` 登记冲突文件、问题类型和建议处理。
4. 涉及业务范围裁决时，写入或更新 `docs/决策/ADR-002-V1范围优先级.md`。
5. 无证据时只写 `PENDING` 或阶段性结论。

## 保留策略

| 文档类型 | 处理方式 |
| --- | --- |
| 高价值旧文档 | 保留并在索引中引用。 |
| V2 规划文档 | 标注为历史背景或 V2 规划。 |
| 冲突文档 | 登记到文档债务台账。 |
| 重复文档 | 优先合并摘要，不直接删除。 |
| 明显临时文件 | 登记为待归档，等待人工确认。 |
