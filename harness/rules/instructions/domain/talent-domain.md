# Talent Domain DDD Instruction

## 领域职责

- 负责达人资料、标签、地址、跟进、认领和保护期。
- 向寄样、商品和业绩链路提供达人事实。

## 领域不负责

- 不负责订单事实采集和订单归属计算。
- 不负责提成、冲正、双轨金额。
- 不替代订单域或业绩域做最终归属。

## V1 规则

- 达人信息必须可支撑寄样、跟进、渠道协作和业绩归属输入。
- 第三方达人接口必须提供真实响应或 BLOCKED 证据。
- 当前 `gender` 筛选缺口按 `docs/领域/达人域.md` 记录处理，不得静默假筛选。

## 禁止越界

- 禁止达人域采集订单事实。
- 禁止达人域计算最终归属、提成或冲正。
- 禁止达人域伪造第三方接口数据。

## 允许调用的 Facade

- 优先使用达人域应用服务、Facade、达人查询 API 或达人资料事件。
- 寄样域消费达人地址时必须保留来源和更新时间。

## 必须执行的测试

- 达人列表 / 详情 / 标签 / 地址 / 跟进 API 或 E2E。
- 权限数据范围和越权负例。
- 第三方接口真实响应或 BLOCKED 证据。

## 完成后必须更新的 state

- `harness/rules/state/snapshots/DOMAIN_STATUS.md` 的达人域状态。
- 达人资料、地址、标签、跟进和接口缺口相关 evidence。

## 失败后必须写入 feedback

- 达人事实缺失、地址联动失败、越权或第三方接口阻塞写入 `harness/feedback/` 或 evidence report。
