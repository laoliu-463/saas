# Config Domain DDD Instruction

## 领域职责

- 负责配置读取、保存、校验、版本和审计。
- 负责系统配置、规则参数、提成比例、复制模板和 `pick_extra` 等配置事实。
- 对业务域提供配置查询能力。

## 领域不负责

- 不负责商品转链、订单归因、提成计算、寄样状态流转。
- 不负责直接触发业务动作。

## V1 规则

- 配置域只提供参数，不执行具体业务规则。
- 规则执行由读取配置的业务域负责。
- 配置变更必须保留操作者、变更前后值和时间。

## 禁止越界

- 禁止配置域计算业绩或归属。
- 禁止配置域直接写商品、订单、寄样或业绩事实。
- 禁止按 dataScope 过滤全局规则中心导致不同操作者看到不同规则输入。

## 允许调用的 Facade

- 优先使用配置域应用服务、Facade、查询 API 或配置读取接口。
- 业务域不得直接改配置表绕过审计。

## 必须执行的测试

- 配置读取 / 保存单测。
- 配置变更审计 API/SQL 验证。
- 规则中心权限和异常分支验证。

## 完成后必须更新的 state

- `harness/state/DOMAIN_STATUS.md` 的配置域状态。
- 相关 evidence report 和配置域风险记录。

## 失败后必须写入 feedback

- 规则执行落错领域、配置审计缺失或数据范围误用写入 `harness/feedback/` 或 evidence report。
