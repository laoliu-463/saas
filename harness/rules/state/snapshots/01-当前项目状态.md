# 当前项目状态

## 当前事实

- 当前技术栈：Spring Boot 3.2 / Java 17 + Vue 3 / Vite / Pinia / Naive UI / TypeScript + PostgreSQL + Redis + Docker Compose。
- 当前环境：`test`、`real-pre`、远端 `real-pre`。
- `test` 是 mock 回归基线。
- `real-pre` 是真实上游 / 生产形态验证环境，不能用 mock 数据证明闭环。

## 已具备能力

- 用户域：登录、角色、菜单、组织和数据范围主链路已具备。
- Authorization Phase 2：本地与远端 `real-pre` 均已具备 4 张授权基础表和 `sys_user.authz_version`；远端提交 `38689b0a` 已通过固定脚本部署且四个核心容器健康。运行模式仍为 `LEGACY(default)`，授权表与 Redis `authz:*` 均为空，新 RBAC 尚未接管业务权限。
- 配置域：配置读取、变更和审计主链路已具备。
- 商品域：商品库、活动商品同步、转链和映射主链路已具备。
- 商品编辑：开始/结束时间已支持日期时间选择；本系统覆盖值写入商品运营状态，清空后回退上游快照；本轮本地构建、定向回归、real-pre 容器健康通过，真实页面/API 正向验收仍因管理员登录 HTTP 401 阻塞。
- 达人域：达人资料、标签、地址和跟进主链路已具备。
- 寄样域：申请、审批、发货和订单事件自动完成链路已具备。
- 订单域：订单事实、退款事实、同步日志和归因输入已具备。
- 业绩域：最终归属、提成、冲正和汇总主链路已具备。
- 分析模块：dashboard、报表和只读汇总主链路已具备。

## 当前未闭环点

- Authorization 远端完整业务验收未闭环：preflight 与 RBAC scope 通过，但 P0 因 SKU 页面断言失败，六角色业务流仅 5/13 steps 通过；订单归因和寄样自动完成缺真实样本。直接 SKU API/proxy 探针为 200，失败 UI 请求未到 backend，根因仍待专项诊断，禁止切换 `SHADOW/ENFORCE`。
- 商品域商品库 full backfill 能力已上线到 real-pre；dry-run、安全确认闸、表结构和展示口径保护已验证通过，但历史商品缺口尚未执行真实写库回补，不能声明“商品库数量已补齐”。
- 商品库服务费率专项已完成本地代码、前后端回归、real-pre 构建/重启/健康检查；普通商品读取上游 `service_ratio`，双佣商品读取 `ad_service_ratio`，缺失值与显式 `0` 已区分。真实页面/API 验收因 admin 登录 HTTP 401、`BLOCKED_AUTH` 暂未闭环。
- 渠道链真实闭环依赖真实通过系统转链产生的订单样本。
- real-pre 历史订单 `pick_source` 大量为空，不能证明渠道归因闭环。
- 寄样自动完成依赖订单归因后的 `channel_id + talent_id + product_id + pay_time`。
- 不能把订单入库等同于业务闭环。

## 关键判断口径

真正闭环必须验证：

- `orders.pick_source`
- `orders.default_channel_id`
- `orders.default_recruiter_id`
- `sample_requests` 状态流转
- `performance_records`
- dashboard / 业绩归属
