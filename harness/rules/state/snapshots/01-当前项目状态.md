# 当前项目状态

## 当前事实

- 当前技术栈：Spring Boot 3.2 / Java 17 + Vue 3 / Vite / Pinia / Naive UI / TypeScript + PostgreSQL + Redis + Docker Compose。
- 当前环境：`test`、`real-pre`、远端 `real-pre`。
- `test` 是 mock 回归基线。
- `real-pre` 是真实上游 / 生产形态验证环境，不能用 mock 数据证明闭环。
- DDD100 当前推进基线已于 2026-06-27 重算并脚本化：`harness/scripts/probes/ddd-migration-metrics.ps1` 当前输出 raw `domain/` share 20.2%、业务迁移代理 26.6%。
- DDD100 架构红线 guard 已于 2026-06-27 新增：冻结 Controller 直连 Mapper/Gateway 债务，零容忍 strict domain-layer Mapper 穿透、前端第三方 HTTP 直连和新增非用户域 DataScope 复制规则。
- DDD100 #34 已补 SysUser CRUD true-route 委托证据：兼容 `SysUserService` 的 getById/create/update/delete/resetPassword 进入用户域 `SysUserCRUDApplicationA/B`。
- DDD100 #35 已将组织成员变更持久化下沉到 `UserGroupMembershipStore`，并显式验证 `dept_id` 可写入与清空。
- DDD-VERIFY-001 商品库查询切片已把 `ProductService.getSelectedLibraryPage` 排序迁入 `ProductLibrarySortPolicy`，活动名称 / 保护期元数据改批量加载；Product 相关测试通过，架构大集合仍有既有 DDD 红线失败，结论为 PARTIAL。

## 已具备能力

- 用户域：登录、角色、菜单、组织和数据范围主链路已具备。
- 配置域：配置读取、变更和审计主链路已具备。
- 商品域：商品库、活动商品同步、转链和映射主链路已具备。
- 达人域：达人资料、标签、地址和跟进主链路已具备。
- 寄样域：申请、审批、发货和订单事件自动完成链路已具备。
- 订单域：订单事实、退款事实、同步日志和归因输入已具备。
- 业绩域：最终归属、提成、冲正和汇总主链路已具备。
- 分析模块：dashboard、报表和只读汇总主链路已具备。

## 当前未闭环点

- 商品域商品库 full backfill 能力已上线到 real-pre；dry-run、安全确认闸、表结构和展示口径保护已验证通过，但历史商品缺口尚未执行真实写库回补，不能声明“商品库数量已补齐”。
- 渠道链真实闭环依赖真实通过系统转链产生的订单样本。
- real-pre 历史订单 `pick_source` 大量为空，不能证明渠道归因闭环。
- 寄样自动完成依赖订单归因后的 `channel_id + talent_id + product_id + pay_time`。
- 不能把订单入库等同于业务闭环。
- DDD100 基线中的 `other/unclassified` 老位置业务代码仍需后续分类，不能把 26.6% 代理指标写成最终 DDD 完成率。
- Controller 直连 Mapper/Gateway 与非用户域直接 DataScope 判断都是冻结债务，不代表已清零；后续迁移必须删除白名单项而不是新增。
- Product query 直接依赖 Mapper、跨域 Mapper 和非用户域 DataScope 既有红线仍在，不能把本轮 ProductService 瘦身写成商品域 DDD 完成。

## 关键判断口径

真正闭环必须验证：

- `orders.pick_source`
- `orders.default_channel_id`
- `orders.default_recruiter_id`
- `sample_requests` 状态流转
- `performance_records`
- dashboard / 业绩归属
