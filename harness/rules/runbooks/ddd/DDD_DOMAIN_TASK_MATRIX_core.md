# DDD Domain Task Matrix — Core（核心业务域）

> 订单 / 商品 / 达人 / 寄样 4 个核心业务域的详细任务清单。
> 拆分自 `DDD_DOMAIN_TASK_MATRIX.md`（GC-OPTIMIZE-003 Step 2）

## 订单域

- O-1 盘点订单域代码、接口、表、同步任务和测试。
- O-2 梳理抖音订单同步入口、参数、响应和错误码。
- O-3 梳理订单事实字段、`raw_payload` 和幂等键。
- O-4 梳理 `pick_source` 和 `colonel_buyin_id` 归因输入。
- O-5 梳理默认渠道和默认招商归因输入。
- O-6 确认订单域不计算提成。
- O-7 确认订单域不写 `performance_records`。
- O-8 确认订单域不直接更新寄样完成状态。
- O-9 收口订单查询的数据范围消费。
- O-10 收口退款 / 售后事实保存。
- O-11 收口订单同步日志和失败证据。
- O-12 定义订单已同步事件生产时机。
- O-13 定义退款事实已同步事件生产时机。
- O-14 验证订单同步幂等。
- O-15 验证 real-pre 真实上游响应或 BLOCKED 证据。
- O-16 补订单域单测。
- O-17 补订单同步集成测试。
- O-18 补订单事件消费方回归验证。
- O-19 更新订单域 state、feedback 和 evidence。

## 商品域

- P-1 盘点商品域代码、接口、表、任务和测试。
- P-2 梳理商品库和活动商品表结构。
- P-3 梳理活动商品同步入口、上游状态和 repair 入口。
- P-4 梳理商品展示规则输入和本地发布控制。
- P-5 梳理转链接口、请求、响应和错误码。
- P-6 梳理 `pick_source_mapping` 落库字段和幂等键。
- P-7 确认商品域不计算最终归属。
- P-8 确认商品域不计算提成。
- P-9 确认商品域不直接写寄样表。
- P-10 收口商品库查询条件和数据范围。
- P-11 收口活动商品人工审核补充信息。
- P-12 收口商品转链事件或映射证据。
- P-13 验证推广中商品自动入库。
- P-14 验证历史状态断链 repair。
- P-15 验证商品库 `productId` 精确查询。
- P-16 验证转链后映射可追溯。
- P-17 验证 real-pre 上游响应或 BLOCKED 证据。
- P-18 补商品域单测。
- P-19 补商品同步集成测试。
- P-20 补转链集成测试。
- P-21 补前端商品库 E2E。
- P-22 更新商品域 state、feedback 和 evidence。

## 达人域

- T-1 盘点达人域代码、接口、表和测试。
- T-2 梳理达人资料、标签、地址和跟进记录。
- T-3 梳理达人认领和保护期规则来源。
- T-4 梳理达人和渠道、商品、寄样关系事实。
- T-5 确认达人域不采集订单事实。
- T-6 确认达人域不计算订单归属。
- T-7 确认达人域不计算提成、冲正或双轨金额。
- T-8 收口达人列表和详情数据范围。
- T-9 收口达人地址供寄样域消费的边界。
- T-10 收口达人标签和跟进审计。
- T-11 验证第三方达人补充接口真实响应或 BLOCKED。
- T-12 验证 `gender` 筛选缺口仍按当前文档处理。
- T-13 补达人域单测。
- T-14 补达人列表 / 详情 E2E。
- T-15 补达人地址和寄样联动验证。
- T-16 补权限越权负例。
- T-17 补异常分支验证。
- T-18 更新达人域 state、feedback 和 evidence。

## 寄样域

- S-1 盘点寄样域代码、接口、表、状态机和测试。
- S-2 梳理寄样申请、审批、发货、签收和交作业状态。
- S-3 梳理状态流转合法性和操作日志。
- S-4 梳理达人地址、商品上下文和渠道关系输入。
- S-5 确认寄样域不同步订单事实。
- S-6 确认寄样域不计算业绩归属。
- S-7 确认寄样域不计算提成或冲正。
- S-8 确认寄样域不依赖物流 API 阻塞主流程。
- S-9 收口寄样列表和详情数据范围。
- S-10 收口审核和发货权限边界。
- S-11 收口订单已同步事件消费逻辑。
- S-12 验证交作业完成命中条件：`channel_id + talent_id + product_id + pay_time`。
- S-13 验证重复订单事件消费幂等。
- S-14 验证订单样本不足时写 BLOCKED / PENDING。
- S-15 补寄样域单测。
- S-16 补状态机集成测试。
- S-17 补订单事件消费测试。
- S-18 补权限越权负例。
- S-19 补寄样 E2E。
- S-20 补异常分支验证。
- S-21 补 real-pre 证据索引。
- S-22 更新寄样域 state、feedback 和 evidence。
