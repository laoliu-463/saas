# Report Style Policy

所有 Harness 报告需遵循以下统一格式（不得超过 200 行）：

## 结论
PASS / PARTIAL / FAIL

## 证据
- **文件**: (影响或检查的对应文件清单)
- **命令**: (执行的关键命令)
- **接口**: (验证的接口名/路径)
- **日志**: (部分关键日志摘要)

## 风险
- 当前仍存在的问题，以及未解决的技术债或阻碍点。

## 下一步
- 可执行动作，明确接下来的操作目标或需要人工介入的事项。

> **注意事项**：
> - 报告需直击重点，禁止冗长的前言或毫无意义的上下文重复。
> - 过程流水账应当在输出本报告前丢弃。
> - 若有多个任务，应分别独立生成短报告并用 `INDEX.md` 索引，严禁输出多任务混杂文档。
