## 4. 登录认证链路

`AuthController` → `AuthService`：锁定 → 用户解析 → BCrypt → 多角色 scope → JWT 双 token → 审计。

`JwtAuthenticationFilter`：黑名单 → request attribute → SecurityContext。

## 5. 用户/角色/菜单/部门模型

7 张核心表；`sys_user_role` 支持一人多角色；`sys_role.permissions` JSON 承载操作权限。

## 6. 多角色实现

dataScope 取 max；ADMIN/OPS_STAFF 强制 ALL；`UserDomainService` 合并 permissions 并集。

## 7. 数据范围实现

**三套并行**：DataScopeAspect | SysUserService.applyDataScopeFilter | PerformanceAccessScope。

self/group/all ↔ PERSONAL/DEPT/ALL；group 仅 dept_id 成员。

## 8. 跨域依赖

Talent/Product/Sample/Order/Data/Merchant 等直接注入 SysUserMapper；OrderController 注入 SysDeptMapper。

## 9. Mapper 横穿

跨域 Service/Controller 直接操作用户域 Mapper，无 Facade 边界。

## 10. DTO/Entity/VO 混用

`auth/dto` + `dto/user` + `vo` 并存；CurrentUser 用 record，管理端用 VO。

## 11. God Service

**SysUserService ~1277 行**（CRUD+scope+组织+角色+分配+事件）。
