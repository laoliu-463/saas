# GIT-BATCH-6 Retro - User Domain Legacy Shells

## What Changed

- Converted legacy user/org services into thin shells over user-domain
  application services.
- Added ports and adapters so org lookup, deletion constraints, leader lookup,
  department repository access, channel-code lookup, and user-department lookup
  are explicit dependencies.
- Added boundary and parity tests to protect behavior while moving logic out of
  legacy services.

## What Worked

- Temp worktree validation caught two real batch-boundary issues before commit:
  missing `SysUserRoleAssignmentApplicationService` and an invalid null-bearing
  `List.of(...)` test input.
- Building the Docker image from the temp candidate worktree kept unrelated
  dirty main-worktree changes out of the deployed backend image.
- Focused tests provide a useful regression net for the next user-domain slice.

## Process Notes

- code-review-graph was used first, but target semantic search had no useful
  function hits and one minimal-context call timed out. Source import tracing
  remains necessary for this batch size.
- `restart-compose.ps1` does not pass the configured compose project name with
  `-p`; direct compose with `-p saas-active` was used to avoid creating a temp
  compose project.
- No harness script upgrade was made in this batch. A future improvement is to
  let restart tooling accept a candidate build context while using the canonical
  real-pre env file and compose project.
