## 2. 核心经验

* **事实优先：** 在 DDD 渐进式重构的长周期项目中，各领域审计任务的状态、报告文件与任务卡的对账必须做到 100% 准确。任何不准确的“任务卡缺失”描述都可能污染后续 Agent 对进度边界的认知，仲裁能够起到强有力的事实保护屏障作用。
* **红线显式声明：** 在跨阶段交接（如从 Phase 0 审查到 Phase 1 测试）时，显式确认“测试先行”与“不编造/Mock真实结算”的规则，有助于约束后置 Agent 的行为规范，控制重构的技术风险。
