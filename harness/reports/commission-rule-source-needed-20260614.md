# COMMISSION-RULE-SOURCE-001 业务侧确认清单

## 结论

- 状态: BLOCKED_BY_COMMISSION_RULE_SOURCE
- 用途: 补齐差异化提成规则来源后，才能进入 COMMISSION-RULE-IMPLEMENT-001。
- 当前证据: 服务费收益已对齐；毛利差异来自招商/渠道提成规则来源缺失。

## 一、最低可用数据

请业务侧至少提供 2026-06-08 至 2026-06-13 每天的以下字段。

| 字段 | 说明 |
|---|---|
| 服务费收益 | 元；需与当前 `serviceFeeProfit` 口径一致 |
| 招商提成 | 元；不能只给总提成 |
| 渠道提成 | 元；不能只给总提成 |
| 毛利 | 元；业务认可基准 |
| 总提成率 | `(招商提成 + 渠道提成) / 服务费收益` |
| 统计口径 | 付款时间 / 创建时间 / 结算时间 |
| 订单范围 | 是否包含退款、取消、失效单 |
| 服务费支出 | 是否已在服务费收益中扣除 |

## 二、最好提供订单级数据

每单至少包含以下字段。

- `order_id`
- `pay_time`
- `product_id`
- `activity_id`
- `talent_id`
- `channel_id`
- `recruiter_id`
- `service_profit`
- `recruiter_commission`
- `channel_commission`
- `gross_profit`
- `recruiter_commission_rate`
- `channel_commission_rate`
- `rule_source`: `global` / `activity` / `product` / `user` / `manual` / `import`

订单级明细用于证明差异化规则能逐单复现，而不是只在日汇总层凑数。

## 三、如果只有规则没有订单明细

请提供规则清单。

| 规则维度 | 必需字段 |
|---|---|
| 活动 | `activity_id`, 招商比例, 渠道比例, 生效开始, 生效结束 |
| 商品 | `product_id`, 招商比例, 渠道比例, 生效开始, 生效结束 |
| 用户 | `user_id`, 招商比例, 渠道比例, 生效开始, 生效结束 |
| 全局 | 兜底招商比例, 兜底渠道比例 |
| 优先级 | 多规则同时命中时的顺序 |
| 来源 | 官方后台 / 人工确认 / 导入文件 / 其他 |

## 四、需要业务确认

1. 正确基准到底按创建时间、付款时间还是结算时间统计？
2. 是否包含退款、取消、失效订单？
3. 服务费支出是否已在服务费收益中扣除？
4. 招商与渠道提成是否允许不对称？
5. 差异化规则优先级是否为：订单级导入 > 商品 > 活动 > 用户 > 全局？（当前代码为 product > activity > user > global；**需业务书面确认**）
6. 是否存在官方后台导出的提成明细？
7. 是否授权只对 2026-06-08..13 做受控历史重算？

## 五、进入实现任务的前置条件

满足以下任一项后，才能进入 COMMISSION-RULE-IMPLEMENT-001。

- A. 2026-06-08..13 每日招商/渠道提成拆分。
- B. 订单级提成明细。
- C. 活动/商品/用户维度提成规则。
- D. 官方后台导出的提成明细。
- E. 业务负责人书面确认的提成规则。

未满足前保持 PARTIAL，不得标记 PASS。
