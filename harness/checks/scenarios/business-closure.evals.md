# Eval: business-closure

## 验收目标

验证 V1 渠道链、招商链、管理链是否形成可证据闭环。

## 前置条件

- `test` 或 `real-pre` 环境健康。
- 账号角色可用：管理员、组长、普通成员。
- real-pre 场景必须通过真实环境预检，不能使用 mock 数据。

## 执行步骤

1. 验证管理链：登录、角色、菜单、数据范围、配置读取。
2. 验证商品库可选品。
3. 验证系统转链生成归因映射。
4. 验证真实订单带回归因；若无真实样本，标记 `PENDING`。
5. 验证订单写入 `default_channel_id` 与 `default_recruiter_id`。
6. 验证寄样申请在订单事件后自动完成。
7. 验证业绩和看板归属正确。

## 通过标准

```text
商品库可选品
-> 系统转链生成归因
-> 真实订单带回归因
-> 订单写入渠道/招商
-> 寄样自动完成
-> 业绩和看板归属正确
```

## 失败含义

- 任一环节没有证据，不能记为 PASS。
- real-pre 无真实订单样本时为 `PENDING`，不是 PASS。
- 真实接口失败或状态错误为 `FAIL` 或 `BLOCKED`。

## 证据要求

- E2E 报告。
- API 响应。
- SQL 查询。
- 后端日志。
- evidence report。
