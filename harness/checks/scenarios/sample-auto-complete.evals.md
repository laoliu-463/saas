# Eval: sample-auto-complete

## 验收目标

验证寄样待交作业状态能被订单同步事件自动完成。

## 前置条件

- 存在待交作业状态样本。
- 订单样本具备 `channel + talent + product + pay_time`。
- real-pre 场景订单必须来自真实同步。

## 执行步骤

1. 查询待交作业状态寄样申请。
2. 同步或构造可验证订单事实。
3. 验证订单事件被寄样域消费。
4. 验证命中 `channel + talent + product + pay_time`。
5. 验证寄样状态变为已完成。
6. 验证记录交作业时间和状态日志。

## 通过标准

- 状态从待交作业变为已完成。
- 交作业时间已记录。
- 状态变化能追溯到订单已同步事件。

## 失败含义

- 订单未归因可能导致寄样自动完成失败。
- 状态无日志或人工改库不算通过。

## 证据要求

- `sample_requests` SQL。
- 订单 SQL/API。
- 后端事件日志。
- E2E 或 API 结果。
