# DDD Domain Task Matrix

## 使用规则

- 本矩阵是后续任务卡索引，不代表任务已经完成。
- 每次只执行一个任务卡，执行前读取对应领域 instruction 和领域合同。
- 任务结果必须进入 evidence report 和 `harness/state/DOMAIN_STATUS.md`。

## 用户域

- U-1 盘点当前用户域代码、接口、表和测试。
- U-2 梳理用户、角色、菜单、组织和数据范围表结构。
- U-3 统一 CurrentUser 来源和调用边界。
- U-4 统一 PermissionContext 构造和传递。
- U-5 实现或收口 DataScopeResolver。
- U-6 实现或收口 PermissionChecker。
- U-7 统一 UserDomainFacade 或等价应用服务出口。
- U-8 改造订单域列表权限消费方式。
- U-9 改造寄样域列表权限消费方式。
- U-10 改造商品域权限消费方式。
- U-11 改造达人域权限消费方式。
- U-12 改造业绩域权限消费方式。
- U-13 改造分析模块权限消费方式。
- U-14 补齐用户域操作日志和审计证据。
- U-15 补用户域单测、权限 E2E 和越权负例。

## 配置域

- C-1 盘点配置域代码、接口、表和测试。
- C-2 梳理系统配置、规则参数、提成比例和模板来源。
- C-3 明确配置域只提供参数，不执行业务规则。
- C-4 收口配置读取 API 和缓存策略。
- C-5 收口配置保存、校验和版本记录。
- C-6 补齐配置变更审计。
- C-7 明确提成比例由业绩域消费和执行。
- C-8 明确复制模板、`pick_extra` 和展示参数的消费方。
- C-9 检查业务域是否反向写配置事实。
- C-10 检查配置域是否触发具体业务动作。
- C-11 补配置域单测。
- C-12 补配置审计 API/SQL 验证。
- C-13 补配置异常分支和权限验证。
- C-14 更新配置域 state、feedback 和 evidence。
