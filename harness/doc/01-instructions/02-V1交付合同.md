# V1 交付合同

## V1 三条主链

### 渠道链

```text
认领达人
-> 商品库选品
-> 复制讲解 / 转链
-> 寄样申请
-> 订单同步
-> 渠道业绩
-> 寄样自动完成
```

### 招商链

```text
同步活动
-> 活动商品入库
-> 商品上架
-> 审核寄样
-> 订单同步
-> 招商业绩
```

### 管理链

```text
用户角色
-> 数据范围
-> 规则配置
-> 各领域读取配置
-> 权限生效
```

## V1 必做

- 用户域提供登录、角色、组织、菜单和 `self/group/all` 数据范围。
- 配置域提供配置读取、配置变更和配置审计，不直接执行业务规则。
- 商品域支持商品库、活动商品同步、转链和 `pick_source_mapping` 落库。
- 达人域支持达人资料、标签、地址、跟进与渠道关系。
- 寄样域支持申请、审批、发货、交作业状态，并通过订单已同步事件判断完成。
- 订单域支持订单同步、订单事实、退款事实、归因输入和同步日志。
- 业绩域支持最终归属、提成、冲正、双轨金额计算和汇总输出。
- 分析模块只读业绩、订单和汇总表，提供看板和导出。

## V1 不做

- 不做独家达人。
- 不做独家商家。
- ~~不做毛利口径扩展或新增财务结算口径。~~ **已撤销**（2026-06-05 用户决策：毛利纳入 V1，公式 = 服务费收益 - 招商提成 - 渠道提成）
- 不做个别品负责人覆盖。
- 不做商品负责人变更历史重算。
- 不做差异化提成。
- 不做 Cookie 池 / 代理池。
- 不做物流 API 自动跟踪作为 V1 必需能力。
- 不做外部抖店快速寄样作为 V1 必需能力。
- 不做数据平台整体导出。
- 不用 mock 数据证明真实业务闭环。

## V1 验收边界

| 验收项 | 证据要求 |
| --- | --- |
| V1 P0 | `npm run e2e:v1-p0`、API/SQL、截图或报告路径 |
| real-pre | `npm run e2e:real-pre:p0`、真实 Token、真实接口响应、证据索引 |
| 后端回归 | Maven 测试结果，或明确未执行原因 |
| 前端回归 | 前端构建、Vitest、E2E 或浏览器证据 |
| 文档重构 | 文件结构、范围标记、旧文档冲突清单 |
