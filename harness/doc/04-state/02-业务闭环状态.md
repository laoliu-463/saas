# 业务闭环状态

## 渠道链

### 当前状态

渠道链主流程具备基础能力，但 real-pre 真实闭环仍依赖真实系统转链订单样本。

### 阻塞点

- 历史订单 `pick_source` 大量为空。
- 真实通过系统转链产生的订单样本不足。
- 寄样自动完成和业绩归属都依赖订单归因结果。

### 通过标准

```text
商品库可选品
-> 系统转链生成 pick_source
-> 真实订单带回归因
-> default_channel_id 不为空
-> default_recruiter_id 不为空
-> 寄样待交作业自动完成
-> 业绩归属正确
-> dashboard / 报表可读取正确结果
```

### 证据要求

- 商品库 API / 页面证据。
- `pick_source_mapping` SQL 证据。
- 第三方转链真实响应。
- 订单表 `pick_source`、`default_channel_id`、`default_recruiter_id`。
- `sample_requests` 状态变化。
- `performance_records` 和 dashboard API。

## 招商链

### 当前状态

招商链大部分可闭环：活动商品同步、活动商品入库、商品上架、寄样审核、订单同步和招商业绩具备主链路能力。

### 阻塞点

- 推广中活动商品历史数据可能存在入库漂移。
- 商品库 repair / backfill 需要通过商品域统一入口验证，禁止裸 SQL 批量直改。

### 通过标准

- 活动商品同步后商品库可查。
- 推广中商品按商品自身上游状态入库并可操作。
- 招商审核寄样状态合法。
- 订单同步后招商业绩归属正确。

## 管理链

### 当前状态

管理链基本可闭环：用户角色、数据范围、配置读取和权限生效已有基础能力。

### 阻塞点

- 权限注解和数据范围覆盖仍需持续审计。
- 自定义角色不应自动获得业务写接口权限。

### 通过标准

- admin / group / self 多账号访问同一资源，数据范围符合预期。
- 配置变更可审计，业务域读取配置后在自身领域验证效果。
- 操作日志、角色菜单和 API 鉴权一致。

## 结论

当前阶段不能把 real-pre 全业务闭环写成 PASS。渠道链仍因真实归因订单样本不足保持 `PENDING` 或 `PARTIAL`，需要继续采集真实系统转链订单样本。
