# Eval: rbac-scope

## 验收目标

验证管理员、组长、普通成员的数据范围和权限边界。

## 前置条件

- 管理员、组长、普通成员测试账号可登录。
- 目标模块存在可区分的数据样本。

## 执行步骤

1. 管理员登录，验证 `all` 范围。
2. 组长登录，验证 `group` 范围。
3. 普通成员登录，验证 `self` 范围。
4. 多角色用户验证权限并集。
5. 数据范围取最宽，不能取更窄导致合法数据不可见。
6. 直接调用越权 API，验证后端拒绝。

## 通过标准

- 不同角色列表、菜单、按钮和 API 返回符合预期。
- 后端权限校验生效。
- 前端隐藏按钮不能替代后端拒绝。

## 失败含义

- 低权限可见或可操作他人数据：P0/P1 权限风险。
- 前端隐藏但后端未校验：权限边界失败。

## 证据要求

- E2E 报告。
- API 响应。
- 用户 / 角色 / 数据范围 SQL 或日志。
