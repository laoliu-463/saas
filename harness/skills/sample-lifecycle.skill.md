# Skill: sample-lifecycle

## 使用场景

用于寄样申请、审批、发货、签收、待交作业、订单触发自动完成、7 天限制和拒绝后重申验证。

## 必读文件

- `docs/领域/寄样域.md`
- `docs/流程/寄样交作业链路.md`
- `docs/领域/订单域.md`
- `docs/09-测试验收总览.md`
- `docs/验收/V1-P0验收清单.md`

## 禁止事项

- 禁止无订单同步证据时把寄样标为真实完成。
- 禁止前端强行覆盖后端状态机。
- 禁止忽略操作者、时间、前后状态和操作日志。
- 禁止把订单归因失败误写成寄样域根因。

## 标准流程

1. 验证商品和达人前置数据。
2. 发起寄样申请。
3. 验证 7 天限制：普通用户受限，组长 / 管理员豁免。
4. 审核通过或拒绝；拒绝后重新申请不受限制。
5. 发货并记录物流信息。
6. 进入签收 / 待交作业状态。
7. 同步订单事实。
8. 验证寄样域消费订单已同步事件。
9. 验证状态变为已完成并记录交作业时间。

## 验证方式

- 寄样 API 响应。
- `sample_requests` SQL。
- 订单 API / SQL。
- 状态变更日志。
- E2E 报告。
