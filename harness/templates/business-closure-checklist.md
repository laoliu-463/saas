# 业务闭环验证清单

## 渠道链

### 前置条件

- real-pre 真实模式开关正确。
- 有真实商品和可转链商品。
- 有渠道账号、达人资料和寄样申请所需数据。
- 有真实通过系统转链产生的订单样本，或明确样本不足。

### 执行步骤

1. 登录渠道账号。
2. 在商品库选品并执行复制讲解 / 转链。
3. 查询 `pick_source_mapping`。
4. 创建或确认寄样申请。
5. 执行订单同步或等待真实订单回流。
6. 查询订单归因字段。
7. 查询寄样状态。
8. 查询业绩明细和 dashboard。

### 通过标准

- 商品库可选品。
- 系统转链生成 `pick_source`。
- 真实订单带回归因。
- `default_channel_id` 和 `default_recruiter_id` 不为空。
- 寄样待交作业自动完成。
- 业绩归属正确。

### 失败含义

- 无 `pick_source_mapping`：商品域 / 转链链路问题。
- 订单无 `pick_source`：真实样本不足或第三方回流字段缺失。
- 有 `pick_source` 但无归因：订单域消费映射或映射数据问题。
- 归因成功但寄样未完成：寄样域事件消费或匹配条件问题。
- 业绩不正确：业绩域计算或配置读取问题。

### 证据要求

- API / SQL 结果。
- 第三方真实响应或 BLOCKED 证据。
- 后端日志。
- `runtime/qa/out/` 或 Playwright 报告。

## 招商链

### 前置条件

- real-pre 真实模式开关正确。
- 有活动商品同步权限和真实活动 / 商品样本。
- 有招商角色账号。

### 执行步骤

1. 同步活动。
2. 验证活动商品入库。
3. 审核商品上架。
4. 审核寄样。
5. 同步订单。
6. 查询招商业绩。

### 通过标准

- 推广中商品进入商品库。
- 商品状态和本地发布状态合法。
- 审核寄样状态流转合法。
- 招商业绩归属正确。

## 管理链

### 前置条件

- admin、group、self 三类账号可登录。
- 有可对比的业务数据。

### 执行步骤

1. 分别登录三类账号。
2. 访问同一业务列表。
3. 比对数据范围。
4. 修改配置并查看审计日志。
5. 再次访问业务域确认配置读取效果。

### 通过标准

- 数据范围符合 `self/group/all`。
- 配置变更有审计。
- 业务域按配置执行，但配置域不直接执行业务动作。
