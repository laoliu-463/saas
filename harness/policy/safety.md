# 安全边界

## 不得做

- 不提交 `.env`、`.env.*`（`.env.example` 除外）、Token、密码、私钥或证书。
- 不直接向 `main` 推送，不在日常流程中直接 SSH 部署。
- 不在 `real-pre` 清库、删除 volume、关闭真实上游或用 mock 数据冒充真实闭环。
- 前端不直接调用抖音 / 抖店接口，不硬编码核心权限和状态机规则。

## 必须升级

涉及数据库迁移、认证权限、财务结算、定时任务、CI/CD 或发布基础设施时，必须补充兼容性、回滚和故障处理说明，并由对应维护者审查。

## 发现违规

立即停止提交、部署或重试，保留命令和日志，先轮换可能暴露的凭据，再通过 PR 或值班渠道处理。
