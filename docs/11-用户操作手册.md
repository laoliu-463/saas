# 星链达客用户操作手册

> 适用范围：抖音团长 SaaS V1 当前试用 / real-pre 环境。本文面向业务用户，说明日常操作路径和常见问题，不包含内部联调、数据库、接口、Git、Docker 和服务器部署细节。

---

## 一、开始使用

### 1.1 访问入口

普通业务用户以管理员通知的访问地址为准。当前试用阶段常见入口如下：

```text
http://1.14.108.159:3001/
```

本机联调人员可能使用 `http://localhost:3001/`。如果浏览器打不开页面，请先确认自己使用的是管理员最近通知的地址。

### 1.2 登录账号

账号由管理员创建、分配或重置。请不要共用账号，也不要把密码、授权链接或后台截图发到群聊和公开文档中。

登录步骤：

1. 打开系统访问地址。
2. 输入管理员分配的用户名和密码。
3. 如果系统提示修改密码，进入个人中心完成密码修改。
4. 登录后，系统会按当前账号角色和数据范围展示菜单。

### 1.3 常见角色

| 角色 | 主要工作 |
| --- | --- |
| 管理员 | 用户、角色、部门、规则、配置、抖店授权和操作日志管理 |
| 招商组长 | 管理招商组商品、审核寄样、查看团队数据 |
| 招商专员 | 跟进负责商品、处理寄样审核、查看本人范围数据 |
| 渠道组长 | 管理渠道达人、查看团队达人和业绩、跟进寄样 |
| 渠道专员 | 查看商品库、维护达人、发起寄样、跟进出单 |
| 运营 | 处理发货、物流单号、签收和异常物流 |

### 1.4 登录后的默认入口

登录后系统会按账号角色进入默认首页；实际可见页面以菜单权限为准。常用入口如下：

| 角色 | 日常入口 |
| --- | --- |
| 运营 | 合作管理 → 发货台 |
| 渠道专员 | 商品库、达人 CRM、合作单、数据看板 |
| 渠道组长 | 数据看板、达人 CRM、合作单、归因排查 |
| 招商专员 | 商品管理 → 商品列表、合作单、数据看板 |
| 招商组长 | 数据看板、商品管理、合作单、归因排查 |
| 管理员 | 数据看板；系统配置从系统管理进入 |

如果看不到某个菜单，通常是当前账号没有该功能权限或数据范围不包含对应数据。

## 二、页面导航

系统顶部是业务模块导航，左侧是当前模块下的子功能。不同角色看到的菜单不同。

| 模块 | 子功能 | 主要用户 |
| --- | --- | --- |
| 商品库 | 商品库 | 招商、渠道 |
| 商品管理 | 活动列表、商品列表 | 招商、管理员 |
| 达人 CRM | 团队公海、我的达人、自然出单达人、达人黑名单 | 渠道、管理员 |
| 合作管理 | 合作单、发货台 | 招商、渠道、运营 |
| 数据看板 | 核心看板、订单明细、独家状态 | 招商、渠道、主管、管理员 |
| 系统管理 | 用户管理、角色管理、部门管理、规则中心、高级配置、提成规则、抖店联调、操作日志 | 管理员 |
| 归因工作台 | 归因概览、订单工作台 | 主管、管理员，通常不作为普通顶部菜单展示 |
| 个人中心 | 账号信息、修改密码、退出登录 | 所有登录用户 |

补充说明：

- 页面数据受账号的数据范围限制：普通成员看本人范围，组长看本组范围，管理员或授权角色看全量范围。
- 管理员具备更高权限，但日常业务仍建议使用对应业务角色账号操作，便于审计。
- 归因工作台主要用于排查订单归属，普通业务操作优先使用数据看板和订单明细。

## 三、业务规则速览

### 3.1 数据范围

系统不会默认展示全平台所有数据。商品、达人、寄样、订单、业绩、看板和导出内容都会按账号权限过滤。

| 范围 | 说明 |
| --- | --- |
| 本人范围 | 当前账号负责的数据 |
| 本组范围 | 当前部门或团队范围内的数据 |
| 全量范围 | 管理员或授权全量角色可见的数据 |

如果你认为数据缺失，请先确认当前登录账号、角色、所属部门和筛选条件是否正确。

### 3.2 real-pre 数据口径

real-pre 是连接真实抖音 / 抖店上游的试用和验证环境，用于真实授权、真实商品、真实订单和真实链路验证。它不是正式全量生产放量环境。

常见情况：

- 未完成抖店授权时，看不到真实商品、订单或授权主体信息。
- 上游当前没有真实样本时，订单、归因、寄样自动完成和业绩看板可能为空。
- 真实推广短链默认由管理员按批准窗口控制，不建议业务用户自行要求开启。
- 空数据不一定是系统故障，可能与授权状态、权限包、时间范围、上游样本或当前账号数据范围有关。

### 3.3 操作审计

用户、权限、商品状态、寄样审核、发货、配置和抖店联调等关键操作会进入操作日志。请使用个人账号操作，避免共用账号导致责任无法追溯。

## 四、按角色操作指南

### 4.1 渠道专员 / 渠道组长

渠道角色的主要工作是找商品、维护达人、发起寄样并跟进出单。

常用路径：

1. 进入商品库，查看可推广商品。
2. 选择商品，复制商品讲解内容；如果真实转链已开启，系统会返回推广短链。
3. 进入达人 CRM，查看团队公海或我的达人。
4. 认领达人后，维护联系方式、收货地址和跟进信息。
5. 进入商品详情或合作单，发起寄样申请。
6. 在合作单中跟进审核、发货、签收、待交作业和完成状态。
7. 在数据看板查看自己或本组的订单与业绩。

权限差异：

- 渠道专员通常只能看到团队公海和我的达人。
- 渠道组长可以查看本组达人、自然出单达人和达人黑名单。
- 拉黑、取消拉黑等风险操作通常由渠道组长或管理员处理。

### 4.2 招商专员 / 招商组长

招商角色的主要工作是维护活动商品、审核寄样和查看商品相关数据。

常用路径：

1. 进入商品管理。
2. 在活动列表查看已分配活动；管理员可查看全部并执行分配。
3. 进入商品列表，查看活动商品、审核状态和商品运营状态。
4. 将符合条件的商品加入商品库，跟进上下架、负责人分配和批量处理。
5. 进入合作管理，处理待审核寄样单。
6. 在数据看板查看权限范围内的订单、服务费、毛利和业绩趋势。

权限差异：

- 招商组长、招商专员默认查看分配给自己的活动和商品。
- 管理员可在活动列表为活动分配招商负责人。
- 寄样审核由招商角色处理，不由渠道角色审核。

### 4.3 运营

运营角色的主要工作是发货、录入物流和处理物流履约。

常用路径：

1. 进入合作管理 → 发货台。
2. 查看待发货合作单。
3. 填写物流公司和物流单号。
4. 可使用批量导入物流单号或批量发货。
5. 发货后跟进快递中、待交作业、已完成、已关闭等状态。
6. 如物流没有自动更新，可在详情中刷新物流状态。

运营通常不处理寄样审核，也看不到与发货无关的审核操作。

### 4.4 管理员

管理员主要负责系统配置、账号权限和抖店授权，不建议代替业务角色做日常业务操作。

| 功能 | 用途 |
| --- | --- |
| 用户管理 | 创建、编辑、禁用用户账号，重置密码 |
| 角色管理 | 管理角色定义、菜单权限和数据范围 |
| 部门管理 | 管理组织结构和人员归属 |
| 规则中心 | 配置商品运营规则 |
| 高级配置 | 管理系统配置参数 |
| 提成规则 | 配置预估和结算相关规则 |
| 抖店联调 | 完成抖音 / 抖店授权、授权状态检查和联调探针 |
| 操作日志 | 查看关键操作审计记录 |

抖店授权流程：

1. 进入系统管理 → 抖店联调。
2. 点击获取授权地址。
3. 在百应授权页完成授权。
4. 回到系统后查询授权状态。
5. 授权有效后，再执行商品同步、订单同步和联调验证。

如果授权缺失、过期或权限包不足，真实商品、真实订单和转链链路可能无法返回数据。

## 五、核心功能说明

### 5.1 数据看板

数据看板用于查看权限范围内的业务结果。

常见指标：

- 订单量
- 订单金额
- 服务费收入
- 技术服务费
- 服务费支出
- 服务费收益
- 招商提成
- 媒介提成
- 毛利
- 预估轨和结算轨数据

订单明细支持按时间、渠道、招商、商品、达人、店铺和订单状态筛选。

导出说明：

- 只有具备导出权限的角色会看到导出按钮。
- 导出的内容仍受当前账号数据范围限制。
- 如果筛选后没有数据，导出文件也不会包含业务记录。

### 5.2 商品库

商品库用于查看可推广商品。

可执行操作：

- 查看商品名称、店铺、价格、佣金率、服务费率和推广时间。
- 在电脑端将鼠标移入商品卡片，可自然展开低频详情并推动下方商品下移；移动端点击商品卡片可展开或收起。详情中可查看并复制招商、寄样要求、推广时间、团长、店铺、活动和库存字段。
- 按活动、商品、店铺、负责人或状态筛选。
- 查看商品详情、SKU、合作设置和操作日志。
- 复制商品讲解内容。
- 在批准窗口内生成或复制推广链接。

推广链接说明：

- 如果系统提示真实推广链接未生成，说明当前环境只允许复制基础简介或真实转链窗口未开启。
- 真实推广链接会触发抖音转链接口，并写入归因映射，因此需要管理员控制开启窗口。
- 生成推广链接后，后续订单可通过推广来源标识进入归因链路。

### 5.3 商品管理

商品管理用于招商侧维护活动和活动商品。

活动列表：

- 管理员：可见全部活动，可为活动分配招商负责人。
- 招商组长 / 招商专员：默认仅展示分配给本人的活动；无分配时页面会提示联系管理员。
- 可查看活动状态、推广时间、商品数、同步时间和活动要求。

商品列表：

- 管理员可从全部活动下拉选择活动商品；招商组长 / 招商专员仅可选择分配给本人的活动。
- 一键同步商品时沿用同一活动选择范围：管理员可同步全部活动，招商角色仅可同步分配给本人的活动；发起后系统在后台同步，页面提示后台同步中，稍后刷新列表查看结果。
- 可查看商品审核状态、商品运营状态和负责人。
- 可将符合条件的商品加入商品库。
- 可管理上下架、暂缓、放弃、分配和批量操作。
- 可查看商品操作日志。

### 5.4 达人 CRM

达人 CRM 用于维护达人经营关系。

| 视图 | 说明 |
| --- | --- |
| 团队公海 | 可认领的公海达人 |
| 我的达人 | 当前账号已认领或负责的达人 |
| 本组达人 | 渠道组长 / 管理员查看本组成员已认领达人 |
| 自然出单达人 | 已经通过订单产生业务结果的达人 |
| 达人黑名单 | 风险达人或暂不合作达人 |

常见操作：

- 新增达人。
- 批量导入达人。
- 查看达人资料、粉丝数、经营标签和业绩贡献。
- 认领或释放达人。
- 维护联系方式和收货地址。
- 从达人发起寄样申请。
- 拉黑或恢复达人。

### 5.5 合作管理 / 寄样

合作单承接寄样申请、审核、发货、签收、交作业和完成状态。

| 状态 | 含义 | 主要处理角色 |
| --- | --- | --- |
| 待审核 | 寄样申请已提交，等待招商审核 | 招商 |
| 待发货 | 审核通过，等待运营发货 | 运营 |
| 快递中 | 已录入物流单号，商品运输中 | 运营 |
| 待交作业 | 物流已签收，等待订单同步证明达人完成作业 | 系统 / 渠道跟进 |
| 已完成 | 系统匹配到相关订单后自动完成 | 系统 |
| 已拒绝 | 招商审核拒绝 | 招商 |
| 已关闭 | 待交作业超时或规则触发关闭 | 系统 |

标准流程：

```text
渠道发起申请 → 招商审核 → 运营发货 → 物流签收 → 待交作业 → 订单同步后自动完成
```

注意事项：

- “已完成”不是达人手动确认收货，而是系统根据订单同步结果推进。
- 物流自动同步依赖物流服务配置和真实物流单样本；异常时可由运营手动刷新或补录。
- 没有真实订单同步证据时，不应把寄样单当作真实完成。

### 5.6 订单与归因

订单明细用于查看订单事实，归因工作台用于主管和管理员排查订单归属。

可查看：

- 订单号、结算时间、商品信息和订单金额。
- 归因状态、排查摘要和归因标识。
- 渠道负责人、达人 UID、达人昵称和归属来源。
- 订单回流、推广参数识别、系统推广映射和业务归因结果。

如果订单未归因，请先确认：

- 商品是否生成过推广链接。
- 是否存在推广来源映射。
- 上游订单是否已同步。
- 当前账号是否有权限查看该订单。

## 六、常见业务流程

### 6.1 渠道推广商品

```text
商品库 → 搜索商品 → 查看商品详情 → 复制讲解内容 / 推广链接 → 发给达人 → 后续在订单和看板中跟进结果
```

如果短链没有生成，先按页面提示确认是否真实转链窗口未开启。

### 6.2 渠道维护达人并申请寄样

```text
达人 CRM → 团队公海 → 认领达人 → 补充联系方式和地址 → 进入商品或合作单 → 发起寄样申请
```

渠道提交后，等待招商审核。审核通过后由运营发货。

### 6.3 招商审核寄样

```text
合作管理 → 合作单 → 待审核 → 查看达人、商品、申请理由和资质信息 → 通过或拒绝
```

通过后进入待发货；拒绝时应填写明确原因，方便渠道补充信息。

### 6.4 运营发货

```text
合作管理 → 发货台 → 待发货 → 填写物流公司和物流单号 → 提交发货
```

批量发货时请先核对表格中的合作单号、物流公司和物流单号。

### 6.5 管理员完成抖店授权

```text
系统管理 → 抖店联调 → 获取授权地址 → 完成百应授权 → 查询授权状态 → 执行联调探针
```

授权完成后仍需要真实商品和真实订单样本，业务看板才会逐步有真实数据。

### 6.6 查看订单和业绩

```text
数据看板 → 选择时间范围 → 按商品、渠道、招商、达人或订单状态筛选 → 查看汇总和明细
```

如果筛选后没有数据，请先确认当前账号的数据范围、时间范围和上游订单是否已经同步。

可执行操作：

## 八、常见问题

### 8.1 看不到某个菜单怎么办？

先确认当前账号角色是否具备权限。菜单按角色、部门和数据范围展示，不同角色看到的顶部导航和左侧菜单不同。

### 8.2 列表里没有数据怎么办？

按顺序检查：

1. 时间筛选是否过窄。
2. 是否选择了过多筛选条件。
3. 当前账号是否只能看本人或本组数据。
4. real-pre 是否已有真实商品、真实达人或真实订单样本。
5. 抖店授权是否有效，权限包是否覆盖当前接口。

### 8.3 为什么复制商品讲解时没有真实推广链接？

真实推广链接需要调用抖音转链接口，并写入归因映射。real-pre 默认按管理员批准窗口控制真实推广写入。看到该提示时，可以先使用基础简介；需要真实短链时联系管理员确认是否允许开启。

### 8.4 为什么寄样签收后还不是已完成？

签收后通常进入待交作业。系统需要同步到匹配的真实订单，才能自动把寄样单推进为已完成。

### 8.5 为什么没有真实订单？

常见原因：

- 抖店未授权或授权已失效。
- 上游当前时间范围没有订单。
- 订单同步任务尚未执行。
- 上游订单缺少归因字段。
- 当前账号无权查看该订单。

### 8.6 为什么没有导出按钮？

导出按钮按角色显示。普通专员账号可能只能查看数据，不能导出。即使有导出权限，导出内容也只包含当前账号可见范围内的数据。

### 8.7 物流没有自动更新怎么办？

先在寄样详情中刷新物流。仍无结果时，联系运营或管理员确认物流公司、单号、手机号和物流服务配置是否正确。

### 8.8 操作失败时如何反馈？

反馈时请提供：

- 当前登录账号。
- 页面名称。
- 操作步骤。
- 失败时间。
- 页面提示语或截图。
- 如果页面出现 traceId，请一并提供。

这些信息可以帮助管理员快速定位问题。

## 九、使用注意事项

- 不要共享账号。
- 不要把密码、授权链接、后台截图外传。
- 不要用管理员账号代替业务账号做日常操作。
- 真实推广写入、真实订单同步、规则配置修改等操作会影响业务数据，执行前请确认权限和操作目的。
- real-pre 的空数据不一定代表系统故障，可能是上游授权、权限包、时间窗口或真实样本不足。
