## 2. 当前执行口径

当前项目不再继续堆页面，而是按两条主线推进：

1. 商品库主链路冻结，只修问题、补回归、补文档。
2. 启动订单回流与归因 MVP，先把后端闭环跑通，再补前端工作台。

当前阶段定义为：

> Mock-first + Gateway 收口 + 订单回流联调阶段

含义是：

- 前端只调用系统内部 API
- 业务层只依赖 Gateway 契约
- Mock / Real 只在 Gateway 实现层区分
- 内部状态流转、日志、落库必须真实生效
- 后续切换真实 SDK 时，不改 Controller，不重写主业务流程

## 3. 当前主链路

### 商品库主链路

`活动商品同步 -> PENDING_AUDIT -> APPROVED -> BOUND -> ASSIGNED -> LINKED -> FOLLOWING`

### 订单回流与归因主链路

`商品转链 -> promotion_link -> pick_source_mapping -> 手动同步订单 -> 订单落库 -> 归因 -> 未归因池 / 看板摘要`
