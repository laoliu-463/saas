# 领域架构总览

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域清单

| 领域 | 合同文档 | 主要职责 | 禁止越界 | 范围 |
| --- | --- | --- | --- | --- |
| 用户域 | [领域/用户域.md](领域/用户域.md) | 用户、角色、组织、`self/group/all` 数据范围 | 不计算业务归属 | V1 必做 |
| 配置域 | [领域/配置域.md](领域/配置域.md) | 配置、规则参数、审计 | 不执行业务规则 | V1 必做 |
| 商品域 | [领域/商品域.md](领域/商品域.md) | 商品、活动、转链、`pick_source_mapping` | 不计算提成 | V1 必做 |
| 达人域 | [领域/达人域.md](领域/达人域.md) | 达人资料、标签、地址、渠道关系 | 不替代订单归属 | V1 必做 |
| 寄样域 | [领域/寄样域.md](领域/寄样域.md) | 寄样申请、审批、发货、交作业 | 不直接判定业绩 | V1 必做 |
| 订单域 | [领域/订单域.md](领域/订单域.md) | 订单事实、退款事实、归因输入 | 不算提成、不应用独家覆盖 | V1 必做 |
| 业绩域 | [领域/业绩域.md](领域/业绩域.md) | 最终归属、提成、冲正、双轨金额 | 不采集第三方事实 | V1 必做 |
| 分析模块 | [领域/分析模块.md](领域/分析模块.md) | 只读汇总、报表、看板 | 不重算业绩归属 | V1 必做 |

## 依赖方向

- [V1 必做] 商品域产生 `pick_source_mapping`，订单域消费为归因输入，业绩域做最终归属。
- [V1 必做] 订单域发布订单已同步事实，寄样域据此判断交作业完成。
- [V1 必做] 配置域提供规则参数，业绩域读取并执行规则。
- [V1 必做] 用户域提供数据范围，所有业务查询按 `self/group/all` 过滤。
- [V1 必做] 分析模块读取业绩和订单汇总，不反向写业务事实。

## 跨域访问原则

- [V1 必做] 其他领域需要读取某领域能力时，优先通过应用服务、Facade、查询 API 或领域事件获取，不直接穿透到对方 Repository 或业务表。
- [V1 必做] 用户域只输出身份、角色、组织和数据范围结果；业务域不得复制一套权限规则。
- [V1 必做] 配置域只输出规则参数、模板和审计事实；具体业务规则由对应业务域执行。
- [V1 必做] 订单域只保存第三方订单事实、退款事实和默认归因输入；不得直接写寄样完成、业绩提成或分析汇总结论。
- [V1 必做] 分析模块只读事实或汇总，不反向更新订单、寄样、业绩或配置事实。

## 架构反馈传感器

- [V1 简化] 当前先把跨域 Repository 穿透、Controller 复杂业务逻辑、分析模块重算归属列为代码审查必查项。
- [V1 简化] `harness/rules/skills/ddd/domain-alignment.skill.md` 用于修改前确认主责领域、关联领域和禁止越界项。
- [V2 预留] ArchUnit 或等价架构测试属于后续专项；未落地前不得把自动架构测试写成已完成证据。
- [V1 必做] 发现边界冲突时，先补证据并写入 ADR 或任务报告，不由 AI 自行裁决业务规则。

## 事件模式

- [V1 必做] V1 以 Spring 本地事件、事务边界内外处理和已存在 Outbox 记录为可执行口径。
- [V1 简化] 事件总表先记录生产者、消费者、触发条件、幂等键和证据，不要求全部 MQ 化。
- [V2 预留] 跨服务拆分、MQ、事件重放和事件版本治理放入后续阶段。

## 架构决策

- [V1 必做] 模块化单体决策见 [决策/ADR-001-模块化单体.md](决策/ADR-001-模块化单体.md)。
- [V1 必做] V1 范围优先级见 [决策/ADR-002-V1范围优先级.md](决策/ADR-002-V1范围优先级.md)。
- [V1 必做] 订单、业绩、分析边界分别见 ADR-003、ADR-004、ADR-005。
