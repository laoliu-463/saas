# 08-Test 演示脚本

更新时间：2026-04-27

## 一、演示目标

用 `test` 环境在本地完整演示以下业务闭环：

1. 商品库选品与转链
2. 订单归因与未归因排查
3. 寄样履约与自动完成
4. 达人 CRM 认领与关联查看

## 二、环境口径

- 后端地址：`http://localhost:8080`
- 前端地址：本地 Vite 页面
- 接口前缀：`/api`
- 调试入口：`/dev/test`
- 测试账号：`admin / admin123`

注意：

- 正确初始化地址：`POST /api/test/seed`
- 正确重置地址：`POST /api/test/reset`
- `/api/test/*` 需要登录态

## 三、推荐演示路径

### 1. 登录系统

1. 打开前端登录页。
2. 使用 `admin / admin123` 登录。
3. 确认左侧菜单可以看到商品库、订单、寄样台、达人 CRM、开发调试台。

### 2. 准备测试数据

1. 打开 `/dev/test`。
2. 先点击“清空业务数据”。
3. 再点击“一键铺设演示数据”。
4. 预期结果：
   - 商品库出现测试商品
   - 达人 CRM 出现测试达人
   - 商品、达人、寄样、归因基础映射已初始化

### 3. 演示商品库

1. 打开 `/product`。
2. 查看商品列表和商品详情。
3. 选择一个商品点击“复制推广链接”。
4. 预期结果：
   - 提示复制成功
   - 后端生成 `pick_source`
   - `pick_source_mapping` 已写入

### 4. 演示订单归因

1. 返回 `/dev/test`。
2. 点击“生成已归因订单”。
3. 打开 `/orders`。
4. 预期结果：
   - 出现一条已确认业绩订单
   - 订单详情中能看到归因渠道、推广链路、商品和达人信息

### 5. 演示未归因排查

1. 回到 `/dev/test`。
2. 点击“生成无标识订单”。
3. 点击“生成映射缺失订单”。
4. 打开 `/orders` 并切换到待排查视角。
5. 预期结果：
   - 可看到“订单未携带推广参数”
   - 可看到“未找到对应推广链接”
   - 详情弹窗中有原因和排查建议

### 6. 演示数据看板

1. 打开 `/dashboard`。
2. 查看 GMV、服务费、订单数、归因成功率等指标。
3. 点击看板卡片跳转到 `/orders`。
4. 预期结果：
   - 统计数据与前面生成的订单场景一致
   - 跳转后保留对应筛选条件

### 7. 演示寄样台

1. 打开 `/sample`。
2. 查看待审核、待发货、快递中、待交作业、已完成等状态。
3. 在 `/dev/test` 使用发货、签收按钮推进状态。
4. 若订单命中寄样关联，确认寄样自动变为已完成。

### 8. 演示达人 CRM

1. 打开 `/talent`。
2. 查看公海达人、私海达人、认领人、保护期信息。
3. 选择一位公海达人点击“认领”。
4. 打开达人详情。
5. 预期结果：
   - 可看到基础资料
   - 可看到关联寄样记录
   - 可看到关联订单产出

## 四、演示结束检查

演示结束后，至少确认以下几点：

1. `/product` 商品可正常查看和转链
2. `/orders` 可看到已归因与未归因订单
3. `/dashboard` 指标与订单数据一致
4. `/sample` 状态流转正常
5. `/talent` 达人列表和详情可正常展示
6. `/dev/test` 所有按钮调用正常，无 404

## 五、异常排查提示

- 若 `/api/test/seed` 返回 `401`：先重新登录
- 若 `/api/test/seed` 返回 `404`：检查是否误用了 `/test/seed`
- 若 `/talent` 无数据：先执行 `/dev/test` 初始化数据，再刷新页面
- 若订单无归因：优先检查是否先生成了推广链接和 mapping
