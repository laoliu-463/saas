## 1. 系统简介

本系统是抖音团长内部管理工具（产品名称：**抖音团长 SaaS**），帮助团队完成以下核心工作：

- **商品管理**：同步抖音活动商品，维护商品库，供渠道选品推广。
- **达人管理**：管理达人资料、标签、收货地址，支持认领和跟进。
- **寄样管理**：渠道申请寄样、招商审核、运营发货、跟踪寄样状态。
- **订单同步**：自动从抖音上游同步订单数据，进行归因和业绩统计。
- **数据看板**：展示订单额、服务费收入、服务费收益、提成、毛利等核心指标。
- **权限管理**：按角色分配菜单和数据可见范围，支持组织架构管理。

系统分三条业务主线：

1. **渠道线**：认领达人 → 商品库选品 → 复制讲解 → 申请寄样 → 查看订单和业绩。
2. **招商线**：同步活动 → 补充商品信息 → 确认上架 → 审核寄样 → 查看订单和业绩。
3. **管理线**：创建员工 → 分配角色 → 配置规则 → 查看数据。

### 系统技术要求

| 项目 | 要求 |
|------|------|
| 浏览器 | 推荐使用 **Chrome 90+**、**Edge 90+** 或 **Firefox 90+**，需支持 JavaScript |
| 网络 | 需要能访问系统服务器（内网或通过公网域名） |
| 分辨率 | 推荐 1920×1080 及以上，最低支持 1280×720 |
| 移动端 | 当前版本为 PC 端 Web 系统，暂不支持移动端 |

## 4. 推荐业务流程

### 招商流程

```
同步活动 → 查看活动商品 → 补充商品信息 → 确认上架 → 商品进入商品库 → 查看订单和业绩
```

1. 管理员或招商组长在"商品管理 → 活动列表"中同步抖音活动。
2. 查看活动下的商品列表，补充商品的合作信息、佣金、服务费率等。
3. 确认商品上架，商品自动进入共享商品库。
4. 渠道从商品库选品推广后，产生的订单会自动同步。
5. 在"数据看板"中查看订单额和业绩。

### 渠道流程

```
认领达人 → 商品库选品 → 复制讲解 → 发给达人推广 → 申请寄样 → 跟进寄样状态 → 查看订单和业绩
```

1. 在"达人 CRM"中添加或认领达人。
2. 进入"商品库"，按类目、关键词搜索商品。
3. 点击"复制讲解"生成推广链接，**必须使用系统生成的链接**，否则订单无法归因到渠道。
4. 将推广内容发给达人。
5. 如需寄样，在商品库点击"快速寄样"或在寄样台发起申请。
6. 在"寄样台"跟进寄样进度。
7. 在"数据看板"查看个人业绩。

### 寄样流程

```
渠道申请 → 招商审核 → 运营发货 → 快递中 → 待交作业 → 达人出单 → 系统自动完成
```

1. 渠道在商品库或寄样台发起寄样申请。
2. 招商组长/专员审核申请，通过或拒绝。
3. 审核通过后，运营在"发货台"录入物流信息。
4. 达人收到样品后开始推广。
5. 达人出单后，系统自动匹配订单并完成寄样。

### 管理流程

```
创建员工 → 分配角色 → 配置提成规则 → 查看数据 → 处理异常
```

1. 管理员在"系统管理 → 用户管理"中创建员工账号。
2. 为员工分配角色（招商/渠道/运营等）。
3. 在"规则中心"或"提成规则"中配置提成比例。
4. 在"数据看板"查看全局数据。

## 5. 商品库使用说明

商品库是渠道选品的核心入口。

### 如何搜索商品

- 进入"商品库"页面，在顶部搜索框输入商品名称或关键词。
- 支持按类目、合作方、推广状态等条件筛选。

### 如何查看商品详情

- 点击商品卡片，可以查看佣金率、服务费率、合作方信息、商品图片等。

### 如何复制讲解

1. 在商品卡片上点击"复制讲解"按钮。
2. 系统自动生成包含推广链接的讲解文案。
3. 复制后发给达人使用。

### 为什么必须使用系统生成的推广链接

- 系统通过推广链接中的标识参数（`pick_source`）来追踪订单来源。
- **只有使用系统链接产生的订单，才能归因到对应渠道**。
- 未使用系统链接的订单，渠道归因会为空，不计入渠道业绩。

### 如何快速寄样

- 在商品卡片上点击"快速寄样"按钮。
- 选择达人并填写收货地址，提交申请。
- 寄样申请会进入审核流程。

### 商品不显示的常见原因

- 商品未上架：招商尚未确认上架。
- 商品已过期：活动已结束，商品自动下架。
- 筛选条件过严：尝试清除筛选条件重新搜索。
- 同步延迟：新同步的商品可能需要几分钟才会显示。

## 6. 活动与商品管理

适用角色：招商组长、招商专员。

### 如何同步活动

1. 进入"商品管理 → 活动列表"。
2. 点击"同步活动"，系统从抖音上游拉取最新活动。
3. 同步后的活动会显示在列表中。

### 如何查看活动商品

1. 在活动列表中点击某个活动，进入商品列表页面。
2. 可以查看该活动下所有商品的审核状态、合作信息。

### 如何分配商品

- 招商组长可以将商品分配给指定招商专员负责。
- 在商品列表中选择商品，点击"分配"按钮。

### 如何补充商品信息

- 招商专员在商品列表中，补充佣金率、服务费率、合作说明等。
- 补充完成后确认上架。

### 如何确认上架

- 商品信息补充完毕后，点击"上架"按钮。
- 上架后商品自动进入共享商品库，渠道可见。

### 商品上架后渠道何时可见

- 上架后一般即时可见。如果商品库中未显示，请稍等几分钟后刷新。

## 7. 达人管理

适用角色：渠道组长、渠道专员。

### 如何添加达人

1. 进入"达人 CRM"页面。
2. 点击"添加达人"。
3. 填写达人基本信息（昵称、抖音 UID 等）。

### 如何认领达人

- 在达人列表中，选择公海达人，点击"认领"。
- 认领后该达人进入你的私海，其他人无法认领。

### 公海 / 私海 / 保护期

- **公海**：未被任何人认领的达人，所有渠道都可以认领。
- **私海**：已被认领的达人，只有认领人和其组长可见。
- **保护期**：认领后有一段保护期，期间其他人不能抢走该达人。

### 如何维护达人信息

- 在达人详情页可以编辑达人资料、添加标签、维护收货地址。
- 收货地址用于寄样时自动填充。

### 如何打标签

- 在达人详情页可以添加自定义标签，方便筛选和分类管理。

### 达人信息不完整时

- 如果达人缺少抖音 UID 等关键信息，寄样时可能无法自动匹配。
- 请尽量补全达人信息。

## 8. 寄样台

寄样台管理所有寄样申请的全生命周期。

### 寄样状态说明

| 状态 | 含义 | 谁处理 |
|------|------|--------|
| 待审核 | 渠道已提交，等待招商审核 | 招商组长/专员 |
| 待发货 | 审核通过，等待运营发货 | 运营 |
| 快递中 | 已录入物流单号，包裹在途中 | 等待收货 |
| 待交作业 | 达人已收到样品，等待出单 | 达人 |
| 已完成 | 达人已出单，寄样流程结束 | 系统自动 |
| 已拒绝 | 招商审核未通过 | — |
| 已关闭 | 寄样被手动关闭或超时 | 管理员 |

### 渠道如何申请寄样

1. 在商品库点击"快速寄样"，或在寄样台点击"新建申请"。
2. 选择商品、达人，确认收货地址。
3. 提交后进入待审核状态。

### 招商如何审核

1. 在寄样台筛选"待审核"状态。
2. 查看申请详情，选择"通过"或"拒绝"。
3. 通过后进入"待发货"状态。

### 运营如何录入物流

1. 运营在"发货台"（`/ops/shipping`）筛选"待发货"寄样。
2. 录入快递公司和物流单号。
3. 保存后状态变为"快递中"。

### 为什么寄样会被限制

- 同一达人 + 同一商品不能重复申请。
- 达人信息不完整时可能无法申请。

### 为什么寄样未自动完成

- 寄样自动完成需要系统匹配到对应的订单（渠道 + 达人 + 商品 + 付款时间）。
- 如果订单没有通过系统推广链接产生（无 `pick_source`），则无法自动匹配。
- 如果匹配条件未满足，寄样会一直停留在"待交作业"状态。

## 9. 订单明细

### 订单数据来源

- 订单来自抖音上游接口自动同步，不是手动录入。
- 订单同步有一定延迟，不是付款后立即出现。
- 系统每 10 分钟同步一次增量订单，每 30 分钟回扫一次近 6 小时窗口。

### 订单筛选

订单可按以下条件筛选：
- 付款时间 / 结算时间
- 活动名称
- 商品信息
- 合作方
- 推广者
- 渠道
- 招商
- 订单状态

### "渠道"字段的来源

- 订单中的"渠道"字段来自系统推广链接归因。
- 如果达人通过系统生成的推广链接成交，订单会自动关联到对应渠道。
- **如果渠道字段为空**，说明该订单没有通过系统链接成交，或当前归因参数缺失。

### 订单字段说明

| 字段 | 说明 |
|------|------|
| 订单 ID | 抖音订单编号 |
| 活动信息 | 所属活动名称 |
| 商品信息 | 商品名称、图片、商品 ID、店铺 |
| 合作方信息 | 合作方名称 |
| 推广者 | 推广达人 |
| 渠道 | 推广渠道（来自系统链接归因） |
| 招商 | 负责该商品的招商人员 |
| 订单状态 | 已付款 / 已结算 / 已退款等 |
| 订单额 | 订单金额 |
| 服务费收入 | 预估/结算服务费收入 |
| 技术服务费 | 平台扣除的技术服务费 |
| 服务费支出 | 预估/结算服务费支出（提成合计） |
| 服务费收益 | 服务费收入 - 服务费支出 - 技术服务费 |
| 招商提成 | 招商人员的提成金额 |
| 渠道提成 | 渠道人员的提成金额 |
| 订单时间 | 付款时间 |

### 订单状态含义

- **已付款**：买家已付款，订单进入预估统计。
- **已结算**：订单完成结算，进入结算统计。
- **已退款**：订单发生退款。

### 当前注意事项

- 订单从付款到系统可见通常有 10 分钟到 1 小时的延迟。
- 如果付款后较长时间仍看不到订单，请联系管理员检查同步日志。

## 10. 数据平台 / 看板

数据平台展示业务核心指标，分为"数据看板"和"订单明细"两个 Tab。

访问路径：
- 数据看板：`/data`
- 订单明细：`/data/orders`
- 归因概览：`/dashboard`（管理员、招商组长、渠道组长可见）
- 订单工作台：`/orders`（管理员、招商组长、渠道组长可见）

### 核心指标说明

| 指标 | 含义 |
|------|------|
| 订单数 | 统计周期内的订单总数 |
| 订单额 | 订单金额合计 |
| 服务费收入 | 预估服务费收入 = 订单额 × 服务费率 |
| 技术服务费 | 平台技术服务费 |
| 服务费支出 | 招商提成 + 渠道提成 |
| 服务费收益 | 服务费收入 - 服务费支出 - 技术服务费 |
| 招商提成 | 招商人员提成合计 |
| 渠道提成 | 渠道人员提成合计 |
| 毛利 | 服务费收益 - 招商提成 - 渠道提成 |

### 预估与结算

- **预估**：基于订单付款金额计算，反映当前业务规模。
- **结算**：基于订单实际结算金额计算，反映真实收益。
- 预估数据实时可见，结算数据需要等上游结算后才能看到。

### 为什么结算数据可能为 0

- 订单从付款到结算通常需要数天到数周。
- 如果系统刚上线或近期没有完成结算的订单，结算数据会显示为 0。
- 这是正常现象，不代表系统有问题。

### 为什么看板和订单有短暂延迟

- 看板数据基于定时汇总，不是实时刷新。
- 新同步的订单可能需要几分钟才会反映到看板。

### 管理层怎么看今日数据

- 进入"数据看板"页面。
- 查看"预估"轨的今日订单数和订单额。
- 关注服务费收入和服务费收益指标。
- 注意：如果当天订单尚未结算，结算轨数据会偏低或为 0。

## 11. 系统管理

适用角色：管理员。

访问路径前缀：`/system/`

### 员工管理

- 路径：系统管理 → 用户管理（`/system/users`）
- 可以创建、编辑、停用员工账号。
- 创建账号时需要指定用户名、姓名、所属部门。

### 角色权限

- 路径：系统管理 → 角色管理（`/system/roles`）
- 查看和编辑角色的数据范围（全部/本组/本人）。
- 为角色分配可见菜单。

### 组织架构

- 路径：系统管理 → 部门管理（`/system/depts`）
- 管理部门结构，设置部门层级。
- 部门影响数据范围：本组数据 = 本部门及子部门的数据。

### 规则配置

- 路径：系统管理 → 规则中心（`/system/rule-center`）
- 配置业务规则参数。

### 提成配置

- 路径：系统管理 → 提成规则（`/system/commission-rules`）
- 配置招商提成和渠道提成的比例和规则。

### 抖店联调

- 路径：系统管理 → 抖店联调（`/system/douyin`）
- 查看抖音 OAuth 授权状态、Token 状态。
- 抖店授权回调地址：`https://real-pre.<域名>/api/douyin/oauth/callback`
- 授权成功跳转：`https://real-pre.<域名>/system/douyin?oauth=success`

### 高级配置

- 路径：系统管理 → 高级配置（`/system/config`）
- 查看和修改系统高级参数（仅限管理员）。

### 操作日志

- 路径：系统管理 → 操作日志（`/system/operation-logs`）
- 查看系统操作记录，追踪谁在什么时间做了什么操作。

### 重要提示

- 修改权限后，用户可能需要**刷新页面或重新登录**才能生效。
- 不同角色的数据范围不同，修改角色会影响该角色下所有用户。
- **不要多人共用一个账号**，会导致数据归属混乱。

## 12. 常见问题 FAQ

### 1. 登录失败怎么办？

- 检查用户名和密码是否正确。
- 确认账号未被停用（联系管理员）。
- 如果提示"服务器异常"，请稍后重试或联系管理员。

### 2. 看不到菜单怎么办？

- 确认当前登录的角色是否正确。
- 联系管理员在"角色管理"中为该角色配置菜单权限。
- 尝试刷新页面或重新登录。

### 3. 为什么看不到别人的数据？

- 数据范围由角色决定。如果角色数据范围是"本人"，则只能看到自己负责的数据。
- 如需查看更多数据，请联系管理员调整角色数据范围。

### 4. 商品库没有商品怎么办？

- 确认招商是否已同步活动并上架商品。
- 尝试清除筛选条件后重新搜索。
- 如果仍为空，联系管理员检查商品同步状态。

### 5. 商品为什么不显示？

- 商品可能未上架（状态为"待处理"或"已隐藏"）。
- 活动可能已过期。
- 新同步的商品可能需要几分钟才显示。

### 6. 复制讲解失败怎么办？

- 检查网络连接。
- 确认商品是否已上架且有有效的推广链接。
- 如果反复失败，联系管理员。

### 7. 为什么渠道归因为空？

- 订单没有通过系统推广链接成交。
- 达人直接使用了抖音原始链接而非系统生成的链接。
- **必须使用系统"复制讲解"功能生成的链接，订单才能归因到渠道。**

### 8. 为什么订单刚产生系统里没有？

- 订单同步有 10 分钟到 1 小时的延迟。
- 系统每 10 分钟同步一次增量订单，另有 6 小时窗口回扫。
- 如超过 2 小时仍未出现，联系管理员检查同步日志。

### 9. 为什么结算数据是 0？

- 订单从付款到结算通常需要数天到数周时间。
- 如果系统刚上线，还没有完成结算的订单，结算数据会显示为 0。
- 这是正常现象，随着时间推移会逐渐有数据。

### 10. 为什么寄样不能重复申请？

- 同一达人 + 同一商品只能有一个有效寄样申请。
- 如需重新申请，请先关闭之前的寄样。

### 11. 为什么寄样没有自动完成？

- 寄样自动完成需要系统匹配到对应订单（渠道 + 达人 + 商品 + 付款时间）。
- 如果订单没有使用系统推广链接，归因数据缺失则无法自动完成。
- 如确认达人已出单但寄样未自动完成，联系管理员排查。

### 12. 为什么看板数据和人工统计有差异？

- 看板数据基于系统同步的订单计算，可能和人工统计有时间差。
- 退款订单会从统计中扣除。
- 部分订单因归因问题可能不在当前角色的统计范围内。

### 13. 修改权限后为什么没生效？

- 用户需要刷新页面或重新登录才能加载新的权限。
- 如果仍不生效，联系管理员确认角色配置是否正确保存。

### 14. 导出失败怎么办？

- 检查导出数据量是否过大，尝试缩小时间范围。
- 检查浏览器是否阻止了文件下载。
- 如反复失败，联系管理员。

## 13. 当前版本使用注意事项

以下内容基于 2026-06-07 远端系统真实核验结果：

### 已上线能力

- 用户登录、角色权限管理（JWT Token 认证，支持 accessToken + refreshToken 双令牌）。
- 商品活动同步（每 5 分钟自动同步）、商品库展示（当前 714 个展示中商品）。
- 达人管理（达人 CRM，支持认领/公海/私海/标签/收货地址）。
- 寄样申请、审核、发货全流程。
- 订单自动同步（当前已有 10,670 个订单，含 6,468 机构订单 + 2,704 结算订单双轨）。
- 数据看板（预估轨指标、订单明细 16 列展示）。
- 操作日志。
- 个人中心（`/profile`，查看个人信息和角色）。

### 当前限制

1. **渠道归因尚未闭环**：当前所有订单的 `pick_source` 为空，意味着没有订单能归因到具体渠道。渠道业绩统计暂时无法验证。需要通过系统"复制讲解"链接产生真实订单后才能闭环。

2. **结算数据为 0**：当前所有订单的结算金额均为 0，因为上游尚未返回结算数据。结算轨指标（结算订单额、结算服务费收入等）暂时显示为 0。

3. **物流以手动录入为主**：当前物流 API 自动跟踪未启用，物流信息需要运营手动录入。

4. **非管理员角色菜单未完整配置**：当前非管理员角色登录后可能看不到菜单，需要管理员在"角色管理"中手动分配菜单权限。

5. **部分高级能力当前版本未启用**：
   - 独家达人 / 独家商家规则。
   - 差异化提成。
   - 物流 API 自动跟踪。
   - 高级数据导出。
   - 自动爬虫增强能力。

6. **订单同步有延迟**：订单从抖音上游同步到系统通常有 10 分钟到 1 小时的延迟。

7. **寄样自动完成依赖归因**：寄样自动完成需要订单归因成功，当前由于归因样本不足，寄样可能无法自动完成。
