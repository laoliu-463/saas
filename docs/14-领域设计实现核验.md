| 状态 | 含义 |

## 四、逐域实现核验

### 一、用户域

需求入口：[01-用户域](./requirements/domain-design/01-用户域.md)

当前判断：**部分实现**。

已落地能力：

- 认证基础能力已具备：登录、刷新 Token、登出。
- 当前用户接口族已补齐：`GET /users/current`、`PUT /users/current/password`、`GET /users/current/data-scope`、`POST /users/current/permissions/check`，并已提供前端 API 封装。
- 前端个人中心 `/profile` 已迁移到当前用户接口族，可查看当前账号、数据范围和角色权限，支持个人改密与权限自检。
- 人员主数据接口已补齐基础下拉：`GET /users/master-data/channels`、`GET /users/master-data/recruiters`、`GET /users/master-data/group-members`，并已提供前端 API 封装。
- 商品分配弹窗已迁移到人员主数据接口，负责人候选通过 `/users/master-data/recruiters` 获取。
- 订单筛选已迁移到人员主数据接口，渠道负责人候选通过 `/users/master-data/channels` 获取，招商负责人候选通过 `/users/master-data/recruiters` 获取。
- 寄样台筛选已迁移到人员主数据接口，渠道负责人候选通过 `/users/master-data/channels` 获取，招商负责人候选通过 `/users/master-data/recruiters` 获取；后端寄样分页/导出支持按渠道负责人和商品招商负责人过滤。
- 登录成功、登录失败、登录锁定、账号停用拒绝和登出已写入 `operation_log`；个人改密已由 `UserDomainService.changePassword` 写入同一操作日志中心。
- 认证审计仅记录用户名、动作、结果和错误原因，不记录密码或 token 原文。
- 系统用户、角色、部门、菜单管理已有后端 Controller 与前端管理页面。
- `@RequireRoles`、`DataScope`、菜单权限与角色权限已在 P1/P2 多轮审查中补强。
- 系统操作日志已有管理入口。

主要缺口：

- 更多业务页权限判断仍需逐步迁移到统一用户域 API。
- 角色数据范围目前主要围绕本人/本组/全部落地，原需求中更完整的组、部门、人员主数据和 `data_scope JSONB` 配置 UI 未完全闭环。
- 前端用户管理仍偏基础账号维护，用户多角色/部门协作和更多业务权限判断迁移仍需补齐。
- 组织架构与业务域下拉的稳定契约仍需进一步统一。

交付影响：

- V1 基础 RBAC、当前用户接口、人员主数据下拉和认证安全审计可交付。
- 若按用户域 V1.0 原文验收，不能宣称用户域完整实现。

已落地能力：

- 达人列表、详情、认领、释放、黑名单、手动补录、刷新、同步资料等基础 CRM 能力已实现。

交付影响：

- 基础达人 CRM 与 V1.1 收尾能力可交付。
- 独家达人机制不能按原文宣称完整实现；**不能宣称真实抖音达人 API 已接通**。

### 六、业绩域

需求入口：[06-业绩域](./requirements/domain-design/06-业绩域.md)

当前判断：**核心计算引擎 + V1 查询接口层已补齐；独家覆盖/完整重算/分析模块仍未完全实现**。

- **`PerformanceController`**（2026-05-23）：`GET /performance/{orderId}`、`POST /performance/batch`、`GET /performance`、`GET /performance/summary`、`GET /performance/export`、`POST /performance/recalculate-month`。
- **`PerformanceSummaryService`**：按 pay/settle cohort 选单，同时汇总 estimate / effective 双轨金额；`serviceFeeExpense = recruiterCommission + channelCommission`。
- **`PerformanceQueryService` + `PerformanceAccessScope`**：按角色过滤 `final_channel_user_id` / `final_recruiter_user_id`；未计算返回 `PERFORMANCE_NOT_CALCULATED`；批量无权限不泄露。

双轨汇总口径：

- effective 轨：cohort 内且（已结算或 `effective_service_fee > 0`），`settle_amount`、`effective_*` 求和。

主要缺口（不阻塞 V1 数据平台闭环）：
