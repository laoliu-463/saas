# 03 Mock 与 Real 网关契约

更新时间：2026-04-25

## 契约原则

Mock 和真实 SDK 分开实现，但必须遵守同一套 Gateway 契约。

要求：

1. 入参一致
2. 出参一致
3. 错误语义一致
4. 业务层调用方式一致
5. 只允许在 Gateway 实现层区分 Mock / Real

## 网关列表

### `DouyinAuthGateway`

能力范围：

- 获取 token
- 刷新 token
- 校验招商团长权限

异常场景：

- token 过期
- 授权主体异常
- 缺少招商团长权限

### `DouyinColonelActivityGateway`

能力范围：

- 查询招商活动列表
- 查询活动商品列表
- 查询活动商品详情

异常场景：

- 活动不存在
- 无权限
- 参数错误
- 限流

### `DouyinProductGateway`

当前能力：

- 查询商品详情
- 查询 SKU
- 查询活动商品列表
- 承接商品能力与活动商品能力的分层

当前状态：

- 接口与 Mock / Real 实现已新增
- Real 的商品详情 / SKU 仍为占位实现

### `DouyinPromotionGateway`

能力范围：

- 商品转链
- 获取推广物料

异常场景：

- 商品不可推广
- 活动已结束
- 达人无权限推广
- 参数错误

### `DouyinOrderGateway`

目标能力：

- 查询订单
- 查询佣金
- 承接后续数据回流

当前状态：

- 尚待独立补齐

## Mock 字段原则

- 第三方字段使用官方英文命名
- 请求参数尽量与官方文档保持一致
- 返回结构尽量贴近真实开放平台
- 错误码和失败信息要可模拟

## 内部 DTO 原则

- Gateway 输出统一内部 DTO
- 前端只消费系统内部 API DTO
- 页面文案和枚举 label 可以中文
- 不允许把第三方原始 JSON 直接穿透到业务层和页面

## Mock 必须覆盖的失败场景

1. 无招商团长权限
2. token 过期
3. 活动不存在
4. 商品不可推广
5. 转链失败
6. 第三方限流
7. 参数错误

## 业务层统一处理要求

- 所有第三方异常统一归一化处理
- 失败结果必须具备日志追踪能力
- Mock 和 Real 的异常处理流程必须一致
- Service 不允许出现 `if mock` 的流程分叉
