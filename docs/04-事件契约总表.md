# 事件契约总表

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 事件治理规则

- [V1 必做] 事件必须记录生产者、消费者、触发条件、关键载荷、幂等键、证据路径。
- [V1 必做] 事件不得绕过领域边界：订单事件只表达订单事实，业绩计算由业绩域触发。
- [V1 简化] V1 可使用 Spring 本地事件或 Outbox 记录，不要求全部外部 MQ。
- [V2 预留] 事件版本、重放、死信、跨服务 MQ 属于后续治理。

## 明确不做

- [V1 不做] 订单事件不携带最终提成结论。
- [V1 不做] 配置变更事件不直接触发具体业务动作，业务域自行读取配置并执行。
- [V1 不做] 分析模块事件不反向修改订单、寄样或业绩事实。
