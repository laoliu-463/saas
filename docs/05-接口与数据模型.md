# 05-接口与数据模型

更新时间：2026-05-03

## 一、接口总览

所有前端接口统一通过 `/api` 前缀访问，前端请求基址为 `/api`。
认证方式统一为：

```http
Authorization: Bearer <token>
```

当前接口文档按以下分组整理：

| 分组 | 路径前缀 | 说明 |
|---|---|---|
| 认证中心 | `/auth` | 登录、Token 刷新、登出 |
| 订单管理 | `/orders` | 订单同步、列表、详情、统计、筛选项 |
| 订单回流与归因 | `/orders/order-attribution-*`、`/dashboard/order-attribution-summary` | 未归因排查与回流摘要 |
| 数据看板 | `/dashboard/summary` | 首页归因概览 |
| 数据平台 | `/data/*`、`/dashboard/metrics`、`/orders/phone-decryptions`、`/orders/exports` | 数据页专用接口 |
| 团长活动管理 | `/colonel/activities` | 团长活动列表与活动商品列表 |
| 活动商品主链路 | `/colonel/activities/{activityId}/products/*` | 商品主链路详情、绑定、分配、审核、转链、跟进、日志 |
| 活动管理 | `/activities` | 本地活动分页与真实活动详情补充 |
| 寄样管理 | `/samples` | 寄样申请、分页、候选达人、状态流转 |
| 达人CRM | `/talents` | 达人列表、公海私海、认领释放、补全 |
| 系统用户 | `/users` | 后台用户管理 |
| 系统角色 | `/roles` | 后台角色管理 |
| 测试工具 | `/test` | 测试环境专用数据初始化与链路推进 |
| 抖音联调 | `/douyin`、`/douyin/webhooks` | 真实 SDK 联调、Token 管理、Webhook 回调 |
| 商品管理（兼容） | `/products`、`/products/picks` | 共享商品库与旧版选品候选兼容接口 |

## 二、统一返回格式

成功响应统一包装为：

```json
{
  "code": 200,
  "msg": "success",
  "data": {},
  "timestamp": 1713628800000
}
```

分页响应统一表现为：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "records": [],
    "total": 100,
    "page": 1,
    "size": 20
  },
  "timestamp": 1713628800000
}
```

## 三、命名与使用口径

### 1. 路径规则

- 资源名优先使用复数：`/orders`、`/samples`、`/talents`、`/users`、`/roles`
- 主链路路径体现层级关系：`/colonel/activities/{activityId}/products/{productId}`
- 子动作后缀统一使用 kebab-case：`audit-result`、`promotion-links`、`manual-fill`

### 2. 分页参数

当前存在两种命名：

- 业务接口部分仍使用 `page` + `pageSize`
- 业务接口部分已经使用 `page` + `size`

当前项目决定是：

- **暂不强行一次性改名**
- 已在 Swagger 中明确标注命名差异
- 后续是否统一为 `page` + `size`，以 `docs/archive/audits/接口整改-决策备忘录.md` 结论为准

### 3. 测试、联调、废弃接口展示规则

- `/test/**`：统一归类到 `测试工具`，接口名称前缀为 `[测试]`
- `/douyin/**`、`/douyin/webhooks/**`：统一归类到 `抖音联调`，接口名称前缀为 `[联调]`
- `/products/**`：统一归类到 `商品管理（兼容）`，其中 `/products` 为共享商品库，`/products/picks` 为旧版选品候选兼容入口
- `/test/**` 仅允许出现在 `local-mock`、`test` 或当前回归型 `real-pre` 环境，不作为未来纯真实 SDK 环境的默认能力

### 4. 枚举说明规则

- 已有代码常量、枚举或映射函数定义的，按代码说明
- 代码中没有明确定义的，Swagger 中统一写：
  - `待确认：取值含义请联系产品`

本文档同样遵守该规则，不臆造未确认枚举值。

## 四、核心接口清单

### 0. 商品管理（兼容）

```http
GET /api/products
GET /api/products/picks
POST /api/colonel/activities/{activityId}/products/{productId}/library-entry
```

说明：

- `GET /api/products`：查询已从选品库沉淀到共享商品库的商品，支持 `page`、`size`、`keyword`
- `GET /api/products/picks`：查询旧版本地选品候选列表，主要用于 Token 缺失时的降级展示
- `POST /api/colonel/activities/{activityId}/products/{productId}/library-entry`：将选品结果加入共享商品库，入库后对全员可见

### 1. 认证中心

```http
POST /api/auth/login
POST /api/auth/refresh
POST /api/auth/logout
```

登录请求：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

登录响应（`LoginResponse`）：

| 字段 | 类型 | 说明 |
|---|---|---|
| `token` | String | Access Token（有效期 2 小时） |
| `refreshToken` | String | Refresh Token（有效期 7 天） |
| `expiresIn` | Long | Access Token 有效期秒数（7200） |
| `refreshExpiresIn` | Long | Refresh Token 有效期秒数（604800） |
| `accessTokenExpiresIn` | Long | Access Token 有效期秒数（与 `expiresIn` 一致） |
| `userId` | Long | 用户 ID |
| `username` | String | 用户名 |

Token 刷新请求：

```json
{
  "refreshToken": "eyJhbGciOi..."
}
```

登出请求：

```json
{
  "accessToken": "eyJhbGciOi...",
  "refreshToken": "eyJhbGciOi..."
}
```

说明：

- Access Token 有效期 2 小时，携带 `type=access` claim
- Refresh Token 有效期 7 天，携带 `type=refresh` claim，使用 SHA-256 哈希存储于 Redis
- 登出后 Access Token 和 Refresh Token 同时加入 Redis 黑名单，立即失效
- 黑名单 key 格式：`auth:blacklist:{tokenHash}`，过期后自动清理

#### DataScope 数据权限

系统支持三级数据权限过滤（`@DataScope` 注解 + `DataScopeAspect`）：

| 枚举值 | 说明 |
|---|---|
| `PERSONAL(1)` | 仅查看本人数据 |
| `DEPT(2)` | 查看本组数据 |
| `ALL(3)` | 查看全部数据 |
