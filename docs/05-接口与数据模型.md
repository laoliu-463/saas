# 05-接口与数据模型

更新时间：2026-04-27

## 一、系统内部 API

### 商品主链路

- `GET /colonel/activities/{activityId}/products`
- `GET /colonel/activities/{activityId}/products/{productId}`
- `PUT /colonel/activities/{activityId}/products/{productId}/audit-result`
- `PUT /colonel/activities/{activityId}/products/{productId}/assignee`
- `POST /colonel/activities/{activityId}/products/{productId}/promotion-links`
- `POST /colonel/activities/{activityId}/products/{productId}/follow`
- `GET /colonel/activities/{activityId}/products/{productId}/operation-logs`

### 订单回流与归因

- `POST /orders/sync`
  - 保留为运维 / 调试接口，前端主流程不直接暴露
- `GET /orders`
  - 订单分页查询
- `GET /orders/{orderId}`
  - 订单详情查询，用于归因排查弹窗
- `GET /orders/unattributed`
  - 未归因订单分页查询
- `GET /orders/filter-options`
  - 订单筛选选项统一接口
- `GET /dashboard/summary`
  - 订单回流与归因摘要接口

### 达人 CRM

- `GET /talents`
- `GET /talents/{talentId}`
- `POST /talents`
- `POST /talents/{talentId}/claim`
- `POST /talents/{talentId}/release`

### Test 调试链路

- `POST /test/reset`
- `POST /test/seed`
- `POST /test/orders/sync`
- `POST /test/orders/generate-attributed`
- `POST /test/orders/generate-no-pick-source`
- `POST /test/orders/generate-missing-mapping`
- `POST /test/logistics/ship/{sampleRequestId}`
- `POST /test/logistics/sign/{sampleRequestId}`

说明：

- 后端统一端口：`8080`
- 前端统一前缀：`/api`
- 正确示例：`http://localhost:8080/api/test/seed`
- 错误示例：`http://localhost:8080/test/seed`

## 二、Gateway 契约

当前已落地：

- `DouyinAuthGateway`
- `DouyinColonelActivityGateway`
- `DouyinProductGateway`
- `DouyinPromotionGateway`
- `DouyinOrderGateway`
- `TalentGateway`
- `LogisticsGateway`

原则：

- Controller 只调用系统内部 Service
- Service 只依赖 Gateway 契约
- Test / Real 只在 Gateway 实现层区分
- 切换真实 SDK 时，不反向污染 Controller、Service 和前端页面

## 五、订单详情返回口径

`GET /orders/{orderId}` 详情接口当前聚合以下信息：

- 订单基础信息
- 归因结果
- `pickSource`
- 推广链接 / 映射信息
- 商品 / 活动 / 招商信息
- 渠道信息
- 达人信息
- 寄样关联信息
- 未归因诊断建议

## 六、当前 Test 场景

### Test 转链

- 生成 `shortId`
- 生成 `shortLink`
- 生成 `promoteLink`
- 写入 `pick_source_mapping`

### Test 订单

- 有映射，可归因
- 无映射，未归因
- 无 `pick_source`，未归因

### Test 达人

- 公海达人
- 已被当前人认领的达人
- 已被他人认领的达人
- 有寄样和订单产出的达人

## 七、当前边界说明

当前 MVP 暂不处理：

- 完整财务结算
- 复杂提成规则
- 自动定时任务全面真实化
- 敏感信息解密链路重构
- 复杂可视化图表
- 人工修改订单归因
- 批量导出与批量编辑

当前先证明一件事：

> 推广链接产生的订单，系统能够同步回流，并区分已归因与未归因，再向渠道、招商、寄样和达人链路输出可排查结果。

## 八、订单筛选选项接口

- `GET /orders/filter-options`
  - 返回统一 `label/value` 结构
  - 字段包括：`orderStatuses`、`attributionStatuses`、`unattributedReasons`、`products`、`channels`、`colonels`
  - 订单页初始化时一次加载，用于回显筛选项与 URL query 状态
