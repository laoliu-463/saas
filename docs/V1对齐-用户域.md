## 一、V1 范围

### V1 做

- U-01 登录 / 登出 / JWT
- U-02 预置六类角色
- U-03 自定义角色 + 菜单 / 操作 / 数据范围
- U-04 数据范围 `self / group / all`
- U-05 员工 CRUD、重置密码
- U-06 部门 + 招商组 / 渠道组
- U-07 渠道 / 招商 / 部门下拉
- U-09 一人多角色（当前项目口径按“权限并集、数据范围取最宽”处理）

### V1 简化

- U-08 操作日志（90 天）按简化口径保留后台可查能力

### V1 不做

- 无

## 二、当前系统对表

| U-01 登录 / 登出 / JWT | 做 | 已有双 Token、登出吊销、Redis 黑名单 | ✅ 符合 |
| U-02 预置六类角色 | 做 | 已有 `admin / biz_leader / biz_staff / channel_leader / channel_staff / ops_staff` | ✅ 符合 |
| U-03 自定义角色 + 菜单 / 操作 / 数据范围 | 做 | 角色管理、菜单能力、数据范围已落地 | ✅ 符合 |

| U-05 员工 CRUD、重置密码 | 做 | 用户管理、重置密码已落地 | ✅ 符合 |

| U-07 渠道 / 招商 / 部门下拉 | 做 | 已供订单、寄样、达人等页面使用 | ✅ 符合 |

## 三、与领域设计的主要偏差

| 类型 | 条目 | 说明 |
|---|---|---|
| 可接受差异 | 角色命名 | 设计文档使用 `recruiter_*`，当前代码使用 `biz_*`，业务含义等价 |
| 做法差异 | U-09 一人多角色 | 设计文档原始口径偏向 V2 才开放，当前项目已按简化规则实现 |
| 简化差异 | U-08 操作日志 | 当前以“后台可查 + 定时清理”为主，不扩展成更复杂审计中心 |

---

## 四、状态机与规则检查

用户域没有复杂业务状态机，当前重点是账号状态、登录安全和数据权限规则：

| 检查项 | 当前情况 | 结论 |
|---|---|---|
| 账号状态 | 正常 / 禁用已有实现，待激活未作为当前主链重点 | ⚠️ 可后补 |
| 登录安全 | 登录、刷新、登出、吊销已落地 | ✅ |
| 数据权限 | `self / group / all` 已稳定使用 | ✅ |
