## 一、V1 范围

### V1 做

- C-01 寄样限制天数 / 开关、超时关闭天数
- C-02 达人认领保护期天数
- C-03 全局招商 / 渠道提成比例
- C-04 默认寄样门槛 JSON
- C-05 复制讲解模板、`pick_extra` 规则
- C-06 配置变更生效机制

### V1 简化

- C-07 配置变更历史

### V1 不做

- C-08 独家达人 / 商家阈值
- C-09 差异化提成
- C-10 预设达人标签库

## 二、当前系统对表

| 功能点 | V1 要求 | 当前实现 | 结论 |
|---|---|---|---|
| C-01 寄样限制 / 超时关闭 | 做 | `sample_limit_days / enabled`、超时关闭参数已有实现 | ✅ 符合 |
| C-02 达人保护期天数 | 做 | `talent_claim_protect_days` 已接入达人保护期逻辑 | ✅ 符合 |
| C-03 全局提成比例 | 做 | 招商 / 渠道全局比例已有实现 | ✅ 符合 |
| C-04 默认寄样门槛 JSON | 做 | `default_sample_requirements` 已接入商品 / 寄样链路 | ✅ 符合 |
| C-05 复制讲解模板、`pick_extra` 规则 | 做 | 当前有固定模板 / 规则底座，但未形成独立配置中心 | ⚠️ 路径差异 |
| C-06 配置变更生效机制 | 做 | 当前通过保存后缓存失效 / 读取最新值生效 | ⚠️ 路径差异 |

| C-08 独家阈值 | 不做 | 代码中可保留配置键，但不作为 V1 启用项 | 🔄 V2 后移 |
| C-09 差异化提成 | 不做 | 当前不作为 V1 配置项启用 | 🔄 V2 后移 |
| C-10 预设标签库 | 不做 | 当前未启用 | 🔄 V2 后移 |

## 三、与领域设计的主要偏差

| 类型 | 条目 | 说明 |
|---|---|---|
| 少做 | C-05 独立配置后台 | 当前还没有独立的模板 / `pick_extra` 配置中心 |
| 做法差异 | C-06 配置变更事件 | 设计文档偏向事件通知，当前实现更接近缓存失效 + 最新值读取 |
| 多做但不启用 | C-08 / C-09 相关配置键 | 当前代码可保留底座，但文档上不能写成 V1 已启用 |

## 四、状态机与规则检查

配置域没有业务状态机，重点是“谁能改、改完如何生效、是否有留痕”：

| 检查项 | 当前情况 | 结论 |
|---|---|---|
| 管理员维护配置 | 已有后台接口 | ✅ |
| 保存前校验 | 已有类型 / 范围 / JSON 校验 | ✅ |
| 生效方式 | 当前以缓存失效 / 最新读取为主 | ⚠️ 路径差异 |
| 变更留痕 | 已有变更日志 | ✅ |
