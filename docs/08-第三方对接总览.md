# 第三方对接总览

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 总原则

- [V1 必做] 前端只调用内部 API。
- [V1 必做] 后端 Gateway / SDK 负责抖音 / 抖店接口封装、鉴权、参数转换、签名、错误码适配和响应标准化。
- [V1 必做] real-pre 验收必须使用真实 Token、真实配置和真实接口响应，不得用 mock 数据冒充。
- [V1 简化] 缺权限包、缺真实样本、触发限流的分支可记录为 BLOCKED，但必须保留请求、响应、时间和环境证据。

## 历史口径处理

- [历史归档] 旧文档中的 Python 爬虫、fake-useragent、Celery 调度只保留在归档中。
- [V1 不做] 不把历史爬虫方案作为当前对接事实。
- [V2 预留] 若后续恢复外部采集能力，必须重新做合规、限流、权限和审计设计。
