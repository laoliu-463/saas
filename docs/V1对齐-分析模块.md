## 一、V1 范围

### V1 做

- A-01 指标卡片（预估 + 结算）
- A-02 时间筛选：付款 / 结算时间
- A-05 汇总更新能力
- A-07 订单明细下钻

### V1 简化

- A-03 趋势图
- A-04 渠道 / 招商排行榜
- A-06 寄样统计

### V1 不做

- A-08 看板整体导出

## 三、与领域设计的主要偏差

| 类型 | 条目 | 说明 |
|---|---|---|
| 做法差异 | A-05 汇总更新 | 设计文档偏向事件 / 汇总表，当前系统以 SQL 直接聚合为主 |
| 简化差异 | A-03 / A-04 / A-06 | 当前只保留基础趋势、基础排行、基础寄样统计 |
| 明确后移 | A-08 | 看板整体导出不进 V1 |

## 四、状态机与规则检查

分析模块没有业务状态机，重点是“指标口径是否稳定、是否与订单 / 业绩一致”：

| 检查项 | 当前情况 | 结论 |
|---|---|---|
| 卡片口径 | 当前已按订单 / 业绩事实聚合 | ✅ |
| 时间筛选 | 付款 / 结算时间切换已具备 | ✅ |
| 汇总实现 | SQL 聚合与原设计路径不同，但业务效果可接受 | ⚠️ |
