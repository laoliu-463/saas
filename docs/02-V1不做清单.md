## 四、实现层「禁止默认开启」检查

对齐时若发现下列情况，视为 **违反 V1 不做清单**：

1. 配置项或 Feature Flag 默认开启独家 / 差异化提成；
2. 订单同步或业绩计算写入 `final_*` 且与 `default_*` 不一致（非测试 Mock）；
3. 看板只展示单行金额却混用付款时间与结算金额轨道；
4. 在 `OrderSyncService` 内计算招商/渠道提成（应仅在业绩域）；
5. 用旧单轨字段（单一 `service_fee`）替代 `estimate_*` / `effective_*` 做 V1 验收。
