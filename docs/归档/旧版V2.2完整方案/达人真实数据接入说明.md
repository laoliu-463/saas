# 达人真实数据接入说明

更新时间：2026-05-24

## 目标

V1-P0 已接入真实达人基础资料采集最小闭环：输入抖音号、主页链接或分享链接后，系统按 Provider 链尝试拉取真实基础资料，并写入 `talent` 表（对外文档称 talents）及 `talent_profile_sync_log`。

## Provider 顺序

1. `configurable_http`：读取 `talent.profile.http.*` 配置，对接官方/第三方 HTTP 数据源
2. `public_web`：低频单条公开主页采集，不做 Cookie 池、代理池、批量抓取
3. `manual`：仅用于失败后人工补充，`data_source=manual`

## 接口

- `POST /api/talents/resolve-profile`：解析输入并尝试拉取真实资料
- `POST /api/talents/{id}/sync-profile`：刷新已有达人资料（失败不覆盖旧成功数据）

## 字段策略

- `talentLevel`、`sales30d` 默认标记 `unsupported`，Provider 未真实返回时不得写入默认值
- 禁止伪造：`LV1`、`0` 等默认值不得作为真实资料写入
- 所有采集记录：`data_source`、`sync_status`、`last_sync_time`、`raw_payload`、`talent_profile_sync_log`

## 寄样门槛

当达人 `unsupported_fields` 包含 `talentLevel` 或 `sales30d` 时，寄样资格检查不得自动通过，需提示填写申请原因。

## 配置项

```yaml
talent:
  profile:
    public-web:
      enabled: true
    http:
      enabled: false
      endpoint: ${TALENT_PROFILE_HTTP_ENDPOINT:}
      method: GET
      token: ${TALENT_PROFILE_HTTP_TOKEN:}
      header-authorization: ${TALENT_PROFILE_HTTP_AUTHORIZATION:}
```

## 百应达人广场本地采集

当真实达人资料来自百应后台而非抖店开放接口时，正式写入 SaaS 仍只允许“普通 Chrome 手动查看 + 本地文件导入”。Playwright 只能作为本机原型探针，用来验证页面选择器、登录态复用、代理出口和字段解析质量；稳定前不接后端、不写数据库、不进入任务表。

### 推荐流程：手动可见内容导入

1. 用普通 Chrome 手动打开百应达人广场：

```text
https://buyin.jinritemai.com/dashboard/servicehall/daren-square
```

2. 在页面内手动完成登录、筛选、排序，确认达人信息是当前账号页面明示可见的内容。
3. 打开 Chrome DevTools Console，粘贴并执行 `runtime/qa/baiying-visible-copy-snippet.js` 的全部内容。
4. 该脚本只读取当前页面已显示 DOM，不点击、不翻页、不请求百应接口；执行后会尽量复制 JSON 到剪贴板，若剪贴板不可用则输出到 Console 变量 `window.__baiyingVisibleTalentsJson`。
5. 用本地导入脚本生成脱敏报告：

```powershell
npm run qa:baiying:talents -- manual-import --input-json <JSON文件路径> --max 20
```

也可以导入人工保存的 HTML：

```powershell
npm run qa:baiying:talents -- manual-import --input-html <HTML文件路径> --max 20
```

报告产物位于 `runtime/qa/out/baiying-talent-collect-*`，重点查看：

- `summary.json`
- `records.redacted.json`
- `records.redacted.csv`
- `report.md`

### 写入 real-pre

人工确认脱敏报告无误后，才允许写入 SaaS real-pre：

```powershell
npm run qa:baiying:talents -- apply --input-json <JSON文件路径> --include-sensitive --saas-token <SaaS access token> --max 20
```

执行口径：

- `manual-import` 和带 `--input-json` / `--input-html` 的 `apply` 只读取本地文件，不访问百应。
- 未提供本地输入文件时，`manual-import` / `apply` 会直接拒绝执行；`capture` / 浏览器 `dry-run` 仅能在显式 `--enable-browser-capture` 的本机原型模式下运行。
- `apply` 会先校验 SaaS 后端为 `REAL-PRE`，并且必须显式传入 SaaS access token。
- 联系方式只处理当前百应账号页面明示可见的内容；默认报告脱敏。
- 只有同时使用 `apply` 与 `--include-sensitive` 时，联系方式才会写入现有 `contactPhone` 字段。
- 脚本不写入 `contactWechat`、`rawPayload`、账号密码、Cookie 或第三方 token。
- 若普通 Chrome 手动访问也出现 `code=9001`、`msg=异常操作` 或“当前网络不稳定”，立即停止采集，走百应官方客服或权限确认。

### 本机 Playwright 原型探针

仅用于本地验证，不作为主采集方案。脚本默认仍禁用浏览器采集，必须显式传 `--enable-browser-capture`，且只允许 `capture` / `dry-run`，不允许直接 `apply`。

首次登录并保存本地授权态：

```powershell
npm run qa:baiying:talents -- capture --enable-browser-capture --manual-ready --save-storage-state runtime/auth/douyin-storage-state.json --max 1
```

复用授权态和固定代理做单条 dry-run：

```powershell
npm run qa:baiying:talents -- dry-run --enable-browser-capture --storage-state runtime/auth/douyin-storage-state.json --proxy-server http://127.0.0.1:7890 --max 1 --delay-ms 20000
```

执行口径：

- `runtime/auth/*` 属于本地运行态，不提交 Git。
- 未传 `--enable-browser-capture` 时，`capture` / `dry-run` 会拒绝直接访问百应。
- `apply` 仍必须传 `--input-json` 或 `--input-html`，不会边爬边写 SaaS。
- 脚本只保存脱敏证据到 `runtime/qa/out/baiying-talent-collect-*`；只有同时传 `--include-sensitive --write-sensitive-report` 才会在本机额外写敏感报告。
- 发现登录页、验证码、风控、`code=9001` 或“异常操作”时停止，不自动绕过验证码，不切换代理继续打。
- 单次默认 `max=1`、页面间隔不低于 15 秒；稳定前不要做批量、多账号、代理池。

### 手动导入文件格式

`manual-import --input-html` 适合保存当前可见页面 HTML 后解析，`manual-import --input-json` 适合人工整理字段。JSON 格式为数组或 `{ "records": [] }`：

```json
[
  {
    "nickname": "示例达人",
    "douyinAccount": "example_douyin_no",
    "talentUid": "7000000000000000000",
    "fansCount": "1.2万",
    "likeCount": "3万",
    "worksCount": "88",
    "ipLocation": "上海",
    "mainCategory": "美妆",
    "contactPhone": "13812345678"
  }
]
```

## SQL 验收

```sql
select id, douyin_account, talent_uid, sec_uid, nickname, fans_count, like_count, works_count,
       ip_location, talent_level, sales_30d, data_source, sync_status, last_sync_time
from talent
order by update_time desc
limit 20;

select id, talent_id, provider_code, sync_status, unsupported_fields, error_code, error_message,
       started_at, finished_at
from talent_profile_sync_log
order by started_at desc
limit 20;
```
