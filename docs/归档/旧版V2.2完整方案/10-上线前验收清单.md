## real-pre P0 统一验收（强制入口）

- 入口：`npm run e2e:real-pre:p0`（串行执行 preflight + `tests/e2e/08` + `tests/e2e/31~36`）。
- 报告路径：`runtime/qa/out/real-pre-p0-YYYYMMDD-HHmmss/summary.json` 与 `report.md`。
- 入口必须使用：上线前对外宣称 real-pre 联调通过时，证据必须是 `real-pre-p0-*` 目录下的报告，不允许仅以 `e2e:real-pre`/`e2e:real-pre:business`/`e2e:real-pre:roles` 等细分报告替代。
- 结论判定（与 README-e2e.md 一致）：
  - **PASS / PASS_NEEDS_CLEANUP**：业务步骤通过；后者表示 cleanup 仅生成 PlanOnly 未执行，需要人工审核 `cleanup-plan.json/sql` 后再决定是否真的执行清理。
  - **BLOCKED**：缺真实 Token / 缺授权 / 缺可复用 `pick_source_mapping` / 关键账号无法登录。
  - **PENDING**：当前窗口缺真实订单 / 缺 `pick_source` 样本 / 缺触发寄样自动完成的真实成交。
  - **FAIL**：环境不是 real-pre、health/页面运行时错误、权限越权、订单同步 failed > 0、业绩公式错、cleanup-plan 命中危险语句。
- 边界：20~24-v1-* 套件只能在 test/mock 跑，不代表 real-pre 联调验收；real-pre 套件不允许跑在 test/mock 库上。
