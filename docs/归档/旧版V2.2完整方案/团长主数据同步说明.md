# 团长主数据同步说明

## 表结构

`colonel_partner`：以 `colonel_buyin_id` 为业务唯一键，扩展联系方式、来源、同步时间、`raw_payload`。

## 同步来源

1. `colonel_activity`
2. `colonelsettlement_order`
3. `pick_source_mapping`

由 `ColonelPartnerSyncService.syncAll()` 聚合并 upsert；定时任务 `ColonelPartnerSyncJob`。

## 人工维护规则

- 自动同步**不得**用空值覆盖已维护的 `contact_name` / `contact_phone` / `contact_wechat`。
- 名称可随同步更新；记录 `last_sync_at`。

## API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/colonel-partners` | 列表（keyword/source/hasContact） |
| GET | `/api/colonel-partners/{id}` | 详情 |
| PUT | `/api/admin/colonel-partners/{id}/contact` | 更新联系方式 |
| POST | `/api/admin/colonel-partners/sync` | 手动同步 |
| GET | `/products/filter-options/colonel-partners` | 商品库筛选项 |

## 商品库筛选

按团长名称筛选时优先 `colonel_partner` 解析 product_id 集合（`resolveProductIdsByColonelName`）。
