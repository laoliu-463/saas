## 寄样域 - 完整设计方案（V1.4 正式版）

------

## 文档说明

本文档基于骨架设计 V1.4 和需求文档 V2.2 设计。

**定位**：管理样品申请的完整生命周期，从渠道申请到招商审核、运营发货、交作业完成。

**原则**：寄样域不关心商品细节、不关心达人详情，通过事件与其他领域通信。

**版本更新**：

- V1.3 新增合作方字段
- V1.4 **修正：媒介改为渠道**（从用户域获取渠道列表）

**版本状态**：✅ 所有问题已确认，可进入开发

## 一、领域定位

### 1.1 职责

| 职责                 | 说明                                            |
| -------------------- | ----------------------------------------------- |
| 管理寄样申请生命周期 | 从申请到完成的完整流程                          |
| 校验申请条件         | 达人门槛、7天重复申请限制                       |
| 管理物流信息         | 录入物流单号、跟踪状态（V1.0手动，V2.0对接API） |
| 判断交作业完成       | 订单产生后自动判断是否完成                      |
| 寄样数据导出         | 导出寄样信息给商家，商家回填物流单号            |
| 发出寄样事件         | 供业绩域等订阅                                  |

### 1.2 边界

**管**：

- 寄样申请的创建、审核、发货、完成、拒绝、关闭
- 申请时的校验规则（门槛、重复申请）
- 物流单号管理
- 寄样数据导出

**不管**：

- 商品信息（从商品域查询）
- 合作方信息（从商品域查询）
- 达人信息（从达人域查询）
- 订单归属（业绩域管）
- 提成计算（业绩域管）

### 1.3 对外提供的能力

| 操作         | 调用方           | 说明                                 |
| ------------ | ---------------- | ------------------------------------ |
| 申请寄样     | 渠道             | 从商品库快速寄样，或从寄样台手动申请 |
| 查询寄样列表 | 渠道/招商/运营   | 支持多维度筛选                       |
| 审核寄样     | 招商             | 通过/拒绝，拒绝需填写原因            |
| 批量审核     | 招商             | 批量通过/拒绝，单次最多100条         |
| 录入物流     | 运营             | 录入物流单号                         |
| 确认签收     | 运营/系统        | 物流签收后更新状态                   |
| 查询我的申请 | 渠道             | 查看自己提交的寄样申请               |
| 导出寄样数据 | 招商/运营/管理员 | 按筛选条件导出Excel                  |

### 1.4 发出的事件

| 事件       | 触发时机             | 携带数据                                 |
| ---------- | -------------------- | ---------------------------------------- |
| 寄样已提交 | 渠道提交申请，待审核 | 申请ID、商品ID、合作方ID、达人ID、渠道ID |
| 寄样已通过 | 招商审核通过         | 申请ID、商品ID、达人ID、渠道ID           |
| 寄样已拒绝 | 招商审核拒绝         | 申请ID、拒绝原因                         |
| 寄样已发货 | 运营录入物流         | 申请ID、物流单号                         |
| 寄样已完成 | 达人出单（≥1单）     | 申请ID、订单ID、达人ID、渠道ID           |
| 寄样已关闭 | 超时30天未出单       | 申请ID、关闭原因                         |

### 1.5 订阅的事件

| 事件         | 来源   | 处理                                                         |
| ------------ | ------ | ------------------------------------------------------------ |
| 订单已同步   | 订单域 | 检查该订单是否对应某寄样申请（达人+商品+渠道），是则标记为已完成 |
| 商品已上架   | 商品域 | 无需处理，但寄样申请时需要商品信息                           |
| 达人信息变更 | 达人域 | 无需处理，但寄样申请时需要达人门槛判断                       |

## 二、核心概念

### 2.1 寄样状态流转

| 状态         | 说明           | 触发操作                      |
| ------------ | -------------- | ----------------------------- |
| **待审核**   | 渠道已提交申请 | 招商审核                      |
| **待发货**   | 招商已审核通过 | 运营录入物流单号              |
| **快递中**   | 商家已发货     | 物流API更新或手动确认         |
| **待交作业** | 达人已签收     | 等待该渠道+该达人+该商品出单  |
| **已完成**   | 已出单（≥1单） | 系统自动判断订单              |
| **已拒绝**   | 招商拒绝申请   | 渠道可重新申请（不受7天限制） |
| **已关闭**   | 超时30天未出单 | 系统自动关闭                  |

### 2.2 状态流转图

```plain
渠道提交申请
      ↓
┌─────────────┐
│   待审核    │ ← 招商审核
└─────────────┘
      │
      ├── 通过 ──→ ┌─────────────┐
      │            │   待发货    │ ← 运营录入物流
      │            └─────────────┘
      │                  ↓
      │            ┌─────────────┐
      │            │   快递中    │ ← 物流签收
      │            └─────────────┘
      │                  ↓
      │            ┌─────────────┐
      │            │  待交作业   │ ← 等待出单
      │            └─────────────┘
      │                  ↓
      │            ┌─────────────┐
      │            │   已完成    │ ← 出单（≥1单）
      │            └─────────────┘
      │
      └── 拒绝 ──→ ┌─────────────┐
                   │   已拒绝    │ ← 可重新申请
                   └─────────────┘

超时30天未出单 ──→ ┌─────────────┐
                    │   已关闭    │
                    └─────────────┘
```

### 2.3 寄样限制规则

| 规则项     | 说明                           | 可配置 |
| ---------- | ------------------------------ | ------ |
| 限制对象   | 同一渠道 + 同一达人 + 同一商品 | -      |
| 限制周期   | 7天内不允许重复申请            | ✅      |
| 例外情况   | 被拒绝的申请不受7天限制        | -      |
| 不受限角色 | 组长、管理员不受此限制         | ✅      |

### 2.4 寄样门槛规则

渠道申请寄样时，系统会检查达人是否符合商品的寄样要求。

| 检查项         | 说明                              | 数据来源 |
| -------------- | --------------------------------- | -------- |
| 达人等级       | 达人等级 ≥ 商品要求的等级阈值     | 达人域   |
| 近30天销售额   | 达人销售额 ≥ 商品要求的销售额阈值 | 达人域   |
| 其他自定义条件 | 商品补充信息中的寄样要求          | 商品域   |

**校验结果处理**：

- 符合条件 → 正常提交
- 不符合条件 → 弹窗提醒，允许填写申请原因后继续提交

### 2.5 交作业判断

| 判断条件     | 说明                                             |
| ------------ | ------------------------------------------------ |
| 触发时机     | 订单同步时                                       |
| 判断逻辑     | 订单中的 渠道ID + 达人ID + 商品ID 匹配某寄样申请 |
| 匹配后处理   | 该申请状态 → 已完成                              |
| 订单数量要求 | ≥1单即完成                                       |

**注意**：一个寄样申请只需要有一单订单即可完成，不需要全部订单都来自该渠道。

## 四、业务规则

### 4.1 申请规则

| 规则           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| 商品必须已上架 | 申请时商品必须在商品库展示中                                 |
| 达人必须存在   | 达人ID必须在达人库中存在                                     |
| 寄样限制检查   | 7天内同一渠道+同一达人+同一商品不重复申请（被拒绝的除外）    |
| 门槛检查       | 达人是否符合商品寄样要求，不符合则弹窗提醒（允许填写原因后继续） |
| 收货地址       | 从达人域自动带入该达人的已保存地址，渠道可修改               |

### 4.2 审核规则

| 规则     | 说明                                 |
| -------- | ------------------------------------ |
| 审核权限 | 该商品的负责人（招商）才能审核       |
| 通过     | 状态 → 待发货                        |
| 拒绝     | 状态 → 已拒绝，需填写拒绝原因        |
| 批量审核 | 支持批量通过/拒绝，**单次最多100条** |

### 4.3 发货规则

| 规则     | 说明                                        |
| -------- | ------------------------------------------- |
| 发货权限 | 运营角色                                    |
| 录入方式 | V1.0 手动录入单号；V2.0 对接物流API自动获取 |
| 状态变化 | 录入后状态 → 快递中                         |

### 4.4 超时关闭规则

| 规则       | 说明                                  |
| ---------- | ------------------------------------- |
| 超时时间   | 从「待交作业」状态开始，30天未出单    |
| 检查频率   | 定时任务每天检查                      |
| 关闭后处理 | 状态 → 已关闭，发出「寄样已关闭」事件 |

## 五、寄样数据导出功能

### 5.1 功能描述

运营或招商可以将寄样申请数据导出为Excel文件，发给商家。商家根据导出信息发货后，回填物流单号，运营再录入系统。

**业务流程**：

```plain
运营/招商进入寄样台
      ↓
设置筛选条件（状态=待发货、时间范围等）
      ↓
点击「导出」按钮
      ↓
系统按筛选条件生成Excel文件
      ↓
运营/招商下载文件，发给商家
      ↓
商家根据文件中的信息发货
      ↓
商家回填物流单号（Excel或系统导入）
      ↓
运营录入物流单号到系统
```

### 5.2 导出权限

| 角色     | 导出权限                       |
| -------- | ------------------------------ |
| 管理员   | ✅ 可导出全部                   |
| 招商     | ✅ 可导出自己负责商品的寄样申请 |
| 招商组长 | ✅ 可导出本组招商负责的寄样申请 |
| 运营     | ✅ 可导出全部（按需配置）       |
| 渠道     | ❌ 不可导出                     |

## 八、与其他领域的关系

### 8.1 依赖关系图

```plain
┌─────────────┐      ┌─────────────┐
│  商品域     │ ───→ │  寄样域     │ 查询商品信息、寄样门槛、合作方信息
└─────────────┘      └──────┬──────┘
                            ↑
┌─────────────┐             │
│  达人域     │ ───────────→│ 查询达人等级、销售额、收货地址
└─────────────┘             │
                            ↓
┌─────────────┐             │
│  用户域     │ ───────────→│ 查询渠道列表（V1.4修正）
└─────────────┘             │
                            ↓
                      ┌─────────────┐
                      │  订单域     │ 订阅订单事件，判断完成
                      └─────────────┘
```

### 8.2 被调用场景

| 调用方 | 调用目的         | 查询内容                               |
| ------ | ---------------- | -------------------------------------- |
| 前端   | 展示寄样列表     | 按状态、商品、合作方、渠道等多维度筛选 |
| 业绩域 | 统计寄样相关指标 | 寄样数量、完成率等                     |

### 8.3 事件驱动场景

| 事件       | 来源   | 寄样域的处理                                                 |
| ---------- | ------ | ------------------------------------------------------------ |
| 订单已同步 | 订单域 | 检查订单(渠道+达人+商品)是否匹配待交作业的申请，匹配则标记完成 |
