## 一、V1 范围

### V1 做

- T-01 达人查询 / 补全（API + 手填兜底）
- T-02 公海 / 私海、多人同时认领
- T-03 保护期、无产出释放、出单重置
- T-04 认领人维护收货地址
- T-05 认领人打标签（最多 3 个、后者覆盖）
- T-06 达人列表 / 详情、按角色数据范围
- T-08 手动 / 定时刷新达人数据

### V1 简化

- T-07 批量导入达人

### V1 不做

- T-09 预设标签库
- T-10 爬虫 Cookie 池 / 代理

## 二、当前系统对表

| 功能点 | V1 要求 | 当前实现 | 结论 |
|---|---|---|---|
| T-01 达人查询 / 补全 | 做 | 列表、详情、手动补录、刷新已有实现 | ✅ 符合 |
| T-02 公海 / 私海、多人认领 | 做 | 多人可同时认领，公海 / 私海已落地 | ✅ 符合 |
| T-03 保护期、无产出释放 | 做 | 认领写 `protectedUntil`、定时释放、有出单跳过释放已落地 | ✅ 符合（子集） |
| T-03 出单重置保护期 | 做（设计） | 订单同步后 `TalentService.extendProtectionOnOrder` 按出单时间重算 `protectedUntil` | ✅ 符合 |
| T-04 收货地址维护 + 寄样带入 | 做（设计） | `talent_claim.recipient_*` + `PUT /talents/{id}/shipping-address`；寄样 `addressSource=claim/manual` | ✅ 符合 |

| T-08 手动 / 定时刷新 | 做 | 单达人刷新、周任务刷新已有 | ✅ 符合 |

| T-10 爬虫 Cookie 池 / 代理 | 不做 | 当前不启用 | 🔄 V2 后移 |

## 三、与领域设计的主要偏差

| 类型 | 条目 | 说明 |
|---|---|---|
| 少做 | T-05 标签上限 | 当前“最多 3 个”还需要更明确校验 |
| 做法差异 | T-01 / T-08 数据补全 | 当前更偏 API + 手动兜底，不强调完整爬虫增强 |
| 简化差异 | T-07 批量导入 | 当前按简化策略后置，不扩大到正式导入工具 |
| 明确后移 | T-09 / T-10 | 预设标签库、Cookie 池 / 代理不进 V1 |

## 四、状态机与规则检查

达人域没有复杂审批状态机，当前重点是认领关系和保护期规则：

| 检查项 | 当前情况 | 结论 |
|---|---|---|
| 多人认领 | 已支持多人同时认领、不互斥 | ✅ |
| 公海 / 私海 | 已有展示与切分逻辑 | ✅ |
| 保护期 | 已接入配置和定时释放 | ✅ |
| 出单后重置保护期 | 订单同步后按 `payTime/createTime + protection_days` 重算有效认领 `expire_time` | ✅ |
