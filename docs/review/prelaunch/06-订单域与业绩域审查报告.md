# 订单域与业绩域上线审查报告

审查日期：2026-05-27 | 审查基线：feature/auth-system @ cfd7236

## 一、订单同步

**状态：PASS（有 P1 改进项）**

- OrderController 提供订单查询、手动同步接口
- 角色限制：BIZ_LEADER, BIZ_STAFF, CHANNEL_LEADER, CHANNEL_STAFF, ADMIN
- 订单同步由定时任务驱动，支持手动触发

**发现：**
- **[P1] R-04 hasPerformanceRecords 全表 COUNT**：`hasPerformanceRecords()` 使用全表 COUNT 判断是否存在业绩记录，在大数据量下性能差
  - 文件：相关 Service
  - 修复：改用 `EXISTS` 子查询或 LIMIT 1

## 二、订单归因

**状态：PASS**

- 订单域只存事实，不算提成，不应用独家覆盖（不变量）
- pick_source 作为订单归属的关键字段，与精选联盟转链形成完整归因链路
- colonel_buyin_id 用于百应平台订单匹配

## 三、业绩归属

**状态：PASS**

- 业绩域负责最终归属、提成、冲正、双轨金额计算（不变量）
- 提成计算在业绩域完成，不在订单域

## 四、分区管理

**状态：PARTIAL**

- PostgreSQL 月分区表（`op_log_yyyy_MM`、订单分区表）
- 订单分区覆盖 2026-04 至 2027-03
- `create_next_month_partitions()` 函数存在

**发现：**
- **[P1] R-05 分区过期风险**：pg_cron 未配置自动创建分区，2027-03 后新分区不会自动创建
  - 文件：`db/init-db.sql`
  - 修复：配置 pg_cron 或在部署脚本中添加分区创建检查

## 五、迁移脚本

**状态：PARTIAL**

- 47 个 SQL 迁移脚本，手动管理（非 Flyway）
- 迁移脚本命名为 `V{version}__{description}.sql`

**发现：**
- **[P1] R-06 非幂等迁移 DML**：`migrate-all.sql` 中包含非幂等的 UPDATE 语句，重复执行会产生不同结果
  - 文件：`db/migrate-all.sql:1092-1099`
  - 修复：改用 INSERT ... ON CONFLICT DO UPDATE 或添加 WHERE 条件

## 六、提成计算

**状态：PASS**

- 配置域负责配置，不执行具体业务规则（不变量）
- 业绩域负责最终归属和提成计算
- 双轨金额计算在业绩域完成
