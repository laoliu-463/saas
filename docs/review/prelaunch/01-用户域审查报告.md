## 一、登录/登出/Token 刷新流程

**状态：PARTIAL**

| 环节 | 验证结果 |
|------|----------|
| 登录 | 查找用户→检查锁定→BCrypt 校验→计算 dataScope→生成 access+refresh JWT→记录操作日志。**通过** |
| 登出 | 校验 refresh token→Redis 黑名单(`auth:refresh:`)→可选黑 access token(`auth:blacklist:`)。**通过** |
| 刷新 | 校验 refresh token 类型→检查 Redis 黑名单→重新查库→生成新 access token。**通过**（但有 P1 问题） |
| 失败计数 | Redis 自增计数器，达到阈值后锁定账户，TTL 控制锁定时长。**通过** |
| 挂起激活限制 | 仅允许 GET /users/current、PUT /users/current/password、POST /auth/logout。**通过** |

**关键发现：**
- **[P1] U-03 Refresh Token 无轮换机制**：`AuthService.refreshToken()` 复用原 refresh token，不签发新的。若 refresh token 被截获，可在 7 天有效期内持续获取新 access token。
  - 文件：`AuthService.java:220`
  - 修复：实现 refresh token rotation，每次刷新签发新 refresh token 并将旧的加入黑名单。

## 四、数据权限过滤

**状态：PARTIAL**

**四场景验证：**

| 场景 | 结果 |
|------|------|
| 管理员看全部 | dataScope=3(ALL)，无过滤条件。**通过** |
| 普通用户看自己 | PERSONAL(1)：追加 `userField = userId`。**通过** |
| 组长看本组 | DEPT(2)：追加 `dept_id = deptId`。**通过** |
| 未登录无法访问 | Spring Security 拦截，返回 401。**通过** |

**实现现状：**
- @DataScope 注解仅用于 3 个 Mapper 方法
- 大部分 Controller 使用手动 `applyDataScope()`（OrderController, DashboardService 等），逻辑重复

**关键发现：**
- **[P1] U-02 数据权限越权**：`SysUserService.assertCanAccess()` 仅检查 PERSONAL 范围，当 dataScope=DEPT 时未校验目标用户是否属于当前用户所在部门。channel_leader 可能操作其他部门用户。
  - 文件：`SysUserService.java:394`
  - 修复：在 assertCanAccess() 中增加 DEPT 范围的部门校验。
- **[P2] U-04 双重数据权限实现**：原订单分页 mapper 同时依赖 @DataScope 注解与 Controller 手动 wrapper 数据范围，存在重复过滤和口径漂移风险。

## 六、安全

**状态：PARTIAL**

**关键发现：**
- **[P1] U-01 JWT 密钥占位值**：`application.yml:64` 使用 `${JWT_SECRET:dev-secret-key-replace-in-production-with-random-64-char-string}`。部署时忘记设置 JWT_SECRET 将以已知密钥签发 JWT。虽有日志警告但不阻断启动。
  - 修复：未设置时直接拒绝启动（fail-fast）。
- **[P2] U-05 SystemEnvController 无 @RequireRoles**：`/system/env` 依赖 `requireAdminWhenProd()` 手动检查，若环境判断失误，非管理员可获取环境信息。
- **[P2] U-06 双重 JWT 校验**：JwtAuthenticationFilter + JwtAuthInterceptor 做完全相同的 JWT 校验，请求处理两次相同逻辑。
- **[P3] U-07 管理员重置密码无需原密码**：`SysUserService.resetPassword()` 直接设置新密码。
- **[P3] U-08 DashboardService.applyScope() DEPT 范围退化**：deptId 为 null 时不过滤，退化为 ALL。
