# 用户域领域合同

> 模板来源：`.claude/templates/领域合同模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域定位

- [V1 必做] 用户域负责用户、角色、组织、菜单和统一数据范围。
- [V1 必做] 用户域对其他业务域提供 `self/group/all` 数据范围，不替业务域判断订单归属、业绩归属或业务状态。

## 职责边界

| 类型 | 内容 | 范围 |
| --- | --- | --- |
| 负责 | 登录用户上下文、角色、组织、菜单、数据范围解析 | V1 必做 |
| 负责 | 管理员、主管、普通渠道等角色的权限基础 | V1 必做 |
| 不负责 | 商品、订单、寄样、业绩的业务状态机 | V1 不做 |
| 不负责 | 订单最终归属、提成、独家覆盖 | V1 不做 |

## 核心事实

- [V1 必做] 数据范围统一抽象为 `self/group/all`。
- [V1 必做] 其他领域查询必须能追溯当前用户、角色、组织和数据范围来源。
- [V1 简化] V1 以当前权限注解、后端查询过滤和 E2E 证据为主。
- [V2 预留] 更复杂的 ABAC / PBAC、细粒度字段权限和组织树治理可后续扩展。

## 对外能力

| 能力 | 使用方 | 证据 | 范围 |
| --- | --- | --- | --- |
| 当前用户信息 | 前端、业务域 | `/api/auth/me` 或当前用户 API | V1 必做 |
| 用户/角色/菜单管理 | 管理端 | 用户、角色、菜单 API | V1 必做 |
| 数据范围解析 | 商品、达人、寄样、订单、业绩、分析 | 权限测试、API/SQL 对比 | V1 必做 |

## 事件契约

| 事件 | 触发 | 消费方 | 范围 |
| --- | --- | --- | --- |
| 用户数据范围已解析 | 查询上下文建立 | 各业务域查询侧 | V1 必做 |
| 用户权限已变更 | 角色、菜单或组织变更 | 审计、缓存失效 | V1 简化 |

## 数据契约

- [V1 必做] 用户、角色、菜单、组织和用户组织关系属于用户域事实。
- [V1 必做] 业务数据只引用用户或组织标识，不复制权限判断逻辑。

## 验收与证据

- [V1 必做] 使用管理员、主管、普通渠道账号分别访问同一列表，验证 `self/group/all` 差异。
- [V1 必做] 验收证据进入 [../验收/验收证据索引.md](../验收/验收证据索引.md)。
- [V1 必做] 权限边界见 [../07-权限与数据范围.md](../07-权限与数据范围.md)。

## 冲突与待确认

- [V1 必做] 若旧文档中存在业务域自行定义数据范围，应改为调用用户域或写入 ADR-002。
