# 商品域领域合同

> 模板来源：`.claude/templates/领域合同模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域定位

- [V1 必做] 商品域负责商品库、活动商品、商品展示输入、转链和 `pick_source_mapping`。
- [V1 必做] 商品域为订单归因提供映射事实，但不计算最终业绩和提成。

## 职责边界

| 类型 | 内容 | 范围 |
| --- | --- | --- |
| 负责 | 商品库、活动商品同步、筛选、展示规则输入 | V1 必做 |
| 负责 | 抖音转链、`pick_source_mapping` 落库 | V1 必做 |
| 负责 | quick sample 等内部寄样申请入口的商品上下文 | V1 简化 |
| 不负责 | 订单最终归属、提成、冲正 | V1 不做 |
| 不负责 | 分析看板重算业绩 | V1 不做 |

## 核心事实

- [V1 必做] 商品域调用转链能力后必须落 `pick_source_mapping`。
- [V1 必做] 订单域消费映射事实作为归因输入，业绩域负责最终归属。
- [V1 必做] 复制讲解入口经 `CopyPromotionApplicationService` 编排，抖音转链经 `DouyinConvertPort` 端口隔离 legacy gateway；转链落库与幂等语义仍由既有服务保持兼容。
- [V1 必做] 商品库“复制简介”按一次用户操作写入富文本和纯文本两种剪贴板表示；富文本包含商品 CDN 图片 URL 与招商入库信息生成的完整简介，纯文本仅作目标端不支持富文本时的降级。不得把独立图片和文字拆成两个复制按钮或两次复制动作。本阶段兼容目标为微信，微信真实粘贴必须同时显示图片和完整简介；飞书允许降级为图片链接和完整文字，暂不要求渲染图片，也不得据此声明跨平台图片兼容通过。
- [V1 必做] 商品库查询支持独立 `productId` 条件，按商品外部 ID 精确匹配；`keyword` 保持商品名称 / 商品 ID / 店铺关键字模糊匹配。
- [V1 必做] 商品库查询默认按置顶优先和上游合作开始时间倒序展示；排序规则属于商品域固定展示规则，不由前端下拉切换。
- [V1 必做] 商品库展示由商品自身上游状态、本地发布控制、推广期和去重规则决定；活动状态不得驱动商品入库或展示。
- [V1 必做] 活动商品以商品自身上游 `status=1/推广中` 作为自动入库和可操作主判断来源：同步和 repair 应补齐 `selectedToLibrary=true`、`auditStatus=2` 和可操作业务状态，历史本地拒绝不得阻断。`manual_disabled=true` 仅表示本地暂停发布，商品库隐藏但活动商品页仍可展示“恢复发布”；非推广中商品不得自动入库。历史状态断链必须通过商品域统一 repair 入口修复，不允许裸 SQL 批量直改。
- [V1 必做] 待审核商品人工审核通过前必须补齐业务补充信息，包括专属价说明、发货信息、商品卖点、推广话术、是否支持投流、奖励说明、参与要求、活动时间和手卡素材；活动商品审核入口和旧商品管理审核通过兼容入口都必须把这些字段透传给商品域服务写入 `audit_payload`，不得由前端绕过或由后端自动伪造。
- [V1 必做] 上游 `product_snapshot.cos_type=1` 表示双佣金时，若审核补充明确 `supportsAds=false`，商品域必须将 `audit_payload.adsRule` 归一化为“不支持投流”，商品列表和详情均展示该说明；不得把双佣金标识推断成支持投流。
- [V1 必做] 商品编辑允许维护本系统推广时间覆盖，时间写入 `product_operation_state.audit_payload`，不修改 `product_snapshot` 上游时间事实；开始时间不得晚于结束时间，清空覆盖值时回退上游时间。
- [V1 必做] 商品库中支持投流的商品必须展示“投流”标签；鼠标悬浮标签时展示招商审核补充中的 `audit_payload.adsRule`，缺失时明确提示招商暂未填写投流说明。
- [V1 必做] 待审核商品人工审核通过或直接入库时，若商品库已有相同 `product_id` 的其他有效关系（`selected_to_library=true`），必须返回业务错误并禁止当前关系进入商品库；当前正在处理的关系不计入重复命中，批量入库逐条执行同一检查。
- [V1 简化] V1 优先保障活动商品同步、商品库、转链、映射闭环。
- [V2 预留] 更复杂的商品展示规则、推荐排序和多平台商品治理后续扩展。

## 数据契约

- [V1 必做] 商品、活动商品和 `pick_source_mapping` 属于商品域核心事实。
- [V1 必做] `pick_source_mapping` 至少能支撑商品、达人/渠道、pick source、时间和第三方响应追溯。
- [V1 必做] 商品域字段级清单以当前 Java 持久化实体和迁移 SQL 为准，不把订单归因、提成、结算或看板重算字段并入商品域。

## 验收与证据

- [V1 必做] 活动商品同步后商品库可查。
- [V1 必做] 商品库按 `productId` 查询时，只返回该商品 ID 对应记录；不存在的商品 ID 返回空分页。
- [V1 必做] 活动商品列表必须同时返回当前筛选总量和活动全量上游状态聚合；前端不得用当前已加载分页行数冒充活动商品总量。
- [V1 必做] 转链后 `pick_source_mapping` 有记录，订单归因链路可引用。
- [V1 必做] real-pre 转链必须保存真实请求 / 响应或阻塞证据。
- [V1 必做] 商品图文复制验收必须同时检查剪贴板包含完整模板文本、HTML 包含商品图片 URL、无独立图片 MIME，并在微信客户端执行一次真实粘贴，确认图片与完整简介同时显示；浏览器侧验证不能替代微信验收。飞书当前以图片链接和完整文字可用作为已接受降级，不纳入本阶段图片渲染门槛。
