# 业绩域领域合同

> 模板来源：`.claude/templates/领域合同模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域定位

- [V1 必做] 业绩域消费订单默认归因输入，按版本化业绩规则计算最终渠道/招商归属、提成、退款调整和双轨金额。
- [V1 必做] 业绩域输出可解释的业绩明细与汇总；订单域不保存最终业绩结论，分析模块不重算归属。

## 职责边界

| 类型 | 内容 | 范围 |
| --- | --- | --- |
| 负责 | 基于订单默认 user/source、独家规则和经审批人工调整计算最终归属、提成、退款调整、双轨金额 | V1 必做 |
| 不负责 | 改写订单默认归因或创建推广映射 | V1 不做 |
| 负责 | 汇总表刷新，供分析模块只读 | V1 必做 |
| 不负责 | 抖店订单原始事实同步 | V1 不做 |
| 不负责 | 商品转链和 `pick_source_mapping` 生成 | V1 不做 |

## 核心事实

- [V1 必做] 订单域是默认渠道/招商输入事实来源；业绩域是 `final_channel_*`、`final_recruiter_*` 和归属规则版本的唯一结论来源。
- [V1 必做] 招商 PERSONAL 数据范围必须按业绩记录 `final_recruiter_user_id` 过滤；招商链接的业绩归链接创建账号，不可回退为活动招商或招商组长。
- [V1 必做] 最终归属优先级为经审批人工调整 > 独家商家 > 独家达人 > 订单默认归因；决策快照、规则版本和默认/最终双列必须保留。
- [V1 必做] 同一订单事实版本只成功计算一次；失败必须记录执行台账并由受控任务重试，不得吞异常后仅留日志。
- [V1 必做] 失败重试调度默认启用；历史 `performance.recalculate-failed.enabled` 不得再作为关闭恢复闭环的开关。
- [V1 必做] 部分退款写入独立 `performance_adjustment_ledger` 比例调整流水，覆盖订单金额、服务费、技术费、服务费支出、达人佣金、利润和两类提成；整单失效不应与已失效基础业绩重复扣减。
- [V1 必做] 执行台账必须保存足以恢复原事件语义的输入快照；退款重试需恢复同一退款流水后才可发布计算完成事件。
- [V1 必做] 双轨金额、提成和毛利口径必须可审计。
- [V1 必做] 服务费收入双轨公式按 2026-06-06 用户口径执行：预估服务费收入 = 预估订单额 × 服务费率（未扣除技术服务费）；结算服务费收入以官方结算服务费字段为准，不用订单额重算且不扣技术服务费。
- [V1 必做] 服务费收益双轨公式按 2026-06-06 用户口径执行：预估服务费收益 = 预估服务费收入 - 预估服务费支出 - 技术服务费；结算服务费收益 = 结算服务费收入 - 结算服务费支出。
- [V1 必做] 技术服务费只参与预估服务费收益扣减；结算服务费收益不再扣减技术服务费。不得用单一“服务费收入 - 技术服务费”公式套用预估 / 结算两轨。
- [V1 简化] V1 优先支持 P0 计算和看板汇总，复杂财务结算后置。

## 对外能力

| 能力 | 使用方 | 证据 | 范围 |
| --- | --- | --- | --- |
| 业绩计算 | 管理端、调度 | 计算日志、业绩明细表 | V1 必做 |
| 冲正处理 | 管理端、分析模块 | 退款事实、冲正记录 | V1 必做 |
| 汇总刷新 | 分析模块、dashboard | 汇总表、看板 API | V1 必做 |
| 规则读取 | 配置域 | 配置 API、规则中心 | V1 必做 |

## 事件契约

| 事件 | 触发 | 消费方 | 范围 |
| --- | --- | --- | --- |
| 订单已同步 | 订单域入库订单事实 | 业绩域计算金额、提成和冲正 | V1 必做 |
| 订单归因事实已更正 | 订单域受控重放已更正默认归因事实版本 | 业绩域读取最新订单事实并按新事件版本重算 | V1 必做 |
| 退款事实已同步 | 订单域入库退款事实 | 业绩域处理冲正 | V1 必做 |
| 业绩归属已计算 | 业绩明细生成 | 分析模块、报表 | V1 必做 |
| 业绩汇总已刷新 | 汇总表刷新 | 分析模块 | V1 必做 |

## 数据契约

- [V1 必做] 业绩明细、归属结论、提成、冲正、双轨金额和汇总表属于业绩域事实。
- [V1 必做] 业绩记录必须保存订单默认 channel/recruiter/source/dept 与最终 channel/recruiter/source/dept、规则版本、决策快照；最终归属可读取规则事实但不得回写订单。
- [V1 必做] `estimate_service_fee` 对应预估服务费收入，按预估订单额和服务费率计算，不扣技术服务费；`effective_service_fee` 对应结算服务费收入，必须来自官方明确结算服务费字段，不用订单额重算且不扣技术服务费。
- [V1 必做] `estimate_service_profit` 对应预估服务费收益，按预估服务费收入、预估服务费支出和技术服务费计算；`effective_service_profit` 对应结算服务费收益，按结算服务费收入和结算服务费支出计算。

## 验收与证据

- [V1 必做] 给定订单默认归因、独家规则和人工调整，业绩明细能证明最终 user/source、规则版本和提成；订单域不得代替此计算。
- [V1 必做] 招商专员 PERSONAL 查询订单和业绩时，必须以业绩 final recruiter 为准；招商链接订单归链接创建账号，不得统计到其招商组长。
- [V1 必做] 给定退款事实，业绩域能证明幂等调整流水、冲正和汇总净额变化。
- [V1 必做] 单笔订单和汇总验收必须分别验证预估服务费收益与结算服务费收益公式，不能只验证其中一轨。
- [V1 必做] 分析模块读取汇总后不重算归属。

## 冲突与待确认

- [V1 必做] 独家覆盖、提成、冲正若存在旧 V2.2 细则差异，先写入 ADR-002，不直接扩大 V1。
- [V1 必做] GROUP 查询以业绩最终部门快照过滤；历史快照缺失的存量记录仅允许本人兼容兜底，不能按当前组织关系回灌。
