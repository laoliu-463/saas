# 业绩域领域合同

> 模板来源：`.claude/templates/领域合同模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域定位

- [V1 必做] 业绩域负责最终归属、提成、冲正和双轨金额计算。
- [V1 必做] 业绩域消费订单事实、商品映射、达人关系和配置参数，输出业绩明细与汇总。

## 职责边界

| 类型 | 内容 | 范围 |
| --- | --- | --- |
| 负责 | 最终归属、提成、冲正、双轨金额 | V1 必做 |
| 负责 | 独家覆盖规则执行 | V1 必做 |
| 负责 | 汇总表刷新，供分析模块只读 | V1 必做 |
| 不负责 | 抖店订单原始事实同步 | V1 不做 |
| 不负责 | 商品转链和 `pick_source_mapping` 生成 | V1 不做 |

## 核心事实

- [V1 必做] 业绩域是最终归属结论来源。
- [V1 必做] 订单域只提供订单事实，配置域只提供规则参数，业绩域负责执行规则。
- [V1 必做] 双轨金额、提成和毛利口径必须可审计。
- [V1 必做] 服务费收入双轨公式按 2026-06-06 用户口径执行：预估服务费收入 = 预估订单额 × 服务费率（未扣除技术服务费）；结算服务费收入以官方结算服务费字段为准，不用订单额重算且不扣技术服务费。
- [V1 必做] 服务费收益双轨公式按 2026-06-06 用户口径执行：预估服务费收益 = 预估服务费收入 - 预估服务费支出 - 技术服务费；结算服务费收益 = 结算服务费收入 - 结算服务费支出。
- [V1 必做] 技术服务费只参与预估服务费收益扣减；结算服务费收益不再扣减技术服务费。不得用单一“服务费收入 - 技术服务费”公式套用预估 / 结算两轨。
- [V1 简化] V1 优先支持 P0 计算和看板汇总，复杂财务结算后置。

## 数据契约

- [V1 必做] 业绩明细、归属结论、提成、冲正、双轨金额和汇总表属于业绩域事实。
- [V1 必做] 业绩域必须保留输入来源，支持追溯订单、商品映射、达人关系和配置版本。
- [V1 必做] `estimate_service_fee` 对应预估服务费收入，按预估订单额和服务费率计算，不扣技术服务费；`effective_service_fee` 对应结算服务费收入，必须来自官方明确结算服务费字段，不用订单额重算且不扣技术服务费。
- [V1 必做] `estimate_service_profit` 对应预估服务费收益，按预估服务费收入、预估服务费支出和技术服务费计算；`effective_service_profit` 对应结算服务费收益，按结算服务费收入和结算服务费支出计算。

## 验收与证据

- [V1 必做] 给定订单事实和映射，业绩明细能证明最终归属和提成。
- [V1 必做] 给定退款事实，业绩域能证明冲正记录和汇总变化。
- [V1 必做] 单笔订单和汇总验收必须分别验证预估服务费收益与结算服务费收益公式，不能只验证其中一轨。
- [V1 必做] 分析模块读取汇总后不重算归属。

## 冲突与待确认
