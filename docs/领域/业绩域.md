# 业绩域领域合同

> 模板来源：`.claude/templates/领域合同模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域定位

- [V1 必做] 业绩域负责最终归属、提成、冲正和双轨金额计算。
- [V1 必做] 业绩域消费订单事实、商品映射、达人关系和配置参数，输出业绩明细与汇总。

## 职责边界

| 类型 | 内容 | 范围 |
| --- | --- | --- |
| 负责 | 最终归属、提成、冲正、双轨金额 | V1 必做 |
| 负责 | 独家覆盖规则执行 | V1 必做 |
| 负责 | 汇总表刷新，供分析模块只读 | V1 必做 |
| 不负责 | 抖店订单原始事实同步 | V1 不做 |
| 不负责 | 商品转链和 `pick_source_mapping` 生成 | V1 不做 |

## 核心事实

- [V1 必做] 业绩域是最终归属结论来源。
- [V1 必做] 订单域只提供订单事实，配置域只提供规则参数，业绩域负责执行规则。
- [V1 必做] 双轨金额、提成和毛利口径必须可审计。
- [V1 简化] V1 优先支持 P0 计算和看板汇总，复杂财务结算后置。

## 对外能力

| 能力 | 使用方 | 证据 | 范围 |
| --- | --- | --- | --- |
| 业绩计算 | 管理端、调度 | 计算日志、业绩明细表 | V1 必做 |
| 冲正处理 | 管理端、分析模块 | 退款事实、冲正记录 | V1 必做 |
| 汇总刷新 | 分析模块、dashboard | 汇总表、看板 API | V1 必做 |
| 规则读取 | 配置域 | 配置 API、规则中心 | V1 必做 |

## 事件契约

| 事件 | 触发 | 消费方 | 范围 |
| --- | --- | --- | --- |
| 订单已同步 | 订单域入库订单事实 | 业绩域计算归属 | V1 必做 |
| 退款事实已同步 | 订单域入库退款事实 | 业绩域处理冲正 | V1 必做 |
| 业绩归属已计算 | 业绩明细生成 | 分析模块、报表 | V1 必做 |
| 业绩汇总已刷新 | 汇总表刷新 | 分析模块 | V1 必做 |

## 数据契约

- [V1 必做] 业绩明细、归属结论、提成、冲正、双轨金额和汇总表属于业绩域事实。
- [V1 必做] 业绩域必须保留输入来源，支持追溯订单、商品映射、达人关系和配置版本。

## 验收与证据

- [V1 必做] 给定订单事实和映射，业绩明细能证明最终归属和提成。
- [V1 必做] 给定退款事实，业绩域能证明冲正记录和汇总变化。
- [V1 必做] 分析模块读取汇总后不重算归属。

## 冲突与待确认

- [V1 必做] 独家覆盖、提成、冲正若存在旧 V2.2 细则差异，先写入 ADR-002，不直接扩大 V1。
