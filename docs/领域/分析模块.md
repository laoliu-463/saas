# 分析模块领域合同

> 模板来源：`.claude/templates/领域合同模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域定位

- [V1 必做] 分析模块负责 dashboard、报表、导出和只读汇总展示。
- [V1 必做] 分析模块只读汇总表，不重算业绩归属。

## 职责边界

| 类型 | 内容 | 范围 |
| --- | --- | --- |
| 负责 | dashboard、报表、趋势、导出 | V1 必做 |
| 负责 | 按用户数据范围读取汇总 | V1 必做 |
| 不负责 | 订单同步、归因、提成、冲正 | V1 不做 |
| 不负责 | 修改订单、寄样、业绩事实 | V1 不做 |

## 核心事实

- [V1 必做] 分析模块消费业绩域和订单域已形成的汇总或事实查询。
- [V1 必做] 分析模块不得通过重新扫描订单来产生新的最终归属。
- [V1 必做] 数据看板经营指标矩阵必须展示总订单数、订单额、服务费收入、技术服务费、服务费支出、服务费收益、招商提成、媒介/渠道提成和毛利的成交/预估与结算轨。
- [V1 简化] V1 以当前 dashboard 和导出能力支撑验收。
- [V2 预留] 更复杂的 BI、指标平台、宽表和数据湖后续设计。

## 对外能力

| 能力 | 使用方 | 证据 | 范围 |
| --- | --- | --- | --- |
| Dashboard 汇总 | 管理端、渠道端 | dashboard API、E2E | V1 必做 |
| 经营指标矩阵 | 管理端、渠道端 | `/dashboard/metrics`、页面测试、SQL/API 对账 | V1 必做 |
| 报表查询 | 管理端 | API/SQL 对比 | V1 必做 |
| 导出 | 管理端 | 导出文件、操作日志 | V1 简化 |

## 事件契约

| 事件 | 触发 | 消费方 | 范围 |
| --- | --- | --- | --- |
| 业绩汇总已刷新 | 业绩域刷新汇总 | 分析模块读取 | V1 必做 |
| 分析快照已生成 | 报表生成 | 前端、验收 | V1 简化 |

## 数据契约

- [V1 必做] 分析模块读取汇总表、报表快照和导出结果。
- [V1 必做] 分析模块展示经营指标矩阵时只能读取 `/dashboard/metrics`、业绩汇总或订单事实只读查询，不反向写入业务事实。
- [V1 不做] 分析模块不写订单最终归属、不写业绩明细。

## 验收与证据

- [V1 必做] dashboard 指标能通过汇总 API 或 SQL 对账。
- [V1 必做] 经营指标矩阵的 9 个字段均需同时展示成交/预估轨与结算轨；毛利公式为 `服务费收益 - 招商提成 - 媒介/渠道提成`。
- [V1 必做] 同一 dashboard 在不同账号下满足 `self/group/all` 数据范围。

## 冲突与待确认

- [V1 必做] 如果旧设计要求 dashboard 重新计算业绩归属，应拆回业绩域并记录 ADR-005。
