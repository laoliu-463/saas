# 达人域领域合同

> 模板来源：`.claude/templates/领域合同模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 领域定位

- [V1 必做] 达人域负责达人资料、标签、地址、跟进和渠道关系。
- [V1 必做] 达人域向寄样、商品和业绩链路提供达人事实，不替代订单域或业绩域做最终归属。

## 职责边界

| 类型 | 内容 | 范围 |
| --- | --- | --- |
| 负责 | 达人基础资料、标签、地址、跟进记录 | V1 必做 |
| 负责 | 达人与渠道、商品、寄样的关系事实 | V1 必做 |
| 不负责 | 订单事实采集和订单归属计算 | V1 不做 |
| 不负责 | 提成、冲正、双轨金额 | V1 不做 |

## 核心事实

- [V1 必做] 达人信息必须可支撑寄样、跟进、渠道协作和业绩归属输入。
- [V1 简化] V1 优先当前系统内达人资料和真实接口可用字段。
- [历史归档] 旧文档中的 `Python + requests + fake-useragent` 采集方案不是当前事实。
- [V2 预留] 达人经营自动化、外部采集策略和更完整画像后续扩展。

## 对外能力

| 能力 | 使用方 | 证据 | 范围 |
| --- | --- | --- | --- |
| 达人列表 / 详情 | 前端、寄样、招商 | 达人 API、E2E | V1 必做 |
| 标签和跟进 | 渠道、招商、管理 | 操作日志、列表筛选 | V1 必做 |
| 达人地址 | 寄样域 | 地址记录、寄样发货 | V1 必做 |
| 第三方达人补充 | 管理端 | real-pre 响应或阻塞证据 | V1 简化 |

## 事件契约

| 事件 | 触发 | 消费方 | 范围 |
| --- | --- | --- | --- |
| 达人资料已更新 | 资料、标签、地址、关系变更 | 寄样域、分析模块 | V1 简化 |

## 数据契约

- [V1 必做] 达人资料、标签、地址、跟进记录和渠道关系属于达人域事实。
- [V1 必做] 达人域可保存外部接口返回字段，但必须标记来源和更新时间。

## 验收与证据

- [V1 必做] 达人列表、详情、标签、地址和跟进记录可通过 API/E2E 验证。
- [V1 必做] 如果依赖第三方达人接口，必须提供真实响应或 BLOCKED 证据。

## 冲突与待确认

- [V1 必做] 旧采集方案与当前 Spring Boot 事实冲突，已按历史归档处理。
- [V1 已知缺口 t4-talent 2026-06-02] `gender` 筛选暂不支持：`Talent` 实体与 `talent` 表当前没有 `gender` 列，前端不再展示 / 发送该筛选；直接调用 API 传入 `gender` 时，`TalentQueryService` 返回参数错误，避免静默假筛选。真正启用前必须先补数据库列、实体字段、数据来源和筛选单测。
