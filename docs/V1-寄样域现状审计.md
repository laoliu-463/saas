## 三、十问专项答复

### 1. 渠道能否从商品库申请寄样？

**能，经寄样台/申请页，非商品详情直链。**

- 商品候选：`GET /samples/product-candidates`（仅 `selected_to_library`）。
- 申请：`POST /samples`（渠道角色）。
- 前端：`/sample/apply`（`Apply.vue`）。

### 2. 7 天限制维度与配置？

| 项 | 实现 |
|----|------|
| 配置键 | `sample.restrict_days`、`sample.restrict_enabled` |
| 维度 | `channel_user_id` + `talent_id` + `product_id` |
| 硬编码 | 无（默认 7 来自 seed） |
| 拒绝豁免 | `status != REJECTED` 不计入 |
| 角色豁免 | `admin`、`channel_leader` |

**说明**：若 V1 定稿为「达人+商品」不含渠道，则当前实现 **更严**（同渠道才可撞限），与技术定稿「渠道+达人+商品」一致。

### 3. 招商审核与批量？

- 单条：`PUT /samples/{id}/status`，`PENDING_SHIP` / `REJECTED`。
- 批量：`/batch-approve`、`/batch-reject`，`@Size(max=100)`。
- 权限：`ensureActionRolePermission` 仅 **ADMIN、BIZ_STAFF** → **BIZ_LEADER 不能审核**。
- 数据范围：招商专员 personal 用 `findPageForAuditor`（按商品 assignee）。

### 4. 手动物流？

运营 `SHIPPING` 录入 `trackingNo`、`shipperCode`，写 `ship_time`，`extra_data.logisticsSource=MANUAL`。  
批量发货 `/batch-ship` 同样逻辑。  
**不依赖**物流 API 即可完成发货。

### 5. 状态机是否清晰？

```text
1 待审核 PENDING_AUDIT
2 待发货 PENDING_SHIP      ← 审核通过
3 快递中 SHIPPING          ← 录单号
4 已签收 DELIVERED         ← 可选
5 待交作业 PENDING_HOMEWORK
6 已完成 COMPLETED
7 已拒绝 REJECTED
8 已关闭 CLOSED            ← Job/超时（V1 应冻结主链路）
```

- `ensureTransition` 防止跳步。
- 人工 **不可** `COMPLETED`/`CLOSED`（仅系统自动）。

### 6. 交作业（P0 重点）？

```java
// OrderSyncPersistenceService.afterOrderPersisted
sampleLifecycleService.completePendingHomeworkByOrder(order);
```

匹配：`channel_user_id` + `talent_uid` + `product.product_id`，`status=5`。  
**缺口**：未检查 `pay_time IS NOT NULL`；`complete_time` 用 `now()`；无 `order_id` 列。

### 7. 寄样台筛选？

当前：**keyword**（达人/单号/商品名）、**status**、角色默认（招商看待审核、运营看待发货）。  
**缺**：合作方、物流单号专筛、申请/交作业时间范围、状态多选、招商/渠道筛选器（部分靠 data_scope 隐含）。

### 8. 导出？

- 接口：`GET /samples/exports`，CSV，最多 **5000** 行，走 `findPageWithScope`。
- 列：单号、达人、商品、状态、招商负责人、物流单号、驳回原因、备注、创建时间。
- **缺**：收件人、电话、地址、合作方、物流待回填说明。
- 权限见下表。

| 角色 | 后端 export | 前端 export 按钮 |
|------|-------------|----------------|
| admin | ✅ | ✅ |
| biz_leader | ✅ | ✅（isLeader） |
| biz_staff | ❌ | ❌ |
| channel_leader | ✅ | ✅（isLeader） |
| channel_staff | ❌ | ❌ |
| ops_staff | ❌ | ❌ |

合同期望「运营可导出、渠道不可导出」→ **channel_leader 可导出需收紧**；**ops 不可导出需确认**。

### 9. 权限与 data_scope？

| 动作 | 角色 |
|------|------|
| 申请 | channel_*、admin |
| 审核 | biz_staff、admin（**非 biz_leader**） |
| 物流 | ops_staff、admin |
| 列表 | `@DataScope` on `channel_user_id`；channel 见本人；dept 见本部门 |
| 运营 | 仅待发货及之后状态 |

### 10. V2 功能是否冻结？

| 功能 | 状态 |
|------|------|
| 自动关闭待交作业/待发货 | **已调度** `SampleLifecycleJob` 每日 02:00，读 `sample.timeout_*_days` |
| 物流 API 跟踪 | Job 默认 **关**；手动 refresh 可选 |
| Excel 导入 | 无 |
