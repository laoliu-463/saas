# 项目总览

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 项目定位

- [V1 必做] 本项目是抖音团长 SaaS 的 V1 交付仓库，目标是支撑渠道、招商、管理、寄样、订单归因、业绩汇总、分析看板的可验证闭环。
- [V1 必做] 文档目标不是重写需求，而是把已有文档中的事实、边界、接口、流程和验收重排为智能体可发现、可执行、可验证的结构。
- [V1 简化] V1 优先保留能支撑 P0 验收和 real-pre 联调的链路，旧 V2.2 的全量能力放入归档或 V2 预留。
- [历史归档] 旧 V2.2 完整方案入口为 [归档/旧版V2.2完整方案.md](归档/旧版V2.2完整方案.md)。

## 当前技术事实

| 项 | 当前事实 | 范围 |
| --- | --- | --- |
| 后端 | Spring Boot 模块化单体 | V1 必做 |
| 数据库 | PostgreSQL | V1 必做 |
| 缓存 | Redis | V1 必做 |
| 本地 / 联调运行 | Docker Compose | V1 必做 |
| 前端 | Vue / TypeScript / Vite | V1 必做 |
| E2E | Playwright | V1 必做 |
| 事件 | Spring 本地事件 + 事务内 / 事务后处理；Outbox 作为既有机制和扩展点 | V1 简化 |
| MQ | 不作为 V1 必需运行依赖 | V2 预留 |
| FastAPI / Celery | 旧设计或历史背景，不是当前事实 | 历史归档 |

## 当前环境口径

| 环境 | 用途 | 关键约束 | 范围 |
| --- | --- | --- | --- |
| test / local mock | 本地 P0 与回归测试 | 可使用 mock 或基线数据，但不得冒充 real-pre | V1 必做 |
| real-pre | 上线前真实联调环境 | `APP_TEST_ENABLED=false`、`DOUYIN_TEST_ENABLED=false`；不得用 mock 数据伪造闭环 | V1 必做 |
| production | 线上部署目标 | 本轮文档只记录上线前准备，不替代生产运维 SOP | V2 预留 |

## 领域边界

- [V1 必做] 用户域提供账号、角色、组织和 `self/group/all` 数据范围。
- [V1 必做] 配置域提供配置事实和配置变更，不执行具体业务规则。
- [V1 必做] 商品域负责活动商品、转链和 `pick_source_mapping`。
- [V1 必做] 达人域负责达人资料、标签、跟进、地址和渠道关系。
- [V1 必做] 寄样域负责寄样申请、审批、发货、交作业状态。
- [V1 必做] 订单域只存订单事实、订单明细、退款和归因输入，不算提成。
- [V1 必做] 业绩域负责最终归属、提成、冲正、双轨金额计算。
- [V1 必做] 分析模块只读汇总表，不重算业绩归属。

## 当前交付准则

- [V1 必做] 开发角度看功能是否闭环，用户角度看业务是否顺手。
- [V1 必做] 每个验收项必须能对应脚本、SQL/API 或证据路径。
- [V1 必做] 冲突不得靠猜测合并，必须进入 [决策/ADR-002-V1范围优先级.md](决策/ADR-002-V1范围优先级.md) 或显式冲突清单。
