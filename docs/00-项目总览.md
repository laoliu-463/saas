# 00-项目总览

更新时间：2026-04-27

## 一、项目定位

本项目是一款专为抖音“团长”业务设计的 SaaS 平台，旨在解决大规模商品推广过程中的招商分配、转链追踪、订单归因及寄样管理等核心痛点。

## 二、业务背景

在传统的团长业务中：
- **招商难追踪**：大量商品通过人工分配给不同的招商人员，进度难以实时把控。
- **归因不准确**：订单回流后，难以准确判定该订单是由哪个渠道、哪位达人、哪条推广链接产生的。
- **寄样管理乱**：达人申请寄样、商家发货、样板回收等流程缺乏系统化支撑。

## 三、核心功能模块

### 1. 商品与招商中心
- **活动商品同步**：通过 API 实时同步抖店官方团长活动的商品池。
- **业务主流程**：涵盖商品同步、初筛、分配招商、就绪等关键状态位。
- **转链中心**：自动化生成带有唯一标识的推广链接，为后续归因埋下伏笔。

### 2. 订单归因工作台
- **增量回流**：自动化拉取订单数据。
- **多维度归因**：支持根据 `pick_source`、`short_id` 等标识自动匹配渠道负责人与达人。
- **业绩摘要**：实时统计已归因业绩，支持导出异常订单进行人工排查。

### 3. 寄样管理
- **全生命周期跟踪**：申请、审核、发货、签收、交作业、结算。
- **物流对接**：支持模拟与真实物流轨迹回填。

### 4. 达人 CRM
- **公海/私海机制**：保护期机制，防止内部撞单。
- **画像补全**：自动获取达人抖音号、UID 及带货表现数据。

## 四、当前进度说明

目前项目处于 **V2.2 维护阶段**：
- **Test 环境**：已实现全链路模拟，支持通过 `Test Console` 一键初始化演示数据。
- **Real 环境**：网关契约已就绪，正处于 SDK 真实联调准备期。
