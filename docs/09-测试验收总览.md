# 测试验收总览

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 验收原则

- [V1 必做] 每个验收项必须对应脚本、SQL/API 或证据路径。
- [V1 必做] test / mock 证据只能证明本地闭环，不能替代 real-pre。
- [V1 必做] real-pre 验收必须有真实 Token、真实接口响应和真实环境日志。
- [V1 必做] 文档重构验收必须证明文件结构、范围标记、归档索引、冲突清单完整。

## 活动分配与推广中入库验收补充

- [V1 必做] 活动分配后端回归：`cd backend; mvn "-Dtest=ColonelActivityControllerTest,ProductDisplayRuleServiceTest,ProductServiceActivityAssignTest" test`。
- [V1 必做] 活动分配前端回归：`cd frontend; npm run test -- activity.test.ts activity-list-display.test.ts product-operation-log-display.test.ts product-assignee-options.test.ts`。
- [V1 必做] 验证点包括：仅管理员调用活动分配接口、候选人来自招商角色、活动列表分配字段回填、分配后短缓存驱逐、推广中活动商品不被同 `product_id` 去重隐藏。

## 状态分类

| 状态 | 含义 | 是否可收口 | 范围 |
| --- | --- | --- | --- |
| PASS | 验收通过且证据完整 | 是 | V1 必做 |
| PASS_NEEDS_CLEANUP | 主链路通过，但有日志、数据或文档清理项 | 条件通过 | V1 简化 |
| BLOCKED | 受外部 Token、权限包、真实样本等阻塞 | 否，需证据 | V1 必做 |
| PENDING | 未执行或证据不足 | 否 | V1 必做 |
| FAIL | 已复现失败 | 否，需缺陷单 | V1 必做 |
