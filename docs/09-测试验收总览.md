# 测试验收总览

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 验收原则

- [V1 必做] 每个验收项必须对应脚本、SQL/API 或证据路径。
- [V1 必做] test / mock 证据只能证明本地闭环，不能替代 real-pre。
- [V1 必做] real-pre 验收必须有真实 Token、真实接口响应和真实环境日志。
- [V1 必做] 文档重构验收必须证明文件结构、范围标记、归档索引、冲突清单完整。

## 主数据与当前用户验收补充（2026-06-02 / t0-master-data）

- [V1 必做] 主数据下拉后端回归：`cd backend; mvn "-Dtest=UserMasterDataServiceTest,ColonelPartnerMasterDataServiceTest,ColonelPartnerMasterDataControllerTest,UserMasterDataControllerTest,CurrentUserControllerTest" test`。
- [V1 必做] 主数据下拉前端单测：`cd frontend; npx vitest run src/api/sys.test.ts src/views/product/product-assignee-options.test.ts src/views/orders/order-user-filter-options.test.ts src/views/sample/sample-user-filter-options.test.ts`。
- [V1 必做] 验证点包括：
  - `/users/master-data/channels|recruiters` 关键词命中 username/realName 模糊匹配；`limit` 50/100 默认与最大；空数据返回空集合；非 admin 用户的 `group-members` 不会因 caller 传 `deptId` 而越权（service 强制回退到当前 deptId）。
  - `/users/current` 响应 `dataScope` 数字与 `dataScopeName` 字符串必须一致；`/users/current/data-scope` 的 `userIds` 在 `all` 范围返回空。
  - `/api/colonel-partners` 列表 SQL 拼接正确（`like + eq + and(isNotNull...or)` 三条件组合），详情缺失返回 404；`/api/colonel-partners/sources` 返回 distinct 来源。
  - 主数据接口的 controller 鉴权注解（`@RequireRoles`）覆盖；service 层不做鉴权（不自动加 caller 角色过滤）。
- [V1 必做] ⚠ `/api/colonel-partners` 当前未在前端接入：前端 `listPartners` 走的是 `/colonel/partners`（`ColonelPartnerController` 暴露，返回 `PartnerVO`）。两者职责不同，**当前团长主数据接口的 V1 consumer 待定**，详见 [05-API契约总表.md](05-API契约总表.md) 的"主数据与当前用户 API 补充事实"段。

## 状态分类

| 状态 | 含义 | 是否可收口 | 范围 |
| --- | --- | --- | --- |
| PASS | 验收通过且证据完整 | 是 | V1 必做 |
| PASS_NEEDS_CLEANUP | 主链路通过，但有日志、数据或文档清理项 | 条件通过 | V1 简化 |
| BLOCKED | 受外部 Token、权限包、真实样本等阻塞 | 否，需证据 | V1 必做 |
| PENDING | 未执行或证据不足 | 否 | V1 必做 |
| FAIL | 已复现失败 | 否，需缺陷单 | V1 必做 |
