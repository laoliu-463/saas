# 三方接口缺失降级联调方案

**版本**：V1.0  
**日期**：2026-05-16  
**适用阶段**：real-pre 前置核查、TEST/mock 主流程回归  

## 1. 结论口径

当前 real-pre 前置阶段已确认：物流接口和达人外部采集接口暂无真实可用服务。处理口径如下：

1. **抖店 SDK 是真实三方主链路联调对象**：订单同步、商品/联盟相关网关、抖店授权与真实上游取证仍按 `docs/03-Test与Real网关契约.md`、`docs/09-真实SDK联调准备清单.md` 执行。
2. **物流接口不阻塞 TEST/mock 主流程**：当前降级为运营手动录入物流单号 + Mock 状态流转。
3. **达人接口不阻塞 TEST/mock 主流程**：当前降级为 Mock 采集 + 采集失败后人工补录。
4. **物流/达人缺失登记为 real-pre 风险**：不得因为降级链路可用而关闭真实接口风险项。
5. **不能标记为真实三方接口联调完成**：物流和达人当前只能标记为 fallback 联调通过，不能标记为真实物流接口或真实达人接口已联调完成。
6. **不阻塞抖店 SDK 主链路 real-pre 联调**：物流/达人当前只做降级验收与风险登记，抖店 SDK 的 Token、活动、商品、转链、订单等主链路仍按 real-pre 计划继续推进。

## 2. 物流降级方案

当前物流 fallback 目标是保证寄样业务在没有真实物流接口时仍能闭环推进：

- 运营角色可手动录入物流单号。
- 手动录入后寄样状态从 `待发货` 进入 `快递中`。
- 物流来源写入 `MANUAL`。
- TEST/mock 工具可执行 Mock 发货和 Mock 签收。
- Mock 发货来源写入 `MOCK`。
- Mock 或手动确认签收后，寄样进入 `待交作业`。
- 非运营角色不能录入物流。
- 页面和内部接口在真实物流接口不可用时不能报错；真实查询失败只能作为日志/风险，不得阻塞当前寄样状态查看与手动推进。

## 3. 达人降级方案

当前达人 fallback 目标是保证达人 CRM 和寄样入口在没有真实达人采集服务时仍可验证：

- 支持 Mock 采集成功样本，自动填充昵称、粉丝数等基础字段。
- 支持 Mock 采集失败样本，记录 `FAILED`，但保留达人记录。
- 采集失败后允许人工补录。
- 人工补录后来源写入 `MANUAL`，补录状态可恢复为 `SUCCESS`。
- 人工补录后的达人可以认领。
- 认领后进入私海，并继续受私海和保护期逻辑约束。
- 人工补录后的达人可发起寄样申请。
- 页面和内部接口在真实达人接口不可用时不能报错；不得要求真实达人服务可用后才能完成 TEST/mock CRM 主流程。

## 4. real-pre 风险登记

物流和达人当前状态必须按风险登记：

| 模块 | 当前降级能力 | real-pre 风险 | 完成口径 |
| --- | --- | --- | --- |
| 物流 | 手动录入 + Mock 发货/签收 | 真实物流接口未完成联调 | 仅 fallback 通过，不是真实联调完成 |
| 达人 | Mock 采集 + 手动补录 | 真实达人采集接口未完成联调 | 仅 fallback 通过，不是真实联调完成 |
| 抖店 SDK | 主链路真实联调 | 按 `docs/09` 跟踪剩余 Token/权限/限流分支 | 真实 SDK 主链路继续推进，不受物流/达人 fallback 阻塞 |

## 5. 回归用例

新增回归脚本：

```bash
node runtime/qa/third-party-fallback-regression.cjs
```

输出目录：

```text
runtime/qa/out/third-party-fallback-regression-<时间戳>/
├── summary.json
└── report.md
```

覆盖用例：

- `LOGISTICS-001` 运营手动录入物流成功。
- `LOGISTICS-002` 非运营录入物流失败。
- `LOGISTICS-003` Mock 签收后进入待交作业。
- `TALENT-001` 采集成功自动填充。
- `TALENT-002` 采集失败手动补录。
- `TALENT-003` 手动补录后可认领和寄样。
- `FALLBACK-001` 关闭真实物流接口不影响寄样流程。
- `FALLBACK-002` 关闭真实达人接口不影响达人 CRM 流程。

## 6. 执行限制

执行本方案和回归脚本时必须遵守：

- 不切 real-pre。
- 不访问真实物流或真实达人三方接口。
- 不清库。
- 不修改生产配置。
- 不把物流/达人标记为真实三方接口联调完成。
- 只允许在 TEST/mock 口径下验证降级链路是否可用。
