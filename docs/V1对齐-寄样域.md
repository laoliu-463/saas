## 一、V1 范围

### V1 做

- S-01 渠道申请寄样（含门槛校验 + 原因）
- S-02 7 天重复申请限制（可配置、组长 / 管理员豁免）
- S-03 招商审核
- S-04 运营录入物流、推进状态
- S-05 交作业：订单匹配渠道 + 达人 + 商品后自动完成
- S-07 寄样台筛选 + 导出
- S-08 物流单号手动录入

### V1 简化 / 底座

- S-09 物流 API 自动跟踪仅保留底座

### V1 不做

- S-06 超时 30 天未出单自动关闭不作为当前 V1 主验收项
- S-10 Excel 批量导入物流单号
- S-11 寄样申请修改 / 撤回

## 三、与领域设计的主要偏差

| 类型 | 条目 | 说明 |
|---|---|---|
| 少做 | S-05 自动完成专项证据 | 当前有实现，但还缺更强业务样本说明 |
| 做法差异 | S-09 物流 API | 当前只保留底座，不把真实物流联动当 V1 主能力 |
| 范围差异 | S-06 自动关闭 | 文档对齐阶段先按“非主验收项”处理，不往前抬成当前主线 |
| 明确后移 | S-10 / S-11 | 批量导入、修改撤回不进 V1 |

## 四、状态机与规则检查

| 状态 | 当前实现 | 结论 |
|---|---|---|
| 待审核 | `PENDING_AUDIT` | ✅ |
| 待发货 | `PENDING_SHIP` | ✅ |
| 快递中 | `SHIPPED` | ✅ |
| 待交作业 | `PENDING_TASK` | ✅ |
| 已完成 | `COMPLETED` | ✅ |
| 已拒绝 | `REJECTED` | ✅ |
| 已关闭 | `CLOSED` | ✅ |

核心规则：

- 7 天重复申请限制受配置和角色豁免影响
- 发货与物流录入推进寄样状态
- 订单命中后寄样进入已完成
