## 三、十问专项答复

### 1. 活动同步是否能落库？

**部分。**  
- **可查**：`GET /colonel/activities` 实时调上游（带缓存）。  
- **可落库**：`colonelsettlement_activity` 在 `hydrateColonelActivityMeta`（转链/解析 buyin 时）`upsertRealActivityMeta`；测试环境可 `app.activities.seed-demo-on-empty` 注入演示活动。  
- **缺口**：无「招商组长点同步 → 全量活动写入 DB」的独立写操作；PRODUCT-CASE-01 若按「表内有全量活动」验收，当前 **不满足**。

### 2. 活动商品同步是否包含合作方 ID？

**以店铺维度部分满足。**  
`product_snapshot.shop_id` / `shop_name` 来自 `DouyinProductGateway.ActivityProductItem`；商品详情可关联 `merchant`。  
领域「合作方 ID」若指独立 partner 主键，当前 **无** 对应字段与 `PartnerService`。

### 3. 商品状态是否支持 待补充 → 可推广 → 展示中？

**能力等价、命名不同。**

| 需求口径 | 代码口径 |
|----------|----------|
| 待补充/待审核 | `PENDING_AUDIT`（同步后 `initStateIfAbsent`） |
| 可推广 | 审核通过 `APPROVED`；分配招商 `ASSIGNED` |
| 展示中 | `selected_to_library=true` + 库列表可见 |
| 已转链 | `LINKED`（转链后） |

上架门槛：`putIntoLibrary` → `requireApprovedAuditForLibraryEntry`。

### 4. 商品库是否支持筛选、排序、置顶？

| 能力 | 结论 |
|------|------|
| 筛选 | 库：**关键词 + status**；活动商品列表：上游 cooperation/product 条件 + 本地快照过滤 |
| 排序 | 库：`selected_at`；活动列表：上游 `sortType` |
| 置顶 | **未实现**（全仓无「置顶」关键词） |

### 5. 复制讲解是否转链并保存 pick_source → channel？

**是。**  
`generatePromotionLink` → `douyinPromotionGateway.generateLink` → `promotion_link` + `pick_source_mapping.user_id`（渠道用户）。  
前置：`selected_to_library=true` 且状态为 `APPROVED`/`ASSIGNED`/`LINKED`（重链）。

### 6. pick_source 映射能否被订单域使用？

**是（订单域已接入）。**  
表 `pick_source_mapping`（`init-db.sql`）；订单归因按 `pick_source` / `pick_extra` / `colonel_buyin_id` 反查渠道。  
商品域责任：转链成功时 **必须** 写入 mapping（当前已实现，buyin 未解析时仍写入但 native 类型可能降级）。

### 7. 快速寄样是否仅为入口？

**当前连入口都未接。**  
寄样门槛写在审核 `audit_payload`；寄样申请走 `SampleController` + 独立寄样台，商品详情 **未** 提供「快速寄样」跳转或 API 封装。

### 8. 是否存在个别品负责人覆盖 default_recruiter？

**存在，且与 V1 冻结冲突。**  
- `assignProduct` 设置 `product_operation_state.assignee_id`。  
- `AttributionService` 在归因时读取该字段作为 `colonelUserId`（默认招商链路）。  
- 另：`MerchantController.override-assignee`、独家商家/达人分支亦影响归属（属配置/达人域交叉，非 P-08 字面，但会改变招商结果）。

**V1 验收建议**：在 TASK-P-08 完成前，**不**将「个别品覆盖活动招商」标为通过。

### 9. 同商品 ID 是否只有一条展示中？

**否。**  
唯一约束在 `(activity_id, product_id)`；同一 `product_id` 参与多个活动时，可有多条 `selected_to_library=true`，库列表 **全部展示**。

### 10. 商品域是否越界计算订单、业绩或达人归属？

| 越界项 | 结论 |
|--------|------|
| 订单金额/业绩 | **未越界**（`ProductService` 无相关逻辑） |
| 订单归因 | **间接**：写 mapping；不写订单表 |
| 达人归属 | **未在商品域计算**；转链可带 `talentId` 入 mapping |
