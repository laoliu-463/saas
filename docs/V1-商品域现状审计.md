## 三、十问专项答复（重审 6/1）

### 1. 活动同步是否能落库？

**部分（lazy 落库已实，缺主动写）。**
- **可查**：`GET /colonel/activities` 实时调上游（60s 缓存）。
- **可落库**：查询时 `syncFromGatewayItem` 循环写 `colonelsettlement_activity`（`ColonelsettlementActivityService.java:130-147`）；测试可 `app.activities.seed-demo-on-empty` 注入演示活动；`assignActivity` 时若不存在会 `upsertListActivitySummary`（`ProductService.java:1525`）。
- **缺口**：无「招商组长点同步 → 全量活动写入 DB」的独立写接口；如要全量同步需新增 `POST /colonel/activities/sync` 类端点。

### 2. 活动商品同步是否包含合作方 ID？是否自动入库？

**主链完整，合作方以店铺维度沉淀。**
- 同步：`ProductService.refreshActivitySnapshots:1590-1662` 全量拉上游并 `upsertSnapshotsWithStats:1366-1417`。
- 自动入库：`applyUpstreamProductLibraryDecision:1419-1468` 判断上游推广中 → 置 `selected_to_library=true`、`bizStatus=APPROVED`、`displayStatus=PENDING`。
- 合作方：`product_snapshot.shop_id`/`shop_name` 来自 `DouyinProductGateway.ActivityProductItem`；商品详情可关联 `merchant` 表。
- 领域「合作方 ID」若指独立 partner 主键，当前**无**对应字段与 `PartnerService`。

### 3. 商品状态是否支持 待补充 → 可推广 → 展示中？

**能力等价、命名不同。** 完整映射：

| 需求口径 | 代码口径 |
|----------|----------|
| 待补充/待审核 | `PENDING_AUDIT`（同步后 `initStateIfAbsent`） |
| 审核通过（可推广） | `APPROVED` |
| 已分配招商 | `ASSIGNED` |
| 展示中 | `selected_to_library=true` + `displayStatus=DISPLAYING` |
| 已转链 | `LINKED` |
| 跟进中 | `FOLLOWING` |
| 拒绝 | `REJECTED`（终态） |

状态机合法转换（`ProductBizStatusService.ensureAllowed:258-271`）：
```
PENDING_AUDIT → APPROVED | REJECTED
APPROVED | BOUND | ASSIGNED → ASSIGNED
APPROVED | ASSIGNED → LINKED
LINKED → FOLLOWING
```

入库门槛：`putIntoLibrary` → `requireApprovedAuditForLibraryEntry`。

### 4. 商品库是否支持筛选、排序、置顶？

| 能力 | 结论 |
|------|------|
| 筛选 | ✅ 40+ 参数（见上表 P-04） |
| 排序 | ✅ 4 优先级：置顶 > 投流 > 高佣 > 晚上架（`ProductService.compareLibraryProducts:358-382`）；`sortBy=latest` 走 `selectPageSorted` SQL 级排序（性能优化见 commit `569a263`） |
| 置顶 | ✅ 24h / 每人最多 10 个；`ProductPinService.pin/unpin/expirePinnedProducts`；清理 job 每 10 分钟 |

### 5. 复制讲解是否转链并保存 pick_source → channel？

**是。** `generatePromotionLink` → `douyinPromotionGateway.generateLink` → `promotion_link` + `pick_source_mapping.user_id`（渠道用户）。前置：`selected_to_library=true` 且状态为 `APPROVED`/`ASSIGNED`/`LINKED`。

### 6. pick_source 映射能否被订单域使用？

**是（订单域已接入）。** 表 `pick_source_mapping`（`init-db.sql:515-524` 多组唯一索引）；订单归因按 `pick_source` / `pick_extra` / `colonel_buyin_id` 反查渠道。商品域责任：转链成功时**必须**写入 mapping（当前已实现）。

### 7. 快速寄样是否仅为入口？

**旧路径可用，新 colonel 路径未接。**
- 旧：`ProductController.quickSample:276-284`（`POST /products/{relationId}/quick-sample` 或 `.../quick-sample-apply`）→ `ProductQuickSampleService.applyQuickSample` 完整实装。
- 新：`ColonelActivityProductController` 暂无对应入口；`ProductDetail.vue` 也未跳转。
- **建议**：补 colonel 路径 `POST /colonel/activities/{aid}/products/{pid}/quick-sample`（仅转发至 `SampleController`）。

### 8. 是否存在个别品负责人覆盖 default_recruiter？

**存在，仍与 V1 冻结冲突。**
- `ProductService.assignProduct:2091-2130` 设置 `product_operation_state.assignee_id`。
- `AttributionService.java:160-165` 归因时 `colonelUserId = state.getAssigneeId()`，并在 L187/203/239/255/267/278/288/301 透传给 `AttributionResult.colonelUserId`。
- 此外 `assignActivity` (`ProductService.java:1500-1570`) 在分配活动招商时**级联更新** `product_operation_state.assignee_id`（L1548-1554），即活动层分配会覆盖个别品层。

**V1 验收建议**：在 TASK-P-08 完成前，**不**将「个别品覆盖活动招商」标为通过；建议要么冻结 `assignee_id` 对 `default_recruiter` 的影响（去掉 L165 读取），要么产品确认 `assignee_id` 即活动招商（语义收敛到一处）。

### 9. 同商品 ID 是否只有一条展示中？

**否。** `product_operation_state` 在 `(activity_id, product_id)` 上**仅索引非唯一**（`migrate-all.sql:86`）；同一 `product_id` 参与多个活动时可有多条 `selected_to_library=true`，库列表全部展示。

### 10. 商品域是否越界计算订单、业绩或达人归属？

| 越界项 | 结论 |
|--------|------|
| 订单金额/业绩 | **未越界**（`ProductService` 无相关逻辑） |
| 订单归因 | **间接**：写 `pick_source_mapping` + `assignee_id`；不写订单表 |
| 达人归属 | **未在商品域计算**；转链可带 `talentId` 入 mapping |
