# ADR-010 仓库阶段口径拍板为 V2

> 范围标记：[V2 必做] 阶段口径收口。
> 关联：ADR-002（V1 范围优先级，保留）；本 ADR 不废止 ADR-002。

## 状态

- [V2 必做] 已采纳。后续发现阶段相关冲突继续追加到本文件。

## 背景

- [V2 必做] `AGENTS.md` 第 5 行写"本仓库是抖音团长内部 SaaS V1 工程"，第 20 行写"V1 核心闭环以三条主链为准"。
- [V2 必做] `CLAUDE.md` 第 14 行写"当前默认交付范围以 `docs/01-V2交付范围与边界.md` 为准"，并以 V2 必做条目列举独家达人/商家全量闭环、历史重算、MQ 化事件、外部 `quick_sample_apply`、真实物流自动化及差异化提名为核心架构演进目标。

- [历史归档] 旧 V2.2 完整方案与早期 V1 闭环文档均不作为当前阶段事实。

## 决策

- [V2 必做] 本仓库当前阶段是 **V2**（演进中），不是 V1。
- [V2 必做] 仓库项目定位与"核心闭环"措辞从 V1 改为 V2；ADR 编号与 ADR-002 中的"范围标记"（V1 必做 / V1 简化 / V1 不做 / V2 预留）继续沿用，不重编号。
- [V2 必做] 当前事实优先级（沿用 ADR-002 决策）：
  1. 当前代码和可运行配置
  2. 当前阶段相关 `docs/*.md`（以 `docs/01-V2交付范围与边界.md` 为交付范围主源）
  3. `harness/` 工程化基座与最新 evidence / retro
  4. `AGENTS.md` / `CLAUDE.md` / `CONTEXT.md`
  5. 旧 V2.2 / 历史归档
- [V2 必做] 发现阶段冲突时，先写入本 ADR，**不自行拍板**。
