# ADR-008 活动商品上游推广中状态优先

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 状态

- [V1 必做] 已采纳。活动商品同步、活动商品列表、商品库展示规则 repair 和前端按钮规则按本文执行。

## 背景

- 活动商品页存在同一类商品在上游已是 `status=1/推广中`，但由于历史本地审核拒绝或业务状态拒绝，进入“推广中”筛选后只显示“查看详情/修改”，缺少暂停发布、合作设置、寄样设置等可操作按钮。
- 旧口径把本地拒绝作为优先判断，会让商品自身上游状态与 SaaS 可操作状态断链；这与“主要商品是推广中就自动同步到商品库并且可以进行暂停发布等操作”的业务目标冲突。
- 暂停发布是用户主动的本地发布控制。如果暂停后把活动商品行变成只读，用户将无法恢复发布，暂停动作本身无法闭环。

## 决策

### 1. 上游商品状态是活动商品可操作主来源

- 商品自身上游 `product_snapshot.status=1` 视为 `officialStatus=PROMOTING`。
- 活动商品同步、活动商品列表视图和 `repair-library-state` 使用同一套规则：`status=1` 自动补齐 `selectedToLibrary=true`、`auditStatus=2` 和可操作业务状态。
- 历史本地 `REJECTED` / `auditStatus=3` 不再阻断上游推广中商品进入活动商品可操作链路。
- 非推广中商品不得自动入库，也不得因为活动状态是推广中而被提升为可操作。

### 2. 本地暂停只控制发布，不覆盖上游推广中事实

- `manual_disabled=true` 时，活动商品规范字段为 `publishStatus=PAUSED`，展示“恢复发布”操作。
- 暂停商品保持 `selectedToLibrary=true`，商品库列表隐藏，`displayStatus=HIDDEN`、`hiddenReason=LOCAL_PAUSED`。
- 恢复发布清除 `manual_disabled` 后进入 `displayStatus=PENDING`，再由商品展示规则根据上游推广状态、推广期和去重规则计算是否展示。

### 3. 活动商品列表必须透出规范字段

- `GET /api/colonel/activities/{activityId}/products` 行数据需要输出：
  `officialStatus`、`reviewStatus`、`publishStatus`、`manualDisabled`、`selectedToLibrary`、`displayStatus`、`hiddenReason`。
- 前端筛选和按钮规则优先使用上游数字状态和规范字段；`statusText` 只作为兼容兜底，不能成为唯一判断来源。

### 4. real-pre 历史数据先 dry-run

- real-pre 历史断链数据本轮只执行 `repair-library-state?dryRun=true` 和 `library/health` 巡检。
- 未经 dry-run 结果保存和人工确认，不直接对 real-pre 历史数据写库。

## 影响

- 后端：`ProductService`、`ProductDisplayRuleService` 需要统一同步、视图和 repair 规则。
- 前端：活动商品筛选和 `product-actions` 需要优先读取上游数字状态与规范字段。
- 文档：API 契约、商品域和本文 ADR 共同作为后续 review 依据。

## 验收

- [V1 必做] 前端单测覆盖 `status=1 + auditStatus=3/bizStatus=REJECTED` 仍显示推广中操作。
- [V1 必做] 前端单测覆盖 `status=1 + manualDisabled=true` 显示恢复发布，不退化成只读。
- [V1 必做] 前端单测覆盖“推广中”筛选优先匹配 `status=1`，不依赖 `statusText`。
- [V1 必做] 后端单测覆盖上游推广中自动入库、历史本地拒绝不阻断、本地暂停保持可恢复、repair dry-run 报告历史断链差异。

## 风险与回滚

- 风险：历史本地拒绝的语义从“阻断展示”变为“被上游推广中覆盖”，需要业务侧确认这是活动商品页的新口径。
- 风险：商品库页仍按展示规则隐藏本地暂停商品，活动商品页允许恢复发布；两处页面展示语义不同，review 时需要区分“商品库展示”和“活动商品操作”。
- 回滚：恢复 `ProductService` 和 `ProductDisplayRuleService` 对本地拒绝的优先阻断，并同步还原前端动作规则与本 ADR；回滚前应确认是否会再次出现活动商品页推广中只读问题。
