# ADR-004 业绩域负责最终归属

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 状态

- [V1 必做] 已采纳。

## 背景

- [V1 必做] 业绩归属需要综合订单事实、商品映射、达人关系、配置规则、退款事实和独家覆盖。
- [V1 必做] 这些规则不能散落在订单域、商品域、配置域或分析模块。

## 决策

- [V1 必做] 业绩域负责最终归属、提成、冲正和双轨金额计算。
- [V1 必做] 配置域提供规则参数，业绩域执行规则。
- [V1 必做] 商品域提供 `pick_source_mapping`，订单域提供订单事实。

## 影响

- [V1 必做] 业绩明细和汇总是最终业绩结论来源。
- [V1 必做] 退款和售后事实进入业绩域后产生冲正。
- [V1 不做] 任何其他领域不得写最终提成结论。

## 验证方式

- [V1 必做] 单笔订单从订单事实、商品映射追溯到业绩明细。
- [V1 必做] 单笔退款从退款事实追溯到冲正记录。
- [V1 必做] 配置变化对业绩计算的影响必须可追溯到配置版本。
