# ADR-005 分析模块只读汇总

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 状态

- [V1 必做] 已采纳。

## 背景

- [V1 必做] Dashboard 和报表需要展示业绩、订单、寄样等指标。
- [V1 必做] 如果分析模块重新计算业绩归属，会与业绩域形成双口径。

## 决策

- [V1 必做] 分析模块只读订单、业绩、寄样等事实或汇总表。
- [V1 必做] 分析模块不得重算业绩归属。
- [V1 必做] Dashboard 指标必须能追溯到业绩汇总或只读事实查询。

## 影响

- [V1 必做] 分析模块输出用于展示和导出，不作为业务事实写回。
- [V1 简化] V1 可使用现有 dashboard 字段兼容，但要注明来源。
- [V2 预留] 独立 BI 指标平台可后续设计，但不得改变领域事实来源。

## 验证方式

- [V1 必做] Dashboard API 与汇总表或业绩 API 对账。
- [V1 必做] 搜索分析模块代码，确认不写订单最终归属或业绩明细。
