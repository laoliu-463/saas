# ADR-007 活动列表与商品库入口路由统一

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 状态

- [V1 必做] 已采纳。代码 + 单测 + E2E 三层覆盖。

## 背景

- 活动列表 `frontend/src/views/product/ActivityList.vue` 的「商品信息」操作原本跳转
  `/product/manage/${row.activityId}`（path param，落到「商品推进池」 `index.vue`），
  **而不是商品库**。
- 商品库 `ProductLibrary.vue` 引入了 `useRoute()` 但**没有读取** `route.query.activityId`，
  也没有任何「已按活动筛选」提示；用户从活动列表点过来后，URL 看起来"对"了，但列表里
  没有真正按活动过滤，造成两个入口契约不一致。
- 字段名排查：`activityId` / `sourceActivityId`（来源活动）/ `recruitActivityId`（招商活动）
  三种语义清晰、未混用；未发现 `activity_id` / `promotionId` / `activityCode` 字段。
- 链接字段排查：`product-library-display.ts` 把 `promotionLink` / `promoteLink` 兜底进
  `productUrl`；`baiyingUrl` 兜底含 `productUrl`。这违反了"商品原链 / 百应 / 转链"三种
  链接字段严格分离的语义。

## 决策

### 1. 入口 URL 契约统一为 query 参数

- 活动列表「商品信息」点击：`/product/library?activityId={row.activityId}`。
- 历史入口 `/product/manage/:activityId` 与 `/product/activity/:activityId`
  改为 `redirect` 到 `/product/library?activityId={id}`，避免 path / query 两套入口并存。
- `ProductLibrary.vue` 单一事实源：`route.query.activityId`，onMounted 时写入
  `filters.value.activityId`，`watch` 同步外部 URL 变化。
- 硬筛选语义：进入即生效；"重置筛选" / "清除活动筛选" 会主动 `router.replace` 清掉 query，
  保证 query 与 filters 不漂移。

### 2. 三种链接字段严格分离

| 字段 | 兜底链 | 用途 |
| --- | --- | --- |
| `productUrl` | `productUrl` → `detailUrl` | 上游原始商品链接（无 `pick_source`） |
| `baiyingUrl` | `baiyingUrl` → `baiyingLink` → `buyinUrl` | 百应后台商品链接（仅作为转链失败时的后路展示） |
| `promotionUrl` | `promotionLink` → `promoteLink` → `shortLink` → `promotionUrl` | 真实转链链接，必须带 `pick_source` |

禁止 `productUrl` 兜底 `promotionLink` / `promoteLink`（违反"商品原链不带 pick_source"）。
禁止 `baiyingUrl` 兜底 `productUrl` / `detailUrl`（违反"商品原链 ≠ 百应后台链"）。
`manual-copy.ts:resolveBaiyingUrl` 同步收紧到 `baiyingUrl / baiyingLink / buyinUrl` 三个键。

### 3. UI 反馈

- 商品库页面在 `route.query.activityId` 非空时，渲染「已按活动筛选」提示条
  (`data-testid="product-library-activity-filter-applied"`)，展示当前活动 ID
  并提供"清除筛选"按钮（`data-testid="product-library-activity-filter-clear"`）。

## 风险与回滚

- 风险：把 `/product/manage/:activityId` 从 `index.vue`（活动商品推进工作台）重定向
  到商品库后，**该 path 的"活动商品推进"语义被合并到商品库**。
  缓解：商品库主链路已具备"按活动筛选"和"按转链状态过滤"能力，活动商品推进的核心
  动作（审核 / 入库 / 转链）可由商品库承担；旧的 `index.vue` 工作台仍可通过
  `/product/manage/products` 进入"商品推进池"完成。
- 回滚：把 `/product/manage/:activityId` 还原为 `component: () => import('../views/product/index.vue')`
  即可恢复历史行为；`/product/library?activityId=` 入口可与 `/product/manage/:activityId`
  并存。
