# ADR-002 V1 范围优先级

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 状态

- [V1 必做] 已采纳，后续发现冲突继续追加到本文件。

## 背景

- [历史归档] 仓库存在旧 V2.2 完整方案和多个专项记录，覆盖范围大于当前 V1 交付。
- [V1 必做] 本轮目标是 Harness Engineering 风格文档重构，不重新发明需求，不扩大业务范围。
- [V1 必做] 用户明确要求 V1 范围优先于旧 V2.2 完整方案。

## 决策

- [V1 必做] 当前事实优先级：当前代码和可运行配置 > 当前 V1 文档 > 旧 V2.2 文档。
- [V1 必做] 发现冲突时，不自行猜测业务规则，先记录冲突、证据和待确认问题。
- [V1 必做] 旧 V2.2 中未进入 V1 的能力放入 `V2 预留` 或 `历史归档`。

## 待人工确认

- [V1 必做] `.env.real-pre` 与 `.env.real-pre.example` 的 profile 名称是否统一为 `real-pre`。
- [V1 必做] OpenAPI Spec 是否需要重新从服务端刷新，补齐 2026-05-22 后接口。
- [V1 必做] quick sample 在 V1 中是否只作为内部寄样入口，还是必须对接外部抖店接口。
- [V1 必做] 物流和达人第三方接口在 V1 P0 中的最低通过标准。

## 验证方式

- [V1 必做] 文档结构检查、关键字检查、git diff 检查。
- [V1 必做] 每个冲突必须能追溯到旧文档、当前代码、配置或测试证据。
