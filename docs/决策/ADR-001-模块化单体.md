# ADR-001 模块化单体

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 状态

- [V1 必做] 已采纳。

## 背景

- [V1 必做] 当前代码和运行事实是 Spring Boot 模块化单体，配套 PostgreSQL、Redis、Docker Compose。
- [历史归档] 旧文档中的 FastAPI、Celery、Python 调度或采集方案属于历史背景。
- [V1 必做] V1 目标是先把业务闭环、领域边界、事件、接口和验收证据收口，而不是重做服务拆分。

## 决策

- [V1 必做] V1 采用模块化单体。
- [V1 必做] 领域通过包、服务、事件和数据边界隔离。
- [V1 简化] 事件以 Spring 本地事件和现有 Outbox 机制为主，不强制引入 MQ。
- [V2 预留] 当业务规模和团队协作需要时，再评估拆分服务、MQ 和独立部署。

## 影响

- [V1 必做] 文档、skills、hooks、MCP 和 subagents 均按 Spring Boot 当前事实组织。
- [V1 必做] 验收优先证明模块边界正确、链路闭环和数据可追溯。
- [V1 不做] 不把旧 FastAPI / Celery 写成当前架构事实。

## 验证方式

- [V1 必做] 检查 `backend/src/**`、`docker-compose.*.yml`、`.env.*` 和测试脚本。
- [V1 必做] 文档检索 `FastAPI|Celery|fake-useragent` 时，仅允许出现在归档、历史或冲突说明语境。
