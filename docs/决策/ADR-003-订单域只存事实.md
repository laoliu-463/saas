# ADR-003 订单域只存事实

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 状态

- [V1 必做] 已采纳。

## 背景

- [V1 必做] 订单同步是第三方事实进入系统的关键入口。
- [V1 必做] 如果订单域同时计算提成、独家覆盖和最终归属，会造成事实和业务结论混杂，难以审计和冲正。

## 决策

- [V1 必做] 订单域只保存订单事实、订单明细、退款 / 售后事实和归因输入。
- [V1 必做] 订单域发布订单已同步和退款事实已同步事件。
- [V1 不做] 订单域不计算提成、不应用独家覆盖、不写最终业绩归属。

## 影响

- [V1 必做] `pick_source`、商品、金额、状态、时间等作为输入保存。
- [V1 必做] 业绩域消费订单事实后输出最终归属、提成和冲正。
- [V1 必做] 分析模块读取业绩汇总，不反向计算订单归属。

## 验证方式

- [V1 必做] 抽取一笔订单，检查订单表不包含最终提成结论。
- [V1 必做] 同一订单的业绩结论应在业绩明细或汇总中找到。
- [V1 必做] 退款事实能触发业绩域冲正，而不是订单域直接改提成。
