## 文档说明

本文档基于骨架设计 V1.4 和需求文档 V2.2 设计。

**定位**：管理活动、合作方（商家/团长）、商品及它们的关联关系。

**核心变化**：

- 活动独立为实体，不是商品的附属字段
- 新增「合作方」实体，管理商家/团长信息
- 新增「活动商品关联表」作为核心管理对象
- 商品库按商品ID去重，展示规则完整定义
- 支持个别品单独分配负责人
- 商品库筛选功能（类目、标签、按需检索、其他筛选）
- 商品排序规则（普通规则 + 人为置顶）
- 复制讲解功能（调用转链接口，生成带归因参数的推广链接）
- 快速寄样功能（选择达人、规格、备注，进入寄样域）

**版本状态**：✅ 所有问题已确认，可进入开发

## 一、领域定位

### 1.1 职责

| 职责                    | 说明                                                   |
| ----------------------- | ------------------------------------------------------ |
| 管理活动基础信息        | 从API同步活动，存储活动名称、时间、状态、默认负责人    |
| 管理合作方（商家/团长） | 从API同步合作方信息，存储合作方ID、名称、类型          |
| 管理活动-商品关联       | 一个活动下有哪些商品，每个商品的补充信息、状态、合作方 |
| 管理商品库展示          | 按商品ID去重，决定哪条关联记录在商品库展示             |
| 支持个别品分配          | 在关联表上覆盖负责人，供业绩域使用                     |
| 提供商品库筛选与排序    | 类目筛选、标签筛选、置顶排序                           |
| 提供复制讲解功能        | 调用转链接口，生成带归因参数的推广链接                 |
| 提供快速寄样入口        | 选择达人、规格、备注，进入寄样域                       |
| 提供合作方查询          | 供业绩域判断独家商家，供招商查看对接商家               |

### 1.2 边界

**管**：

- 活动的基础信息（同步、存储、过期判断）
- 合作方的基础信息（同步、存储）
- 活动-商品关联的完整生命周期
- 商品库展示规则（3个月保护期、优势切换）
- 商品补充信息（寄样要求、话术、投流信息、标签等）
- 商品库筛选与排序
- 推广链接转链与归因映射

**不管**：

- 订单归属计算（业绩域管）
- 寄样申请流程（寄样域管）
- 达人归属（达人域管）
- 商品数据从哪来（API对接单独处理）

### 2.4 独家商家规则（基于合作方）

| 项目         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| **判断维度** | 合作方ID + 招商ID                                            |
| **触发条件** | 该招商对接的该合作方产生的服务费 ≥ 该招商个人总服务费的 **70%** |
| **生效周期** | 当月满足 → 整月享受；不满足 → 下月初失效                     |
| **效果**     | 该合作方的所有订单服务费收益归该招商                         |
| **数据来源** | 订单表中的商家ID（合作方ID）                                 |

**注意**：同一合作方可能对应多个商品ID，独家商家是基于合作方ID的，不是基于商品ID。

| ------ | -------------------------------- | ---------- |

| 展示标记 | 含义                           | 渠道是否可见 |

## 三、状态机

### 3.1 活动商品关联的状态流转

```plain
API同步（活动+商品+合作方）
      ↓
┌─────────────────────────────────────────────────────────┐
│  创建关联记录，系统状态 = 待补充                          │
└─────────────────────────────────────────────────────────┘
      ↓
  招商补充信息 + 确认上架
      ↓
┌─────────────────────────────────────────────────────────┐
│  系统状态 = 可推广                                        │
└─────────────────────────────────────────────────────────┘
      ↓
  系统执行商品库展示规则（按商品ID分组竞争）
      ↓
    ┌───────────────┴───────────────┐
    ↓                               ↓
┌──────────────┐               ┌──────────────┐
│  展示标记     │               │  展示标记     │
│  = 展示中     │               │  = 已隐藏    │
└──────────────┘               └──────────────┘
```

### 3.2 状态转换表

| 当前系统状态 | 当前展示标记 | 触发动作          | 新系统状态 | 新展示标记 | 条件               |
| ------------ | ------------ | ----------------- | ---------- | ---------- | ------------------ |
| 待补充       | 待定         | 补充信息+确认上架 | 可推广     | 待定       | 信息完整           |
| 可推广       | 待定         | 执行展示规则      | 可推广     | 展示中     | 被选中             |

| 可推广       | 展示中       | 执行展示规则      | 可推广     | 已隐藏     | 被其他记录覆盖     |
| 可推广       | 展示中       | 活动过期          | 可推广     | 已隐藏     | 关联活动结束       |

## 四、商品库展示规则（核心）

### 4.1 基础条件

一条活动商品关联记录要参与商品库展示竞争，必须同时满足：

- 系统状态 = 可推广
- 抖音状态 = 推广中
- 关联的活动未过期

### 4.2 展示规则总流程

```plain
对于同一个商品ID，收集所有满足基础条件的关联记录
              ↓
         ┌───┴───┐
         ↓       ↓
       0条      1条或多条
         ↓       ↓
     不展示   需要选择展示哪条
                  ↓
             按优先级选择：
              1. 有投流信息的优先
              2. 佣金率高的优先
              3. 上架时间晚的优先
                  ↓
             选中的记录 → 展示标记 = 展示中
             其他记录 → 展示标记 = 已隐藏
                  ↓
             应用排序规则：
              1. 置顶商品优先（按置顶时间倒序）
              2. 普通商品按入库时间倒序
                  ↓
             应用筛选条件：
              用户选择的类目、合作方、标签等
```

### 4.3 3个月保护期规则

同一商品ID，从上架时间（首次有任一关联记录被选中展示的时间）起，3个月内不允许**更换**展示的记录。

| 时间             | 情况                                          |
| ---------------- | --------------------------------------------- |
| 1月1日           | 商品A首次上架（关联记录X展示）                |
| 1月2日 - 3月31日 | 保护期内，新的关联记录Y即使有优势，也不能切换 |
| 4月1日           | 保护期结束                                    |
| 4月2日           | 可以重新执行展示规则，可能切换到关联记录Y     |

### 4.4 优势覆盖规则（保护期内例外）

**在保护期内**，如果新关联记录**有优势**，可以**强制切换**展示，并隐藏旧记录。

**优势判断标准**（满足任一即可）：

| 条件         | 说明                         |
| ------------ | ---------------------------- |
| 佣金率更高   | 新记录佣金 > 旧记录佣金      |
| 服务费率更低 | 新记录服务费 < 旧记录服务费  |
| 有投流信息   | 新记录有投流信息，旧记录没有 |

**优势覆盖后的效果**：

- 新记录展示标记 = 展示中
- 旧记录展示标记 = 已隐藏
- 保护期**不重置**（仍以首次上架时间为准）

### 4.5 活动过期处理

| 场景                           | 处理方式                                                     |
| ------------------------------ | ------------------------------------------------------------ |
| 展示中的记录关联的活动过期     | 该记录展示标记 → 已隐藏，发出「商品已隐藏」事件，原因=活动过期 |
| 同一商品ID下其他记录的活动过期 | 不影响，只要还有可展示的记录即可                             |
| 活动延期后                     | 原记录的关联活动恢复有效，重新执行展示规则判断               |

### 4.6 展示规则触发时机

| 触发时机           | 执行范围                 |
| ------------------ | ------------------------ |
| 招商确认上架       | 该商品ID下的所有关联记录 |
| 定时任务（每小时） | 所有状态为可推广的商品   |
| 抖音状态变更       | 该商品ID下的所有关联记录 |
| 活动延期/过期      | 该活动下的所有关联记录   |
| 保护期到期         | 该商品ID下的所有关联记录 |

### 5.3 招商视角的合作方

## 七、复制讲解功能

### 7.1 功能描述

渠道点击「复制讲解」按钮，系统自动复制**通用模板**内容到剪贴板，**不指定达人**。

模板内容可配置（管理员可在系统管理中配置）：

- 可选择包含哪些字段
- 可增加自定义文案
- 可调整字段顺序

### 7.2 默认模板

```plain
【抖音】{商品标题}
【合作方】{合作方名称}
【售价】{售价}元
【佣金率】{佣金率}%
【推广链接】{转链后的推广链接}
```

### 7.3 可配置字段

| 字段占位符     | 说明                   | 是否默认包含 |
| -------------- | ---------------------- | ------------ |
| `{商品标题}`   | 商品名称               | ✅            |
| `{合作方名称}` | 商家/团长名称          | ✅            |
| `{售价}`       | 商品售价               | ✅            |
| `{佣金率}`     | 佣金率                 | ✅            |
| `{服务费率}`   | 服务费率               | ❌            |
| `{推广链接}`   | 转链后的推广链接       | ✅            |
| `{商品ID}`     | 商品ID                 | ❌            |
| `{活动名称}`   | 活动名称               | ❌            |
| `{招商负责人}` | 招商负责人             | ❌            |
| `{自定义文案}` | 管理员可配置的固定文案 | ❌            |

### 7.4 推广链接转链流程

```plain
渠道点击「复制讲解」
      ↓
系统获取当前登录用户（渠道ID）
      ↓
调用抖音API：buyin.instPickSourceConvert
      ├── product_url: 商品原始链接
      └── pick_extra: channel_{user_id}（限制20字符）
      ↓
抖音返回转链链接
      └── product_url: ...&pick_source={抖音生成的归因码}
      ↓
系统存储映射关系：pick_source → channel_id
      ↓
返回转链后的链接
      ↓
拼接复制内容，写入剪贴板
```

### 7.5 转链接口详情

| 项目           | 值                                                  |
| -------------- | --------------------------------------------------- |
| **接口名称**   | `buyin.instPickSourceConvert`                       |
| **功能**       | 商品选品来源转链                                    |
| **核心作用**   | 将普通商品链接转换为带 `pick_source` 参数的推广链接 |
| **归因有效期** | `pick_source` 参数有效期为 **3个月**                |

**请求参数**：

| 参数名        | 类型   | 必填 | 说明                                               |
| ------------- | ------ | ---- | -------------------------------------------------- |
| `product_url` | String | 是   | 商品原始链接                                       |
| `pick_extra`  | String | 否   | 自定义参数（只允许数字、字母和下划线，限制长度20） |

### 7.6 归因映射表

用于存储 `pick_source` 与渠道的对应关系，供订单同步时使用。

| 字段        | 类型     | 说明                   |
| ----------- | -------- | ---------------------- |
| pick_source | string   | 主键，抖音返回的归因码 |
| channel_id  | string   | 渠道ID                 |
| product_id  | string   | 商品ID                 |
| 创建时间    | datetime |                        |
| 过期时间    | datetime | 3个月后自动过期        |

## 八、快速寄样功能

### 8.1 功能描述

渠道点击「快速寄样」按钮，弹出寄样申请表单，填写完成后进入寄样域。

### 8.2 快速寄样表单

| 字段     | 类型      | 必填 | 说明                                           |
| -------- | --------- | ---- | ---------------------------------------------- |
| 选择达人 | 多选下拉  | 是   | 从该渠道的**私海达人**中选择（自己认领的达人） |
| 商品规格 | 下拉/单选 | 是   | 商品的多规格选项（如有）                       |
| 备注     | 文本框    | 否   | 申请备注，如：加急、特殊要求等                 |

**说明**：

- 达人列表只显示该渠道**自己认领的达人**（状态=保护中或已合作）
- 商品信息从当前商品卡片自动带入
- 达人可多选（一次申请可对多个达人发起寄样）
- 收货地址从达人域自动带入，渠道可修改

### 8.3 快速寄样流程

```plain
渠道点击「快速寄样」
      ↓
弹出寄样申请表单
      ├── 达人列表（私海达人，多选）
      ├── 商品规格（自动带入）
      ├── 收货地址（从达人域自动带入）
      └── 备注
      ↓
渠道填写后提交
      ↓
系统校验：
  ├── 达人是否符合商品寄样门槛
  ├── 寄样限制检查（7天内是否重复申请）
  └── 校验失败则弹窗提醒
      ↓
校验通过 → 创建寄样申请记录
      ↓
发出「寄样已提交」事件
      ↓
通知寄样域处理
      ↓
关闭弹窗，刷新页面
```

## 十一、业务规则汇总

| 规则         | 说明                                                 |
| ------------ | ---------------------------------------------------- |
| 招商确认上架 | 补充信息后需点确认，触发展示规则判断                 |
| 抖音状态检查 | 只有「推广中」才能参与展示竞争                       |
| 活动过期检查 | 活动过期后，关联记录不能展示                         |
| 3个月保护期  | 从首次上架起算，期内不允许切换展示（除非优势覆盖）   |
| 优势覆盖     | 保护期内新记录有优势可强制切换                       |
| 展示优先级   | 投流优先 > 高佣金优先 > 晚上架优先                   |
| 隐藏后恢复   | 保护期结束后重新执行规则，可能切换回旧记录           |
| 负责人覆盖   | 关联表的实际负责人覆盖活动的默认负责人               |
| 置顶规则     | 负责该商品的招商可置顶，上限10个，有效期24小时       |
| 复制讲解     | 调用转链接口，生成带归因参数的推广链接               |
| 快速寄样     | 渠道选择私海达人、规格、备注，进入寄样域             |
| 筛选全局可见 | 所有筛选标签（货品标签、商品标签、其他筛选）全局可见 |
| 合作方同步   | 从活动商品API自动同步商家信息                        |
