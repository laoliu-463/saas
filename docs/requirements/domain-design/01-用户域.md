------

## 用户域 - 完整设计方案（V1.0 正式版）

------

## 文档说明

本文档基于骨架设计 V1.5 和需求文档 V2.2 设计。

**定位**：管理用户身份、角色权限、组织架构（组/部门），为全系统提供「谁能做什么、能看什么数据」。

**原则**：

- 权限在网关或用户域统一校验，业务域接收已解析的数据范围
- 渠道/招商人员主数据由本域维护，供订单域、寄样域下拉使用
- 支持预置角色与自定义角色

**版本状态**：✅ 所有问题已确认，可进入开发

## 一、领域定位

### 1.1 职责

| 职责 | 说明 |
| ---- | ---- |
| 用户账号管理 | 创建、禁用、重置密码 |
| 认证与会话 | 登录、登出、Token 刷新 |
| 角色与权限 | 菜单、操作、数据范围配置 |
| 组织架构 | 部门、招商组、渠道组 |
| 人员主数据 | 渠道/招商/组长/运营列表，供各域下拉 |
| 数据范围解析 | 输出 self/group/all 及具体用户 ID 列表 |

### 1.2 边界

**管**：

- users、roles、groups、departments
- 登录态与权限包
- 操作日志（登录、权限变更，保留90天）

**不管**：

- 业务规则参数（配置域）
- 提成与业绩（业绩域）
- 各业务实体 CRUD

### 1.3 对外提供的能力

| 操作 | 调用方 | 说明 |
| ---- | ------ | ---- |
| 登录/登出 | 前端 | JWT 认证 |
| 获取当前用户 | 前端 | 用户信息 + 权限包 |
| 用户 CRUD | 管理员 | 员工管理 |
| 角色权限配置 | 管理员 | 菜单/操作/数据范围 |
| 查询渠道列表 | 订单域/寄样域/前端 | 下拉筛选 |
| 查询招商列表 | 订单域/寄样域/前端 | 下拉筛选 |
| 查询部门列表 | 各域 | 部门筛选 |
| 解析数据范围 | 各业务域 | 列表查询前注入过滤条件 |
| 校验操作权限 | 各域/网关 | check_permission |

### 1.4 发出的事件

| 事件 | 触发时机 | 携带数据 |
| ---- | -------- | -------- |
| 用户已创建 | 新员工入职 | 用户ID、角色、组别 |
| 用户已禁用 | 离职/停用 | 用户ID |
| 用户组别变更 | 调组 | 用户ID、原组、新组 |
| 角色权限已更新 | 保存权限配置 | 角色ID |

### 1.5 订阅的事件

| 事件 | 来源 | 处理 |
| ---- | ---- | ---- |
| 无 | - | V1.0 用户域不依赖其他领域事件 |

## 二、核心概念

### 2.1 预置角色

| 角色 | role_code | 默认数据范围 | 核心职责 |
| ---- | --------- | ------------ | -------- |
| 管理员 | admin | all | 系统配置、人员管理 |
| 招商组长 | recruiter_lead | group | 活动绑定、商品分配、组内业绩 |
| 招商 | recruiter | self | 商品审核、寄样审核 |
| 渠道组长 | channel_lead | group | 组内渠道管理、组内业绩 |
| 渠道 | channel | self | 达人认领、寄样申请 |
| 运营 | ops | 可配置 | 物流录入、发货 |

支持**自定义角色**（`roles.is_system = false`），管理员可配置菜单与权限。

### 2.2 权限模型（JSONB 存储于 roles 表）

```json
{
  "menus": [
    "product_library",
    "activity_manage",
    "talent_crm",
    "sample",
    "order",
    "data_platform",
    "admin"
  ],
  "operations": {
    "sample": ["create", "audit", "ship", "export"],
    "order": ["view", "export"],
    "talent": ["claim", "import", "tag"],
    "product": ["audit", "pin", "assign"]
  },
  "data_scope": "self"
}
```

| 字段 | 说明 |
| ---- | ---- |
| menus | 可见菜单模块 |
| operations | 各模块操作权限 |
| data_scope | self / group / all |

### 2.3 组织结构

```plain
公司
  └── 部门（department）
        ├── 招商组（recruiter_group）
        │     ├── 招商组长（leader）
        │     └── 招商（members）
        └── 渠道组（channel_group）
              ├── 渠道组长（leader）
              └── 渠道（members）
运营人员可挂部门，不强制归属业务组
```

| 概念 | 说明 |
| ---- | ---- |
| 部门 | 行政/报表维度，如「招商一部」 |
| 招商组 | 招商组长 + 若干招商，用于 group 数据范围 |
| 渠道组 | 渠道组长 + 若干渠道，用于 group 数据范围 |
| 组长 | 组的 leader_id，拥有 group 范围 + 部分管理操作 |

### 2.4 数据范围解析

`get_user_data_scope(user_id)` 返回：

| data_scope | 返回内容 | 业务域用法 |
| ---------- | -------- | ---------- |
| all | 无 ID 限制 | 不过滤 |
| self | `{ user_ids: [当前用户] }` | channel_id / recruiter_id = 自己 |
| group | `{ user_ids: [组内所有成员] }` | IN 组内成员 ID |

**与骨架 5.3 一致**：

| 优先级 | 规则 |
| ------ | ---- |
| 1 | 管理员 → 全部 |
| 2 | 组长 → 本组 |
| 3 | 普通成员 → 仅自己 |

### 2.5 认证方案

| 项目 | 说明 |
| ---- | ---- |
| 协议 | JWT Access Token + Refresh Token |
| Access 有效期 | 2 小时 |
| Refresh 有效期 | 7 天 |
| 密码存储 | bcrypt |
| Refresh 黑名单 | Redis |
| 登录失败 | 5 次锁定 15 分钟（可配置） |

## 三、状态机

### 3.1 用户状态

```plain
待激活 → 正常 → 已禁用
```

| 状态 | 说明 |
| ---- | ---- |
| 待激活 | 新建账号，首次登录需改密（可选） |
| 正常 | 可登录使用 |
| 已禁用 | 不可登录，历史数据保留 |

## 四、业务规则汇总

### 4.1 用户管理规则

| 规则 | 说明 |
| ---- | ---- |
| 用户名唯一 | 登录名不可重复 |
| 禁用不删数据 | 禁用后保留历史关联 |
| 重置密码 | 仅管理员可操作，生成临时密码或邮件（V2.0） |
| 一人一角 | V1.0 每用户一个角色；V2.0 可多角色 |

### 4.2 组长特权（与各域联动）

| 场景 | 规则 |
| ---- | ---- |
| 寄样7天限制 | 组长、管理员不受限（寄样域校验） |
| 数据导出 | 管理员 + 组长可导出 |
| 批量审核寄样 | 招商角色，单次最多100条 |

### 4.3 操作日志

| 项目 | 说明 |
| ---- | ---- |
| 记录范围 | 登录、权限变更、用户创建/禁用 |
| 保留时间 | 90 天 |
| 存储 | `audit_logs` 表，按月分区（可选） |

## 七、对外接口定义

### 7.1 认证

| 接口 | 用途 | 入参 | 出参 | 权限 |
| ---- | ---- | ---- | ---- | ---- |
| `login` | 登录 | username, password | access_token, refresh_token, user | 公开 |
| `logout` | 登出 | refresh_token | success | 登录用户 |
| `refresh_token` | 刷新 | refresh_token | 新 access_token | 登录用户 |
| `get_current_user` | 当前用户 | - | user + permissions | 登录用户 |
| `change_password` | 改密 | old, new | success | 登录用户 |

### 7.2 用户管理

| 接口 | 用途 | 入参 | 出参 | 权限 |
| ---- | ---- | ---- | ---- | ---- |
| `list_users` | 用户列表 | 筛选+分页 | 列表 | 管理员 |
| `create_user` | 创建用户 | 用户信息 | user_id | 管理员 |
| `update_user` | 更新用户 | user_id, 字段 | success | 管理员 |
| `disable_user` | 禁用 | user_id | success | 管理员 |
| `reset_password` | 重置密码 | user_id | 临时密码 | 管理员 |

### 7.3 角色权限

| 接口 | 用途 | 入参 | 出参 | 权限 |
| ---- | ---- | ---- | ---- | ---- |
| `list_roles` | 角色列表 | - | 列表 | 管理员 |
| `get_role` | 角色详情 | role_id | 含 permissions | 管理员 |
| `update_role_permissions` | 更新权限 | role_id, permissions | success | 管理员 |
| `create_custom_role` | 自定义角色 | name, permissions | role_id | 管理员 |

### 7.4 人员主数据（供各域下拉）

| 接口 | 用途 | 入参 | 出参 | 权限 |
| ---- | ---- | ---- | ---- | ---- |
| `list_channels` | 渠道列表 | keyword, page | 渠道用户列表 | 登录用户 |
| `list_recruiters` | 招商列表 | keyword, page | 招商用户列表 | 登录用户 |
| `list_departments` | 部门列表 | - | 部门树/列表 | 登录用户 |
| `list_group_members` | 组员列表 | group_id | 成员列表 | 组长/管理员 |

### 7.5 权限校验

| 接口 | 用途 | 入参 | 出参 | 权限 |
| ---- | ---- | ---- | ---- | ---- |
| `get_user_data_scope` | 数据范围 | user_id | scope + user_ids | 内部/各域 |
| `check_permission` | 操作权限 | user_id, resource, action | bool | 内部/各域 |

## 八、与其他领域的关系

```plain
┌──────────┐
│  用户域  │
└────┬─────┘
     │ list_channels / list_recruiters / get_user_data_scope
     ├──────────────────────────────────────────→ 订单域
     ├──────────────────────────────────────────→ 寄样域
     ├──────────────────────────────────────────→ 达人域（认领人=渠道用户）
     ├──────────────────────────────────────────→ 商品域（负责人=招商用户）
     ├──────────────────────────────────────────→ 业绩域（列表权限）
     └──────────────────────────────────────────→ 配置域（仅管理员可写）
```

| 消费方 | 用途 |
| ------ | ---- |
| 订单域 | 渠道/招商/部门下拉；订单列表数据范围 |
| 寄样域 | 渠道/招商下拉；寄样列表权限 |
| 达人域 | 认领人、标签更新人 = 渠道用户 ID |
| 商品域 | 招商负责人、置顶权限 |
| 业绩域 | 业绩列表/导出权限 |
| 分析模块 | 看板查询前注入 data_scope |
