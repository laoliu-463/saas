# 达人真实数据接入说明

更新时间：2026-05-20

## 目标

V1-P0 已接入真实达人基础资料采集最小闭环：输入抖音号、主页链接或分享链接后，系统按 Provider 链尝试拉取真实基础资料，并写入 `talent` 表（对外文档称 talents）及 `talent_profile_sync_log`。

## Provider 顺序

1. `configurable_http`：读取 `talent.profile.http.*` 配置，对接官方/第三方 HTTP 数据源
2. `public_web`：低频单条公开主页采集，不做 Cookie 池、代理池、批量抓取
3. `manual`：仅用于失败后人工补充，`data_source=manual`

## 接口

- `POST /api/talents/resolve-profile`：解析输入并尝试拉取真实资料
- `POST /api/talents/{id}/sync-profile`：刷新已有达人资料（失败不覆盖旧成功数据）

## 字段策略

- `talentLevel`、`sales30d` 默认标记 `unsupported`，Provider 未真实返回时不得写入默认值
- 禁止伪造：`LV1`、`0` 等默认值不得作为真实资料写入
- 所有采集记录：`data_source`、`sync_status`、`last_sync_time`、`raw_payload`、`talent_profile_sync_log`

## 寄样门槛

当达人 `unsupported_fields` 包含 `talentLevel` 或 `sales30d` 时，寄样资格检查不得自动通过，需提示填写申请原因。

## 配置项

```yaml
talent:
  profile:
    public-web:
      enabled: true
    http:
      enabled: false
      endpoint: ${TALENT_PROFILE_HTTP_ENDPOINT:}
      method: GET
      token: ${TALENT_PROFILE_HTTP_TOKEN:}
      header-authorization: ${TALENT_PROFILE_HTTP_AUTHORIZATION:}
```

## SQL 验收

```sql
select id, douyin_account, talent_uid, sec_uid, nickname, fans_count, like_count, works_count,
       ip_location, talent_level, sales_30d, data_source, sync_status, last_sync_time
from talent
order by update_time desc
limit 20;

select id, talent_id, provider_code, sync_status, unsupported_fields, error_code, error_message,
       started_at, finished_at
from talent_profile_sync_log
order by started_at desc
limit 20;
```
