# 业务闭环总览

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 总体闭环

- [V1 必做] 商品进入商品库后，通过活动商品同步和转链形成可推广商品。
- [V1 必做] 商品域落 `pick_source_mapping`，为订单归因提供事实输入。
- [V1 必做] 达人域维护达人资料、标签、地址和渠道关系。
- [V1 必做] 寄样域管理寄样申请、审批、发货和交作业，并通过订单已同步事件判断交作业完成。
- [V1 必做] 订单域同步订单和退款事实，只保留事实，不计算提成。
- [V1 必做] 业绩域读取订单事实、商品映射、达人关系和配置，计算最终归属、提成、冲正和双轨金额。
- [V1 必做] 分析模块只读汇总结果，用于 dashboard、报表和验收查询。

## 模块边界

- [V1 必做] 前端负责 UI、交互、表单轻量校验、接口调用和错误提示。
- [V1 必做] 后端负责核心业务规则、权限、状态机、幂等、审计和持久化事实。
- [V1 必做] SDK / Gateway 负责第三方接口封装、鉴权、参数转换、签名、错误码适配和响应标准化。
- [V1 必做] 数据库负责持久化事实、约束关键一致性和支撑查询。
- [V1 必做] 测试负责验证核心链路、异常分支、权限边界、数据一致性和回归风险。

## 不进入 V1 的闭环

- [V1 不做] 不要求完整实现旧 V2.2 中所有运营自动化。
- [V1 不做] 不把旧 Python / Celery 定时任务作为当前业务闭环。
- [V2 预留] 更复杂的异步工作流、MQ 编排和跨服务拆分进入后续阶段。
