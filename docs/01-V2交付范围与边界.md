# V2 交付范围与边界

> 范围标记：`必做`、`技术基线`、`历史归档`。

## 必做 (功能开发升级)

- [必做] **独家机制**：业绩域与订单域引入独家达人/独家商家覆盖判定，支持独家代理的优先结算与订单绑定。
- [必做] **履约自动化**：打通外部抖店的 `quick_sample_apply` 接口，引入物流 API 自动跟踪，实现寄样状态的无人化流转与自动确认。
- [必做] **复杂财务与结算**：支持基于达人等级、商品类目、特定活动档期的差异化提成比例配置；支持跨月对账及保证金扣减逻辑。
- [必做] **历史重算**：支持当商品负责人变更或规则调整时，对指定时间区间的历史业绩进行重算与冲正。
- [必做] **数据导出与分析**：开放全站级、跨租户级的数据拉取接口，支持复杂数据大屏与离线分析；完善经营指标矩阵。
- [必做] **原 V1 基础功能保持**：用户域权限控制、商品库同步转链、达人 CRM、基础订单事实同步、寄样基础流转等原 V1 核心链路持续维护。
- [必做] test 和 real-pre 的验收必须有脚本、API/SQL、证据路径，不得使用 mock 数据冒充 real-pre 真实闭环。

## 技术基线 (架构遵循不变)

> 架构层面保持稳定，为业务扩展提供可靠支撑。

- [技术基线] **事件驱动**：继续采用 Spring 本地事件机制，**不引入** 复杂的分布式 MQ（如 RabbitMQ/Kafka）。
- [技术基线] **微服务约束**：维持当前单体模块化（单体架构）划分，**不进行** 跨服务网络级物理拆分。
- [技术基线] **基础设施**：沿用 PostgreSQL、Redis，保持 Docker Compose 为核心部署方式，暂**不引入** K8s 生产级弹性运维治理。
- [技术基线] 第三方接口按 P0 链路优先，缺权限、缺 Token、缺真实样本的分支保留阻塞证据。

## 冲突处理规则

- [必做] 当前代码、可运行配置、测试结果优先于旧文档描述。
- [必做] V2 全新业务能力（如独家、自动物流、复杂提成等）已正式解禁，原 V1 时代的一切“不做”硬性限制均作废。
- [必做] 发现业务规则冲突时，必须写入 [决策/ADR-002-V1范围优先级.md](决策/ADR-002-V1范围优先级.md) 补充决议。
- [历史归档] 已废弃或不符合 V2 最新演进的内容，应统一清理或归入归档目录。

## 验收边界

| 验收项 | 证据要求 | 范围 |
| --- | --- | --- |
| P0 核心链路 | `npm run e2e:v1-p0`、API/SQL 证据、截图或报告 | 必做 |
| real-pre | `npm run e2e:real-pre:p0`、真实 Token、真实接口响应、证据索引 | 必做 |
| 后端回归 | Maven 测试结果或明确未运行原因 | 必做 |
| 前端回归 | 前端构建 / E2E / 浏览器证据 | 必做 |
| 文档同步 | 文件结构、范围标记、规则校验更新 | 必做 |
