## 4. 状态和权限说明

- 官方状态字段：前端统一类型为 `ProductOfficialStatus`，当前兼容后端已有 `status/statusText/bizStatus`。
- 发布状态字段：前端统一类型为 `ProductPublishStatus`，当前通过 `publishStatus/published/selectedToLibrary/promotionLink` 兼容推断。
- 审核状态字段：优先使用 `reviewStatus=PENDING/APPROVED/REJECTED`，字段缺失时兼容 `auditStatus` 与 `bizStatus=PENDING_AUDIT/APPROVED/REJECTED`。
- 推广中口径：当前“推广中”页面状态以本地审核通过 + 已发布为准，不完全等于抖音官方 `officialStatus`。当 `reviewStatus=APPROVED` 且 `publishStatus=PUBLISHED` 时，即使官方状态仍为 `PENDING_REVIEW`，前端也归入推广中操作口径。
- 按钮显示规则：`getProductActions(row, context)` 统一计算，先看 row 上的 `canXXX` 字段；字段缺失时按角色做保守 fallback。
- 权限边界：前端只控制展示和交互，真实权限、状态机、幂等和审计必须由后端校验。

## 4.1 后端状态流转说明

- `POST /api/products/manage/{relationId}/approve`：复用商品审核服务，审核通过后设置本地审核通过、加入商品库，并触发 `ProductDisplayRuleService.applyForProductId(productId)`。
- 展示规则：本次允许本地已审核通过的待官方推广商品进入展示竞争；`officialStatus/status=0` 不再阻止本地商品库展示，但拒绝、终止、到期仍不进入展示。
- `POST /api/products/manage/{relationId}/reject`：拒绝原因必填；拒绝后取消商品库选择并把 `displayStatus` 置为 `HIDDEN`，商品库不可见。
- 多活动同商品展示竞争仍由现有展示规则处理；完整 DDD 拆表、3 个月保护期和优势覆盖规则后续继续审计。
