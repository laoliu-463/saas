## 二、MCP / OAS 事实来源建议

### 1. 推荐优先级

1. **后端已启动时**：直接读 live OpenAPI
2. **后端未启动时**：读仓库内离线 OAS 文件
3. **需要查真实抖音联调辅助接口时**：单独读 `sdk-debug` 分组，不和业务主链混用

### 2. Live / Offline 事实源

- live `business`：
  - `http://localhost:8080/api/v3/api-docs/business`
  - `http://localhost:8081/api/v3/api-docs/business`
- live `sdk-debug`：
  - `http://localhost:8080/api/v3/api-docs/sdk-debug`
  - `http://localhost:8081/api/v3/api-docs/sdk-debug`
- offline：
  - `docs/openapi/openapi-business.json`
  - `docs/openapi/openapi-sdk-debug.json`

### 3. 本轮使用方式

- **MCP / OAS 继续负责接口事实**：
  - 路径
  - 方法
  - 字段
  - 角色注解
  - 请求/响应结构
- **Playwright + TypeScript 负责测试执行**：
  - `tests/e2e/**/*.spec.ts`
  - `runtime/qa/*.cjs`
  - 后续收口到 `runtime/qa/e2e/**`
- **不再规划 Apifox 场景执行**

## 三、接口取舍规则

1. **V1 主矩阵优先业务接口**
   - `/auth/**`
   - `/menus/**`
   - `/users/**`
   - `/roles/**`
   - `/configs/**`
   - `/talents/**`
   - `/samples/**`
   - `/colonel/activities/**`
   - `/orders/**`
   - `/dashboard/**`
   - `/data/**`
2. **`/douyin/**` 只做联调辅助**
   - 用于 Token、上游能力、Webhook、真实回流排查
   - 不直接当成页面主链路验收接口
3. **Deprecated 接口不作为主链**
   - `/products/**` 只留给兼容和回退参考
4. **当前用户上下文不新增需求**
   - 当前没有 `/me`
   - 登录响应 + 菜单树已经是当前接口核对口径
5. **接口存在不等于验收通过**
   - `completed_by_order`
   - `pick_source` 真实归因
   - 默认招商归因
   - 保护期改前改后
   - 地址自动带入
   - 数据范围多账号
   仍必须通过 E2E / DB / 日志 / 真实样本补证

## 六、待核对接口与暂不扩张项

### 1. 待核对接口

- `POST /colonel/activities/{activityId}/sync`
- `POST /colonel/activities/{activityId}/products/sync`

处理方式：**继续挂为前端调用待核对，不纳入当前 E2E 通过判断**。

### 2. 暂不纳入当前主计划

- 达人爬虫增强
- Cookie 池 / 代理池
- 预设标签库
- 独家机制
- 自动物流轨迹增强
- Excel 批量导入物流单号
- 个别品负责人覆盖
- 高级看板 / 高级分析

这些项继续按 V1 简化或 V2 后移口径处理，不回流为当前主验收条件。

## 七、一句话结论

> 当前项目已经具备以 MCP / OAS 为事实底稿、以 Playwright + TypeScript 为执行器的 E2E 落地条件；登录、配置、商品、订单、看板、寄样、达人这些业务接口都可以直接进入脚本化验收，但仍不能把 `/douyin/**` 调试接口、达人爬虫增强或 V2 能力重新拉回 V1 主矩阵。
