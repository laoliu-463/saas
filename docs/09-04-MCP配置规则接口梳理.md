# 09-04-MCP配置规则接口梳理

更新时间：2026-05-18

> 本文档是 [09-MCP接口梳理-第一版](./09-MCP接口梳理-第一版.md) 的第四份拆分稿，专门收口 **系统配置 -> 寄样限制 -> 达人保护期 -> 提成比例 -> 角色数据范围 configured** 链路。  
> 本轮只读取 **Apifox MCP / business 离线 OAS / 前端调用 / 后端源码**，不改业务代码，不跑测试，不把 V2 后移项重新拉回 V1。

## 一、梳理范围

本轮只拆以下 5 组：

1. 系统配置接口
2. 寄样限制配置接口
3. 达人保护期配置接口
4. 招商 / 渠道提成比例配置接口
5. 角色数据范围与运营 `configured` 相关配置

统一边界：

- **主事实源**：`SAAS-business-offline` -> `docs/openapi/openapi-business.json`
- **辅助事实源**：前端 `src/api/sys.ts`、`system/ConfigList.vue`、`system/RoleList.vue`、`system/UserList.vue`；后端 `SysConfigController`、`SysConfigService`、`BusinessRuleConfigService`、`ConfigDefinitionRegistry`、`SampleController`、`SampleLifecycleService`、`TalentService`、`CommissionService`、`AuthService`、`SysRoleService`
- **不纳入主链**：
  - `/douyin/**` 和 `sdk-debug`
  - 独家达人 / 独家商家作为 V1 归因主逻辑
  - 按活动 / 商品 / 人员差异化提成作为 V1 主验收
  - 把“接口存在”直接写成“验收通过”

## 二、当前结论

1. 当前 business OAS 暴露的是通用配置 CRUD：`/configs`、`/configs/{id}`、`/configs/grouped`；没有独立的“寄样限制配置”“达人保护期配置”“提成比例配置”专用接口。
2. 寄样 7 天限制、寄样限制开关、寄样自动关闭天数、寄样默认标准、达人保护期都有 `SystemConfigKeys` 和 `ConfigDefinitionRegistry` 校验，并被 `BusinessRuleConfigService` 消费。
3. 提成比例配置键存在于 `system_config` 种子数据，且 `CommissionService` 会直接读取 `commission.business_default_ratio`、`commission.channel_default_ratio` 并输出到 `GET /dashboard/metrics`。
4. 提成比例不在 `SystemConfigKeys` / `ConfigDefinitionRegistry` 里，因此通用 `/configs` 能读写，但当前缺少配置项专属校验、缓存失效与改前 / 改后专项证据。
5. `commission_config` 表存在，但当前 business OAS 没有独立 `/commission-config` 类接口；V1 只按全局提成比例收口，不把差异化提成拉回 V1。
6. 7 天寄样限制豁免当前不是配置项，而是代码硬编码：`ADMIN` 与 `CHANNEL_LEADER` 豁免。若“组长”需泛化到招商组长，仍需业务确认。
7. `ops_staff` 初始化 `data_scope=1`，且 `AuthService` 登录 / 刷新保持角色配置数据范围；寄样发货场景通过接口角色判断获得全量处理能力。
8. `RoleList.vue` 当前只提供 `1(self)` 与 `3(all)`，不提供 `2(group)` 选择；`UserList.vue` 已支持多角色选择并提交完整 `roleIds[]`。

## 三、状态标记

| 状态 | 含义 |
|---|---|
| `已确认主链` | 已在当前 business OAS、controller / service 与前端入口或业务消费点中形成主口径 |
| `待核对` | 当前有实现迹象，但还不足以直接写成主链已闭环 |
| `辅助接口` | 对排查、配置或补证有用，但不是规则配置主入口 |
| `V1 不采用` | 当前 V1 不以该接口 / 能力作为主链验收口径 |

| 提成比例 | `PUT /configs/{id}` 修改 `commission.*_default_ratio` -> `GET /dashboard/metrics` | `bizCommission/channelCommission` 与配置比例和订单聚合手算一致 | 验证 P0-7 |
