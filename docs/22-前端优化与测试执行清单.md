## 0. 全局基线与统一规范

| 项 | 约定 |
|---|---|
| 分页默认 | `pageSize=20`，最大 100；`page=1` 起步 |
| 列表 P95 | < 500ms（后端），前端骨架 >= 200ms 才显示，避免闪烁 |
| 防抖 | 关键字搜索 300ms、远程下拉 250ms、滚动加载 80ms |
| 节流 | 窗口 resize / 滚动指标计算 100ms |
| 短 TTL 缓存 | Card/Trend 后端 Redis 60s；前端列表 `keepAlive` <= 30s |
| Toast | `useMessage()`：success/info/warning/error 四级；致命错误改 `useDialog().error()` |
| 加载占位 | 列表用 `n-skeleton text :repeat`；卡片用骨架卡（已存在 `metric-cards skeleton-card`） |
| 空态 | 统一 `PageEmpty.vue`，必须给次级操作按钮（重置筛选 / 重新加载） |
| testid | 全部统一前缀：`<module>-<element>-<action>`，已用 `data-testid` |
| 鉴权过滤 | 路由层 `meta.roles` + 列表项二次过滤（角色 + DataScope） |
| 错误码 | `error?.response?.data?.msg || error.message || '默认中文兜底'` |

E2E 公共契约：

- 角色 storage state：`helpers/test-data.ts -> storageStates.{admin,bizLeader,bizStaff,channelLeader,channelStaffOps,opsStaff}`
- 选择器：`helpers/selectors.ts -> testIds.*`
- 截图归档：`helpers/screenshot.ts -> capturePage(page, testInfo, name)`
- 数据种子：`helpers/api-assertions.ts -> seedTestData()`

## 1. 商品域

### 1.1 商品库 `ProductLibrary.vue`

路由：`/product`、`/product/library`
角色可见：`BIZ_LEADER / BIZ_STAFF / CHANNEL_LEADER / CHANNEL_STAFF`

| 关键组件 | 文件 |
|---|---|
| 商品卡片 | `views/product/components/ProductCard.vue` |
| 筛选区 | `views/product/components/ProductFilters.vue` |
| 详情抽屉 | `views/product/ProductDetail.vue` |
| 操作日志抽屉 | `views/product/components/ProductOperationLogDrawer.vue` |

#### 加载商品库

- 动作：进入商品库或点击刷新。
- 状态前后：`loading=false` -> `loading=true` -> `products.length>0` 或 `PageEmpty`。
- API：`GET /api/products?page=1&size=12&...filters`（`api/product.ts -> getProducts`）。
- 前端可视化反馈：`n-spin` 全屏遮罩 + 卡片 skeleton；右上角“刷新商品”按钮 loading。
- 测试用例：商品库 >= 1 条时 `productGrid` 可见；空列表时 `PageEmpty` 标题为“暂无商品数据”。
- Mock 数据：固定 24 条样本，含 `cover/title/shopName/sales30d/gmv30d/bizStatus(LINKED/APPROVED/PENDING_AUDIT)/selectedToLibrary=true`。

#### 关键字 + 多维筛选

- 动作：调整 `商品名/ID`、`shopKeyword`、`categoryName`、`systemTag`、`commission`、`salesRange`、`allianceStatus`、`promotionLink`、`hasSample`。
- 状态前后：筛选条件变化 -> 300ms 防抖 -> 重新 `fetchProducts(true)`。
- API：`GET /api/products`。
- 前端可视化反馈：筛选行不必整体 loading；搜索按钮绑定 `:loading="loading"`；选项接口失败时下拉 `loading=false` 并显示错误 tag。
- 测试用例：键入“水”后 300ms 内只触发 1 次 `/api/products` 请求。
- Mock 数据：`/api/products` 按 `keyword` 模糊过滤，`size=12` 强制截断。

#### 加载更多 / 分页

- 动作：点击加载更多。
- 状态前后：`hasMore=true` -> `loadingMore=true` -> 追加列表 -> `loadingMore=false`。
- API：`GET /api/products?page=N&size=12`。
- 前端可视化反馈：底部显示“已加载 X / Y 件”；`loadingMore` 时按钮 loading；到底显示“已全部加载”。
- 测试用例：连续点击直到 `no-more-inline` 出现。
- Mock 数据：分页响应 `{records, total}`，`total > 12*N` 直到第 N+1 页返回空。

#### 复制专属推广链接

- 动作：渠道角色点击复制推广链接。
- 状态前后：`promotion.status=PENDING|FAILED|SUCCESS` -> 调用 -> 更新该卡片 `promotion`。
- API：`POST /api/colonel/activities/{activityId}/products/{productId}/promotion-links`，`scene=PRODUCT_DETAIL`。
- 前端可视化反馈：卡片按钮 loading；成功 `message.success('转链成功，短链：...')`；写入剪贴板；失败 `message.error`。
- 测试用例：`CHANNEL_STAFF` 可见 `product-copy-link`；`BIZ_LEADER` `toHaveCount(0)`。
- Mock 数据：成功返回 `{shortLink:"https://v.douyin.com/abc/", mappingId, pickSource, createdAt}`；失败返回 `{code:"PROMOTION_FAILED"}` 并显示“链接生成失败”红 tag。

#### 置顶 / 投流 / 状态标识

- 动作：根据商品状态渲染卡片 tag 与角标。
- 状态前后：商品数据进入卡片 -> `cardTags(item)` 输出状态标签。
- 触发事件·API：商品列表和详情接口返回状态字段。
- 前端可视化反馈：
  - `bizStatus === 'PENDING_AUDIT'` -> 黄 badge “待审核”
  - `['LINKED','FOLLOWING']` -> 绿 badge “已转链”
  - `REJECTED` -> 红 badge “审核拒绝”
  - `activityExpired` -> “活动过期”
  - `alertTags.includes('库存不足')` -> “库存不足”
  - `!hasMaterial` -> “缺少话术素材”
  - `!assigneeName` -> “待分配负责人”
  - `selectedToLibrary` -> “商品库可见”
  - `supportsAds === true` -> `n-tag type=info round` “可投流”
  - `pinned === true` -> `card-image` 右上角红色置顶徽标
- 测试用例：状态机覆盖 6 状态各 1 条样本并截图回归。
- Mock 数据：`multi.json` 覆盖 6 个 `bizStatus`，含过期、库存不足、可投流、置顶样本。

### 1.5 复制讲解 + 快速寄样

#### 复制讲解

数据来源：`auditSupplement` + `promotionMaterialPack`。

模板：

```text
【商品】{title}（{shopName}）
【售价】{priceText}  【佣金率】{activityCosRatioText}  【近30天】{sales30d}
```

- 动作：一键复制讲解 + 短链。

- 测试用例：mock 转链 200 / 500 两套；断言剪贴板内容。
- Mock 数据：`auditSupplement` 全字段非空 / 半空 / 全空三套样本。

  - 成功 toast “寄样申请已提交，状态为待审核”。
  - 远程下拉空结果显示“暂无匹配商品”。
- 校验：`productId`、`talentId`、`receiverName`、`receiverPhone`、`receiverAddress`、`reason` 必填；手机号建议 `/^1\d{10}$/`。
- 测试用例：选商品 + 选达人 + 填必填后提交；`productId` 缺失时表单校验拦截。
- Mock 数据：
  - `/api/samples/product-candidates` 返回 5 条 `{id, productName, productId}`。

## 8. 跨模块共性优化

### 8.1 加载占位 / 骨架屏

| 场景 | 策略 |
|---|---|
| 列表首次加载 | 200ms 阈值后显示 `n-skeleton`，避免闪烁 |
| 卡片网格 | 6~12 个骨架卡 |
| 抽屉 / Modal 内 | `n-spin :show="detailLoading"` 包裹 |
| 异步下拉 | `<n-select loading>` |
| 表格 | `<n-data-table loading>` |

### 8.2 防抖 / 节流

| 场景 | 类型 | 时长 |
|---|---|---|
| 关键字输入 -> 远程搜索 | debounce | 300ms |
| select remote search | debounce | 250ms |
| 滚动加载更多 | throttle | 80ms |
| 窗口 resize | throttle | 100ms |

实现建议：统一在 `utils/debounce.ts` 提供 `useDebouncedRef` / `useThrottledFn`。

### 8.3 Toast / 错误提示

统一兜底：

- success：操作成功后 1.5s 自动关闭。
- info：纯信息提示，3s。
- warning：业务校验拦截，4s，可手动关闭。
- error：接口失败 / 致命，5s + 复制 traceId 按钮。

错误信息提取：

```typescript
const msg = err?.response?.data?.msg
  || err?.response?.data?.message
  || err?.message
  || '请求失败，请稍后再试'
message.error(msg)
```

### 8.4 缓存策略

| 层级 | 数据 | TTL | 失效 |
|---|---|---|---|
| 后端 Redis | dashboard summary / metrics | 60s | 订单同步事件触发 invalidate |
| 后端 Redis | 商品库列表 | 60s | 审核/分配/转链事件 invalidate |
| 前端内存 | 系统配置 `getConfigGrouped` | 整个会话 | 用户主动刷新 |
| 前端内存 | 角色 `getRoleAll` / 部门 `getDeptTree` | 整个会话 | 系统管理页修改后强制刷新 |
| 前端 sessionStorage | 当前用户 + roleCodes | session | 登出清空 |

### 8.5 性能基线

| 指标 | 目标 | 验证 |
|---|---|---|
| 路由切换 P95 | < 200ms | `router.afterEach` 已埋点 `[router timing] afterEach durationMs` |
| 列表首屏 P95 | < 500ms | 后端契约；前端打点 `performance.now()` |
| 商品卡片网格渲染 | 12 张 < 80ms | 用 `requestAnimationFrame` 测帧 |
| 表格分页 | 默认 20，最大 100 | 全局 `pageSize` 上限护栏 |
| Dashboard 双轨更新 | < 30s | 订单同步事件触发 invalidate + 前端 30s 内可手动刷新 |

### 8.6 异常场景统一兜底

| 场景 | 表现 | 处理 |
|---|---|---|
| 接口 401 | 自动登出 | `request.ts` 拦截 -> `authStore.logout()` -> 跳 `/login` |
| 接口 403 | toast “无操作权限” | 不跳页 |
| 接口 5xx | 错误兜底 + traceId | toast + 列表空态保持上次数据 |
| 接口超时 30s | abort | 显示“网络较慢，请重试” + 重试按钮 |
| 空列表 | `PageEmpty` 组件 | 必带次级 CTA |
| 校验失败 | 表单红字 + `message.warning` | 不调接口 |
