# 07-Test 全链路验收

更新时间：2026-04-27

## 一、验收目标

确保在 `test` 环境下，当前系统能稳定完成：

1. 演示数据初始化
2. 商品转链
3. 订单归因与订单排查
4. 寄样履约闭环
5. 达人 CRM 查看与认领

## 二、验收前提

- 后端以 `test` profile 启动
- 前端开发环境可正常访问
- 登录账号：`admin / admin123`
- 调试台页面：`/dev/test`
- 接口统一前缀：`/api`

## 三、推荐验收路径

### 1. 初始化 Test 数据

- [ ] 登录系统
- [ ] 打开 `/dev/test`
- [ ] 先执行 `reset`
- [ ] 再执行 `seed`
- [ ] 页面无报错，商品、达人、寄样、订单基础数据已生成

### 2. 商品转链闭环

- [ ] 打开 `/product`
- [ ] 查看商品列表与商品详情可正常打开
- [ ] 选择一个商品执行“复制推广链接”
- [ ] 页面提示复制成功
- [ ] 后续可用于生成已归因订单

### 3. 订单归因闭环

- [ ] 回到 `/dev/test`
- [ ] 执行 `generate-attributed`
- [ ] 可选执行 `generate-no-pick-source`
- [ ] 可选执行 `generate-missing-mapping`
- [ ] 打开 `/orders`
- [ ] 已归因、未归因列表可正常展示
- [ ] 订单详情可看到归因信息、商品信息、达人信息、寄样关联和诊断建议

### 4. Dashboard 看板闭环

- [ ] 打开 `/dashboard`
- [ ] 订单数、已归因订单数、未归因订单数、GMV、服务费可正常展示
- [ ] 点击指标后可跳转 `/orders`
- [ ] 跳转后筛选状态保持正确

### 5. 寄样履约闭环

- [ ] 打开 `/sample`
- [ ] 查看已存在寄样单
- [ ] 对待发货寄样执行模拟发货
- [ ] 对快递中寄样执行模拟签收
- [ ] 命中已归因订单后，寄样可自动进入完成态

### 6. 达人 CRM 闭环

- [ ] 打开 `/talent`
- [ ] 达人列表可正常展示
- [ ] 公海 / 私海状态可区分
- [ ] 可认领公海达人
- [ ] 可释放自己认领的达人
- [ ] 达人详情可查看关联寄样记录与订单产出

## 四、通过标准

- [ ] 上述路径全部无阻塞报错
- [ ] 页面状态变化符合预期
- [ ] `/product`、`/orders`、`/dashboard`、`/sample`、`/talent` 均可正常访问
- [ ] `backend mvn test` 通过
- [ ] `frontend npm.cmd run build` 通过

当前后端测试基线：

- `394 tests, 0 failures, 0 errors`

## 五、补充说明

- `/api/test/*` 需要登录态，未登录返回 `401` 属于正常行为
- `test` 环境是当前演示、回归和联调前基线，不应被真实 SDK 联调改坏
