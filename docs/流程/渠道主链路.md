# 渠道主链路

> 模板来源：`.claude/templates/业务流程模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 流程目标

- [V1 必做] 渠道用户能在自身数据范围内查看商品、转链、跟进达人、申请寄样、查看订单和业绩。
- [V1 必做] 全流程必须受用户域 `self/group/all` 数据范围约束。

## 参与角色

- [V1 必做] 普通渠道、渠道主管、管理员。
- [V1 必做] 商品域、达人域、寄样域、订单域、业绩域、分析模块、用户域。

## 前置条件

- [V1 必做] 用户已登录，用户域可解析当前角色和数据范围。
- [V1 必做] 商品库或活动商品已有可推广商品。
- [V1 必做] real-pre 场景下第三方 Token 和环境开关已通过预检。

## 主流程

1. [V1 必做] 渠道登录系统，用户域返回当前用户和数据范围。
2. [V1 必做] 渠道按范围查看商品列表。
3. [V1 必做] 渠道对商品执行转链，商品域落 `pick_source_mapping`。
4. [V1 必做] 渠道维护达人资料、跟进或寄样申请。
5. [V1 必做] 订单域同步订单事实，保留 `pick_source` 等归因输入。
6. [V1 必做] 业绩域根据订单事实、映射、达人关系和配置计算最终归属。
7. [V1 必做] 分析模块只读汇总，展示渠道业绩。

## 异常与分支

- [V1 必做] 转链失败时必须保留第三方错误码和请求上下文。
- [V1 必做] 订单未回流时不能伪造业绩结论，只能标记待同步或阻塞。
- [V1 简化] 缺达人外部信息时可使用系统内达人资料，但需标记来源。
- [V1 不做] 前端不得绕过后端权限直接展示他人订单或业绩。

## 数据与事件

- [V1 必做] 关键数据：用户数据范围、商品、`pick_source_mapping`、达人、寄样、订单事实、业绩明细、业绩汇总。
- [V1 必做] 关键事件：商品转链已完成、订单已同步、业绩归属已计算、业绩汇总已刷新。

## 验收证据

- [V1 必做] `npm run e2e:v1-p0` 覆盖渠道主链路。
- [V1 必做] API/SQL 证明不同账号的数据范围差异。
- [V1 必做] 转链记录、订单事实、业绩汇总可串联。
