# 寄样交作业链路

> 模板来源：`.claude/templates/业务流程模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 流程目标

- [V1 必做] 寄样从申请、审批、发货到交作业完成形成可验证闭环。
- [V1 必做] 交作业完成以订单已同步事件为核心证据。

## 参与角色

- [V1 必做] 渠道、招商、管理员。
- [V1 必做] 寄样域、达人域、商品域、订单域、分析模块。

## 前置条件

- [V1 必做] 商品和达人资料可用。
- [V1 必做] 寄样申请入口可用。
- [V1 必做] 订单域可发布订单已同步事件。

## 主流程

1. [V1 必做] 渠道或招商基于商品和达人发起寄样申请。
2. [V1 必做] 管理端审批寄样申请。
3. [V1 必做] 管理端发货并记录物流信息。
4. [V1 必做] 订单域同步相关订单事实。
5. [V1 必做] 寄样域消费订单已同步事件，判断交作业完成。
6. [V1 必做] 分析模块读取寄样状态和汇总展示。

## 异常与分支

- [V1 必做] 没有订单同步证据时，不得把交作业标为真实完成。
- [V1 简化] 物流接口不可用时可用手工物流事实，但需保留来源。
- [V2 预留] 自动催办、样品库存、异常赔付后续设计。

## 数据与事件

- [V1 必做] 关键数据：寄样申请、达人地址、物流单、订单事实、寄样状态。
- [V1 必做] 关键事件：寄样状态已变更、订单已同步。

## 验收证据

- [V1 必做] E2E 覆盖申请、审批、发货、订单回流后的状态变化。
- [V1 必做] SQL/API 能证明状态变化来源于订单已同步事件。
