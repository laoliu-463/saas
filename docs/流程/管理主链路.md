# 管理主链路

> 模板来源：`.claude/templates/业务流程模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 流程目标

- [V1 必做] 管理员能配置规则、管理用户权限、查看审计和验收关键看板。
- [V1 必做] 管理动作必须留下可追溯证据。

## 参与角色

- [V1 必做] 管理员、主管、运维。
- [V1 必做] 用户域、配置域、分析模块、各业务域审计入口。

## 前置条件

- [V1 必做] 管理员账号可登录。
- [V1 必做] 用户域可提供 `all` 或授权范围。
- [V1 必做] 配置域和操作日志可查询。

## 主流程

1. [V1 必做] 管理员登录并获取管理菜单。
2. [V1 必做] 管理员维护用户、角色、组织和菜单。
3. [V1 必做] 管理员维护配置和规则参数。
4. [V1 必做] 配置域记录配置变更日志。
5. [V1 必做] 管理员查看订单、业绩、寄样和 dashboard 汇总。
6. [V1 必做] 管理员导出或记录验收证据。

## 异常与分支

- [V1 必做] 低权限账号尝试管理操作时必须被后端拒绝。
- [V1 必做] 配置变更失败必须返回明确错误，不产生半更新。
- [V1 简化] 操作日志以关键操作覆盖为主，完整审计湖进入 V2。

## 数据与事件

- [V1 必做] 关键数据：用户、角色、菜单、配置、配置变更日志、操作日志、汇总表。
- [V1 必做] 关键事件：用户权限已变更、配置已变更、业绩汇总已刷新。

## 验收证据

- [V1 必做] 管理员和普通用户同页面权限差异可通过 E2E 证明。
- [V1 必做] 配置变更有 API 响应、数据库记录和操作日志。
