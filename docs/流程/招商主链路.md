# 招商主链路

> 模板来源：`.claude/templates/业务流程模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 流程目标

- [V1 必做] 招商角色能维护活动商品、达人合作、寄样推进和订单转化证据。
- [V1 必做] 招商链路输出的商品、达人、寄样、订单事实能被业绩域消费。

## 参与角色

- [V1 必做] 招商、渠道、管理员。
- [V1 必做] 商品域、达人域、寄样域、订单域、业绩域、配置域。

## 前置条件

- [V1 必做] 活动商品或商品库可同步。
- [V1 必做] 达人资料可维护或从第三方补充。
- [V1 必做] 寄样审批和发货权限已配置。

## 主流程

1. [V1 必做] 招商同步或维护活动商品。
2. [V1 必做] 招商维护达人资料、标签、地址和跟进记录。
3. [V1 必做] 招商或渠道发起寄样申请。
4. [V1 必做] 管理端完成审批和发货。
5. [V1 必做] 商品转链形成 `pick_source_mapping`。
6. [V1 必做] 订单域同步订单事实。
7. [V1 必做] 业绩域计算最终归属和提成。

## 异常与分支

- [V1 必做] 活动商品为空时记录第三方响应，不伪造商品。
- [V1 必做] 达人接口无权限时记录 BLOCKED 证据。
- [V1 简化] 物流接口缺真实样本时允许手工录入物流事实，但必须标记来源。
- [V2 预留] 自动招商、达人评分和批量触达后续设计。

## 数据与事件

- [V1 必做] 关键数据：活动商品、达人资料、达人地址、寄样申请、映射、订单事实、业绩结果。
- [V1 必做] 关键事件：商品已同步、达人资料已更新、寄样状态已变更、订单已同步。

## 验收证据

- [V1 必做] 商品同步日志和商品表可查。
- [V1 必做] 达人、寄样、订单、业绩之间能通过 API/SQL 串联。
- [V1 必做] real-pre 对接项必须落入 [../验收/验收证据索引.md](../验收/验收证据索引.md)。
