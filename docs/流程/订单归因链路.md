# 订单归因链路

> 模板来源：`.claude/templates/业务流程模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 流程目标

- [V1 必做] 从商品转链到订单同步再到业绩归属形成可审计链路。
- [V1 必做] 订单域只存事实，业绩域输出最终归属。

## 参与角色

- [V1 必做] 商品域、订单域、业绩域、配置域、达人域、用户域。

## 前置条件

- [V1 必做] 商品域已生成 `pick_source_mapping`。
- [V1 必做] 订单域能同步到包含归因输入的订单事实。
- [V1 必做] 业绩域可读取配置、达人关系和订单事实。

## 主流程

1. [V1 必做] 商品域调用转链接口并落 `pick_source_mapping`。
2. [V1 必做] 订单域同步订单事实，保存订单金额、商品、时间、`pick_source` 等输入。
3. [V1 必做] 订单域发布订单已同步事件。
4. [V1 必做] 业绩域读取订单事实、商品映射、达人关系和配置。
5. [V1 必做] 业绩域应用最终归属和独家覆盖规则。
6. [V1 必做] 业绩域写入业绩明细和汇总。
7. [V1 必做] 分析模块读取汇总展示，不重算归属。

## 异常与分支

- [V1 必做] 找不到 `pick_source_mapping` 时，订单域仍保存订单事实，业绩域将归属标记为待处理或异常。
- [V1 必做] 退款事实由订单域保存，业绩域负责冲正。
- [V1 不做] 订单域不得在同步时直接计算提成。
- [V1 不做] 分析模块不得重新扫描订单生成最终归属。

## 数据与事件

- [V1 必做] 关键数据：`pick_source_mapping`、订单事实、退款事实、达人关系、配置版本、业绩明细、业绩汇总。
- [V1 必做] 关键事件：商品转链已完成、订单已同步、退款事实已同步、业绩归属已计算。

## 验收证据

- [V1 必做] 选定订单能追溯到商品转链映射和最终业绩明细。
- [V1 必做] 退款订单能看到订单事实和业绩冲正结果。
- [V1 必做] 证据写入 [../验收/验收证据索引.md](../验收/验收证据索引.md)。
