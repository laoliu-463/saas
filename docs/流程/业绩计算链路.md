# 业绩计算链路

> 模板来源：`.claude/templates/业务流程模板.md`。范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 流程目标

- [V1 必做] 基于订单事实、商品映射、达人关系和配置，计算最终归属、提成、冲正和双轨金额。
- [V1 必做] 输出业绩明细和汇总，供分析模块只读。

## 参与角色

- [V1 必做] 业绩域、订单域、商品域、达人域、配置域、分析模块。

## 前置条件

- [V1 必做] 订单事实已同步。
- [V1 必做] `pick_source_mapping` 可追溯。
- [V1 必做] 规则配置可读取。

## 主流程

1. [V1 必做] 业绩域消费订单已同步事件或按任务扫描待计算订单事实。
2. [V1 必做] 业绩域读取商品映射、达人关系和配置版本。
3. [V1 必做] 业绩域执行最终归属和独家覆盖规则。
4. [V1 必做] 业绩域计算提成、毛利、双轨金额。
5. [V1 必做] 业绩域处理退款 / 售后事实产生冲正。
6. [V1 必做] 业绩域刷新汇总表。
7. [V1 必做] 分析模块读取汇总并展示。

## 异常与分支

- [V1 必做] 输入缺失时写入待处理或异常原因，不静默吞掉订单。
- [V1 必做] 规则配置缺失时返回可追踪错误。
- [V1 不做] 订单域、配置域、分析模块不得代替业绩域完成最终计算。
- [V2 预留] 复杂结算、账期锁定、跨平台财务结算后续设计。

## 数据与事件

- [V1 必做] 关键数据：订单事实、退款事实、`pick_source_mapping`、配置版本、业绩明细、汇总表。
- [V1 必做] 关键事件：订单已同步、退款事实已同步、业绩归属已计算、业绩汇总已刷新。

## 验收证据

- [V1 必做] 单笔订单可从订单事实追溯到业绩明细。
- [V1 必做] 单笔退款可追溯到冲正结果。
- [V1 必做] dashboard 汇总能与业绩汇总表对账。
