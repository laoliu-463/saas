# Skill: frontend-ux

## 使用场景

用于前端页面、表单、状态展示、错误提示、权限按钮和业务操作路径优化。

## 必读文件

- `docs/02-业务闭环总览.md`
- `docs/05-API契约总表.md`
- 当前页面所属领域合同和流程文档
- `docs/09-测试验收总览.md`

## 禁止事项

- 禁止前端直接调用第三方开放接口。
- 禁止前端硬编码核心业务规则。
- 禁止只隐藏按钮替代后端权限校验。
- 禁止让页面 fallback 掩盖后端错误或业务状态缺失。

## 标准流程

1. 明确用户角色、页面路径和业务动作。
2. 查接口契约和后端权限规则。
3. 确认前端只做展示、交互、轻量校验和错误提示。
4. 验证请求、响应、错误码和空状态。
5. 检查桌面 / 移动端布局是否有遮挡、溢出和不可点击区域。
6. 用 Playwright 或浏览器截图记录证据。

## 验证方式

- 前端构建。
- Vitest / Playwright。
- 浏览器 Console 和 Network。
- 页面截图。
- 后端 API 响应。
