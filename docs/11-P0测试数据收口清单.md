# 11-P0 测试数据收口清单

更新时间：2026-04-21

## 一、收口目标

确保在 `real` 环境上线前，`test` 环境中的 P0 级核心业务数据能够达到“演示级”拟真度。

## 二、数据准备清单

### 1. 拟真商品池 (Realistic Product Pool)
- [ ] 覆盖 5 个以上主流类目（美妆、服饰、食品、数码、家居）。
- [ ] 包含 20 个以上带有真实图片 URL 和真实标题的商品。
- [ ] 设置阶梯佣金与固定佣金的对比数据。

### 2. 拟真达人库 (Realistic Talent Pool)
- [ ] 预置 10 名以上具有真实抖音 UID 格式的达人。
- [ ] 粉丝数分布覆盖 KOC (1w-10w) 到大 V (100w+)。
- [ ] 补全达人画像：带货分、垂直类目标签、所属 MCN。

### 3. 典型订单案例 (Typical Order Cases)
- [ ] **归因成功案例**：订单包含正确的 `pick_source`。
- [ ] **标识缺失案例**：模拟从抖音自然进入下单，无归因参数。
- [ ] **映射丢失案例**：模拟链接已失效或 mapping 被删除后的下单。
- [ ] **跨期订单案例**：模拟在商品分配负责人变更前后的下单归因。

### 4. 寄样闭环数据
- [ ] 预置不同快递公司（顺丰、中通）的模拟物流单号。
- [ ] 预置已签收且已产生归因订单的寄样记录。

## 三、验证方法

1. 执行 `POST /api/test/seed` 重新铺数。
2. 登录前端，检查“商品库”是否显示真实的商品图文。
3. 执行订单模拟，检查 Dashboard 中的归因率是否符合预期（例如：模拟 100 笔订单，80 笔归因成功，20 笔进入排查池）。

## 四、执行负责人

- **后端实现**：`TestDataService` 逻辑更新。
- **数据采集**：提供拟真 JSON 数据包。
