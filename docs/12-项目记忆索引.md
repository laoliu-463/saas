## 当前项目记忆

- 当前阶段是 V2 演进中，V1 基础链路继续维护；旧 V1 “不做”限制不再自动约束 V2 新能力。
- 技术栈：Spring Boot / Java 17、Vue 3 / Vite / Pinia / Naive UI / TypeScript、PostgreSQL、Redis、Docker Compose、Playwright。
- 默认本地修改环境是 `real-pre`；远端 `real-pre` 部署必须由用户明确要求。
- `test` 是 mock 回归基线；`real-pre` 是真实上游 / 生产形态验证环境，不允许 mock 数据冒充闭环。
- Agent 默认中文回复、证据优先、小步可回滚；探索代码前优先使用 code-review-graph。
- 代码或文档变更需要 evidence；docs-only 可跳过构建、重启和业务 E2E，但不能把未验证项写成 PASS。

## 业务与边界记忆

- 渠道链：认领达人 -> 商品库选品 -> 复制讲解 / 转链 -> 寄样申请 -> 订单同步 -> 渠道业绩 -> 寄样自动完成。
- 招商链：同步活动 -> 活动商品入库 -> 商品上架 -> 审核寄样 -> 订单同步 -> 招商业绩。
- 管理链：用户角色 -> 数据范围 -> 规则配置 -> 各领域读取配置 -> 权限生效。
- 订单域只存订单事实、退款事实和归因输入；业绩域负责最终归属、提成、冲正和双轨金额计算。
- 配置域负责配置事实，不执行具体业务规则；分析模块只读汇总表，不重算业绩归属。
- SDK / Gateway 负责第三方接口封装、鉴权、参数转换、签名和错误码适配，前端不得直连抖音 / 抖店开放接口。

## 财务口径记忆

- 预估服务费收入 = 预估订单额 * 服务费率，不扣技术服务费。
- 结算服务费收入以官方结算服务费字段为准，不用订单额重算，也不扣技术服务费。
- 预估服务费收益 = 预估服务费收入 - 预估服务费支出 - 技术服务费。
- 结算服务费收益 = 结算服务费收入 - 结算服务费支出。
- `OrderDualTrackAmountResolver` 已按 first-positive priority 处理 raw 一级 / 二级 `estimated_commission`，避免重复累加。

## 补迁移流程

1. 用户从 ChatGPT Project 导出或复制项目记忆、项目说明、会话摘要。
2. 将导出内容放入临时导入区，例如 `runtime/imports/chatgpt-project-memory/`，不要直接写入最终事实文档。
3. 先做敏感信息扫描，删除或脱敏 Token、密码、私钥、OAuth code、服务器凭证。
4. 按事实类型分类：阶段口径、业务规则、架构决策、部署状态、验证结果、风险、待办。
5. 与当前代码、ADR、evidence、领域文档交叉验证；不能验证的条目标记为 `待确认`。
6. 可验证且仍有效的内容写入本文或对应领域 / 决策 / 验收文档。
7. 重新执行 docs-only 安全检查、harness 限制检查，并生成 evidence / retro。
