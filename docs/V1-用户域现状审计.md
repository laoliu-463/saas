## 三、五问专项答复

### 1. 当前是一人一角还是一人多角？

**一人多角（数据模型已支持）。**

- 关系表：`sys_user_role`（`init-db.sql` L77-88）。
- 写入：`SysUserService.create` / `assignRoles` → `replaceUserRoles`。
- Seed **演示账号** 仍为每用户 1 角色（`init-db.sql` L1043-1076），**不能**用 seed 直接验收多角。

### 2. 若已多角色，是否真的做了权限并集？

| 维度 | 是否并集 | 说明 |
|------|----------|------|
| 菜单 | **是** | 多角色菜单 ID 合并（`SysMenuService.findUserTree`） |
| 接口角色 | **是（OR）** | `RoleGuardAspect`：任一 `roleCodes` 命中即可（`RoleGuardAspectTest.shouldProceedWhenMultiRoleAndOneMatches` 已验证） |
| 操作权限 JSON | **否** | `SysRole.permissions` 未参与鉴权 |
| JWT roleCodes | **是** | 登录返回全部 `roleCodes` 列表 |

### 3. data_scope 是否按 all > group > self 取最宽？

**是（数值 max，含特例）。**

```java
// AuthService.login — 取各角色 data_scope 最大值
int dataScope = roles.stream().map(SysRole::getDataScope).filter(...).max(Integer::compareTo).orElse(1);
if (roleCodes.contains(OPS_STAFF) || roleCodes.contains(ADMIN)) dataScope = 3;
```

- DB/角色：1=仅自己，2=本组，3=全部（`SysRole.data_scope` 注释）。
- JWT → `JwtAuthInterceptor` → `DataScope.fromCode`；code=2 → DEPT（TASK-U-04：已移除同码 GROUP 枚举，V1 组=部门，由 `dept_type` 区分）。
- **单元测试已补**：`AuthServiceTest.login_multiRole_shouldTakeMaxDataScope`（biz_staff=1 + channel_leader=2 → 2）；`DataScopeTest`（7 条，含 fromCode/code 唯一性/无 GROUP 验证）。

### 4. 招商组、渠道组是否真实存在？

**存在，但建模为「部门表承载组」。**

| 概念 | 实现 |
|------|------|
| 招商组 | `sys_dept.dept_type='recruiter'`，seed `dept_code='BIZ'` |
| 渠道组 | `dept_type='channel'`，seed `CHANNEL` |
| 运营组 | `dept_type='dept'`，seed `OPS` |
| 组员 | 用户 `sys_user.dept_id` 指向组 UUID |
| 组长 | `sys_dept.leader` 字段（弱约束） |

**不存在**：独立 `group`/`team` 表、部门管理 REST API、组员任职历史。

### 5. list_channels / list_recruiters / list_departments / list_group_members 是否存在？

| 设计名 | 当前等价物 | 存在？ |
|--------|------------|--------|
| `list_channels` | `GET /depts?deptType=channel`（TASK-U-01 已建）；订单 filter-options .distinct channel_user_name 作为补充 | **已具备** |
| `list_recruiters` | `GET /depts?deptType=recruiter`（TASK-U-01 已建）；`GET /users/assignable` 作为负责人下拉 | **已具备** |
| `list_departments` | `GET /depts`（TASK-U-01 已建） | **已具备** |
| `list_group_members` | `GET /users/assignable` 按角色+dept 过滤 | **已具备** |

前端：`frontend/src/api/sys.ts` → `getUserPage`、`getAssignableUserOptions`、`getDeptOptions`（TASK-U-05 补充）。
