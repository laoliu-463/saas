## 三、已完成的源码修复

### 3.1 业绩域 P0 — profit 公式 bug 修复

**文件**：`OrderPerformanceCalculator.java`

**问题**：`profit = serviceFee - tech - talentCommission`（多减了 talentCommission）

**正确公式**：`profit = serviceFee - tech`

**原因**：talentCommission 是独立给二级团长的佣金，不应从主运营利润中扣除。

```java
// 修复前
BigDecimal profit = serviceFee.subtract(tech).subtract(talentCommission);

// 修复后
BigDecimal profit = serviceFee.subtract(tech);
```

## 六、跨域 E2E 三链验收计划

### 6.1 渠道链

```
认领达人（TalentService.claim）
  → 商品选品（ProductService.listSelectedLibrary）
  → 转链（PickSourceMappingService.generatePromotionLink）
  → 寄样申请（SampleController.apply）
    → 7 天限制校验
    → 达人门槛校验
    → 地址自动带出
  → 招商审核（SampleController.audit）
  → 运营发货（SampleController.ship）
  → 订单同步（OrderSyncPersistenceService.afterOrderPersisted）
    → 交作业完成（SampleLifecycleService.completePendingHomeworkByOrder）
    → 业绩计算（OrderPerformanceCalculator.calculate）
  → 渠道业绩（DashboardService.getSummary）
```

### 6.2 招商链

```
同步活动（ColonelActivityController.sync）
  → 同步商品（DouyinProductGateway）
  → 招商审核上架（ProductBizStatusService）
  → 寄样审核（SampleController.audit）
  → 订单同步（OrderSyncPersistenceService.afterOrderPersisted）
    → 交作业完成（SampleLifecycleService.completePendingHomeworkByOrder）
    → 业绩计算（OrderPerformanceCalculator.calculate）
  → 招商业绩（DashboardService.getSummary）
```

### 6.3 管理链

```
配用户（SysUserService.create）
  → 配角色（SysUserService.assignRoles）
  → 配数据范围（DataScope 合并最宽）
  → 配规则（BusinessRuleConfigService）
    → sample.restrict_days → checkSevenDaysLimit()
    → talent.protection_days → TalentService.getTalentProtectionDays()
    → commission ratios → CommissionService.loadDefaultRates()
    → sample.default_standard → SampleEligibilityService.evaluate()
  → 验证权限生效（RoleGuardAspect）
  → 验证数据范围生效（DataScopeAspect）
```
