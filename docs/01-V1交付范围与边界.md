# V1 交付范围与边界

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## V1 必做

- [V1 必做] 用户域提供登录用户、角色、组织、菜单和 `self/group/all` 数据范围。
- [V1 必做] 配置域提供配置读取、配置变更、配置审计，不直接执行业务规则。
- [V1 必做] 商品域支持商品库、活动商品同步、转链、`pick_source_mapping` 落库。
- [V1 必做] 达人域支持达人资料、标签、地址、跟进与渠道关系的主链路能力。
- [V1 必做] 寄样域支持申请、审批、发货、交作业状态和通过订单已同步事件判断完成。
- [V1 必做] 订单域支持订单同步、订单事实、退款事实、归因输入和可审计日志。
- [V1 必做] 业绩域支持最终归属、提成、冲正、双轨金额计算、经营毛利和汇总输出。
- [V1 必做] 分析模块只读业绩 / 订单 / 汇总表，提供看板、经营指标矩阵和导出，不重算归属。
- [V1 必做] test 和 real-pre 的验收必须有脚本、API/SQL、证据路径。
- [V1 必做] real-pre 验收不得使用 mock 数据冒充真实闭环。

## V1 简化

- [V1 简化] 事件优先采用 Spring 本地事件和现有 Outbox 机制，不强制引入 MQ。
- [V1 简化] 第三方接口按 P0 链路优先，缺权限、缺 Token、缺真实样本的分支保留阻塞证据。
- [V1 简化] 看板以真实汇总和已落库事实为准，复杂 BI 分析能力不作为 V1 必需。
- [V1 简化] 部署以 Docker Compose 的 test / real-pre 可验证环境为主，生产级弹性运维进入 V2。

## V1 不做

- [V1 不做] 不把 FastAPI、Celery、Python 爬虫写成当前运行事实。
- [V1 不做] 不在订单域计算提成、最终归属或独家覆盖。
- [V1 不做] 不让配置域执行具体业务规则。
- [V1 不做] 不让分析模块重算业绩归属。
- [V1 不做] 不用 mock 数据伪装 real-pre 闭环。
- [V1 不做] 不因为旧 V2.2 文档存在而扩大本轮交付范围。

## V2 预留

- [V2 预留] 更完整的达人采集、达人经营自动化和外部平台爬取策略。
- [V2 预留] 完整 MQ 化、跨服务拆分、分布式一致性治理。
- [V2 预留] 更复杂的财务结算、商家结算和多账期治理。
- [V2 预留] 生产级运维体系、告警、容量和灰度治理。
- [V2 预留] 旧 V2.2 中未进入 V1 的完整运营平台能力。

## 冲突处理规则

- [V1 必做] 当前代码、可运行配置、测试结果优先于旧文档描述。
- [V1 必做] V1 范围优先于旧 V2.2 完整方案。
- [V1 必做] 发现冲突时，不能自行猜测业务规则，必须写入 [决策/ADR-002-V1范围优先级.md](决策/ADR-002-V1范围优先级.md)。
- [历史归档] 已迁移但不再作为当前事实的内容保留在 [归档/旧版V2.2完整方案.md](归档/旧版V2.2完整方案.md)。

## 验收边界

| 验收项 | 证据要求 | 范围 |
| --- | --- | --- |
| V1 P0 | `npm run e2e:v1-p0`、API/SQL 证据、截图或报告 | V1 必做 |
| real-pre | `npm run e2e:real-pre:p0`、真实 Token、真实接口响应、证据索引 | V1 必做 |
| 后端回归 | Maven 测试结果或明确未运行原因 | V1 必做 |
| 前端回归 | 前端构建 / E2E / 浏览器证据 | V1 必做 |
| 文档重构 | 文件结构、范围标记、归档索引、冲突清单 | V1 必做 |
