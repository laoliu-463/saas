## 四、逐域代码核验

### 1. 用户域

需求入口：[01-用户域](./requirements/domain-design/01-用户域.md)

当前判断：**部分实现**。

已实现代码证据：

- `backend/src/main/java/com/colonel/saas/auth/controller/AuthController.java` 提供 `/auth/login`、`/auth/refresh`、`/auth/logout`。
- `backend/src/main/java/com/colonel/saas/controller/SysUserController.java` 提供系统用户分页、详情、创建、更新、删除、重置密码、角色分配。
- `backend/src/main/java/com/colonel/saas/controller/SysRoleController.java` 提供角色 CRUD、启用角色查询、角色菜单读取与分配。
- `backend/src/main/java/com/colonel/saas/controller/SysDeptController.java` 提供部门管理。
- `backend/src/main/java/com/colonel/saas/controller/SysMenuController.java` 提供当前用户菜单树与管理员菜单管理。
- `backend/src/main/java/com/colonel/saas/entity/SysRole.java` 已有 `dataScope` 与 `permissions` 字段。
- `backend/src/main/java/com/colonel/saas/controller/CurrentUserController.java` 已提供当前用户、个人改密、数据范围解析和权限检查接口。
- `backend/src/main/java/com/colonel/saas/service/UserDomainService.java` 已聚合当前用户角色权限包，支持 `self/group/all` 数据范围解析和基于 `sys_role.permissions.operations` 的权限检查。
- `backend/src/main/java/com/colonel/saas/controller/UserMasterDataController.java` 与 `backend/src/main/java/com/colonel/saas/service/UserMasterDataService.java` 已提供渠道、招商、本组成员基础下拉接口。
- `frontend/src/api/sys.ts` 已封装当前用户接口族和人员主数据接口，供个人中心、业务权限判断和业务下拉统一接入。
- `frontend/src/views/profile/UserProfile.vue` 已通过 `/users/current`、`/users/current/data-scope`、`/users/current/permissions/check`、`/users/current/password` 展示当前用户信息、数据范围、权限自检和个人改密。
- `frontend/src/views/layout/Header.vue` 已提供个人中心入口，`frontend/src/router/index.ts` 已注册 `/profile` 路由。
- `frontend/src/views/product/components/ProductAssignDialog.vue` 已通过 `frontend/src/views/product/product-assignee-options.ts` 调用 `/users/master-data/recruiters` 加载招商负责人候选。
- `frontend/src/views/orders/index.vue` 已接入渠道负责人和招商负责人远程筛选，下拉候选通过 `frontend/src/views/orders/order-user-filter-options.ts` 读取 `/users/master-data/channels` 与 `/users/master-data/recruiters`。
- `frontend/src/views/sample/index.vue` 已接入渠道负责人和招商负责人远程筛选，下拉候选通过 `frontend/src/views/sample/sample-user-filter-options.ts` 读取 `/users/master-data/channels` 与 `/users/master-data/recruiters`。
- `backend/src/main/java/com/colonel/saas/controller/SampleController.java` 的寄样分页与导出已支持 `channelUserId/recruiterUserId`，其中渠道负责人落 `sample_request.channel_user_id`，招商负责人通过 `product_operation_state.assignee_id` 关联商品负责人。
- `backend/src/main/java/com/colonel/saas/auth/service/AuthService.java` 已将登录成功、登录失败、登录锁定、账号停用拒绝和登出写入 `operation_log`，认证审计只保留用户名、动作、结果和错误原因。
- `backend/src/main/java/com/colonel/saas/service/UserDomainService.java` 的个人改密已写入 `operation_log`，与认证审计共用 90 天操作日志中心。

未完全实现或未闭环：

- 更多业务页权限判断还未全面迁移到当前用户接口族。
- `SysUserCreateRequest.deptId` 可为空，人员主数据与组织协作不是强闭环。
- 前端用户管理使用单角色选择：`frontend/src/views/system/UserList.vue` 的 `selectedRoleId` 会把 `formData.roleIds` 收敛为单个角色。
- 角色 `permissions` JSONB 字段存在，但当前角色页主要围绕基础信息和菜单配置，操作权限/数据范围 UI 不完整。

交付口径：

- V1 可说“基础账号、角色、菜单、部门、登录登出、当前用户接口、个人中心、人员主数据基础下拉、商品分配人员下拉、订单人员筛选、寄样台人员筛选、认证安全审计和管理员权限可用”。
- 不能说“用户域 V1.0 完整实现”。

### 2. 商品域

需求入口：[02-商品域](./requirements/domain-design/02-商品域.md)

当前判断：**主链路部分闭环**。

已实现代码证据：

- `backend/src/main/java/com/colonel/saas/controller/ProductController.java` 的商品库分页已接收 `salesRange/promotionLink/allianceStatus/commission/hasSample/assignee/systemTag/decision` 等筛选参数。
