| 工具 | 版本 | 说明 |

## 1. 数据库初始化

### 1.1 创建数据库

```sql
CREATE DATABASE colonel_saas;
```

### 1.2 执行初始化脚本

按顺序执行以下 SQL 文件：

```
backend/src/main/resources/db/
├── init-db.sql                              -- 建表DDL（先执行）
├── alter-order-attribution-mvp-v2.sql       -- 订单归因扩展
├── alter-product-biz-status.sql             -- 商品状态扩展
├── alter-product-following.sql             -- 商品跟单扩展
├── alter-product-main-chain.sql            -- 商品主链扩展
├── alter-talent-crm-gap-fill.sql           -- 达人CRM补全
├── alter-talent-enrich.sql                 -- 达人丰富扩展
├── alter-order-pick-source-length.sql      -- 转链长度扩展
└── alter-test-existing-volumes-20260504.sql
```

### 1.3 初始化测试数据（可选）

```bash
# Windows PowerShell
./backend/scripts/init-test-db.ps1

# 或手动在数据库执行 init-test-db.sql 中的测试数据
```

## 2. 后端启动

### 2.1 配置数据库连接

编辑 `backend/src/main/resources/application-dev.yml`：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/colonel_saas
    username: your_username
    password: your_password
  data:
    redis:
      host: localhost
      port: 6379
```

### 2.2 启动后端

```bash
cd backend

# 开发模式（推荐，自动重载）
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 或打包后运行
mvn clean package -DskipTests
java -jar target/colonel-saas-*.jar --spring.profiles.active=dev
```

后端启动后访问：`http://localhost:8080/api`

---

## 3. 前端启动

```bash
cd frontend

# 安装依赖
pnpm install

# 开发模式
pnpm dev
```

前端启动后访问：`http://localhost:3000`

## 5. 环境说明

### 5.1 四种环境配置

| Profile | 说明 | 数据源 |
|---------|------|--------|
| `local-mock` | 本地 Mock 模式，使用模拟数据 | 本地 PostgreSQL + Mock Gateway |
| `dev` | 开发模式 | 本地 PostgreSQL + Mock Gateway |
| `test` | 测试模式 | 测试数据库 + Mock Gateway |
| `prod` | 生产模式 | 生产数据库 + 真实抖音API |

**当前默认 profile 为 `local-mock`**，无需接入真实抖音账号即可体验全部功能。

### 5.2 切换到真实抖音API（可选）

仅在 `prod` 环境下有效：

```yaml
# application-prod.yml 或环境变量
douyin:
  app:
    app-id: your_app_id
    client-key: your_client_key
    client-secret: your_client_secret
  test:
    enabled: false          # 关闭Mock，使用真实API
  real:
    upstream-mode: live     # live 或 contract
```

## 6. 常见问题

### 6.1 启动报 Redis 连接失败

确保 Redis 已启动且 `application-dev.yml` 中 `redis.host` 配置正确。

### 6.2 启动报数据库连接失败

确保 PostgreSQL 已启动，且 `init-db.sql` 已执行，数据库用户名密码与配置一致。

### 6.3 前端 `pnpm dev` 报 `vite not found`

```bash
npm install -g vite
# 或
pnpm add -g vite
```

### 6.4 后端启动报 port already used

8080 端口被占用，修改 `application.yml` 中的 `server.port` 或关闭占用进程。
