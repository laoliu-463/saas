# 18-抖店店铺侧订单接口联调备忘录

> 历史说明（2026-05-15 补记）
>
> 本文为店铺侧订单接口联调备忘录归档。当前项目口径已移除“订单手机号解密”功能，因此文中关于 `order.batchDecrypt`、敏感数据解密权限和收货人信息补全的内容，仅保留为历史背景，不再构成当前范围。

更新时间：2026-05-08

## 1. 文档定位

本文档记录抖店开放平台「店铺侧」订单接口的联调要点，供权限包申请下来后快速对照执行。

与项目当前已联通的「团长侧」订单接口 `buyin.instituteOrderColonel` 做对照区分。

---

## 2. 接口概览

| 接口 | 文档ID | 用途 | 权限类型 |
|------|--------|------|---------|
| `order.searchList` | 1342 | 店铺订单列表查询 | 店铺授权 |
| `order.orderDetail` | 1343 | 店铺订单详情查询 | 店铺授权 + 敏感数据 |

---

## 3. 与团长侧接口的区别

| 维度 | `buyin.instituteOrderColonel`（团长侧） | `order.searchList / order.orderDetail`（店铺侧） |
|------|----------------------------------------|----------------------------------------------|
| 视角 | 团长 | 店铺 |
| 权限 | 团长身份授权 | 店铺授权 + 敏感数据权限包 |
| 收货人信息 | **不返回** | 返回密文，需单独解密 |
| 当前状态 | ✅ 已联通（real-pre） | ❌ `30001` 权限不足 |
| 对应业务 | 订单主同步 | 收货人手机/地址补全 |

---

## 4. 订单号字段区分

抖店订单体系有三个容易混淆的订单号：

| 字段 | 含义 | 适用接口 |
|------|------|---------|
| `shop_order_id` | 店铺父订单号 | `order.searchList` / `order.orderDetail` |
| `sku_order_id` | 商品子订单号 | 同上 |
| `order_id` | 团长侧订单号 | `buyin.instituteOrderColonel` |

> **注意**：不要把 `buyin.instituteOrderColonel` 返回的 `order_id` 直接传给店铺侧接口，两者字段体系不同。

---

## 5. 接口一：订单列表查询 `order.searchList`

### 5.1 基本信息

- **接口路径**：`/order/searchList`
- **文档ID**：1342
- **用途**：根据时间范围、订单状态、订单类型等条件，分页拉取店铺订单列表
- **业务流程**：
  ```
  order.searchList (1342)
          ↓
  获取 shop_order_id / order_id / sku_order_id 等订单字段
          ↓
  order.orderDetail (1343)
          ↓
  落库 / 同步订单状态 / 解密收货信息 / 后续发货处理
  ```

### 5.2 核心入参

| 参数 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| `orderType` | Integer | 否 | 订单类型，如普通订单、虚拟商品订单等 |
| `tradeType` | Integer | 否 | 交易类型，如普通、拼团、预售等 |
| `orderStatus` | Integer | 否 | 订单状态 |
| `createTimeStart` | Long | **是** | 下单开始时间，**秒级时间戳** |
| `createTimeEnd` | Long | **是** | 下单结束时间，**秒级时间戳** |
| `page` | Integer | 否 | 页码，**从 0 开始** |
| `size` | Integer | 否 | 单页大小，**最大 100** |

示例请求：

```json
{
  "createTimeStart": 1714492800,
  "createTimeEnd": 1714579200,
  "page": 0,
  "size": 50
}
```

### 5.3 核心返回字段

| 字段 | 说明 |
|------|------|
| `page` | 当前页 |
| `size` | 单页大小 |
| `total` | 总订单数 |
| `shop_order_list` | 订单信息列表 |

---

## 6. 接口二：订单详情查询 `order.orderDetail`

### 6.1 基本信息

- **接口路径**：`/order/orderDetail`
- **文档ID**：1343
- **用途**：根据店铺父订单号查询订单详细信息
- **前置条件**：通常先调 `order.searchList` 获取 `shop_order_id`

### 6.2 核心入参

| 参数 | 类型 | 是否必填 | 说明 |
|------|------|---------|------|
| `shop_order_id` | String | **是** | 店铺父订单号 |

示例请求：

```json
{
  "shop_order_id": "6917533224570258991"
}
```

---

## 7. 鉴权参数

两个接口共用同一套公共鉴权参数：

```text
app_key
access_token
method
timestamp
param_json
sign
sign_method
```

- `sign_method` 推荐使用 `hmac-sha256`（md5 已不推荐）
- `method` 分别为 `order.searchList` 和 `order.orderDetail`

---

## 8. 高频踩坑点

### 8.1 时间范围不能太大

- 最大支持查询**最近 90 天**内数据
- 不要一上来查全量，按天/小时切片查询

推荐切片方式：

```text
2024-05-01 00:00:00 ~ 2024-05-01 05:59:59
2024-05-01 06:00:00 ~ 2024-05-01 11:59:59
2024-05-01 12:00:00 ~ 2024-05-01 17:59:59
2024-05-01 18:00:00 ~ 2024-05-01 23:59:59
```

### 8.2 分页从 0 开始

- `page` 从 **0** 开始，不是 1

### 8.3 size 最大 100

- 单页大小限制 100 以内，不要传超过 100

### 8.4 5 万条分页上限

- `size * page` 最大不能超过 5 万条
- 超过需按时间切片，不要只靠分页翻页

### 8.5 签名封装

签名逻辑建议单独封装：

```text
DouDianSignUtil      - 签名计算
DouDianClient        - 公共参数构建、请求发送
DouDianOrderApi      - 订单列表/详情
```

### 8.6 日志要求

联调期间保留完整日志，注意脱敏：

```text
可打印：method、timestamp、sign_method、param_json（脱敏后）
禁止打印：access_token（完整）、app_secret
```

---

## 9. 推荐同步流程（对应项目）

```text
1. 定时任务触发订单同步
2. 根据上次同步时间生成 createTimeStart / createTimeEnd
3. 调用 order.searchList（1342）
4. 遍历 shop_order_list
5. 判断订单是否已存在
6. 新订单 → 落库
7. 已存在订单 → 更新状态、金额、售后等字段
8. 对关键订单调用 order.orderDetail（1343）补全收货人信息
9. 调用 order.batchDecrypt 解密手机号/地址（历史背景）
10. 保存同步游标
11. 记录同步日志
```

## 10. 当前项目阻塞状态

| 步骤 | 接口 | 阻塞原因 | 处理方式 |
|------|------|---------|---------|
| 订单主同步 | `buyin.instituteOrderColonel` | 无 | ✅ 已联通，10单已落库 |
| 收货人信息补全 | `order.orderDetail` | `30001 isv.app-permissions-insufficient` | 申请「订单管理接口权限包 + 店铺授权」 |
| 敏感数据解密 | `order.batchDecrypt` | 密文字段来源缺失（orderDetail 未通） | 历史背景，不再继续推进 |

**解除阻塞前提**：
1. 在抖店开放平台申请「订单管理接口权限包」
2. 对目标店铺完成店铺授权（历史记录中曾包含敏感数据解密权限）

## 10.1 2026-05-08 首轮店铺侧订单探针记录

执行环境：

- 环境：`real-pre`
- 后端入口：`http://localhost:8081/api`
- 应用：`7623...7199`
- Token 状态：`hasAccessToken=true`、`hasRefreshToken=true`
- 授权主体确认：`GET /api/douyin/institution-info` 返回 HTTP 200，上游 `code=10000 / msg=success`

本轮调用方式：

- 使用现有联调 raw 入口 `POST /api/douyin/promotion-link-probes/raw`
- 通过 `method=order.searchList` / `method=order.orderDetail` 透传到抖店开放平台
- 请求中未打印 `access_token`、`app_secret`，仅记录脱敏应用号、接口名、参数摘要与上游错误码

本轮结果：

| 场景 | 请求摘要 | 结果 |
|------|----------|------|
| 正常查询列表 | 最近 6 小时、`page=0`、`size=10` | HTTP 200，本系统包装成功返回；上游业务失败 `30001 / isv.app-permissions-insufficient` |
| `size=100` | 最近 6 小时、`page=0`、`size=100` | 同上，权限校验先于分页上限校验 |
| `page=1` | 最近 6 小时、`page=1`、`size=10` | 同上，暂无法验证分页连续性 |
| 90 天边界 | `createTimeStart` 超过 90 天 | 同上，权限校验先于 `40004 / isv.parameter-invalid:1036` |
| 随机订单详情 | `shop_order_id=9999999999999999999` | HTTP 200，本系统包装成功返回；上游业务失败 `30001 / isv.app-permissions-insufficient` |
| 空订单号详情 | `shop_order_id=""` | 同上，权限校验先于参数错误校验 |

代表性上游取证：

- `order.searchList`：`errorCode=30001`，`subCode=isv.app-permissions-insufficient`，`message=应用无权限调用该接口，请先申请接口权限包`，`logId=20260508092056DED47E3D93D2AB82FE65`
- `order.orderDetail`：`errorCode=30001`，`subCode=isv.app-permissions-insufficient`，后端日志可见同类 `logId=20260508092043EC725D96D86098F11B59`

同步复核：

- 团长侧 `POST /api/orders/sync` 仍可用，本轮 6 小时窗口返回 HTTP 200、`failed=0`、`unattributed=10`
- `GET /api/orders?page=1&size=5` 返回 HTTP 200、订单总数 `21`
- `GET /api/dashboard/metrics?timeField=createTime` 返回 HTTP 200，今日订单 `10`、今日 GMV `197.69`

结论：

> 当前店铺侧订单接口已经完成到达上游的首轮探针；阻塞点仍是应用未获得 `order.searchList` / `order.orderDetail` 所需订单管理接口权限包及店铺授权。关于收货人信息补全和 `order.batchDecrypt` 成功样本验证的描述，现仅保留为历史背景。

## 10.2 2026-05-08 三方联调 SOP 防护补查

对照本轮三方接口联调 SOP，已补查并收口以下底座风险：

| 检查项 | 当前结论 |
|--------|----------|
| 网络可达 | 本机到 `openapi-fxg.jinritemai.com:443` TCP 可达 |
| 鉴权机制 | 当前 `DouyinApiClient` 通过抖店 SDK `SignUtil.sign(..., "2")` 生成签名，真实调用统一走 `app_key / method / timestamp / sign / access_token` |
| 环境区分 | 当前继续使用 `real-pre` 的 `8081 / 5433 / 6380` 拓扑，未污染 `test` 基线 |
| 超时设置 | 抖店真实 `RestTemplate` 默认改为 `connectTimeout=3s`、`readTimeout=5s`，并可通过 `DOUYIN_CONNECT_TIMEOUT / DOUYIN_READ_TIMEOUT` 覆盖 |
| 日志脱敏 | 网络异常不再打印原始 `RestTemplate` exception cause，避免完整请求 URL 中的 `access_token / sign` 被上层日志输出 |
| Webhook 日志 | 回调入口只记录事件名、bodyLength、签名是否存在 / 长度与异常类型，不记录原始 body、手机号、token 或签名值 |
| 错误码保留 | 上游业务错误继续保留 `errorCode / subCode / logId / endpoint / message`，用于平台提单与复盘 |

本轮新增验证：

- `DouyinConfigTest`：锁定抖店 HTTP 默认超时为 `3s / 5s`，并验证可配置覆盖
- `DouyinApiClientTest#post_shouldNotLogOrThrowSensitiveRequestUrlWhenTransportFails`：模拟网络异常 URL 中包含 `access_token / sign`，验证日志与上抛异常均不泄漏敏感 URL
- `DouyinWebhookControllerTest`：覆盖正常 JSON 与异常 JSON 回调体，验证日志不输出手机号、token、签名值或原始 body
