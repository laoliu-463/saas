# 24-达人经营平台补全一期记录

更新时间：2026-05-04

## 一、背景

当前 `/talent` 已具备基础 CRM 能力，但与需求文档中的经营台口径仍有明显差距。为了不破坏现有 `local-mock` 主链路，本次只补“达人经营平台第一阶段”，先把最核心的经营池切分、筛选和状态动作闭环补齐。

## 二、本次范围

纳入本期：

- 团队公海
- 我的达人
- 自然出单达人
- 达人黑名单
- 经营筛选（关键词、类目、地区、认领状态、粉丝区间）
- 列表经营字段展示
- 认领 / 释放 / 拉黑 / 解除拉黑
- 详情中的经营字段与黑名单状态展示

暂不纳入本期：

- 客户端找达人
- 团长管理
- 商家管理
- 合作管理
- 推广效果
- 产品壳层导航扩展

## 三、实现口径

### 1. 前端

- 保持 `/talent` 单路由，不新增新的经营产品壳
- 使用视图切换承载 `团队公海 / 我的达人 / 自然出单达人 / 达人黑名单`
- 新增经营筛选面板与状态动作组件
- 在列表和详情中展示主类目、经营区间、黑名单状态等经营字段

### 2. 后端

- 新增 `TalentPageQuery` 统一达人分页查询参数
- `/talents` 支持 `view / category / claimStatus / minFans / maxFans / region / ownerKeyword`
- 新增 `/talents/{id}/blacklist` 与 `/talents/{id}/unblacklist`
- 查询服务补齐四个经营视图过滤和经营字段衍生逻辑
- `talent` 表补充黑名单字段初始化

## 四、本次验证

- 后端定向测试：`TalentControllerTest`、`TalentServiceTest`、`TalentQueryServiceTest` 通过
- 前端构建：`npm run build` 通过
- 新增脚本：`runtime/qa/talent-platform-smoke.cjs`

## 五、当前取舍

1. 自然出单达人当前按“列表聚合后订单数大于 0”判定，先服务演示与经营筛选，不在本轮引入更复杂的合作关系规则。
2. 经营区间字段优先使用现有达人与订单聚合数据衍生，不额外新建完整统计域。
3. 黑名单先作为达人状态治理动作接入，不在本轮扩展到更复杂的审批与恢复流程。

## 六、后续建议

1. 二期单独拆分“客户端找达人”和“合作管理 / 推广效果”
2. 若真实 SDK 联调后字段更丰富，再把经营区间从衍生值切换到真实指标
3. 结合浏览器烟雾脚本补齐更稳定的达人认领/释放/拉黑回归基线
