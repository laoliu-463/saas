# 26-20260504 本地规则中心验收记录

更新时间：2026-05-04

## 一、目标

当前轮次按最新口径暂不推进真实 SDK 联调，先补齐本地 Mock 中“系统配置已经有后台页面，但尚未真正驱动业务规则”的缺口。

本轮只做三件事：

1. 管理员修改 `/system/config`
2. 寄样与达人主链路立即读取新规则
3. 用可见浏览器慢速验证“改规则 -> 业务生效”

## 二、已完成补齐

### 1. 新增统一规则读取服务

- `backend/src/main/java/com/colonel/saas/service/BusinessRuleConfigService.java`

统一负责从 `system_config` 读取并解析：

- `sample.restrict_days`
- `sample.restrict_enabled`
- `sample.timeout_homework_days`
- `sample.timeout_pending_ship_days`
- `talent.protection_days`
- `talent.exclusive.service_fee_ratio`
- `talent.exclusive.monthly_samples`

### 2. 已接入业务主链路

- 寄样重复申请限制：从固定 7 天改为读取配置
- 达人认领保护期：从固定 30 天改为读取配置
- 寄样自动关闭：从固定 `30 / 15` 天改为读取配置
- 达人独家判断阈值：改为读取配置

### 3. 顺手修复的本地阻断点

- `SampleController.createSample()` 新建寄样单时补齐显式 UUID

原因：

- 可见浏览器验收中，重复寄样规则验证被寄样主键填充异常提前打断，前台只会看到“服务器异常”
- 修复后，才能真实验证规则中心是否生效
