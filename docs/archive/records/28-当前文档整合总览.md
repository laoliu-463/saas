## 二、当前统一结论

当前项目最准确的阶段表述是：

> 本地 Mock / test 核心业务闭环已完成，认证安全体系与权限边界已大幅补齐，real-pre 已进入真实上游联调模式；当前重点是真实订单归因闭环、商品详情 / SKU、店铺侧订单权限、Webhook 业务消费、M1.6 数据看板真实化和 M1.7 部署验证。

1. 不能再说“真实 SDK 完全未接入”。Auth、授权主体、活动、活动商品、真实转链、团长侧订单同步已经有 real-pre 证据。

## 六、当前环境口径整合

### 1. 标准本机拓扑

| 用途 | 前端 | 后端 | 说明 |
| --- | --- | --- | --- |
| 本地 / test 基线 | `3000` | `8080` | 本机前端 + test 后端 / 本地 Mock 口径 |
| real-pre | `3001` | `8081` | 真实上游联调与独立拓扑验证 |

约束：

- 不要混起第二个 `3001` 本机 Vite。
- 不要把额外手工后端混到 `8080`。
- 真实联调前先确认 Docker / 端口 / Redis / Postgres 来源。

### 2. real-pre 当前口径

最新文档和记录显示，real-pre 已从早期“local-mock profile + Test Gateway 开关”推进到真实联调模式：

- `SPRING_PROFILES_ACTIVE=real`
- `DOUYIN_TEST_ENABLED=false`
- `APP_TEST_ENABLED=false`
- `ORDER_SYNC_ENABLED=true`
- Gateway 日志显示 `RealDouyinOrderGateway, upstreamMode=live`

但这仍然不是生产环境。它是本机真实上游联调环境，敏感配置和真实订单数据要按准生产级别保护。
