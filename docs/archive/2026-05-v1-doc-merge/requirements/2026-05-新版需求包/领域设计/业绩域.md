## 业绩域 - 完整设计方案（V1.1 正式版）

------

## 文档说明

本文档基于骨架设计 V1.5 和需求文档 V2.2 设计。

**定位**：管理订单的最终业绩归属、提成与毛利计算、独家商家资格；为数据平台与各业务域提供业绩数据。

**原则**：

- 订单默认归因来自订单域，业绩域只做覆盖与计算
- 独家达人资格由达人域判定与落库，业绩域订阅事件并应用
- 独家商家资格由业绩域判定与落库
- 提成比例由配置域提供，业绩域只读
- **双轨金额**：对订单域存入的预估轨、结算轨各算一遍提成/毛利（与看板两行一致）

**版本更新**：

- V1.1 对齐看板：**服务费收益**为提成基数；预估/结算双轨字段；修正时间维度说明（cohort 时间 vs 金额轨道）

**版本状态**：✅ 可进入开发

**关联文档**：`订单域.md` §2.4、`重要/抖音开放平台Python_SDK接口文档.md` §9.4

## 一、领域定位

### 1.1 职责

| 职责 | 说明 |
| ---- | ---- |
| 计算最终业绩归属 | 默认归因 + 独家达人/独家商家规则覆盖 |
| 计算提成与毛利 | 按配置域提成比例计算招商/渠道提成 |
| 管理独家商家资格 | 月度判定、生效/失效周期、发出资格事件 |
| 维护业绩记录 | 每单一条业绩记录，支持重算 |
| 维护归属缓存 | 独家达人/独家商家/商品负责人缓存 |
| 提供业绩查询 | 数据平台、订单列表扩展、各域统计 |

### 1.2 边界

**管**：

- 最终渠道ID、最终招商ID及归因方式
- 招商提成、渠道提成、毛利
- 独家商家记录与月度计算任务
- 业绩记录的生命周期（生效/冲正/失效）

**不管**：

- 订单同步与默认归因存储（订单域）
- 独家达人条件判定与记录（达人域）
- 提成配置写入（配置域）
- 看板汇总表维护（分析模块）
- 用户账号与权限（用户域）

### 1.3 对外提供的能力

| 操作 | 调用方 | 说明 |
| ---- | ------ | ---- |
| 查询单笔订单业绩 | 前端/订单列表 | 最终归属 + 提成明细 |
| 查询业绩列表 | 数据平台 | 多维筛选、分页 |
| 查询业绩汇总指标 | 数据平台/分析模块 | 卡片指标（支持双时间口径） |
| 批量查询订单业绩 | 订单列表 BFF | 批量补全提成字段 |
| 查询独家商家 | 商品域/招商 | 合作方独家状态 |
| 查询我的独家商家 | 招商 | 本人生效中的独家合作方 |
| 重算单笔订单业绩 | 管理员 | 规则或归属变更后 |
| 重算指定月份 | 管理员 | 批量重算（慎用） |

### 1.4 发出的事件

| 事件 | 触发时机 | 携带数据 |
| ---- | -------- | -------- |
| 业绩已计算 | 业绩记录写入或更新 | 订单ID、最终渠道/招商、**预估轨+结算轨**提成与毛利、是否有效 |
| 独家商家资格生效 | 月初计算后新增资格 | 合作方ID、招商ID、生效月份 |
| 独家商家资格失效 | 资格到期失效 | 合作方ID、招商ID、失效月份 |

### 1.5 订阅的事件

| 事件 | 来源 | 处理 |
| ---- | ---- | ---- |
| 订单已同步 | 订单域 | 计算或更新该订单业绩记录 |
| 独家资格生效 | 达人域 | 更新「达人→独家渠道」缓存 |
| 独家资格失效 | 达人域 | 清除或更新独家渠道缓存 |
| 商品负责人变更 | 商品域 | 更新商品负责人缓存；未结算订单可触发重算 |
| 合作方同步完成 | 商品域 | 刷新合作方名称等冗余字段 |
| 配置已更新 | 配置域 | 刷新提成比例等配置缓存 |
| 寄样已提交/已通过/已完成 | 寄样域 | V1.0 可选：仅用于寄样相关统计；不驱动单笔业绩计算 |

## 二、核心概念

### 2.1 归属计算流水线（单笔订单）

```plain
订单已同步（事件 §订单域 2.8：双轨金额 + 默认归因）
      ↓
① 确定最终归属（独家覆盖，渠道/招商各一）
      ↓
② 读取配置域：招商/渠道提成比例
      ↓
③ 对【预估轨】【结算轨】分别计算（见 2.4，公式相同、输入不同）
      ↓
④ UPSERT performance_records，发出「业绩已计算」
```

**与骨架 5.1 一致**：独家达人与独家商家为**独立维度**，可同时生效。

### 2.2 独家商家机制（本域负责）

| 项目 | 说明 |
| ---- | ---- |
| 判断维度 | 合作方ID + 招商ID |
| 触发条件 | 上月该招商在该合作方的服务费 ≥ 该招商个人总服务费 × 阈值（默认70%，配置域） |
| 统计口径 | **已结算**订单，按 `settle_time`；统计时使用**默认招商**维度，避免循环依赖 |
| 生效 | 当月满足条件 → **当月整月**享受独家资格 |
| 失效 | 当月不满足条件 → **下月初**失效 |
| 判断时机 | 每月1日凌晨定时任务，基于上月数据 |
| 效果 | 该合作方所有后续订单的招商维度归该独家招商 |

### 2.3 独家达人机制（达人域负责，本域应用）

| 项目 | 说明 |
| ---- | ---- |
| 判定与落库 | 达人域（`exclusive_talents` 表 + 月度任务） |
| 本域职责 | 订阅「独家资格生效/失效」事件，维护 Redis 缓存 |
| 应用规则 | 订单达人存在生效中独家渠道 → 覆盖默认渠道 |
| 人工特批 | 寄样不达标时渠道组长可审核通过 → **达人域** `approve_exclusive_talent` 接口；本域只读生效记录 |

### 2.4 双轨提成公式（对齐数据平台看板）

对 **预估轨**、**结算轨** 各执行一遍（`track ∈ {estimate, effective}`）：

```plain
服务费收入(轨)   = 订单.estimate_service_fee 或 订单.effective_service_fee
技术服务费(轨)   = 订单.estimate_tech_service_fee 或 订单.effective_tech_service_fee

服务费收益(轨)   = 服务费收入(轨) - 技术服务费(轨)    ← 提成基数

招商提成(轨)     = 服务费收益(轨) × 招商提成比例
渠道提成(轨)     = 服务费收益(轨) × 渠道提成比例

毛利(轨)         = 服务费收益(轨) - 招商提成(轨) - 渠道提成(轨)
```

| 看板行 | 汇总字段 |
| ------ | -------- |
| 预估 / 成交 | `sum(*_estimate)` 或 `sum(estimate_*)` |
| 结算 | `sum(*_effective)` 或 `sum(effective_*)` |

**服务费支出**（看板可选）：V1.0 可展示为 `招商提成 + 渠道提成`（两轨分别汇总）；若业务定义为 0 则仅展示占位。

**归属**：`final_channel_id` / `final_recruiter_id` 两套金额共用，不因轨道变化。

**退款/失效**：

| 订单状态 | 业绩处理 |
| -------- | -------- |
| 已退款 | `is_valid=false`；两轨提成、毛利置 0；`is_reversed=true` |
| 已失效 | `is_valid=false`，不计入汇总 |
| 待结算 | **预估轨**有值；**结算轨**多为 0，结算后随订单更新重算 |

### 2.5 看板查询的两个维度（勿混淆）

| 维度 | 含义 | 实现 |
| ---- | ---- | ---- |
| **A. 时间 cohort** | 哪些订单进入统计 | `time_filter_type=pay` 用 `pay_time`；`=settle` 用 `settle_time` |
| **B. 金额轨道** | 每个指标显示哪一行 | 预估行 SUM `estimate_*`；结算行 SUM `effective_*` |

示例：筛选「4月 + 付款时间」→ cohort 为 4 月付款的订单 → 卡片上「总订单数·成交」= COUNT(cohort)，「总订单数·结算」= COUNT(cohort 中 `effective_service_fee>0` 或已结算)。

独家商家月度判定：仅用 **结算轨** + `effective_service_fee` + `settle_time` + **默认招商**维度。

### 2.6 归因方式枚举

| 维度 | 枚举值 | 说明 |
| ---- | ------ | ---- |
| 渠道 | `pick_source` | 默认 pick_source 映射 |
| 渠道 | `exclusive_talent` | 独家达人覆盖 |
| 渠道 | `unattributed` | 无法归因 |
| 招商 | `activity_owner` | 活动默认负责人 |
| 招商 | `product_owner` | 个别品负责人覆盖 |
| 招商 | `exclusive_merchant` | 独家商家覆盖 |
| 招商 | `unattributed` | 无法归因 |

## 三、状态机

### 3.1 业绩记录状态

```plain
待计算 → 已生效 →（订单退款）→ 已冲正
                →（订单失效）→ 已失效
```

| 状态 | 说明 |
| ---- | ---- |
| 待计算 | 订单同步到达，尚未完成计算 |
| 已生效 | 正常计入统计 |
| 已冲正 | 退款冲减，提成归零 |
| 已失效 | 订单无效，不计入统计 |

### 3.2 独家商家记录状态

```plain
待生效 → 生效中 → 已失效
```

| 状态 | 说明 |
| ---- | ---- |
| 待生效 | 已计算待次月生效（如需要） |
| 生效中 | 当前月份有效 |
| 已失效 | 资格结束 |

## 四、业务规则汇总

### 4.1 归属优先级（与骨架一致）

| 优先级 | 规则 | 作用维度 |
| ------ | ---- | -------- |
| 1 | 独家达人 | 渠道 |
| 2 | 独家商家 | 招商 |
| 3 | 默认归因 | 渠道/招商 |

### 4.2 重算规则

| 场景 | 是否重算历史单 | 说明 |
| ---- | -------------- | ---- |
| 商品负责人变更 | 仅未结算单 | 已结算单保持历史快照 |
| 提成比例变更 | 管理员手动触发 | 按月批量重算 |
| 独家资格变更 | 新订单即时生效 | 历史单不重算 |
| 订单状态变为退款 | 自动冲正 | 订阅订单已同步更新 |

### 4.3 数据可见范围

| 角色 | 可见范围 |
| ---- | -------- |
| 管理员 | 全部业绩 |
| 招商组长 | `final_recruiter` 属于本组招商 |
| 招商 | `final_recruiter` = 自己 |
| 渠道组长 | `final_channel` 属于本组渠道 |
| 渠道 | `final_channel` = 自己 |
| 运营 | 按需配置 |

导出：管理员 + 组长（与需求文档一致）。

| 毛利（预估/结算） | `estimate_gross_profit` / `effective_gross_profit` |

### 5.2 指标卡片 `get_performance_summary`

每个指标返回 **两行**：`estimate`（成交/预估）、`effective`（结算）。

| 看板指标 | 预估行字段 | 结算行字段 |
| -------- | ---------- | ---------- |
| 总订单数 | COUNT(cohort) | COUNT(cohort 且 effective_service_fee>0 或已结算) |
| 订单总额 | SUM(pay_amount) | SUM(settle_amount) |
| 服务费收入 | SUM(estimate_service_fee) | SUM(effective_service_fee) |
| 技术服务费 | SUM(estimate_tech_service_fee) | SUM(effective_tech_service_fee) |
| 服务费收益 | SUM(estimate_service_profit) | SUM(effective_service_profit) |
| 招商提成 | SUM(estimate_recruiter_commission) | SUM(effective_recruiter_commission) |
| 渠道提成 | SUM(estimate_channel_commission) | SUM(effective_channel_commission) |
| 毛利 | SUM(estimate_gross_profit) | SUM(effective_gross_profit) |

**筛选参数**：`time_filter_type`（pay/settle）、时间范围、订单状态、商品、达人、招商、渠道、活动。

> 分析模块看板可订阅「业绩已计算」写汇总表，聚合逻辑与本表一致（见 `分析模块.md`）。

## 九、与其他领域的关系

```plain
┌──────────┐     订单已同步      ┌──────────┐
│  订单域  │ ─────────────────→ │  业绩域  │
└──────────┘                    └────┬─────┘
                                     │ 业绩已计算
┌──────────┐  独家资格生效/失效     │
│  达人域  │ ─────────────────→     │
└──────────┘                        ↓
┌──────────┐  商品负责人变更   ┌──────────┐
│  商品域  │ ─────────────────→ │ 分析模块 │
└──────────┘                    └──────────┘
┌──────────┐  读提成配置
│  配置域  │ ─────────────────→ 业绩域
└──────────┘
┌──────────┐  数据范围
│  用户域  │ ─────────────────→ 业绩域（列表权限）
└──────────┘
```

| 调用方 | 调用目的 |
| ------ | -------- |
| 订单列表 BFF | 批量补全提成字段 |
| 数据平台 | 业绩列表、指标卡片 |
| 分析模块 | 订阅「业绩已计算」更新汇总表 |
| 商品域 | 查询合作方是否独家商家 |
