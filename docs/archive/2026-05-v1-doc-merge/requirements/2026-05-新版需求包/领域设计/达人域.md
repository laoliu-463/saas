## 文档说明

本文档基于骨架设计 V1.4 和需求文档 V2.2 设计。

**定位**：管理达人是谁、达人归谁管（认领关系）、达人的基础信息、状态、标签和收货地址。

**核心特点**：

- 达人认领：**多人可同时认领，不互斥**
- 独家达人：基于服务费占比 + 月寄样数量
- 达人标签：**认领人可打标签，后者覆盖，最多3个，所有人可见**
- 达人地址：**认领人可保存/修改达人的收货地址，用于寄样**
- 数据采集：API优先 → 爬虫兜底 → 手动填写

**版本状态**：✅ 所有问题已确认，可进入开发

## 一、领域定位

### 1.1 职责

| 职责             | 说明                                          |
| ---------------- | --------------------------------------------- |
| 管理达人基础信息 | 达人是谁、昵称、账号、等级、月销等            |
| 管理达人认领关系 | 多人可同时认领，各自独立保护期                |
| 管理达人收货地址 | 认领人可保存/修改达人的收货地址（用于寄样）   |

| 提供达人查询补全 | 输入账号，API/爬虫自动补全信息                |

### 1.2 边界

**管**：

- 达人基础信息（API/爬虫采集 + 手动录入）
- 达人认领记录（认领、保护期、释放）
- 达人收货地址（认领人维护，用于寄样）
- 独家达人资格判断与周期管理
- 达人标签（认领人维护）

**不管**：

- 寄样申请（寄样域管，但会来查达人地址和门槛）
- 订单业绩归属（业绩域管，但独家达人时需特殊处理）

## 二、核心概念

### 2.1 达人认领模型

**核心**：多人可同时认领同一达人，不互斥。

| 概念               | 说明                                                 |
| ------------------ | ---------------------------------------------------- |
| **公海**           | 未被任何人认领的达人，所有渠道可见                   |
| **私海（认领后）** | 认领后进入认领人的私海，但**其他渠道也能看到并认领** |
| **认领关系**       | 一人对一达人一条记录，认领人之间相互独立             |
| **保护期**         | 每人认领后各自有30天保护期，互不影响                 |
| **联系方式**       | 所有认领人填写的联系方式，对所有认领人可见           |

### 2.2 保护期规则

| 规则                         | 说明                                     |
| ---------------------------- | ---------------------------------------- |
| 认领后立即进入保护期         | 30天内，认领人可见该达人完整信息         |
| 保护期内无产出，到期自动释放 | 30天未产生订单，认领记录标记为「已释放」 |
| 每次产生订单，重置保护期     | 从订单时间起重新计算30天                 |
| 保护期时长可配置             | 默认30天，管理员可调整                   |

### 2.3 达人收货地址

| 规则项         | 说明                                           |
| -------------- | ---------------------------------------------- |
| **谁可以维护** | **认领人**（认领该达人的渠道）                 |
| **用途**       | 寄样申请时自动带入，减少重复填写               |
| **存储位置**   | 达人认领记录表（每个渠道独立维护自己的地址）   |
| **修改后影响** | 后续寄样申请使用新地址，不影响已提交的申请     |
| **可见性**     | 仅维护者自己可见（地址是渠道和达人之间的约定） |

**示例**：

```plain
达人「张三」
  - 被渠道A认领 → 渠道A填写地址「北京市朝阳区xxx」
  - 被渠道B认领 → 渠道B填写地址「上海市浦东新区xxx」
  
渠道A寄样时 → 自动带入「北京市朝阳区xxx」
渠道B寄样时 → 自动带入「上海市浦东新区xxx」
```

### 2.4 独家达人机制

#### 触发条件（两个条件需同时满足）

| 条件           | 说明                                              | 阈值（可配置） |
| -------------- | ------------------------------------------------- | -------------- |
| **服务费占比** | 该渠道产生的服务费 ≥ 本团该达人总服务费的 **70%** | 70%            |
| **月寄样数量** | 该渠道给该达人的月寄样数量 ≥ **10个**             | 10个           |

**寄样统计口径**：寄样状态为「待发货」及之后

#### 生效/退出机制

| 场景           | 规则                                    |
| -------------- | --------------------------------------- |
| 当月满足条件   | **整月享受独家资格**（从当月1日到月底） |
| 当月不满足条件 | **下月初生效**（失去独家资格）          |
| 判断时机       | 每月1日凌晨定时任务计算上月数据         |

#### 独家达人效果

**该达人所有后续订单的服务费收益，全部归该渠道所有（无论订单通过谁的推广链接产生）**

### 2.5 达人标签

| 规则项           | 说明                                             |
| ---------------- | ------------------------------------------------ |
| **谁可以打标签** | **认领人**（认领该达人的渠道）                   |
| **标签数量**     | 每个达人最多 **3个标签**                         |
| **标签更新规则** | **后者覆盖**（最后修改者生效）                   |
| **标签可见性**   | **所有人可见**（管理员、招商、其他渠道都能看到） |
| **V1.0 实现**    | 开放输入，渠道自由填写文字                       |
| **V2.0 升级**    | 增加预设标签库，渠道从预设中选择                 |

### 2.6 达人等级

| 项目     | 说明                               |
| -------- | ---------------------------------- |
| 数据来源 | 抖音API优先 → 爬虫兜底 → 手动填写  |
| 等级体系 | LV0 - LV7（抖音官方等级）          |
| 用途     | 寄样域判断达人是否符合商品寄样门槛 |
| 更新频率 | 首次采集 + 每周定时更新 + 手动刷新 |

### 2.7 数据来源优先级

| 场景         | 处理方式                       |
| ------------ | ------------------------------ |
| 渠道输入账号 | API优先 → 爬虫兜底 → 手动填写  |
| 批量导入     | 系统逐条查询补全，失败记录标记 |
| 定时刷新     | 每周更新活跃达人的等级、月销   |
| 手动刷新     | 渠道点击按钮触发单条更新       |

## 三、状态机

### 3.1 达人认领记录状态流转

```plain
渠道认领达人
      ↓
┌─────────────┐
│   保护中    │ ← 认领后进入30天保护期
└─────────────┘
      │
      ├── 产生订单 ──→ 重置保护期（继续保护中）
      │
      └── 30天无订单 ──→ 自动释放
                              ↓
                        ┌─────────────┐
                        │   已释放    │
                        └─────────────┘
                              ↓
                        渠道可重新认领
```

### 3.2 状态转换表

| 当前状态   | 触发动作   | 新状态 | 条件               |
| ---------- | ---------- | ------ | ------------------ |
| 无认领记录 | 渠道认领   | 保护中 | 无限制             |
| 保护中     | 保护期到期 | 已释放 | 30天无订单         |
| 保护中     | 产生订单   | 保护中 | 重置保护期到期时间 |
| 已释放     | 重新认领   | 保护中 | 重新开始保护期     |

| 规则           | 说明                     |
| -------------- | ------------------------ |
| 认领人可打标签 | 只有当前认领人能修改标签 |

| 后者覆盖       | 后修改的覆盖之前的       |
| 所有人可见     | 所有角色都能看到标签     |
| V1.0自由输入   | V2.0升级为预设标签库     |

### 4.5 独家达人规则

| 规则           | 说明                         |
| -------------- | ---------------------------- |
| 判断周期       | 每月1日计算上月数据          |
| 服务费占比阈值 | ≥70%（可配置）               |
| 月寄样数量阈值 | ≥10个（可配置）              |
| 生效周期       | 满足条件的整月               |
| 退出周期       | 不满足条件的下月初           |
| 独家效果       | 该达人所有订单服务费归该渠道 |

### 4.6 数据可见范围

| 角色     | 可见达人范围                             | 可见收货地址     |

| 管理员   | 所有达人 + 所有认领记录 + 所有标签       | 所有地址         |
| 招商组长 | 本组内渠道认领的达人                     | 本组渠道的地址   |

| 渠道     | 自己认领的达人 + 公海达人 + 所有人的标签 | **仅自己的地址** |
| 运营     | 按需配置                                 | 按需配置         |

| `get_talent_address`    | 获取达人收货地址  | 达人ID                   | 收货地址                    | 认领人/寄样域 |
| `list_talents`          | 达人列表          | 筛选条件、分页           | 达人列表                    | 按角色过滤    |

**标签更新规则**：

- 不传的标签字段 = 清空
- 最多传3个标签
- 调用者必须是该达人的**当前认领人**
- 后者覆盖（新标签覆盖旧标签）

**地址更新规则**：

- 调用者必须是该达人的**当前认领人**
- 更新后发出「达人地址更新」事件
- 地址仅维护者自己可见
