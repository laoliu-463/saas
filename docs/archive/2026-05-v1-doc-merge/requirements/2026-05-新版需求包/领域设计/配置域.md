## 配置域 - 完整设计方案（V1.0 正式版）

------

## 文档说明

本文档基于骨架设计 V1.5 和需求文档 V2.2 设计。

**定位**：管理系统级业务规则与模板参数，为各业务域提供可热更新的只读配置；写操作仅管理员可用。

**原则**：

- 改配置不改代码，界面化维护
- 配置变更发出事件，各域刷新本地/Redis 缓存
- V1.0 提成比例为全局统一；V2.0 扩展差异化提成表

**版本状态**：✅ 所有问题已确认，可进入开发

## 一、领域定位

### 1.1 职责

| 职责 | 说明 |
| ---- | ---- |
| 系统参数管理 | KV 形式存储各类业务阈值与开关 |
| 提成规则配置 | 全局招商/渠道提成比例（V1.0） |
| 寄样规则配置 | 限制天数、开关、超时关闭天数 |
| 独家阈值配置 | 达人/商家服务费占比、月寄样数量 |
| 达人保护期配置 | 认领保护天数 |
| 默认寄样门槛 | 全局默认寄样要求 JSON |
| 推广模板配置 | 复制讲解文案模板、pick_extra 规则 |
| V2.0 扩展 | 预设标签库、按活动/商品差异化提成 |

### 1.2 边界

**管**：

- 配置项的 CRUD（管理员）
- 配置校验与版本记录
- 配置变更事件

**不管**：

- 用户账号（用户域）
- 独家资格判定执行（达人域/业绩域读配置后执行）
- 单笔业绩计算（业绩域）
- 转链 API 调用（商品域读配置后调用抖音）

### 1.3 对外提供的能力

| 操作 | 调用方 | 说明 |
| ---- | ------ | ---- |
| 读取单个配置 | 各业务域 | get_config |
| 批量读取配置 | 各业务域 | get_configs |
| 读取全部配置 | 管理端 | list_all_configs |
| 更新配置 | 管理员 | update_config，带校验 |
| 读取提成比例 | 业绩域 | 专用接口或 get_configs |
| 读取复制模板 | 商品域 | copy_template 等 |

### 1.4 发出的事件

| 事件 | 触发时机 | 携带数据 |
| ---- | -------- | -------- |
| 配置已更新 | 管理员保存配置 | config_key, old_value, new_value, operator_id |

### 1.5 订阅的事件

| 事件 | 来源 | 处理 |
| ---- | ---- | ---- |
| 无 | - | V1.0 配置域不依赖其他领域事件 |

## 二、核心概念

### 2.1 配置项清单（V1.0）

| config_key | 默认值 | 类型 | 消费方 | 说明 |
| ---------- | ------ | ---- | ------ | ---- |
| `sample_limit_days` | 7 | int | 寄样域 | 同一渠道+达人+商品重复申请间隔天数 |
| `sample_limit_enabled` | true | bool | 寄样域 | 是否启用寄样限制 |
| `sample_auto_close_days` | 30 | int | 寄样域 | 待交作业超时自动关闭天数 |
| `exclusive_talent_fee_ratio` | 0.70 | decimal | 达人域 | 独家达人服务费占比阈值 |
| `exclusive_talent_monthly_samples` | 10 | int | 达人域 | 独家达人月寄样数量阈值 |
| `exclusive_merchant_fee_ratio` | 0.70 | decimal | 业绩域 | 独家商家服务费占比阈值 |
| `talent_claim_protect_days` | 30 | int | 达人域 | 达人认领保护期天数 |
| `recruiter_commission_rate` | 待业务确认 | decimal | 业绩域 | 招商提成比例（如 0.30） |
| `channel_commission_rate` | 待业务确认 | decimal | 业绩域 | 渠道提成比例（如 0.30） |
| `default_sample_requirements` | JSON | 商品域/寄样域 | 默认寄样门槛（销售额、等级等） |
| `copy_template` | text | 商品域 | 复制讲解文案模板 |
| `pick_extra_rule` | JSON | 商品域 | 转链 pick_extra 生成规则 |
| `login_max_failures` | 5 | int | 用户域 | 登录失败锁定次数 |
| `login_lock_minutes` | 15 | int | 用户域 | 锁定时长 |

### 2.2 default_sample_requirements 结构示例

```json
{
  "min_sales_30d": 30000,
  "min_talent_level": "LV1",
  "description": "近30天销售额≥3W，达人等级≥LV1"
}
```

### 2.3 copy_template 占位符

| 占位符 | 说明 | 替换来源 |
| ------ | ---- | -------- |
| `{product_name}` | 商品名称 | 商品域 |
| `{product_id}` | 商品ID | 商品域 |
| `{commission_rate}` | 佣金率 | 商品域 |
| `{service_fee_rate}` | 服务费率 | 商品域 |
| `{promotion_link}` | 推广链接 | 转链接口 |
| `{custom_text}` | 管理员固定文案 | 本配置 |

### 2.4 pick_extra_rule 结构示例

```json
{
  "format": "channel_{channel_id}_product_{product_id}",
  "encode": "none"
}
```

### 2.5 V2.0 差异化提成（预留）

`commissions` 表结构（V2.0 启用）：

| 字段 | 说明 |
| ---- | ---- |
| 配置ID | 主键 |
| 维度类型 | global / activity / product / user |
| 维度ID | 对应实体 ID，global 时为空 |
| 提成类型 | recruiter / channel |
| 比例 | decimal |
| 生效开始 | datetime |
| 生效结束 | datetime |

V1.0 仅使用 `system_configs` 中的全局比例；表结构可提前建表留空。

### 2.6 V2.0 预设标签库（预留）

| config_key | 说明 |
| ---------- | ---- |
| `preset_talent_tags` | JSON 数组，达人域 V2.0 读取 |

## 三、配置更新流程

```plain
管理员在「系统管理-规则配置」修改参数
      ↓
配置域校验（类型、范围）
      ↓
写入 system_configs + config_change_logs
      ↓
发出「配置已更新」事件
      ↓
各域 Consumer 刷新缓存：
  - 寄样域：sample_limit_days
  - 达人域：独家阈值、保护期
  - 业绩域：提成比例、独家商家阈值
  - 商品域：copy_template
  - 用户域：登录锁定参数
```

**注意**：提成比例变更**不自动**重算历史业绩，需管理员在业绩域手动触发重算。

## 四、业务规则汇总

### 4.1 校验规则

| config_key | 校验 |
| ---------- | ---- |
| sample_limit_days | 整数 1~90 |
| sample_auto_close_days | 整数 7~90 |
| exclusive_*_fee_ratio | 0.50 ~ 1.00 |
| exclusive_talent_monthly_samples | 整数 1~100 |
| talent_claim_protect_days | 整数 7~90 |
| recruiter_commission_rate | 0 ~ 1，两位小数 |
| channel_commission_rate | 0 ~ 1，两位小数 |
| recruiter + channel 合计 | V1.0 建议 ≤ 1（告警不阻断） |

### 4.2 权限

| 操作 | 权限 |
| ---- | ---- |
| 读取配置 | 各域内部接口 / 登录用户读公开项 |
| 更新配置 | 仅管理员 |
| 查看变更日志 | 管理员 |

### 4.3 变更日志

每次 update 写入 `config_change_logs`：

| 字段 | 说明 |
| ---- | ---- |
| 配置键 | config_key |
| 旧值 | old_value |
| 新值 | new_value |
| 操作人 | user_id |
| 操作时间 | datetime |

保留 1 年，可归档。

## 七、对外接口定义

### 7.1 读取接口（各域调用）

| 接口 | 用途 | 入参 | 出参 | 权限 |
| ---- | ---- | ---- | ---- | ---- |
| `get_config` | 读单个 | key | value + type | 内部 |
| `get_configs` | 读多个 | keys[] | key-value map | 内部 |
| `get_commission_rates` | 读提成 | - | recruiter_rate, channel_rate | 内部/业绩域 |

### 7.2 管理接口

| 接口 | 用途 | 入参 | 出参 | 权限 |
| ---- | ---- | ---- | ---- | ---- |
| `list_all_configs` | 全部配置 | group 可选 | 列表含说明 | 管理员 |
| `update_config` | 更新 | key, value | success | 管理员 |
| `batch_update_configs` | 批量更新 | items[] | success | 管理员 |
| `list_config_change_logs` | 变更历史 | key, 分页 | 日志列表 | 管理员 |

## 八、与其他领域的关系

```plain
┌──────────┐  get_config / 配置已更新
│  配置域  │ ─────────────────────────────→ 寄样域
│          │ ─────────────────────────────→ 达人域
│          │ ─────────────────────────────→ 业绩域
│          │ ─────────────────────────────→ 商品域
│          │ ─────────────────────────────→ 用户域
└──────────┘
     ↑ 仅管理员写
┌──────────┐
│  用户域  │
└──────────┘
```

| 消费方 | 读取的配置 |
| ------ | ---------- |
| 寄样域 | sample_limit_*、sample_auto_close_days |
| 达人域 | exclusive_talent_*、talent_claim_protect_days |
| 业绩域 | exclusive_merchant_fee_ratio、commission rates |
| 商品域 | copy_template、pick_extra_rule、default_sample_requirements |
| 用户域 | login_max_failures、login_lock_minutes |
