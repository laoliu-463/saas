## 当前建议用法

如果你要给老板或开发同学看：

1. 先看 [22-V1已确认范围-当前系统落地差距-验收证据清单](./22-V1已确认范围-当前系统落地差距-验收证据清单.md)
2. 需要逐领域深挖时，再看 [24-V1领域对齐总表](./24-V1领域对齐总表.md)
3. 还要继续拆到单领域时，再看 `23-01 ~ 23-09` 工作稿
4. 最后根据争议项回跳到原始需求包逐份核对

如果你要继续补验收：

1. 以 [V1交付范围表](./requirements/2026-05-新版需求包/V1交付范围表.md) 定范围
2. 以 [技术落地设计](./requirements/2026-05-新版需求包/技术落地设计.md) 定原始实现口径
3. 以 [22-V1已确认范围-当前系统落地差距-验收证据清单](./22-V1已确认范围-当前系统落地差距-验收证据清单.md) 作为当前项目执行清单
4. 涉及条目级 / 领域级冲突时，再展开 [24-V1领域对齐总表](./24-V1领域对齐总表.md)
