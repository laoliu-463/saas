# 01-业务闭环

更新时间：2026-05-18

## 一、V1 业务闭环目标

V1 的目标不是把旧版 V2.2 的所有高级能力一次做完，而是先跑通三条基础业务链：

1. 渠道链
2. 招商链
3. 管理链

判断标准不是“页面有了、接口通了”，而是业务角色能不能顺着主线把动作做完。

## 二、渠道链

### 主流程

渠道认领达人  
↓  
渠道维护达人地址 / 标签  
↓  
渠道在商品库选品  
↓  
渠道复制推广链接  
↓  
系统生成 `pick_source` 映射  
↓  
渠道申请寄样  
↓  
招商审核寄样  
↓  
运营 / 商家发货  
↓  
订单同步  
↓  
订单默认归因到渠道  
↓  
业绩统计

### 当前 V1 关键点

- 达人认领是渠道入口。
- 商品库是渠道选品入口。
- 推广转链必须产生 `pick_source_mapping`。
- 寄样不是孤立表单，要能进入状态流转。
- 订单归因先按默认渠道归因，不启用独家覆盖。
- 业绩先按默认归因 + 双轨金额统计。

## 三、招商链

### 主流程

招商组长同步活动  
↓  
同步活动商品  
↓  
分配 / 补充商品信息  
↓  
商品上架到商品库  
↓  
渠道推广  
↓  
寄样审核  
↓  
订单同步  
↓  
默认招商归因 = 活动绑定招商  
↓  
招商业绩统计

### 当前 V1 关键点

- 活动和活动商品是招商链起点。
- 商品上架后才进入共享商品库。
- 默认招商归因以活动 / 商品绑定负责人为主。
- 个别品负责人覆盖不作为 V1 P0 阻断项。
- 招商链先跑默认归因，不启用独家商家覆盖。

## 四、管理链

### 主流程

管理员配置人员、角色、数据范围  
↓  
管理员配置寄样限制、保护期、提成比例  
↓  
商品、达人、寄样、订单、业绩读取配置  
↓  
不同角色看到不同数据范围

### 当前 V1 关键点

- 权限和数据范围是主链的一部分，不是额外附件。
- 规则配置要能影响实际业务动作。
- V1 重点规则是：
  - 寄样 7 天限制
  - 达人保护期
  - 全局提成比例
  - 复制讲解模板 / `pick_extra` 规则

## 五、V1 统一简化口径

| 主题 | V1 口径 |
|---|---|
| 订单归因 | 只做默认渠道、默认招商归因 |
| 业绩归属 | `final_channel = default_channel`，`final_recruiter = default_recruiter` |
| 物流 | 手动录入物流信息和手动推进状态为主 |
| 看板 | 基础指标、基础下钻即可 |
| 提成 | 全局配置为主，不做差异化提成 |
| 达人数据 | 基础字段 + 手动兜底 |
| 角色能力 | 先保证主线角色可完成动作，再收紧边界 |

## 六、V2 / 后续规划

以下能力不作为当前 V1 主线必交内容：

- 独家达人链路
- 独家商家链路
- 差异化提成
- 高级排行榜
- 趋势分析增强
- 物流 API 自动轨迹
- Excel 批量导入物流单号
- 爬虫 Cookie 池 / 代理池

这些能力即使在旧版 V2.2 或领域设计里有完整描述，当前也统一归到“后续规划”。

## 七、当前结论

当前业务文档应统一理解为：

> V1 先跑通渠道链、招商链、管理链三条基础主线；独家机制、自动物流、高级分析等能力后置，不作为当前主线验收阻断。
