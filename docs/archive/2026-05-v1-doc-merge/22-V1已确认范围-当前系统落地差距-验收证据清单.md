## 二、状态定义

| 状态 | 含义 | 使用边界 |
|---|---|---|
| ✅ 已验收 | 有页面、接口、数据库、QA 报告或 real-pre 证据可复核 | 可以写入验收结论 |
| 🟡 已实现待验收 | 代码或页面有对应入口，但缺少完整业务链证据 | 不能写成已验收 |
| ⚠️ 有底座未联调 | 底层能力或 fallback 已有，但真实环境、真实样本或业务副作用未验证 | 默认不阻断 P0，需说明边界 |
| ❌ 不进 V1 / 缺口 | 范围表明确不进 V1，或 V1 必做但当前无实现 | 必须在备注中区分“不做”和“缺口” |

使用原则：

- “代码里有类 / service / controller”只能证明有实现入口，不能证明业务已验收。
- “TEST/mock 通过”不能替代 real-pre 或真实三方证据。
- “real-pre 单样本通过”不能替代所有异常分支、真实订单归因样本和生产发布结论。
- V1 主线从“做了多少”切换为“怎么验收，证据在哪里”。
