# 06-部署与对接计划

更新时间：2026-05-18

## 一、当前定位

当前阶段的部署与对接目标不是直接推生产，而是先保证：

1. `test` 环境可作为 V1 功能和权限验证基线稳定使用。
2. `real-pre` 环境可作为真实 SDK / 真实上游联调环境稳定切换。
3. 文档口径统一按当前 Spring Boot 实现描述，不再混写旧版 FastAPI / Celery 技术方案。

## 二、当前环境

| 环境 | 用途 | 当前口径 | 端口 / 数据库 |
|---|---|---|---|
| `local-mock` | 历史 Mock 口径与回滚参考 | 不再作为独立主线环境扩展 | 历史参考，不再单独定义 |
| `test` | 系统功能 / 权限测试 / 本地 Mock 联调 | 当前默认功能验证环境 | `3000 / 8080 / saas_test` |
| `real-pre` | 真实 SDK / 真实上游联调 | 当前真实联调环境 | `3000 / 8080 / saas_real_pre` |
| `prod` | 正式环境规划 | 本期不展开 | 后续单独规划 |

补充说明：

- 当前只允许一套 active 环境运行。
- `test` 和 `real-pre` 共用 `docker-compose.yml`，通过不同 `.env` 切换。
- `local-mock` 继续保留为历史脚本、历史报告和思路参考，不再作为新的主线部署口径。

## 三、当前技术栈

| 层级 | 当前实现 |
|---|---|
| 后端 | Spring Boot |
| 数据库 | PostgreSQL |
| 缓存 | Redis |
| 前端 | Vue 3 + Vite |
| 部署 | Docker Compose |
| 三方对接 | Gateway 方式隔离 Mock / Test / Real |

统一说明：

- 新版需求包中的 `FastAPI + Celery` 只作为原始设计背景。
- 当前部署和对接文档必须以 Spring Boot 实际工程形态为准。

## 四、对接优先级

### P0：当前主链优先

1. Token / 授权主体
2. 活动同步
3. 活动商品同步
4. 商品转链
5. `pick_source_mapping`
6. 订单同步
7. 默认渠道归因
8. 默认招商归因

### 非 P0：当前后置

1. Webhook 完整业务消费
2. 物流 API 自动跟踪
3. 达人爬虫增强
4. 店铺侧能力
5. 高级分析和高级看板

## 六、当前环境切换约束

1. 同一时间只允许一个 active 环境。
2. 固定容器名为：
   - `saas-frontend`
   - `saas-backend`
   - `saas-postgres`
   - `saas-redis`
3. 固定端口为：
   - 前端 `3000`
   - 后端 `8080`
   - PostgreSQL `5432`
   - Redis `6379`
4. 不再额外混起第二个本机 `3000` Vite 或第二个 `8080` 后端。
5. 前端只调用内部 `/api`，不直连第三方。

## 七、当前对接边界

1. 抖音联盟 / 团长侧能力是当前主链。
2. `real-pre` 是真实联调预备环境，不等同生产环境。
3. 物流当前按“手动录入 + 手动推进状态”作为 V1 默认口径。
4. 达人外部数据当前按“基础字段 + 手动兜底”作为 V1 默认口径。
5. 店铺侧订单、店铺侧物流、电子面单等能力不纳入当前 V1 主链。

## 八、当前建议用法

### 做功能联调

优先使用 `test`：

- 适合验证页面、接口、权限、规则和主链业务动作
- 适合不依赖真实三方样本的本地回归

### 做真实三方联调

使用 `real-pre`：

- 验证 Token、活动、活动商品、转链、订单同步
- 只做真实 SDK 相关动作
- 不把 `real-pre` 的证据反向写成“已可生产发布”
