## 一、V1 范围

### V1 做

- P-01 同步机构团长活动列表
- P-02 同步活动下商品、合作方信息
- P-03 招商补充信息、审核上架商品库
- P-04 商品库筛选
- P-05 商品排序 + 招商置顶
- P-06 复制讲解（转链 + `pick_source` 映射）
- P-07 快速寄样入口
- P-09 商品库展示规则
- P-10 活动 / 商品抖音状态同步
- P-11 合作方列表 / 详情查询

### V1 简化

- P-12 定时同步商品详情（价格 / 佣金）

### V1 不做

- P-08 个别品分配负责人覆盖活动默认招商
- P-13 活动商品批量导入
- P-14 合作方手动维护联系方式

## 三、与领域设计的主要偏差

| 类型 | 条目 | 说明 |
|---|---|---|
| 少做 | P-05 置顶规则 | 当前已具备排序底座，但 24h / 最多 10 个的规则还需单独确认 |
| 做法差异 | P-09 商品展示规则 | 当前以入库状态和展示判断收口，不完全照搬原文档术语 |
| 简化差异 | P-12 定时同步 | 当前以手动刷新 / 同步触发为主，不单独强调定时任务 |
| 明确后移 | P-08 / P-13 / P-14 | 不纳入当前 V1 验收 |

## 四、状态机与规则检查

| 设计口径 | 当前实现 | 结论 |
|---|---|---|
| 待补充 / 待审核 | `PENDING_AUDIT` 等待补充与审核 | ✅ |
| 可推广 / 展示中 | 审核通过后进入商品库展示路径 | ✅ |
| 已隐藏 | 当前有展示控制 / 下架判断底座 | ⚠️ 路径差异 |
| 跟进推进 | 已有分配、转链、跟进等主链动作 | ✅ |

核心规则：

- 先审核、后上架
- 复制讲解必须产生 `pick_source_mapping`
- 商品域只负责默认负责人链路，不扩到 V1 的个别品负责人覆盖

## 六、差异清单

### 必须修

- 无

### 需要确认

- P-05 招商置顶规则是否按当前版本继续收口
- P-11 合作方列表 / 详情查询的页面边界与查询口径

### V2 后移

- P-08 个别品负责人覆盖
- P-13 活动商品批量导入
- P-14 合作方手动维护联系方式

### 不再验收

- 不把个别品负责人覆盖写成当前 V1 默认招商归因能力
