# 09-01-MCP商品主链接口梳理

更新时间：2026-05-18

> 本文档是 [09-MCP接口梳理-第一版](./09-MCP接口梳理-第一版.md) 的第一份拆分稿，专门收口 **商品主链 5 组接口**。  
> 本轮只读取 **MCP / 离线 OAS / 前端调用 / 后端代码**，不改业务代码，不跑测试，不把 V2 后移项重新拉回 V1。

## 一、梳理范围

本轮只拆以下 5 组：

1. 活动列表 / 活动详情
2. 活动商品 / 商品同步
3. 商品审核 / 补充信息 / 上架
4. 商品库查询 / 商品详情
5. 商品转链 / `pick_source` mapping

统一边界：

- **主事实源**：`SAAS-business-offline` -> `docs/openapi/openapi-business.json`
- **辅助事实源**：前端 `src/api`、`src/views/product` 与后端 controller / service
- **不纳入主链**：
  - `/douyin/**`
  - 已明确废弃的 `/products/**` 之外的 V2 扩展诉求
  - 未在 OAS / controller 中确认的前端遗留调用

## 二、当前结论

1. 商品主链当前的**主入口**已经是 `/colonel/activities/**`。
2. 商品库展示链路当前仍然**实际依赖** `/products/**`，但该组接口在 OAS 中已明确标记为“商品管理（已废弃）”。
3. 当前没有发现以下两个前端调用对应的 business OAS / controller：
   - `POST /colonel/activities/{activityId}/sync`
   - `POST /colonel/activities/{activityId}/products/sync`
4. 当前可见的“商品同步 / 状态刷新”实现口径，更接近：
   - `GET /colonel/activities`
   - `GET /colonel/activities?refresh=true`
   - `GET /colonel/activities/{activityId}/products?refresh=true`
   - `GET /activities/{activityId}/douyin-detail`
5. 当前没有独立的 business OAS 接口直接查询 `pick_source_mapping`；要补证时，需要依赖：
   - 转链接口返回值
   - 推广记录历史接口
   - 订单详情 / 未归因排查页
   - 数据库证据

## 三、状态标记

| 状态 | 含义 |
|---|---|
| `已确认主链` | 已在当前 business OAS、controller、前端链路中形成一致口径 |
| `待核对` | 当前有实现迹象，但还不足以直接写成已确认主链 |
| `兼容旧链` | 接口仍在使用，但已被标记为旧链 / 废弃兼容 |
| `辅助接口` | 对排查、补水、修复有用，但不是商品主链的直接主入口 |
| `sdk-debug 辅助` | 仅在 `/douyin/**` 调试域使用，不进入本轮主表 |
| `V1 不采用` | 当前 V1 不以该接口 / 能力作为主链验收口径 |

## 七、九个问题的当前答案

1. **活动从哪里查？**
   - 主查 `GET /colonel/activities`。
2. **活动商品从哪里查？**
   - 主查 `GET /colonel/activities/{activityId}/products`。
3. **商品怎么进入商品库？**
   - 走 `POST /colonel/activities/{activityId}/products/{productId}/library-entry`。
4. **招商审核 / 补充信息对应哪个接口？**
   - 走 `PUT /colonel/activities/{activityId}/products/{productId}/audit-result`。
5. **渠道看到的商品库用哪个接口？**
   - 当前仍是 `GET /products`，属于兼容旧链；候选页兜底是 `GET /products/picks`。
6. **转链接口是哪一个？**
   - 主链是 `POST /colonel/activities/{activityId}/products/{productId}/promotion-links`。
7. **`pick_source mapping` 有没有接口能查？**
   - 没有独立 business OAS 读接口；当前只能通过推广记录、订单归因页或数据库补证。
8. **哪些前端调用和 OAS 不一致？**
   - `POST /colonel/activities/{activityId}/sync`
   - `POST /colonel/activities/{activityId}/products/sync`
9. **哪些接口是旧 `/products/**`，不再作为 V1 主入口？**
   - `GET /products`
   - `GET /products/picks`
   - `GET /products/{id}`
   - `POST /products/{id}/promotion-links`
   - `GET /products/{productId}/promotion-links/history`
   - 以及同组其它 `/products/**` 兼容接口
