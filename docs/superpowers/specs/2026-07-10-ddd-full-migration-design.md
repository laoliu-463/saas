# DDD 全量收口与渐进式迁移设计

**日期：** 2026-07-10

## 目标

完成当前 DDD 证据矩阵中的代码侧未完成项，并把仍由 Legacy Service 承担的核心业务职责逐步迁移到 domain application、policy、facade、port 和 event 边界；保留外部 API 和既有业务行为，直到新链路具备等价证据后再删除旧入口。

## 当前基线

- 178 张 DDD 任务卡：132 DONE、38 PARTIAL、8 BLOCKED、0 TODO。
- `cross-domain-mapper-legacy-whitelist.txt` 和 `architecture-redline-legacy-whitelist.txt` 当前有效项均为 0。
- 仍有 137 个工作区 dirty 文件，其中包含现有 DDD 代码、测试、文档和证据；不得 reset、checkout 或覆盖。
- 矩阵中的历史宽口径记录曾为 312 tests、7 failures；本次批准后在当前 dirty 工作区重新执行同一宽口径命令，得到 363 tests、0 failures、0 errors、1 skipped。后续仍需在每个迁移批次后重新执行，不能沿用历史结果。
- real-pre 当前缺有效抖音 access token 和可复跑真实订单/寄样样本，业务闭环只能保持 BLOCKED，不能用本地测试冒充。

## 方案选择

采用渐进式绞杀迁移：

1. 先修复可由代码和测试完成的红灯，形成稳定基线。
2. 每次只迁移一个主责领域或一个 Legacy 热点，保留兼容 façade。
3. 通过行为等价测试、架构守卫和调用方扫描确认新入口可用。
4. 等真实运行态证据和零调用方证据成立后，再删除旧实现。

不采用一次性大爆炸重构，因为当前工作区 dirty、Legacy 聚合点仍多、真实上游证据缺失；也不采用只补文档的方案，因为它不能降低实际迁移债务。

## 迁移边界

### 用户与配置

用户域只解释身份、角色、组织和 `self/group/all` 数据范围；配置域只提供配置事实、版本和审计。订单、业绩、商品、达人、寄样不得复制用户权限规则或执行配置业务规则。

### 订单、业绩与分析

订单域负责订单、退款、同步和归因输入事实；业绩域负责最终归属、提成、冲正和汇总；分析模块只读订单/业绩事实或汇总，不重算归属。

### 商品、达人与寄样

商品域负责活动商品、共享商品库、转链和 `pick_source_mapping`；达人域负责资料、认领、保护期和关系事实；寄样域负责申请、审核、物流状态机和基于订单已同步事件的完成判断。

## 执行批次

### 批次 0：基线和证据治理

记录当前分支、HEAD、dirty 文件分类、矩阵状态和宽口径失败清单；不清理既有用户改动。所有后续修改必须使用明确的文件范围和单卡证据。

### 批次 1：代码侧红灯

优先修复订单失败证据、业绩权限范围、佣金规则乐观锁、配置消费、业绩单元闭环、E-7 幂等键透传，以及 U-14/U-15、O-9、Y-4/Y-6/Y-17、A-6/A-8、P-10、T-4、S-1~S-4 等可本地完成项。每项先补失败测试，再做最小实现。

### 批次 2：核心 Legacy 热点迁移

依次处理 `ProductService`、`SampleApplicationService`、`TalentService`、`OrderSyncService`、`DataApplicationService` / `DashboardService`。每个热点按查询、命令、策略、持久化端口和事件边界拆分；Legacy 类只保留兼容委派，不新增业务规则。

### 批次 3：真实运行态与前端领域化

在授权和真实样本具备后执行订单同步、归因、寄样自动完成、业绩、Dashboard 的 API/SQL/页面对账；同时收口前端 API client、store 和页面的领域边界、全量 Vitest 和认证 E2E。

### 批次 4：清理与最终验收

逐项确认无调用方、迁移证据完整、白名单仍为 0 后删除旧代码；更新领域状态、矩阵、evidence、retro、Harness 限制检查，并提交、推送当前分支。

## 每批验收门禁

代码修改后必须执行：编译、定向测试、宽口径相关测试、固定 `agent-do`、Docker 重启、健康检查、业务验证、evidence report、retro summary 和 Harness 限制检查。任何缺失项均标记为 PARTIAL 或 BLOCKED，不写成 PASS。

## 外部依赖处理

抖音授权、真实订单、真实转链和真实寄样样本属于外部运行条件。代码、测试、日志和本地运行态可由本迁移流程完成；授权或样本缺失时只记录阻塞证据和解阻条件，不生成 mock 结果替代真实闭环。
