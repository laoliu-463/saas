# 合作单操作栏增强设计

日期：2026-07-16
更新：2026-07-18
状态：用户已确认，可实施

## 目标

在现有 `CooperationWorkbench.vue` 表格操作列中固定竖排展示七项操作：

1. 通过
2. 拒绝
3. 修改订单
4. 查看进度
5. 复制链接
6. 复制订单
7. 备注

不可执行项继续显示，但置灰并展示后端返回的原因。合作单继续复用寄样域和现有状态机，不新增业务领域。

## 业务规则

- 通过、拒绝：仅待审核可用，复用现有状态接口；拒绝原因必填。
- 修改订单：仅申请人或管理员可操作；待审核、待发货、快递中、已拒绝可修改。
- 待交作业、已完成、已关闭仍显示修改入口，但必须置灰。
- 可修改字段仅限申样备注、收货人、电话和地址；达人、商品、数量、规格只读。
- 收货信息读取合作单申请人在有效达人认领记录中保存的真实地址。
- 保存时在同一事务中更新合作单地址快照和申请人的达人地址，并使用乐观锁。
- 查看进度复用现有详情、物流轨迹和操作日志。
- 复制链接由商品域生成真实推广文本；转链失败不得伪造成功。
- 复制订单由寄样域按合作单真实事实生成固定十三行文本。
- 备注按“合作单 + 当前用户”私有保存，最多 200 字；管理员不可读取他人内容。
- 管理员拥有全部人工操作权限，但不能绕过系统专属的完成、关闭流转。

## 接口

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `GET` | `/samples/{id}/edit-context` | 获取只读事实和申请人真实地址 |
| `PUT` | `/samples/{id}/cooperation-details` | 乐观锁更新备注与地址 |
| `POST` | `/samples/{id}/promotion-copy` | 生成推广复制文本 |
| `GET` | `/samples/{id}/order-copy` | 生成订单复制文本 |
| `GET/PUT` | `/samples/{id}/private-note` | 当前用户私有备注 |

## 数据与边界

- 仅新增 `sample_private_note` 表，使用 `(sample_request_id, user_id)` 有效行唯一约束。
- 寄样域负责操作矩阵、受限编辑、订单文本和私有备注。
- 商品域负责真实转链和推广文本。
- 达人域负责申请人有效认领地址和近 30 天橱窗销量事实。
- 前端只消费后端能力矩阵，不硬编码权限或状态机。

## 验证

- 后端：七项顺序、状态/角色权限、地址原子双写、乐观锁、复制文本、备注隔离。
- 前端：竖排样式、固定顺序、禁用原因、编辑/备注弹窗、两种复制降级。
- 集成：迁移幂等、真实地址读取和更新一致性。
- 本地 `real-pre`：构建、容器重启、健康检查、业务验证和 evidence。
