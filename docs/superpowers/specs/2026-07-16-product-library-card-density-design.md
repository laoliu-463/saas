# 商品库卡片信息密度优化设计

## 目标

- 商品卡片横向与纵向间距由 16px 收紧到 8px。
- 商品名称取消两行截断，直接完整换行展示。
- 商品 ID 在卡片中直接完整展示，并保留复制能力。
- 卡片快捷按钮只保留“复制 ID”；删除“复制图文链接”“刷新”“详情”。
- 保持图片区悬浮主操作“复制简介”“快速寄样”，保持点击卡片进入详情的既有路径。

## 现状与证据

- 页面网格与虚拟网格都使用 `PRODUCT_LIBRARY_GRID_GAP=16`。
- 卡片标题使用 `-webkit-line-clamp: 2`，因此长名称被截断。
- 商品 ID 只通过一个写着 `ID` 的复制按钮出现，ID 内容本身未展示。
- 快捷区当前包含复制 ID、复制图文链接、刷新、详情四个按钮。
- 本地 real-pre 只读数据检查：商品名称最大 64 字符、95 分位 35 字符；商品 ID 最大 19 字符。

## 方案比较

### 方案 A：仅增加悬浮提示

改动最小，但商品名称和 ID 仍未直接完整展示，不满足需求，排除。

### 方案 B：卡片改为任意高度并移除虚拟网格

可以无上限展开内容，但会回退当前 100 条以上列表的虚拟渲染能力，性能风险较高，排除。

### 方案 C：保留固定行虚拟网格并扩大信息区（采用）

标题取消截断、ID 独立成行；根据 real-pre 最大长度把统一卡片高度从 432px 调整到可容纳四行名称和一行 ID 的高度。页面网格和虚拟网格继续共享同一高度与间距常量，避免虚拟行重叠。

## 组件设计

### `ProductSelectionCard.vue`

- 标题行只承载完整商品名称，移除标题右侧快捷按钮组。
- 标题下增加商品 ID 行：`商品 ID` 标签、完整 ID 文本、唯一的“复制 ID”按钮。
- ID 使用可换行的等宽文本，极端值也不产生横向溢出。
- 删除 `refresh` 事件声明；保留 `detail`、`copyBrief`、`quickSample` 业务事件。
- 保留卡片点击进入详情，不再额外显示“详情”快捷按钮。

### `ProductLibrary.vue`

- 删除两处 `@refresh` 监听。
- 删除不再可达的单卡刷新处理函数。
- 保留页面顶部“刷新商品”，因为本次删除范围是卡片下方快捷按钮。

### `product-library-layout.ts`

- 网格间距统一改为 8px。
- 卡片统一高度按测试样本调整，页面容器总高度、虚拟窗口偏移继续由共享常量计算。

## 测试与验收

- 组件测试先断言商品名称无截断样式、完整 ID 可见、快捷区仅存在复制 ID。
- 组件测试断言复制 ID 仍写入完整值。
- 页面测试断言不再向卡片绑定 refresh 事件。
- 虚拟网格测试断言新的共享高度与 8px 行间距计算一致。
- 执行相关 Vitest、前端全量测试、类型检查与生产构建。
- 使用本地 real-pre 重启前端容器，验证 `/healthz`、商品库页面、Console 和关键交互。
- evidence 结论必须区分自动化通过与浏览器业务验证结果。

## 边界与回滚

- 不修改商品接口、后端业务规则、权限规则、转链或寄样流程。
- 不删除悬浮区“复制简介”“快速寄样”。
- 回滚时恢复共享布局常量、卡片模板/CSS 和页面事件接线即可；无数据迁移与历史数据影响。
