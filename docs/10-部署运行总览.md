# 部署运行总览

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 当前运行口径

- [V1 必做] 当前运行以 Docker Compose 管理 PostgreSQL、Redis、后端和前端。
- [V1 必做] 本机常用端口组为 `3000/8080` 与 `3001/8081`，不得混起第二个 `3001` Vite 或额外 `8080` 后端。
- [V1 必做] real-pre 典型端口为前端 `3001`、后端 `8081`、PostgreSQL `5433`、Redis `6380`。
- [V1 必做] real-pre 必须关闭 mock：`APP_TEST_ENABLED=false`、`DOUYIN_TEST_ENABLED=false`。
- [V1 简化] profile 名称存在旧文档口径差异，实际运行前以 `.env.real-pre`、`.env.real-pre.example`、`docker-compose.real-pre.yml` 和启动日志为准。

## 环境清单

| 环境 | Compose / 配置 | 用途 | 验证 | 范围 |
| --- | --- | --- | --- | --- |
| test | `docker-compose.test.yml`、`.env.test` | 本地 mock / P0 回归 | E2E、API、SQL | V1 必做 |
| real-pre | `docker-compose.real-pre.yml`、`.env.real-pre` | 真实联调和上线前验收 | preflight、真实接口、日志 | V1 必做 |
| local-dev | `.env.local-dev.example` 等 | 日常开发 | 构建、后端测试 | V1 简化 |
| production | 未在本轮文档展开 | 线上部署 | 需单独 SOP | V2 预留 |

## 部署前检查

- [V1 必做] 环境变量不包含过期 Token、测试开关误开或端口冲突。
- [V1 必做] PostgreSQL 和 Redis 容器健康。
- [V1 必做] 后端 profile、数据库连接、Redis 连接、第三方配置在启动日志中可证。
- [V1 必做] 前端目标 API 地址指向当前环境后端。
- [V1 必做] real-pre 跑验收前执行预检命令。

## 明确不做

- [V1 不做] 不把旧 FastAPI / Celery 部署方案写为当前部署事实。
- [V1 不做] 不使用 mock 数据证明 real-pre 闭环。
- [V2 预留] 生产环境高可用、灰度发布、告警体系和容量规划另行建设。
