# 物流接口

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 对接目标

- [V1 简化] 支撑寄样发货、物流单号记录和必要的物流查询。
- [V1 必做] 真实物流接口不可用时，必须记录权限、样本或接口阻塞证据。

## 当前事实

- [V1 必做] 寄样域负责寄样物流事实和状态流转。
- [V1 简化] V1 可采用手工物流事实 + 第三方接口阻塞证据的组合。
- [V1 简化] 快递100查询已使用 Redis 保存跨实例限频状态，同一公司编码 + 运单号的安全查询间隔为 31 分钟；Redis 不可用时失败关闭，不绕过限频。
- [V1 简化] `QUERY_THROTTLED` 是跳过结果：不得覆盖已有物流状态、轨迹、原始报文或错误信息，也不得统计为成功。
- [V1 简化] 快递100回调入口为 `/public/logistics/kuaidi100/callback` 与兼容入口 `/api/public/logistics/kuaidi100/callback`；签名为大写 MD5(`param + callbackSalt`)。

## 数据流

1. [V1 必做] 寄样审批通过后进入发货。
2. [V1 必做] 管理端录入或接口获取物流单号。
3. [V1 简化] 如有接口权限，则 Gateway 调用物流接口查询状态。
4. [V1 必做] 寄样域记录物流事实和状态变更。
5. [V1 简化] 轮询前通过 Redis `SET NX` 获取 31 分钟运单级限频窗口；未获取到窗口时仅计 skipped。

## 验收证据

- [V1 必做] 发货操作有 API 响应和寄样状态记录。
- [V1 简化] 真实物流接口有请求 / 响应或 BLOCKED 证据。
- [V1 简化] 回调验收必须同时包含公网 HTTPS 可达、错误签名被拒绝、正确签名进入业务校验三项证据；不得输出回调盐值。
- [V1 必做] 不使用无法追溯的页面文字替代数据库或接口证据。

## 不做

- [V1 不做] 不把物流接口缺失包装成已完成真实对接。
- [V2 预留] 快递公司映射、自动签收、异常赔付后续设计。
