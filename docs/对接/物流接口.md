# 物流接口

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 对接目标

- [V1 简化] 支撑寄样发货、物流单号记录和必要的物流查询。
- [V1 必做] 真实物流接口不可用时，必须记录权限、样本或接口阻塞证据。

## 当前事实

- [V1 必做] 寄样域负责寄样物流事实和状态流转。
- [V1 简化] V1 可采用手工物流事实 + 第三方接口阻塞证据的组合。
- [V2 预留] 完整自动物流、异常轨迹、签收回调后续设计。

## 数据流

1. [V1 必做] 寄样审批通过后进入发货。
2. [V1 必做] 管理端录入或接口获取物流单号。
3. [V1 简化] 如有接口权限，则 Gateway 调用物流接口查询状态。
4. [V1 必做] 寄样域记录物流事实和状态变更。

## 验收证据

- [V1 必做] 发货操作有 API 响应和寄样状态记录。
- [V1 简化] 真实物流接口有请求 / 响应或 BLOCKED 证据。
- [V1 必做] 不使用无法追溯的页面文字替代数据库或接口证据。

## 不做

- [V1 不做] 不把物流接口缺失包装成已完成真实对接。
- [V2 预留] 快递公司映射、自动签收、异常赔付后续设计。
