# 达人信息获取

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 对接目标

- [V1 简化] 在权限允许时从第三方补充达人资料。
- [V1 必做] 系统内达人资料、标签、地址和跟进能力必须可用。

## 当前事实

- [V1 必做] 达人域负责达人资料事实。
- [V1 简化] 第三方达人信息获取受权限包和接口稳定性约束，可记录 BLOCKED。
- [历史归档] 旧 `Python + requests + fake-useragent` 采集方案不是当前事实。
- [V2 预留] 外部采集策略、合规审计、画像评分后续设计。

## 数据流

1. [V1 必做] 系统内维护达人基础资料。
2. [V1 简化] 如第三方权限可用，Gateway 获取达人补充字段。
3. [V1 必做] 达人域保存来源、更新时间和关键字段。
4. [V1 必做] 寄样、商品和业绩链路读取达人事实，不直接依赖第三方实时查询。

## 验收证据

- [V1 必做] 系统内达人列表、详情、标签、地址可通过 API/E2E 验证。
- [V1 简化] 第三方达人接口需提供真实响应或 BLOCKED 证据。
- [V1 必做] 证据写入 [../验收/验收证据索引.md](../验收/验收证据索引.md)。

## 不做

- [V1 不做] 不把旧爬虫方案写成当前对接事实。
- [V1 不做] 不让业务链路依赖不可验证的实时外部达人页面。
