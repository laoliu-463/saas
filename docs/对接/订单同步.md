# 订单同步

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 对接目标

- [V1 必做] 从抖店侧同步订单和退款事实，落入订单域。
- [V1 必做] 订单同步后发布订单已同步事件，供寄样域和业绩域消费。

## 当前事实

- [V1 必做] 订单域只存事实，不算提成，不应用独家覆盖。
- [V1 必做] 订单同步必须具备幂等、时间窗口、错误记录和真实响应证据。
- [V1 简化] V1 先保证 P0 时间窗口和真实样本，复杂补偿策略后续扩展。
- [历史归档] 旧 Celery 口径不是当前运行事实。

## 数据流

1. [V1 必做] 管理端或任务触发订单同步。
2. [V1 必做] Gateway 使用真实 Token 调用订单接口。
3. [V1 必做] 订单域保存订单事实和退款事实。
4. [V1 必做] 订单域发布订单已同步 / 退款事实已同步事件。
5. [V1 必做] 寄样域判断交作业完成，业绩域计算归属和冲正。

## 验收证据

- [V1 必做] 第三方订单接口请求 / 响应摘要。
- [V1 必做] 订单表、订单明细、退款表 SQL/API 证据。
- [V1 必做] 同步日志包含时间窗口、数量、成功 / 失败摘要。
- [V1 必做] 业绩域结果单独验收，不能用订单表替代。

## 不做

- [V1 不做] 不在订单同步时直接写最终提成。
- [V1 不做] 不用 mock 订单证明 real-pre 订单回流。
- [V2 预留] 大规模补偿、死信、事件重放和账期锁定后续设计。
