# 活动商品同步

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 对接目标

- [V1 必做] 从抖音 / 抖店侧同步活动商品，落入商品域，支撑招商和渠道推广。
- [V1 必做] 同步结果必须能通过商品 API、数据库和日志验证。

## 当前事实

- [V1 必做] 活动商品同步属于商品域能力。
- [V1 必做] 第三方字段进入内部模型时必须保留来源和必要原始标识。
- [V1 简化] V1 优先保证 P0 商品字段、筛选、列表和转链所需字段。
- [V2 预留] 更完整的商品展示规则、推荐排序和多平台商品源后续治理。

## 数据流

1. [V1 必做] 管理端或同步任务触发活动商品同步。
2. [V1 必做] 后端 Gateway 调用第三方接口。
3. [V1 必做] 商品域标准化字段并入库。
4. [V1 必做] 前端通过内部 API 查询商品。
5. [V1 必做] 验收通过 API/SQL/日志证明数据来自同步结果。

## 验收证据

- [V1 必做] 第三方请求 / 响应摘要。
- [V1 必做] 商品表或商品 API 返回同步商品。
- [V1 必做] 商品列表 E2E 可展示同步结果。
- [V1 必做] real-pre 空数据必须记录真实响应，不手工伪造。

## 不做

- [V1 不做] 不让前端直接调用第三方商品接口。
- [V1 不做] 不把 mock 商品作为 real-pre 同步成功证据。
