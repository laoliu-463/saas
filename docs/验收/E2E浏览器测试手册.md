# E2E 浏览器测试手册

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 测试目标

- [V1 必做] 用浏览器路径验证用户真实操作闭环。
- [V1 必做] E2E 不能替代 API/SQL 证据，必须与后端事实互相印证。

## 常用命令

| 命令 | 用途 | 范围 |
| --- | --- | --- |
| `npm run e2e:v1-p0` | V1 P0 本地 / test 闭环 | V1 必做 |
| `npm run e2e:real-pre:p0:preflight` | real-pre 预检 | V1 必做 |
| `npm run e2e:real-pre:p0` | real-pre P0 浏览器验收 | V1 必做 |
| `npx playwright show-report` | 查看测试报告 | V1 简化 |

## 覆盖路径

| 路径 | 重点断言 | 后端证据 | 范围 |
| --- | --- | --- | --- |
| 登录 / 权限 | 菜单、按钮、数据范围 | `/api/auth/me`、角色 API | V1 必做 |
| 商品 | 商品列表、筛选、转链 | 商品 API、`pick_source_mapping` | V1 必做 |
| 达人 | 列表、详情、标签、跟进 | 达人 API、操作日志 | V1 必做 |
| 寄样 | 申请、审批、发货、状态 | 寄样 API、状态表 | V1 必做 |
| 订单 | 同步、列表、详情 | 订单 API、同步日志 | V1 必做 |
| 业绩 | 明细、汇总、冲正 | 业绩 API、汇总表 | V1 必做 |
| Dashboard | 指标展示和范围过滤 | dashboard API、汇总表 | V1 必做 |

## 证据保存

- [V1 必做] Playwright report、截图、trace、console、Network 响应按失败用例保存。
- [V1 必做] 关键验收证据登记到 [验收证据索引.md](验收证据索引.md)。
- [V1 不做] 不用截图替代数据库事实或第三方真实响应。

## 失败处理

- [V1 必做] 先确认复现路径、账号、环境、API 响应、后端日志。
- [V1 必做] 不能仅根据页面现象判断根因。
- [V1 必做] 失败项进入缺陷或冲突清单，不得静默改验收口径。
