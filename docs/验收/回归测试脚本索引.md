# 回归测试脚本索引

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

## 根目录脚本

| 命令 | 作用 | 适用场景 | 范围 |
| --- | --- | --- | --- |
| `npm run e2e:v1-p0` | V1 P0 浏览器验收 | 本地 / test 验收 | V1 必做 |
| `npm run e2e:real-pre:p0:preflight` | real-pre 预检 | 真实联调前 | V1 必做 |
| `npm run e2e:real-pre:p0` | real-pre P0 E2E | 上线前联调 | V1 必做 |
| `npm run test:all` | 聚合测试入口 | 回归验证 | V1 简化 |
| `npx playwright show-report` | 查看 E2E 报告 | 失败排查 | V1 简化 |

## 后端脚本

| 命令 | 作用 | 适用场景 | 范围 |
| --- | --- | --- | --- |
| `cd backend; mvn test` | 后端测试 | 后端逻辑、文档影响代码时 | V1 必做 |
| `cd backend; mvn clean test` | 清理后全量测试 | 重大变更后 | V1 简化 |

## 前端脚本

| 命令 | 作用 | 适用场景 | 范围 |
| --- | --- | --- | --- |
| `cd frontend; npm run build` | 前端构建 | 前端代码或接口契约变更 | V1 必做 |
| `cd frontend; npm run dev` | 本地开发 | 手工验证 | V1 简化 |

## 文档重构检查

| 命令 | 作用 | 范围 |
| --- | --- | --- |
| `git status --short` | 确认只改文档和 `.claude` | V1 必做 |
| `rg -n "FastAPI|Celery|fake-useragent" docs .claude CLAUDE.md` | 检查历史口径是否只出现在归档 / 冲突语境 | V1 必做 |
| `Get-ChildItem -Recurse -File docs,.claude` | 检查目标结构 | V1 必做 |

## 说明

- [V1 必做] 只改文档时可以不跑业务测试，但必须说明未跑原因。
- [V1 必做] 如果文档修改改变了接口、事件或验收口径，应至少跑结构检查和相关 E2E。
