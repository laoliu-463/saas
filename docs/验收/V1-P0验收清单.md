# V1 P0 验收清单

> 范围标记：`V1 必做`、`V1 简化`、`V1 不做`、`V2 预留`、`历史归档`。

- [V1 必做] 主脚本：`npm run e2e:v1-p0`

## 不通过条件

- [V1 必做] 验收项没有脚本、API/SQL 或证据路径。
- [V1 必做] 前端展示通过但后端没有事实记录。
- [V1 必做] test/mock 结果被写成 real-pre 真实闭环。
- [V1 必做] 订单域出现提成或最终归属计算口径。

## 输出要求

- [V1 必做] PASS / FAIL / BLOCKED / PENDING 状态必须明确。
- [V1 必做] BLOCKED 必须说明外部原因、请求时间、响应摘要和下一步。
- [V1 必做] FAIL 必须能复现并给出最小复现路径。
