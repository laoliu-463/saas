## 一、V1 三条业务链

### 1. 渠道链

达人认领  
→ 商品库选品  
→ 复制讲解 / 转链  
→ 生成 `pick_source_mapping`  
→ 申请寄样  
→ 招商审核  
→ 运营发货  
→ 订单同步  
→ 默认归因到渠道  
→ 业绩统计

### 2. 招商链

同步活动  
→ 同步活动商品  
→ 审核 / 上架  
→ 维护合作方与商品信息  
→ 渠道推广  
→ 订单同步  
→ 默认归因到招商  
→ 业绩统计

### 3. 管理链

人员 / 角色 / 数据范围  
→ 规则配置  
→ 各域读取规则  
→ 影响商品、达人、寄样、订单、业绩行为

---

## 二、当前串联情况

| 链路 | 当前实现 | 主要差异 |
|---|---|---|
| 渠道链 | 主链已基本贯通 | 真实 `pick_source` 样本、达人地址带入、寄样自动完成仍需专项确认 |
| 招商链 | 活动 / 商品同步、审核、上架已有底座 | 个别品负责人覆盖不进 V1 |
| 管理链 | 用户、权限、配置已具备主线能力 | 一人多角色和部分配置简化边界需固定 |

---

## 三、关键跨域关系

| 来源领域 | 目标领域 | 关系 | 当前情况 |
|---|---|---|---|
| 用户域 | 全域 | 角色、数据范围、人员下拉 | ✅ |
| 配置域 | 达人域 | 保护期 | ✅ |
| 配置域 | 寄样域 | 7 天限制、超时规则 | ✅ |
| 配置域 | 商品域 | 复制讲解模板、`pick_extra` 规则 | ⚠️ 需继续对齐 |
| 商品域 | 订单域 | `pick_source_mapping`、默认招商基础 | ✅ |
| 达人域 | 寄样域 | 地址带入寄样申请 | ⚠️ 需专项确认 |
| 订单域 | 寄样域 | 订单命中后寄样完成 | ⚠️ 需专项确认 |
| 订单域 | 业绩域 | 双轨金额、默认归因事实来源 | ✅ |
| 业绩域 | 分析模块 | 统计口径与看板展示 | ✅ |

## 四、当前不纳入 V1 主链的跨域能力

- 独家达人 / 独家商家跨域覆盖
- 个别品负责人覆盖默认招商
- 物流 API 自动跟踪
- Excel 批量导入物流
- 高级排行 / 高级经营分析

## 五、差异清单

### V1 需要确认

### V2 后移
