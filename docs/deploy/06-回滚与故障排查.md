# 回滚与故障排查

## 适用场景

本文用于 real-pre 部署失败、端口不可达、授权失败、订单未回流或需要回滚应用版本时使用。

## 当前仓库实际情况

- 回滚脚本：`scripts/rollback-real-pre.sh`。
- 备份脚本：`scripts/backup-db.sh`。
- 健康检查脚本：`scripts/health-check.sh`。
- Compose project：`saas-active`。
- 服务名：`postgres-real-pre`、`redis-real-pre`、`backend-real-pre`、`frontend-real-pre`。
- 默认先回滚应用，不轻易回滚数据库。

## 前置条件

- 可以 SSH 登录服务器。
- `/opt/saas/app` 是 Git 仓库或可切换代码版本。
- `/opt/saas/env/.env.real-pre` 存在且可读。
- Docker 和 Compose 可用。

## 执行步骤

### 1. 部署前记录 commit

```bash
cd /opt/saas/app
git rev-parse --short HEAD
git log --oneline -5
```

### 2. 数据库备份

优先使用仓库脚本：

```bash
cd /opt/saas/app
ENV_FILE=/opt/saas/env/.env.real-pre BACKUP_DIR=/opt/saas/backups ./scripts/backup-db.sh
```

手动备份示例：

```bash
cd /opt/saas/app
docker compose \
  --env-file /opt/saas/env/.env.real-pre \
  --project-name saas-active \
  -f docker-compose.real-pre.yml \
  exec -T postgres-real-pre pg_dump -U saas -d saas_real_pre -F c > /opt/saas/backups/saas_real_pre-$(date +%Y%m%d-%H%M%S).dump
```

### 3. 查看状态和日志

```bash
cd /opt/saas/app
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml ps
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=300 backend-real-pre
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=300 frontend-real-pre
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=300 postgres-real-pre
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=300 redis-real-pre
```

### 4. 重启服务

```bash
cd /opt/saas/app
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml restart backend-real-pre
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml restart frontend-real-pre
```

如需重建：

```bash
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml up -d --build
```

不要执行 `down -v`，避免删除 real-pre 数据卷。

### 5. 应用回滚到上一 commit

推荐用脚本：

```bash
cd /opt/saas/app
ENV_FILE=/opt/saas/env/.env.real-pre ./scripts/rollback-real-pre.sh 上一个稳定commit
```

手动回滚：

```bash
cd /opt/saas/app
git fetch origin
git checkout 上一个稳定commit
ENV_FILE=/opt/saas/env/.env.real-pre ./scripts/deploy-real-pre.sh
```

默认策略：先回滚应用，不轻易回滚数据库。只有迁移破坏表结构且有备份证据时，才评估数据库恢复。

## 故障分类与排查

### 1. 服务器端口不通

```bash
ss -lntp | grep -E '3001|8081|80|443'
curl -v http://127.0.0.1:8081/api/system/health
curl -v http://127.0.0.1:3001/healthz
```

判断：本机通但公网不通，优先查云安全组和宝塔防火墙。

### 2. Nginx 反代失败

```bash
curl -v https://real-pre.xxx.com/api/system/health
tail -n 200 /www/wwwlogs/real-pre.xxx.com.error.log
```

判断：502 多为后端/前端端口不可达；404 多为反代路径错误。

### 3. SSL 证书失败

```bash
curl -I http://real-pre.xxx.com
curl -I https://real-pre.xxx.com
```

判断：HTTP 不通或 DNS 未生效会导致文件验证失败。

### 4. 后端启动失败

```bash
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=300 backend-real-pre
```

重点看：DB、Redis、JWT、抖音配置、profile。

### 5. 前端页面白屏

```bash
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=300 frontend-real-pre
curl -fsS http://127.0.0.1:3001/healthz
```

浏览器再看 Console 和 Network。

### 6. 数据库连接失败

```bash
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml ps postgres-real-pre
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=200 postgres-real-pre
```

判断：已有 volume 下改 `DB_PASSWORD` 不会自动改库内密码。

### 7. Redis 连接失败

```bash
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=200 redis-real-pre
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=200 backend-real-pre | grep -i redis
```

判断：`REDIS_PASSWORD` 需要和 Redis 容器启动参数一致。

### 8. 抖音 Token 失败

```bash
docker compose --env-file /opt/saas/env/.env.real-pre --project-name saas-active -f docker-compose.real-pre.yml logs --tail=300 backend-real-pre | grep -E "Douyin|Token|oauth"
```

判断：code 过期、client 配置错误、权限不足、主体未授权都可能导致失败。

### 9. 百应回调失败

```bash
curl -I https://real-pre.xxx.com/api/douyin/oauth/callback
```

没有 code/state 时失败是正常的；重点确认路径能到后端，不是 Nginx 404/502。

### 10. 真实订单没有回流

```sql
select count(*) from colonelsettlement_order;
select order_id, create_time, pick_source, colonel_buyin_id
from colonelsettlement_order
where deleted = 0
order by create_time desc
limit 20;
```

判断：没有真实订单样本时 P0 应保持 PENDING，不写 PASS。

## 验收标准

- 回滚前有 commit 记录。
- 回滚前有数据库备份或明确说明首次部署无可备份容器。
- 回滚后 `/api/system/health` 和 `/healthz` 通过。
- 故障结论能对应到日志、命令输出或数据库证据。

## 常见问题

| 问题 | 处理 |
| --- | --- |
| 想清空 volume 重来 | 禁止直接执行，除非人工确认并有备份 |
| 回滚后数据库仍异常 | 先确认是否迁移破坏 schema，再评估备份恢复 |
| Jenkins 部署失败 | 先用手动脚本复现，Jenkins 不作为第一根因 |
| 日志没有明确错误 | 增加命令证据，不用猜测下结论 |
