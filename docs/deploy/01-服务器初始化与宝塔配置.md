# 服务器初始化与宝塔配置

## 适用场景

本文用于第一次准备服务器，安装基础组件、规划目录、开放必要端口，并明确宝塔只承担站点、SSL、Nginx 反代、防火墙和运维入口职责。

## 当前仓库实际情况

- 项目代码建议放在 `/opt/saas/app`。
- 真实环境变量文件建议放在 `/opt/saas/env/.env.real-pre`。
- Docker Compose 文件是 `docker-compose.real-pre.yml`。
- Compose project 固定为 `saas-active`。
- 后端宿主端口默认 `8081`，前端宿主端口默认 `3001`。
- PostgreSQL 和 Redis 只应在 Docker 网络内访问，不应开放公网端口。

## 前置条件

- Linux 服务器一台，建议最低 `2 vCPU / 4GB RAM / 40GB 磁盘`。这是部署建议，不是仓库硬编码约束。
- root 或 sudo 权限。
- 服务器能访问 GitHub，或准备本地打包上传。
- 如果使用宝塔，先完成宝塔面板安装并限制面板访问 IP。

## 端口规划

| 端口 | 用途 | 公网策略 |
| --- | --- | --- |
| `22` | SSH | 允许，建议限制来源 IP |
| `80` | HTTP / Let's Encrypt 验证 | 允许 |
| `443` | HTTPS | 允许 |
| 宝塔面板端口 | 面板管理 | 允许但必须限制来源 IP，建议修改默认端口 |
| `3001` | real-pre 前端临时调试 | 仅第一轮端口测试临时开放，反代成功后关闭 |
| `8081` | real-pre 后端临时调试 | 仅第一轮端口测试临时开放，反代成功后关闭 |
| `5432` | PostgreSQL 容器内端口 | 禁止公网开放 |
| `6379` | Redis 容器内端口 | 禁止公网开放 |
| `18080` | Jenkins 可选 | 第二阶段再开放，且限制来源 IP |

## 目录规划

```bash
sudo mkdir -p /opt/saas/app
sudo mkdir -p /opt/saas/env
sudo mkdir -p /opt/saas/logs
sudo mkdir -p /opt/saas/backups
sudo mkdir -p /opt/saas/runtime/qa/out
sudo chown -R "$USER":"$USER" /opt/saas
```

| 目录 | 用途 |
| --- | --- |
| `/opt/saas/app` | 项目代码 |
| `/opt/saas/env` | `.env.real-pre`，禁止提交 Git |
| `/opt/saas/logs` | 部署日志、容器日志快照 |
| `/opt/saas/backups` | PostgreSQL 备份 |
| `/opt/saas/runtime/qa/out` | 部署和 E2E 证据 |

## 执行步骤

### 1. 安装 Git

Ubuntu / Debian：

```bash
sudo apt-get update
sudo apt-get install -y git curl ca-certificates gnupg lsb-release
git --version
```

CentOS / Rocky：

```bash
sudo dnf install -y git curl ca-certificates
git --version
```

如果系统仍使用 `yum`：

```bash
sudo yum install -y git curl ca-certificates
git --version
```

### 2. 安装 Docker / Docker Compose

Ubuntu / Debian 示例：

```bash
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list >/dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker
docker --version
docker compose version
```

CentOS / Rocky 提示：

```bash
sudo dnf install -y dnf-plugins-core
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker
docker --version
docker compose version
```

如需普通用户执行 Docker：

```bash
sudo usermod -aG docker "$USER"
```

执行后重新登录 SSH。

### 3. 宝塔基础设置

宝塔安装完成后建议：

1. 修改默认面板端口。
2. 面板入口限制为你的固定 IP。
3. 安装 Nginx。
4. 安装 Docker 管理插件可选，但部署命令仍以服务器 Shell 中的 `docker compose` 为准。
5. 不在宝塔数据库面板创建 PostgreSQL / Redis；这两个服务由 Compose 管理。

### 4. 防火墙检查

Ubuntu / Debian 常见 `ufw`：

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw status
```

CentOS / Rocky 常见 `firewalld`：

```bash
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
sudo firewall-cmd --list-all
```

端口测试阶段如需临时开放 `3001/8081`，只允许短时间开放，并在 Nginx 反代成功后关闭。

## 验收标准

```bash
docker --version
docker compose version
git --version
ls -ld /opt/saas /opt/saas/app /opt/saas/env /opt/saas/logs /opt/saas/backups /opt/saas/runtime/qa/out
```

期望：

- Docker 服务运行。
- `docker compose version` 可用。
- `/opt/saas` 目录存在且当前部署用户可写。
- 公网未开放 `5432/6379`。

## 常见问题

| 问题 | 排查命令 | 判断和处理 |
| --- | --- | --- |
| Docker 命令无权限 | `groups` | 当前用户不在 `docker` 组，执行 `sudo usermod -aG docker "$USER"` 后重新登录 |
| Docker 未启动 | `systemctl status docker` | 执行 `sudo systemctl enable --now docker` |
| Compose 不存在 | `docker compose version` | 安装 `docker-compose-plugin` |
| 宝塔面板暴露风险 | 云安全组 / 宝塔防火墙 | 修改面板端口并限制来源 IP |
| 端口被占用 | `ss -lntp` | 先确认占用进程，再决定是否停止 |
