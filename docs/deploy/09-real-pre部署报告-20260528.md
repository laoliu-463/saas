## 构建方式

采用**本地构建 + SCP + 服务器打包**策略（避免服务器端缓慢的镜像拉取）：

1. **后端**：本地 `mvn clean package -DskipTests` → 生成 `colonel-saas.jar`（79MB）→ SCP 到服务器
2. **前端**：本地 `npx pnpm install && pnpm build` → 生成 `dist/` → SCP 到服务器
3. **服务器**：`docker compose up -d --build` 仅打包预构建产物（耗时 < 10s）

# 查看后端日志
ssh saas "cd /opt/saas && docker compose --project-name saas-active -f docker-compose.real-pre.yml logs -f backend-real-pre"

# 重启后端（修改 .env 后）
ssh saas "cd /opt/saas && docker compose --env-file .env.real-pre --project-name saas-active -f docker-compose.real-pre.yml restart backend-real-pre"

# 停止全部服务
ssh saas "cd /opt/saas && docker compose --project-name saas-active -f docker-compose.real-pre.yml down"

# 停止并删除数据卷（⚠️ 会丢失数据库数据）
