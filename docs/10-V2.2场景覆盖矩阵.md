# 10-V2.2 场景覆盖矩阵

更新时间：2026-05-05

## 一、文档范围

本文件仅基于《抖音团长 SaaS 系统 - 完整需求方案文档 V2.2》整理。

明确说明：

- 只记录 V2.2 文档中已经出现的业务场景
- 不额外补充密码修改、站内消息、审批通知等文档外需求
- 当前状态判断以“现有代码 + 当前 `local-mock` / `test` / `real-pre` 已验证基线”为准
- 本地页面级回归默认以 `3000/8080` 可见浏览器分批验收为准：每个测试用例单独新开浏览器并截图归档

## 二、覆盖状态说明

- `已覆盖`：当前项目已进入现有主链路，可在 `local-mock` 稳定演示，并已具备 `real-pre` 浏览器回归结果
- `部分覆盖`：已有基础能力，但未达到 V2.2 完整业务口径
- `未进入本阶段`：V2.2 提到，但当前不作为主线交付
- `待澄清`：V2.2 本身口径存在歧义，需要后续补规则

## 六、待澄清问题

以下问题来自 V2.2 文档本身，不是当前代码额外引入的问题：

1. 商品审核拒绝后的“退回”对象是谁，是招商组长、待分配池，还是仅标记不可见？
2. 推广链接生成失败、归因标识失效、官方平台无返回时，渠道侧如何感知和重试？
3. 未归因订单是否允许人工补归因？
4. 当前实现口径已调整为“未认领当前用户的达人仍可出现在团队公海，用户认领后进入自己的私海”；当前已按“他人已认领但我仍可继续认领”的业务口径收口。
5. 达人保护期内“无产出”的判定口径是什么，订单、寄样、合作记录是否都算？
6. “团队长可审核成为独家达人”中的团队长具体是哪类角色？
7. 人工审核独家达人由谁发起、审核通过后何时生效？
8. 寄样发货责任方是运营、商家，还是运营仅录入商家物流信息？
9. 手动确认签收的角色是谁？
10. 数据平台导出能力存在口径冲突，需要确认 V1.0 是否开放导出。
11. 删除员工后，其历史业绩、客户、达人、寄样和操作日志如何处理。

## 七、当前结论

当前项目并不是“V2.2 全量场景全部完成”，而是已经完成了其中一组最关键、最适合 `local-mock` 演示和回归，并已在 real-pre 做过页面级扩展验证的核心闭环。

更准确的表述应该是：

> 当前项目已完成 V2.2 中商品转链、订单归因、寄样履约、达人 CRM 和数据看板相关核心场景的本地 Mock 闭环；其中商品中心当前执行口径已演进为“选品库 + 商品库”双层结构，并已完成“加入商品库后共享可见”的浏览器回归，达人 CRM 已完成“多人可同时认领同一达人”的可见专项回归。当前数据看板已补上基础提成计算与活动级比例覆写能力，但独家机制、批量处理、导出和完整规则中心仍属于后续阶段。
