## 一、文档范围

本文件仅基于《抖音团长 SaaS 系统 - 完整需求方案文档 V2.2》整理。

明确说明：

- 里程碑与测试数量等业务事实以主干文档为准；本矩阵仅做场景覆盖对照，不与 `docs/04` 抢「当前进度」定义权

- 不额外补充密码修改、站内消息、审批通知等文档外需求

## 二、覆盖状态说明

- `已覆盖`：当前项目已进入现有主链路，可在 `local-mock` 稳定演示，并已具备 `real-pre` 浏览器回归结果
- `部分覆盖`：已有基础能力，但未达到 V2.2 完整业务口径
- `未进入本阶段`：V2.2 提到，但当前不作为主线交付
- `待澄清`：V2.2 本身口径存在歧义，需要后续补规则

## 六、待澄清问题

以下问题来自 V2.2 文档本身，不是当前代码额外引入的问题：

1. 商品审核拒绝后的“退回”对象是谁，是招商组长、待分配池，还是仅标记不可见？
2. 推广链接生成失败、归因标识失效、官方平台无返回时，渠道侧如何感知和重试？
3. 未归因订单是否允许人工补归因？
4. 当前实现口径已调整为“未认领当前用户的达人仍可出现在团队公海，用户认领后进入自己的私海”；当前已按“他人已认领但我仍可继续认领”的业务口径收口。
5. 达人保护期内“无产出”的判定口径是什么，订单、寄样、合作记录是否都算？
6. “团队长可审核成为独家达人”中的团队长具体是哪类角色？
7. 人工审核独家达人由谁发起、审核通过后何时生效？
8. 寄样发货责任方是运营、商家，还是运营仅录入商家物流信息？
9. 手动确认签收的角色是谁？
10. 数据平台导出能力已开放到“活动列表 / 订单明细 CSV”口径，但是否需要继续扩展为统一导出中心仍待确认。
11. 删除员工后，其历史业绩、客户、达人、寄样和操作日志如何处理。

## 七、当前结论

当前项目并不是“V2.2 全量场景全部完成”，而是已经完成了其中一组最关键、最适合 `local-mock` 演示和回归，并已在 real-pre 做过页面级扩展验证的核心闭环。

更准确的表述应该是：

> 当前项目已完成 V2.2 中商品转链、订单归因、寄样履约、达人 CRM 和数据看板相关核心场景的本地 Mock 闭环，并已在 real-pre 通过准入、单样本业务闭环与六类角色账号业务流程验收。当前状态可以表述为 `real-pre 准入通过 -> P3 单样本业务闭环通过 -> P3-5 全角色账号业务流程通过 -> P3-6 SYNC_OK_NO_SAMPLE -> P3-7 Dashboard 与 PostgreSQL 口径对账 PASS -> P3-8 异常分支 PARTIAL_PASS -> 当前约 94%`。其中 P3-6 当前为“部分收口，等待真实 pick_source 订单样本”，P3-7 已证明 Dashboard Summary 与 PostgreSQL 聚合口径一致，P3-8 当前为“第一批只读异常证据已冻结，剩余项继续等待 live 样本或安全路径”。系统已经具备 real-pre 业务可用性，但还不能宣称可生产发布；真实订单归因样本与异常分支证据仍需补齐。

## 八、组长角色权限收口（2026-05-06）

本节用于固定“招商组长 / 渠道组长”权限边界，后续所有改动均以本节为验收基线。

补充说明：

- `admin` 是后端超级角色，可覆盖类级 / 方法级 `@RequireRoles` 做系统管理、联调排障和权限兜底；下列组长 / 运营权限口径描述的是普通业务角色的前台入口和业务动作边界
- `colonel_leader` 当前不再作为前台主口径单独扩展；若历史账号或旧数据仍带该角色，执行时按 `biz_leader` 兼容处理
- 前端菜单、首页落地与路由展示统一按 `biz_leader` 口径收口；后端活动获取、商品分配与负责人候选接口仍保留 `colonel_leader` 兼容，避免旧账号在本地/real-pre 验收时出现“接口可调、页面无入口”的断裂

### 8.1 招商组长（`biz_leader`）口径

- 可见模块：`商品库`、`活动管理`、`商品管理`、`寄样管理`、`数据看板`
- 不可见模块：`达人 CRM`、`系统管理`
- 可执行动作：查看共享商品库、活动绑定/重试、待审商品分配审核负责人、已入库商品分配招商负责人（含重分配）、寄样列表查看与导出、看板与订单明细查看导出
- 禁止动作：商品审核通过/拒绝、寄样审核通过/拒绝、系统配置修改、角色/员工管理写操作、手动触发真实订单同步

### 8.2 渠道组长（`channel_leader`）口径

- 可见模块：`商品库`、`达人 CRM`、`寄样台`、`数据看板`
- 不可见模块：`系统管理`
- 可执行动作：选品与转链、达人认领与跟进、寄样申请、寄样/看板导出、组内数据查看
- 禁止动作：商品审核通过/拒绝、寄样审核通过/拒绝、达人归属覆盖、系统配置修改
- 特殊规则：渠道组长本人申请寄样时豁免“7天同达人同商品限制”

### 8.3 运营（`ops_staff`）口径

- 可见模块：`寄样发货台`
- 不可见模块：`商品库`、`达人 CRM`、`寄样审核页`、`数据看板`、`独家状态`
- 可执行动作：查看`待发货 / 快递中 / 待交作业 / 已完成 / 已关闭`寄样单、录入物流单号、确认签收进入待交作业

## 九、组长权限回归清单（防回归）

### 9.1 招商组长回归（`biz_leader`）

- 登录后可进入：`/product`、`/product/manage`、`/sample`、`/data`、`/data/orders`
- 登录后不可进入：`/talent`、`/system/*`
- 商品页无“审核”按钮；待审核商品仅可“分配审核人”，审核通过入库后可“分配/重新分配招商”
- 寄样详情无“审核通过/拒绝”按钮
- 访问达人归属覆盖接口返回 403：`POST /talents/{id}/override-assignee`
- 访问商家归属覆盖接口返回 403：`POST /merchants/{id}/override-assignee`
- 调用商品审核接口返回 403：`PUT /products/{id}/audit-result`、`PUT /colonel/activities/{activityId}/products/{productId}/audit-result`
- 调用真实订单手动同步接口返回 403：`POST /orders/sync`

### 9.2 渠道组长回归（`channel_leader`）

- 登录后可进入：`/product`、`/talent`、`/sample`、`/data`、`/data/orders`
- 登录后不可进入：`/system/*`
- 商品页可转链，不可审核
- 寄样详情无“审核通过/拒绝”按钮
- 访问达人归属覆盖接口返回 403：`POST /talents/{id}/override-assignee`
- 访问商家归属覆盖接口返回 403：`POST /merchants/{id}/override-assignee`
- 寄样导出可用：`GET /samples/exports`
- 同达人同商品 7 天内重复申请寄样可通过（组长豁免）

### 9.3 负向接口校验（统一）

- 招商组长与渠道组长访问寄样审核动作均返回 403：`POST /samples/batch-approve`、`POST /samples/batch-reject`
- 招商组长与渠道组长访问商品审核动作均返回 403（仅招商可审）

## 十、运营权限回归清单（2026-05-06）

- 登录后可进入：`/ops/shipping`、`/sample/:id`（仅通过发货台详情进入）
