## 四、阅读顺序建议

### 做需求核对

1. 先读本文档，确认七个领域入口。
2. 再读对应领域原文，例如 [01-用户域](./requirements/domain-design/01-用户域.md)。
3. 最后读 [14-领域设计实现核验](./14-领域设计实现核验.md) 与 [24-七域需求代码核验报告](./24-七域需求代码核验报告.md)，确认当前实现差距和代码证据。

### 做代码开发

1. 先读对应领域原文。
2. 再读 [04-开发进度](./04-开发进度.md) 和 [10-V2.2场景覆盖矩阵](./10-V2.2场景覆盖矩阵.md)。
3. 涉及接口、环境或三方联调时补读 [03-Test与Real网关契约](./03-Test与Real网关契约.md)、[05-接口与数据模型](./05-接口与数据模型.md)、[06-部署与对接计划](./06-部署与对接计划.md)、[09-真实SDK联调准备清单](./09-真实SDK联调准备清单.md)。
4. 以当前代码和测试结果确认最终事实。
