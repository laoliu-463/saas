# 已知风险

[V1 必做] real-pre 真实 `pick_source` 新单样本仍需持续观察。

[V1 必做] real-pre 当前缺少完整渠道转链样本：`pick_source_mapping` 中 8 条记录均为 `colonel_native_*` 种子，`activity_id`、`product_id`、`promotion_link_id`、`converted_url` 为空；`promotion_link` 表为空。不能把这些种子映射当作商品转链闭环证据。

[V1 必做] `buyin.colonelMultiSettlementOrders` 非空样本依赖真实上游窗口。

[V1 必做] Dashboard 不能等同完整业绩结算域。

[V1 必做] 真实达人外部采集受风控/API 权限影响。

[V1 必做] 真实物流 API 尚未完成 sandbox/real-pre 闭环。

[V1 必做] 订单查询 `DataScope.DEPT` 范围下仅按 `colonelsettlement_order.dept_id`（招商主部门）做权限过滤，未匹配 `channel_dept_id`。后果：纯渠道部门用户（sys_dept.dept_type=channel_group）在 DEPT 数据范围下可能漏看本部门订单。命中位置：`OrderController.applyDataScope` / `applyQueryDataScope`。修复方向：按用户所属部门的 `dept_type` 决定比对 `dept_id` 还是 `channel_dept_id`，或采用 `(dept_id = ? OR channel_dept_id = ?)` 的 OR 兜底。暂以"部门筛选下拉"作为业务补救手段（admin/ALL 范围用户可手动选渠道部门筛选），未做权限层根因修复。
