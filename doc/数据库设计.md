## 一、设计原则

| 原则 | 说明 |
|------|------|
| 主键 | UUID v4，支持水平分库分表扩展 |
| 外键 | 按模块内聚，本模块内建 FK，跨模块仅逻辑关联 |
| 审计字段 | 所有业务表含 `create_time`、`update_time`、`create_by`、`update_by` |
| 软删除 | 核心业务表使用 `deleted` 标记（默认 0），不物理删除 |
| 金额单位 | 分（整数），避免浮点精度问题 |
| 时间分区 | `orders` 和 `logs` 表按月 RANGE 分区 |
| JSONB | 用于扩展配置字段和动态属性 |

## 二、ER 关系总览

```
users ──N:N── roles (sys_user_role)
    │
    └── N:1 ── departments          (招商组 / 渠道组)

products ──N:1── activities
       └── N:N── talents (talent_claims)

talents ──1:N── exclusive_talents    (独家达人)
       └── 1:1 ── pick_source_mapping (归因记录，含short_id透传机制)

merchants ──1:N── exclusive_merchants (独家商家记录)

sample_requests ──N:1── talents
               ──N:1── products
               ──N:1── users (招商)

orders (月分区) ──N:1── products
              ──N:1── users (colonel/招商)
              ──N:1── activities
              ──1:N── order_details              (解密后的敏感信息)

commission_settlement ──N:1── orders
                     ──N:1── users (colonel)

commission_config ──N:1── users (提成配置，可为空=全局)

order_decrypt_records ──N:1── orders

system_configs
douyin_tokens
```
