# 错误判断标准与解决方案

**版本**：V1.0
**更新日期**：2026-04-23
**状态**：执行依据

---

## 一、错误分类体系

### 1.1 错误层级

| 层级 | 分类 | 来源 | 处理策略 |
|------|------|------|----------|
| L1 | 系统级错误 | Spring Boot / JVM | 日志记录 + 告警 + 兜底 |
| L2 | 业务级错误 | 抖音 SDK 返回 | 按错误码分流 |
| L3 | 数据级错误 | 入参校验 / 持久化 | 快速失败 + 明确提示 |
| L4 | 集成级错误 | 抖音 API 调用 | 重试 + 熔断 |

## 二、抖音 SDK 错误码

### 2.1 Token 相关错误

| 错误码 | sub_code | 含义 | 解决方案 |
|--------|----------|------|----------|
| 10000 | — | 成功 | — |
| 31007 | — | 授权码失效或过期 | 重新获取授权码 |
| 31021 | — | 授权码已使用 | 重新获取授权码 |
| 31006 | — | 授权记录失效 | 重新授权 |
| 31020 | — | 授权记录不存在 | 重新授权 |
| 31005 | — | 授权已撤销 | 重新授权 |
| 31012 | — | 并发刷新/生成冲突 | 降低并发，等待后重试 |
| 31009 | — | refresh_token 不存在 | 重新授权 |
| 31008 | — | refresh_token 过期 | 重新授权 |
| 31011 | — | 应用不存在或被删除 | 检查应用配置 |
| 31003 | — | grant_type 参数错误 | 检查请求参数 |

**处理流程：**

```
收到 Token 错误
    │
    ├── 31012（并发冲突）─→ 等待 1 秒重试（最多 3 次）
    ├── 31007/31021（授权码失效）─→ 提示用户重新授权
    ├── 31009/31008（refresh_token 失效）─→ 提示用户重新授权
    └── 其他 ─→ 记录日志 + 告警
```

### 2.2 业务参数错误

| 错误码 | sub_code | 含义 | 解决方案 |
|--------|----------|------|----------|
| 40004 | isv.parameter-invalid:257 | 参数校验失败 | 检查参数格式与范围 |
| 40004 | isv.parameter-invalid:1031 | 无效 size | size 必须在 1~100 |
| 40004 | isv.parameter-invalid:282 | 无效 time_type | 仅支持 settle/update |
| 40004 | isv.parameter-invalid:1033 | 未指定时间范围或订单号 | 提供 orderIds 或时间范围 |
| 40004 | isv.parameter-invalid:1036 | 无效时间区间 | 时间跨度不超过 90 天 |
| 40004 | isv.parameter-invalid:284 | 无效 cursor | cursor 必须是数值字符串 |
| 40004 | isv.parameter-invalid:1043 | 无效活动状态 | status 必须在 0/1/2/3/4/5/7 |

**处理流程：**

```
收到参数错误
    │
    ├── 40004 错误码
    │   ├── 记录 sub_code 和 sub_msg
    │   ├── 返回给用户明确提示
    │   └── 不重试（参数问题不会自动恢复）
    │
    └── 其他 400xx 错误
        └── 记录日志 + 告警
```

### 2.3 业务失败错误

| 错误码 | sub_code | 含义 | 解决方案 |
|--------|----------|------|----------|
| 50002 | isv.business-failed:4197 | 需【招商团长】角色授权 | 联系商家确认授权角色 |
| 50002 | isv.business-failed:500 | 活动不存在 | 检查 activity_id 是否正确 |
| 20000 | isp.service-error:256 | 服务异常（可重试） | 等待后重试（指数退避） |

**处理流程：**

```
收到业务失败
    │
    ├── 50002:4197（权限不足）
    │   ├── 提示用户检查授权
    │   └── 记录授权缺失
    │
    ├── 50002:500（活动不存在）
    │   └── 提示检查 activity_id
    │
    └── 20000（服务异常）
        └── 等待 5 秒重试（最多 3 次）
```

## 三、后端统一异常处理

### 3.1 全局异常处理器

`GlobalExceptionHandler.java` 统一处理：

| 异常类型 | HTTP 状态码 | 错误码 | 说明 |
|----------|-------------|--------|------|
| `DouyinApiException` | 502 | 来自 sub_code | 抖音 API 错误 |
| `ValidationException` | 400 | VALIDATION_ERROR | 参数校验失败 |
| `BusinessException` | 400 | BUSINESS_ERROR | 业务逻辑错误 |
| `UnauthorizedException` | 401 | UNAUTHORIZED | 未认证 |
| `ForbiddenException` | 403 | FORBIDDEN | 无权限 |
| `ResourceNotFoundException` | 404 | NOT_FOUND | 资源不存在 |
| `RedisLockException` | 409 | LOCK_FAILED | 分布式锁获取失败 |
| 其他 | 500 | INTERNAL_ERROR | 内部错误 |

### 3.2 统一响应格式

```json
{
  "code": "ERROR_CODE",
  "msg": "错误描述",
  "data": null,
  "traceId": "xxx"
}
```

## 四、参数校验规则

### 4.1 抖音 API 参数规范

| 接口 | 参数 | 校验规则 |
|------|------|----------|
| 活动列表 | status | 仅允许 0/1/2/3/4/5/7 |
| 活动列表 | search_type | 仅允许 0/1/2 |
| 活动列表 | sort_type | 仅允许 0/1 |
| 活动列表 | page | >= 1 |
| 活动列表 | page_size | 1~20 |
| 订单列表 | size | 1~100 |
| 订单列表 | cursor | 数值字符串 |
| 订单列表 | timeType | 仅支持 settle/update |
| 订单列表 | startTime/endTime | 成对存在，跨度 <= 90 天 |
| 订单列表 | orderIds | 最多 100 个 |
| 商品状态 | products | 1~50 个，非空白 URL |

### 4.2 校验失败处理

- 快速失败，返回 400 状态码
- 错误信息明确指出哪个参数、什么规则
- 不调用外部 API，直接返回

## 五、重试策略

### 5.1 可重试错误

| 错误类型 | 条件 | 重试策略 |
|----------|------|----------|
| 网络超时 | 连接超时 / 读取超时 | 立即重试，最多 3 次 |
| 限流 | 429 Too Many Requests | 等待 10 秒重试 |
| 服务异常 | 20000 / 500 | 等待 5 秒重试，最多 3 次 |
| 并发冲突 | 31012 | 等待 1 秒重试，最多 3 次 |

### 5.2 不可重试错误

| 错误类型 | 条件 | 处理策略 |
|----------|------|----------|
| 参数错误 | 40004 | 记录日志，返回用户提示 |
| 授权失效 | 31007/31009/31008 | 提示用户重新授权 |
| 资源不存在 | 404 / 50002:500 | 记录日志，提示检查 ID |

### 5.3 熔断策略

- 连续失败 5 次，熔断 60 秒
- 熔断期间跳过调用，直接返回「服务繁忙」
- 熔断恢复后逐步放行

## 六、日志规范

### 6.1 日志级别

| 场景 | 级别 | 内容 |
|------|------|------|
| 接口调用成功 | INFO | method、耗时、响应码 |
| 参数校验失败 | WARN | 参数名、规则、实际值 |
| 可重试错误 | WARN | 错误码、重试次数 |
| 不可重试错误 | ERROR | 完整错误信息、堆栈 |
| 限流触发 | WARN | 调用方标识 |
| 授权失效 | ERROR | 需人工介入 |

### 6.2 日志字段

```
时间 | 级别 | traceId | 模块 | 操作 | 耗时(ms) | 错误码 | 描述
```

## 七、错误知识库

### 7.1 常见问题速查

| 问题 | 现象 | 原因 | 解决 |
|------|------|------|------|
| token 获取失败 | 10003 错误 | code 已使用或过期 | 重新获取授权码 |
| 活动列表为空 | 返回空数组 | 团长无活动或授权不足 | 检查授权角色 |
| 订单同步卡住 | 长时间无新订单 | 同步时间窗口问题 | 检查 Redis 锁状态 |
| 归因结果异常 | 佣金比例不对 | 未匹配独家配置 | 检查有效期和商家/达人 ID |
| 寄样状态不变 | 状态停留在中间态 | 自动闭环条件不满足 | 检查订单匹配逻辑 |

### 7.2 错误定位 Checklist

```
1. 确认错误层级（L1~L4）
2. 检查日志 traceId
3. 检查抖音 sub_code 和 sub_msg
4. 确认参数是否符合规范
5. 检查 Token 是否有效
6. 检查授权是否包含所需权限
```
