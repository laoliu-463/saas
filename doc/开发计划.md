## 二、当前代码实况

### 后端已完成

1. **权限与系统管理**
   - SysUser / SysRole CRUD
   - JWT 登录认证
   - RBAC + DataScope 切面（数据隔离）

2. **抖音接入与订单**
   - DouyinApiClient + DouyinTokenService（官方 token.create / token.refresh）
   - ActivityApi / ProductApi / OrderApi / PromotionApi / TalentApi 封装
   - OrderSyncService + OrderSyncJob（滑窗 + 分布式锁）
   - DouyinTokenRefreshJob（自动刷新定时任务）

3. **业务模块**
   - 商品库、达人 CRM 基础版、寄样台（含状态机）
   - 寄样自动闭环（订单驱动 + 30 天超时关闭）
   - 独家商家 / 独家达人实体 + Service
   - 月度评估定时任务（ExclusiveEvalJob）

4. **测试**
   - `mvn test` 全绿

### 前端已完成

- 登录、商品、达人、寄样、看板、系统管理等核心页面
- RBAC 路由守卫

## 三、联调优先级

| 优先级 | 任务 | 说明 |
|--------|------|------|
| **P0** | 第三方 SDK 真联调 | 当前仅本地/Mock 验证，真实接口待验收 |
| P1 | M1.6 看板真实口径 | 指标字段与真实订单口径对齐 |
| P2 | M1.7 Docker 部署 | docker-compose 全链路回归 |

**联调完成定义（doc/API_INTEGRATION.md）：**
- 真实环境能稳定获取 token
- 真实接口返回字段映射通过
- 订单同步任务可在真实返回下稳定跑通
- 异常码（尤其 31012）处理符合预期

## 四、执行顺序

```
1. M1.3 真联调验收（SDK 最小闭环）
   └─ token.create → activity.list → order.sync → 入库 + 归因
2. M1.6 看板真实化验收
   └─ 指标口径与真实数据对齐
3. M1.7 部署验收
   └─ docker-compose 全链路
4. V2.0 功能验收与补齐
```
