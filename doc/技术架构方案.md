## 四、核心模块设计

### 4.1 爬虫模块（Java + Hutool）

```java
@Service
public class TalentCrawlerService {

    private final HttpRequest httpRequest = HttpRequest.create();

    /**
     * 采集达人信息
     * 使用Hutool HttpUtil进行请求，Jsoup解析HTML
     */
    public void crawlTalentInfo(String seedUrl) {
        // 1. 获取页面内容
        String html = HttpUtil.get(seedUrl);

        // 2. 解析HTML提取数据
        Document doc = Jsoup.parse(html);
        String name = HtmlUtil.clean(doc.select(".name").first().html(), WhitespaceAndTagFilter.ALL);
        String fans = doc.select(".fans-count").first().text();

        // 3. 构建达人对象
        Talent talent = Talent.builder()
                .name(name)
                .fansCount(parseFans(fans))
                .crawledAt(new Date())
                .build();

        // 4. 存储到数据库
        saveTalent(talent);
    }
}
```

**Hutool工具链：**
- `HttpUtil` - HTTP请求（GET/POST/文件下载）
- `HtmlUtil` - HTML清理、防XSS
- `Jsoup` - HTML解析（需额外引入）
- `StrUtil` - 字符串处理
- `DateUtil` - 日期处理
- `CollUtil` - 集合操作

### 4.3 抖音API调用封装

```java
@Component
public class DouyinApiClient {

    private final RedisTemplate<String, String> redisTemplate;

    /**
     * 获取有效Token（自动刷新）
     */
    public String getValidToken() {
        String token = redisTemplate.opsForValue().get("douyin:token");
        if (StrUtil.isBlank(token) || isTokenExpired(token)) {
            token = refreshToken();
        }
        return token;
    }

    /**
     * 封装抖音API调用
     */
    public <T> T callApi(String method, Map<String, Object> params, Class<T> responseType) {
        String token = getValidToken();

        HttpRequest request = HttpUtil.createPost(DOUYIN_API_URL);
        request.form("method", method);
        request.form("access_token", token);
        request.form("param", JSONUtil.toJsonStr(params));

        String response = request.execute().body();
        return JSONUtil.toBean(response, responseType);
    }
}
```

## 五、性能优化策略

| 优化点 | 策略 | 目标 |

## 六、与原方案对比

| 对比项 | 原方案（FastAPI+Celery+Python爬虫） | 新方案（Spring Boot+Hutool） |
|--------|-----------------------------------|------------------------------|
| 后端框架 | Python FastAPI | Java Spring Boot |
| 爬虫 | Python独立进程 | Java Hutool（内置） |
| 异步任务 | Celery + Redis | Spring @Scheduled |
| ORM | SQLAlchemy | MyBatis-Plus |
| 部署 | 微服务架构 | 单体 + Docker Compose |
| 维护成本 | 双语言栈 | 统一Java栈 |

**优势：**
1. 统一技术栈，降低维护复杂度
2. Spring Boot生态完善，文档丰富
3. Hutool工具库轻量且功能强大
4. Java企业级稳定性

**需注意：**
1. Python爬虫可复用部分代码逻辑
2. 抖音API调用需重新封装Java版本
3. 原API_INTEGRATION.md接口定义不变，仅实现层调整

## 七、Git 管理规范

### 7.1 分支策略

采用 **Git Flow** 为主、**Trunk-Based** 为辅的混合策略：

```
main (生产环境)
  └── release/v1.0 (发布分支)
  └── feature/v0.5-xxx (功能开发)
  └── fix/xxx (热修复)
```

| 分支类型 | 命名规范 | 生命周期 | 合并目标 |
|----------|----------|----------|----------|
| main | `main` | 永久 | - |
| release | `release/v{version}` | 版本发布 | main |
| feature | `feature/v{version}-{模块名}` | 功能开发 | develop |
| hotfix | `fix/{issue-id}` | 热修复 | main + develop |

**示例：**
```bash
feature/v0.5-product-crud      # V0.5商品模块CRUD
feature/v1.0-douyin-api         # V1.0抖音API对接
feature/v2.0-exclusive-talent   # V2.0独家达人功能
fix/order-sync-500-error        # 修复订单同步500错误
```

### 7.2 提交规范

**Commit Message 格式：**
```
<类型>(<模块>): <简短描述>

[可选正文：详细说明、影响范围、关联Issue]
[可选-footer：breaking changes、关闭的Issue]
```

**类型标识：**

| 类型 | 说明 | 示例 |
|------|------|------|
| feat | 新功能 | `feat(order): 添加订单批量解密接口` |
| fix | Bug修复 | `fix(product): 修复商品列表分页异常` |
| refactor | 重构（无功能变化） | `refactor(auth): 提取Token刷新逻辑` |
| docs | 文档更新 | `docs: 更新API接口文档` |
| test | 测试相关 | `test: 添加订单模块单元测试` |
| chore | 构建/工具变更 | `chore: 升级Hutool版本` |
| perf | 性能优化 | `perf(order): 优化订单批量插入性能` |
| ci | CI/CD变更 | `ci: 添加GitHub Actions构建流程` |

**示例：**
```bash
# 简单提交
git commit -m "feat(product): 添加商品上下架功能"

# 详细提交
git commit -m "feat(order): 实现订单批量解密接口

- 新增OrderDecryptService服务类
- 支持最多50条订单数据解密
- 关联抖音order.batchDecrypt接口

Closes #123"
```

### 7.5 版本发布流程

| 阶段 | 操作 | 说明 |
|------|------|------|
| 1. 准备 | `git checkout -b release/v1.0 main` | 创建发布分支 |
| 2. 修复 | 在release分支修复Bug | 不引入新功能 |
| 3. 封版 | 更新版本号、Changelog | 确认发布内容 |
| 4. 合并 | PR到main，标记tag | `git tag v1.0.0` |
| 5. 部署 | 触发CI/CD流水线 | 自动部署 |
| 6. 清理 | 删除release分支 | 保留tag |

**版本号规范：** `v{主版本}.{次版本}.{修订版本}`
- 主版本：不兼容的API变更
- 次版本：向后兼容的功能新增
- 修订版本：向后兼容的Bug修复

### 7.6 Git Hooks 配置

使用 **pre-commit** 和 **commit-msg** 钩子保证规范执行：

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-added-large-files

  - repo: https://github.com/commitizen-tools/commitizen
    rev: v3.5.0
    hooks:
      - id: commitizen
        stages: [commit-msg]
```
