## 三、核心设计

### 3.1 认证与权限

**JWT + RBAC + DataScope 三层模型：**

1. **JWT Token**：用户登录后颁发，有效期可配置，存储用户 ID 和角色
2. **RBAC 角色权限**：角色 → 菜单 → 按钮三级映射，支持超级管理员
3. **DataScope 数据范围**：按商户/部门隔离数据，SQL 拦截注入 `merchant_id` 条件

**相关文件：**
- `config/SecurityConfig.java` — 安全配置
- `security/DataScopeAspect.java` — 数据权限 AOP
- `common/exception/GlobalExceptionHandler.java` — 统一异常处理

### 3.2 抖音 SDK 封装

**Token 生命周期管理：**
- `token.create` — 授权码换取 token（支持工具型/自用型）
- `token.refresh` — 自动刷新（TTL 14 天，过期前 1 小时触发）
- `DouyinTokenRefreshJob` — 定时任务自动刷新

**API 分层封装：**
- `DouyinApiClient` — 统一 HTTP 调用层（post/postWithoutAuth）
- `ActivityApi` — 团长活动（createOrUpdate / colonelActivityDetail / instituteColonelActivityList）
- `ProductApi` — 商品查询（materialsProductStatus / colonelActivityProduct）
- `OrderApi` — 订单查询（colonelMultiSettlementOrders / 滑窗同步）

### 3.3 订单同步与归因

**滑窗同步机制：**
- 定时任务每 5 分钟触发
- Redis 分布式锁防重
- 滑动时间窗口拉取增量订单
- 批量插入数据库

**归因优先级（doc/rules/attribution-logic.md）：**
1. 独家商家（按商家维度归因）
2. 独家达人（按达人维度归因）
3. pick_source 映射（商品来源渠道）

### 3.4 寄样闭环

**状态机：**
```
申请 → 待审核 → 审核通过(待发货) → 已发货 → 已签收 → 待交作业 → 已交作业(完成)
                         ↓                              ↓
                    审核拒绝                       超时关闭(30天)
```

**自动闭环规则：**
- 订单入库后，自动将对应寄样记录从「待交作业」推进至「已交作业」
- 「待交作业」超过 30 天无订单，自动关闭

### 3.5 数据库设计

**月度分区表策略（doc/rules/partition-table.md）：**
- `colonelsettlement_order` — 按 `create_time` 月度 RANGE 分区
- `sample_log` — 按 `create_time` 月度 RANGE 分区

**核心实体：**
- `ColonelsettlementOrder` — 团长结算订单
- `Talent` — 达人
- `Product` — 商品
- `Sample` — 寄样记录
- `Merchant` — 商家
- `ExclusiveTalent` — 独家达人配置
- `ExclusiveMerchant` — 独家商家配置
- `PickSourceMapping` — 渠道映射

## 四、部署架构

### 4.1 开发环境

```yaml
# docker-compose.yml
services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: colonel_saas
      POSTGRES_USER: saas
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build: ./backend
    ports:
      - "8080:8080"
    depends_on:
      - postgres
      - redis

  frontend:
    build: ./frontend
    ports:
      - "3000:80"
    depends_on:
      - backend
```

## 五、性能优化

| 优化点 | 策略 | 目标 |
|--------|------|------|
| 订单查询 | 月度分区表 + 复合索引 | 5 万条/天查询 < 500ms |
| 商品库 | Redis 缓存商品列表 | 首屏加载 < 1s |
| 高并发 | Nginx 限流 + 数据库连接池 | 支持 50+ 并发用户 |
| 订单同步 | 增量轮询 + 批量写入 | 同步延迟 < 5 分钟 |
| 爬虫采集 | 异步线程池 + 限流 | 不影响主流程 |
