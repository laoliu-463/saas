# 环境切换与生产检查清单

## 1. 环境策略

- `test` 环境：用于联调与回归，默认 `mock` 数据源
- `prod` 环境：用于生产运行，强制 `real` 数据源

## 2. 核心配置

### test（Mock）

```env
SPRING_PROFILES_ACTIVE=test
TALENT_ENRICH_MODE=mock
TALENT_PUBLIC_PAGE_CRAWL_ENABLED=false
DB_NAME=colonel_saas_test
REDIS_DATABASE=1
```

### prod（Real）

```env
SPRING_PROFILES_ACTIVE=prod
TALENT_ENRICH_MODE=real
TALENT_PUBLIC_PAGE_CRAWL_ENABLED=false
DB_NAME=colonel_saas
REDIS_DATABASE=0
```

## 3. 已落地保护

- `talent.enrich.mode` 已配置化（`mock|real`）
- `MockTalentProvider` 仅在 `mock` 模式生效
- `ThirdParty/InternalBusiness` 仅在 `real` 模式生效
- `prod` + `mock` 会在启动阶段直接失败（防误上线上）

## 4. 切换步骤

1. 更新环境变量（profile/mode/DB/Redis）
2. 重启后端服务
3. 登录后验证达人列表页顶部环境标识
4. 执行最小回归：
   - 新增达人
   - 自动补全
   - 手动补录
   - 单个刷新
   - 批量刷新

### Docker 启动方式

- 测试环境（test + mock）：

```bash
docker compose -f docker-compose.yml -f docker-compose.test.yml up -d
```

说明：测试环境覆盖文件已开启后端 DevTools 自动重启（代码改动后无需手动重启容器）。

- 生产环境（prod + real）：

```bash
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

说明：生产环境不启用 DevTools 自动重启。
